## v1.4 Requirements (Phase 16 — Assessment Monitoring)

### Assessment Monitoring (MON)

- [x] **MON-01**: Monitoring tab groups sessions by assessment (Title + Category + Schedule.Date) — one row per group instead of one row per user — ✅ Shipped v1.4
- [x] **MON-02**: Each group row shows completion progress — "X/Y completed" with a visual progress bar — ✅ Shipped v1.4
- [x] **MON-03**: Each group row shows pass rate — "Z passed (N%)" based on completed sessions — ✅ Shipped v1.4
- [x] **MON-04**: HC can click "View Details" on a group row to navigate to a dedicated detail page showing each user's name, status, score, and pass/fail result — ✅ Shipped v1.4
- [x] **MON-05**: Monitoring includes recently closed assessments (closed within last 30 days) in addition to Open + Upcoming — ✅ Shipped v1.4
- [x] **MON-06**: Groups are sorted by schedule date (soonest first for Open/Upcoming, most recent first for closed) — ✅ Shipped v1.4

## v1.5 Requirements (Phase 17 — Question and Exam UX)

Requirements were defined in `.planning/phases/17-question-and-exam-ux-improvements/17-CONTEXT.md` and plan must_haves.

### Package Management (PKG)

- [x] **PKG-01**: HC can create multiple packages per assessment (e.g., Paket A, Paket B, Paket C) — ✅ Shipped v1.5
- [x] **PKG-02**: HC can import questions via Excel file upload (.xlsx) using ClosedXML parser — ✅ Shipped v1.5
- [x] **PKG-03**: HC can import questions via paste from Excel (tab-separated) — ✅ Shipped v1.5
- [x] **PKG-04**: Import format: Question | Option A | Option B | Option C | Option D | Correct (letter) — parser accepts `D`, `D.`, `D. text`, `OPTION D` — ✅ Shipped v1.5
- [x] **PKG-05**: HC can preview exam content (PREVIEW MODE — import order, no timer, no submit, correct answers highlighted) — ✅ Shipped v1.5
- [x] **PKG-06**: HC can delete a package and all its questions — ✅ Shipped v1.5

### Exam UX (EXM)

- [x] **EXM-01**: System randomly assigns each user one package when exam starts (idempotent on resume) — ✅ Shipped v1.5
- [x] **EXM-02**: Questions shuffled per user (Fisher-Yates); option order shuffled per user per question — ✅ Shipped v1.5
- [x] **EXM-03**: Exam paged layout — 10 questions per page, Prev/Next navigation — ✅ Shipped v1.5
- [x] **EXM-04**: Countdown timer visible; turns red at 5 minutes remaining; auto-submits at 0 — ✅ Shipped v1.5
- [x] **EXM-05**: Header shows assessment name and "X/N answered" progress — ✅ Shipped v1.5
- [x] **EXM-06**: Collapsible question number panel (answered = green, unanswered = grey) — ✅ Shipped v1.5
- [x] **EXM-07**: Skipping allowed — user can advance without answering all questions on current page — ✅ Shipped v1.5
- [x] **EXM-08**: Pre-submit summary page shows all questions + selected answers; unanswered rows highlighted — ✅ Shipped v1.5
- [x] **EXM-09**: Grading uses PackageOption.Id (ID-based) — not displayed A/B/C/D letters — ✅ Shipped v1.5
- [x] **EXM-10**: Legacy assessments (no packages) continue to grade via existing path — ✅ Shipped v1.5
