## v1.9 Requirements

### Schema (SCHEMA)

- ✅ **SCHEMA-01**: ProtonTrack exists as a first-class table; ProtonKompetensi references it via FK; all code uses ProtonTrackId — no TrackType+TahunKe string fields remain — v1.9 (Phase 33)

### Catalog Management (CAT)

- ✅ **CAT-01**: HC/Admin can view the Proton Catalog Manager page from navigation — v1.9 (Phase 34)
- ✅ **CAT-02**: HC/Admin can select a track from a dropdown and see the full Kompetensi → SubKompetensi → Deliverable tree with expand/collapse — v1.9 (Phase 34)
- ✅ **CAT-03**: HC/Admin can add a new Kompetensi to the selected track inline without a page reload — v1.9 (Phase 35)
- ✅ **CAT-04**: HC/Admin can add a new SubKompetensi under a Kompetensi inline without a page reload — v1.9 (Phase 35)
- ✅ **CAT-05**: HC/Admin can add a new Deliverable under a SubKompetensi inline without a page reload — v1.9 (Phase 35)
- ✅ **CAT-06**: HC/Admin can rename any Kompetensi, SubKompetensi, or Deliverable in-place via pencil-icon edit — v1.9 (Phase 35)
- ✅ **CAT-07**: HC/Admin can delete any catalog item only after seeing the affected active coachee count in a confirmation modal — deletion cascades to all child items — v1.9 (Phase 36)
- ~~**CAT-08**: HC/Admin can drag Kompetensi, SubKompetensi, or Deliverable rows to reorder within their level — the new order persists immediately~~ — **CANCELLED** (SortableJS incompatible with nested-table tree structure; user decision to remove entirely)
- ✅ **CAT-09**: HC/Admin can add a new Track via modal without a page reload — new track appears in the dropdown immediately — v1.9 (Phase 34)
