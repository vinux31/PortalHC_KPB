### `tests/helpers/dbSnapshot.ts` (generic-util, request-response)

**Analog:** `.planning/seeds/313-timer-fixtures.sql` (consumer pattern) + RESEARCH § Pattern 2 (template code)

**Why partial match:** Project belum punya TypeScript wrapper untuk sqlcmd. Phase 313.1 invoke sqlcmd langsung via dokumentasi comment (line 13: `sqlcmd -S "localhost\SQLEXPRESS" -d HcPortalDB_Dev -E -C -I -i ...`). Phase 315 jadi yang pertama wrap sqlcmd di Node — gunakan flag set yang sudah proven dari Phase 313.1 + tambah `-b` (fail-on-error).

**Flag pattern to copy** (`.planning/seeds/313-timer-fixtures.sql` line 13):
```
sqlcmd -S "localhost\SQLEXPRESS" -d HcPortalDB_Dev -E -C -I -i <script.sql>
```
→ TS wrapper SQLCMD_BASE_ARGS WAJIB:
```ts
const SQLCMD_BASE_ARGS = [
  '-S', 'localhost\\SQLEXPRESS',
  '-d', 'HcPortalDB_Dev',
  '-E', '-C', '-I',
  '-b',   // NEW (Phase 315) — fail on T-SQL error; Phase 313.1 jalan via SSMS/sqlcmd manual, kita butuh exit code propagation
];
```

**Hostname guard pattern** (NEW — `CLAUDE.md` § Develop Workflow compliance):
Project policy: "Jangan edit kode/DB langsung di server Dev/Prod." Wrapper WAJIB reject `-S` arg non-`localhost`. Template di RESEARCH § Pattern 2 lines 430-433:
```ts
const sIdx = args.indexOf('-S');
if (sIdx >= 0 && !/^localhost/i.test(args[sIdx + 1])) {
  return reject(new Error(`Refusing to target non-localhost SQL Server: ${args[sIdx + 1]}`));
}
```

**Connection string source of truth:** `appsettings.Development.json` (verified RESEARCH line 13: `Server=localhost\SQLEXPRESS;Database=HcPortalDB_Dev;Integrated Security=True`). Hardcode di `SQLCMD_BASE_ARGS`, jangan baca dari env.

**Exports contract** (4 functions per RESEARCH § Pattern 2):
- `backup(snapshotPath: string): Promise<void>` — `BACKUP DATABASE ... WITH INIT, FORMAT`
- `restore(snapshotPath: string): Promise<void>` — SINGLE_USER → RESTORE WITH REPLACE → MULTI_USER (catatan: drop `-d HcPortalDB_Dev` saat restore karena USE master)
- `execScript(sqlPath: string): Promise<void>` — `sqlcmd -i <file>`
- `queryScalar(sql: string): Promise<number>` — `sqlcmd -Q "SET NOCOUNT ON; <sql>" -h -1 -W` + regex parse `^-?\d+`

**Error handling pattern:** `spawn` close handler reject(stderr || stdout) saat exit code ≠ 0. Path resolution lewat `path.join` (jangan literal Windows backslash di TS literal — pakai forward slash, SQL Server accept both).
