// Phase 322: History tab Riwayat Training client-side filter (parity sama Riwayat Assessment)
function filterTrainingRows() {
    var workerFilter = document.getElementById('trainingWorkerFilter')?.value.toLowerCase() || '';
    var rows = document.querySelectorAll('#trainingHistoryTable .training-history-row');
    rows.forEach(function(row) {
        var worker = row.getAttribute('data-worker') || '';

    });
}

**Rationale (CONTEXT D-21 Strategy D Hybrid):**
- Param `category`/`statusFilter`/`search` overlap nama tapi semantic-different per tab (D-01 + RESEARCH.md): Tab 1 `category` = distinct-DB string OJT/dll; Tab 2 `category` = Kategori Training hardcoded 8 enum; Tab 1 `statusFilter` = lifecycle Open/Upcoming/Active; Tab 2 `statusFilter` = completion Sudah/Belum.
- Inject blind dari ViewBag-shared values → semantic mismatch saat tab switch URL bookmark = Bug 2 reproduce.
- **Strategy D:** Wrapper Tab 1 full preserve (Tab 1 = default landing, dominant URL bookmark use case). Wrapper Tab 2 only `section`/`unit`/`page` (Tab 2-specific domain). Wrapper Tab 3 only `page` (Tab 3 client-side filter own di sub-tab).
- **Effect:** Bug 2 prevention by-design — Tab 2 partial TIDAK pernah dapet `category=OJT` saat user switch dari Tab 1, karena wrapper Tab 2 hx-vals TIDAK inject `category`.

**Trade-off documented:**
- URL bookmark Tab 1 (`?category=OJT&statusFilter=Active`) fully preserved ✅ (D-10)
- URL bookmark Tab 2 (`?tab=training&section=Operation&unit=KPB`) partial preserved (`section`+`unit` ✅; `category`/`statusFilter`/`search` Tab 2 not preserved — user filter ulang di form Tab 2 setelah landing)
- URL bookmark Tab 3 dropped (Tab 3 client-side filter own)
- → Tab 1 use case dominan; Tab 2 partial degradation acceptable per D-21 RISK section CONTEXT.md.
