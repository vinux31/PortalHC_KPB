## v2.5 Requirements — User Infrastructure & AD Readiness

### PROF — Profile & Settings

- [x] **PROF-01**: Profile page menampilkan data real user login (Nama, NIP, Email, Position, Section, Unit, Directorate, Role, JoinDate)
- [x] **PROF-02**: Field kosong menampilkan placeholder "Belum diisi", bukan blank/error
- [x] **PROF-03**: Avatar initials dinamis dari FullName user (bukan hardcoded "BS")
- [x] **PROF-04**: Settings page: Change Password functional via ChangePasswordAsync
- [x] **PROF-05**: Settings page: User bisa edit FullName dan Position; NIP/Email/Role/Section read-only
- [x] **PROF-06**: Item non-functional (2FA, Notifications, Language) dihapus atau di-mark "Belum Tersedia" disabled

### AUTH — Authentication

- [x] **AUTH-01**: Config toggle `Authentication:UseActiveDirectory` di appsettings.json (dev=false, prod=true)
- [x] **AUTH-02**: `IAuthService` interface + `LdapAuthService` menggunakan DirectoryEntry ke `LDAP://OU=KPB,OU=KPI,DC=pertamina,DC=com` dengan samaccountname filter
- [x] **AUTH-03**: `LocalAuthService` implementation wrapping existing PasswordSignInAsync
- [x] **AUTH-04**: Program.cs register IAuthService berdasarkan config toggle via DI
- [x] **AUTH-05**: Login page: Email + Password (identik kedua mode); AD mode tampilkan hint kecil "Login menggunakan akun Pertamina" di bawah form
- [x] **AUTH-06**: User belum terdaftar di DB → ditolak: "Akun Anda belum terdaftar. Hubungi HC." (no auto-provisioning, HC pre-registers via ManageWorkers)
- [x] **AUTH-07**: AD user login: sync FullName (displayName) dan Email (mail) saja; skip null values; Role/SelectedView TIDAK pernah diubah
- [x] **AUTH-08**: NuGet package System.DirectoryServices ditambahkan ke csproj

### USR — Admin User Management

- [x] **USR-01**: ManageWorkers CRUD (list, create, edit, delete, import, export, detail) accessible dari /Admin/ManageWorkers
- [x] **USR-02**: Old /CMP/ManageWorkers dihapus — clean break, no redirect
- [x] **USR-03**: Standalone "Kelola Pekerja" button di navbar dihapus — akses via Kelola Data hub
- [x] **USR-04**: Kelola Data hub di-reorganize: ManageWorkers card prominent, stale "Segera" items cleaned up

### USTR — User Structure

- [x] **USTR-01**: ~~ApplicationUser punya field AuthSource ("Local"/"AD") + EF migration~~ — Created in Phase 69, removed in Phase 72 (global config routing replaces per-user AuthSource)
- [x] **USTR-02**: Role-to-SelectedView mapping di-extract ke shared helper UserRoles.GetDefaultView()

### AUTH-HYBRID — Hybrid Authentication (added Phase 72)

- [x] **AUTH-HYBRID**: HybridAuthService enables AD-first + local fallback so dedicated Admin KPB user works in production AD mode
