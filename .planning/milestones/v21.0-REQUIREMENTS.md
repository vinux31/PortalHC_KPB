## v1 Requirements

### Org Label CRUD (ORG-LABEL)

- [x] **ORG-LABEL-01**: Tabel `OrganizationLevelLabels` dibuat via EF migration dengan kolom Level (PK), Label, UpdatedAt, UpdatedBy. Seed default 3 baris (0=Bagian, 1=Unit, 2=Sub-unit) masuk `Data/SeedData.cs` klasifikasi permanent+prod-required.
- [x] **ORG-LABEL-02**: Service `IOrgLabelService` + `OrgLabelService` tersedia dengan in-memory cache, key `OrgLabels:All`, manual invalidate, fallback string `"Level {N}"` bila level tidak ada di tabel.
- [x] **ORG-LABEL-03**: Endpoint `GET /Admin/GetLevelLabels` mengembalikan JSON `{ "0": "Bagian", "1": "Unit", ... }` untuk konsumsi JS, auth: authenticated user.
- [x] **ORG-LABEL-04**: Page `/Admin/ManageOrgLevelLabels` dapat diakses role Admin + HC, menampilkan tabel level (auto-detect max depth + buffer 1), edit modal per row, delete hanya untuk level tertinggi yang tidak dipakai. _(Phase 341)_
- [x] **ORG-LABEL-05**: Setiap UPDATE/INSERT/DELETE label tercatat ke `AuditLog` existing dengan UserId + before/after value. _(Phase 341)_
- [x] **ORG-LABEL-06**: Validation enforced server-side: label required + non-whitespace, unique across all rows, max 50 char. _(Phase 341)_
- [x] **ORG-LABEL-07**: Auto-detect max level via `MAX(OrganizationUnits.Level)` + buffer 1 untuk allow add child slot baru.

### ManageOrganization Page Fixes (ORG-TREE)

- [x] **ORG-TREE-01**: Dropdown induk di modal Tambah/Edit Unit terurut pre-order DFS — parent → semua keturunannya → sibling parent berikutnya (fix Bug B).
- [x] **ORG-TREE-02**: Validasi duplicate name unique **per-parent**, bukan global. "Operations" boleh ada di 2 Bagian beda.
- [x] **ORG-TREE-03**: Parent nonaktif visible di dropdown induk dengan suffix " (nonaktif)" + styled grey, tidak disembunyikan.
- [x] **ORG-TREE-04**: Inline JS string interpolation di `openDeleteModal` diganti pakai `data-name` attribute + event delegation (fix Bug #3 escape kotor).
- [x] **ORG-TREE-05**: Icon color palette extended ke level 3-5 (cycling), tidak cap hardcoded ke level 2 (fix Bug #4).
- [x] **ORG-TREE-06**: Modal Tambah/Edit Unit menampilkan path breadcrumb saat user pilih induk ("RFCC → LPG Treating → (unit baru)") — render real-time on `select` change.
- [x] **ORG-TREE-07**: Endpoint `POST /Admin/PreviewEditCascade` mengembalikan count affected Users/Mappings/Kompetensi/Guidance sebelum apply. Modal konfirmasi tampil sebelum aktual EditOrganizationUnit submit bila ada impact.
- [x] **ORG-TREE-08**: Card header tree tampilkan legend warna ↔ tier label (`▣ Bagian ▣ Unit ▣ Sub-unit`), bersumber dari `OrgLabelService`.
- [x] **ORG-TREE-09**: Title modal Tambah/Edit Unit dynamic per induk level — "Tambah Bagian" untuk root, "Tambah Unit" untuk anak L0, dst. Fallback `"Level N"`.
- [x] **ORG-TREE-10**: Setiap tree row menampilkan badge tier label, bersumber dari `OrgLabelService.GetLabel(node.level)`.

### Integrasi App-wide (ORG-INTEG)

- [x] **ORG-INTEG-01**: View Razor di 7 area page (CMP, CDP, Worker, CoachMapping, ProtonData, Renewal, DocumentAdmin) mengganti hardcoded string "Bagian"/"Unit" pakai `@inject IOrgLabelService OrgLabels` + `@OrgLabels.GetLabel(N)`.
- [x] **ORG-INTEG-02**: Controller string yang masuk ke response/TempData/ViewBag (display label) dynamic via service. Audit log message body dan literal di test tetap pakai string statis.
- [x] **ORG-INTEG-03**: Tidak ada regresi pada fungsi existing — tree CRUD, drag-reorder, toggle active, dan semua page yang di-integrasi tetap berfungsi normal.

### Testing (TEST)

- [x] **TEST-01**: Unit test xUnit `OrgLabelService.GetLabel` cover happy path + fallback `"Level N"` untuk level di luar config.
- [x] **TEST-02**: Unit test xUnit `OrgLabelController` cover permission denial non-Admin/non-HC + validation reject (empty, duplicate, >50 char).
- [x] **TEST-03**: Unit test xUnit pre-order DFS sort correctness untuk tree dengan 3 level + multi-root.
- [x] **TEST-04**: Unit test xUnit duplicate-name per-parent (existing test diperluas — same name boleh di parent berbeda, ditolak di parent sama).
- [x] **TEST-05**: Integration test migration apply + seed default + service first read.
- [x] **TEST-06**: Playwright E2E 5 scenario: tree load + legend visible, dropdown pre-order + inactive parent shown, cascade warning modal count akurat, label CRUD live rename → tree updated, label baru kelihatan di 2+ page integrasi (CMP + Worker form).
