## Phase Details

### Phase 340: Foundation — Tabel + Service + Cache

**Goal:** Layer dasar dipakai phase berikutnya untuk akses label dynamic.

**Requirements:** ORG-LABEL-01, 02, 03, 07

**Success criteria:**
1. EF migration `AddOrganizationLevelLabel` apply tanpa error + seed default 3 baris hadir di tabel.
2. `OrgLabelService.GetLabel(0..2)` mengembalikan "Bagian", "Unit", "Sub-unit". `GetLabel(99)` mengembalikan fallback `"Level 99"`.
3. Endpoint `GET /Admin/GetLevelLabels` mengembalikan JSON dict dari 3 level seed.
4. Cache invalidation triggered saat label di-update (mock test).
5. `GetMaxUsedLevel()` mengembalikan max level dari OrganizationUnits saat ini (sesuai DB lokal).

### Phase 341: Label CRUD Page

**Goal:** HC/Admin dapat rename label tier via browser tanpa edit kode atau restart aplikasi.

**Requirements:** ORG-LABEL-04, 05, 06

**Success criteria:**
1. Page `/Admin/ManageOrgLevelLabels` dapat diakses role Admin + HC.
2. Edit modal label "Bagian" → "Direktorat" → submit → toast success + tabel reload menampilkan label baru.
3. AuditLog entry tercatat dengan UserId + before/after label.
4. Validation reject empty input + duplicate label across levels + label >50 char.
5. Delete tombol hanya muncul untuk level tertinggi yang tidak dipakai unit; disabled untuk level yang masih dipakai (tooltip "Tidak bisa dihapus, masih dipakai").

### Phase 342: ManageOrganization Page Fixes

**Goal:** Page Admin/ManageOrganization clean dari 4 bug + 4 inovasi UX.

**Requirements:** ORG-TREE-01..10

**Success criteria:**
1. Modal Tambah Unit dropdown induk menampilkan urutan pre-order (parent → keturunannya → sibling), bukan flat per level.
2. Validasi name `Operations` bisa dibuat di 2 Bagian beda; ditolak bila ada di parent yang sama.
3. Parent nonaktif tetap muncul di dropdown dengan suffix " (nonaktif)" + grey style; user bisa pilih.
4. Modal title dynamic ("Tambah Bagian" / "Tambah Unit" / "Tambah Sub-unit") + tree row badge per level + legend di card header.
5. Edit nama unit yang punya >0 user terkait → modal warning muncul dengan count akurat sebelum aktual submit.

### Phase 343: Integrasi App-wide

**Goal:** Label tier dynamic terapply di semua page Portal HC, tidak hanya di page ManageOrganization.

**Requirements:** ORG-INTEG-01, 02

**Success criteria:**
1. Audit Grep `"Bagian"|"Unit"` per file di 7 area target (CMP/CDP/Worker/CoachMapping/ProtonData/Renewal/DocumentAdmin) selesai dengan keputusan eksplisit per occurrence (ganti vs skip).
2. Setelah label "Bagian" diubah jadi "Direktorat" via page CRUD, label baru muncul di minimal 3 page integrasi (CMP filter + Worker form + CDP assignment).
3. View Razor pakai `@inject IOrgLabelService` consistent (tidak ada hardcode tersisa di 7 area target untuk display string).
4. Audit log entry message body tetap menggunakan literal "Bagian"/"Unit" untuk konsistensi debug.
5. Test strings di xUnit + Playwright pakai label final saat test ditulis (deterministik, tidak random).

### Phase 344: Test + UAT

**Goal:** Verifikasi quality + tidak ada regresi.

**Requirements:** TEST-01..06, ORG-INTEG-03

**Success criteria:**
1. Unit test xUnit `OrgLabelService` + `OrgLabelController` + pre-order DFS + per-parent dup-name PASS.
2. Integration test migration + seed PASS di DB lokal fresh.
3. Playwright E2E 5 scenario PASS di `http://localhost:5277` (lihat REQ TEST-06 untuk skenario list).
4. Manual UAT 5 scenario (spec §7) PASS di lokal — HC rename label → cek di 7 area, Admin add Bagian baru → title dynamic, "Operations" di 2 Bagian beda OK, nonaktifkan parent → edit anak pindah parent OK, Edit Bagian besar → warning muncul dengan count benar.
5. Regression smoke: tree drag-reorder + toggle active + delete unit + add unit existing flow tetap berfungsi sama seperti pre-v21.0.
