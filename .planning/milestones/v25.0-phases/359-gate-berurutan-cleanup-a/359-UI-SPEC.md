## Spacing Scale

Inherit Bootstrap 5.3 spacer scale already used app-wide (1rem base = 16px). Use existing utility classes (`mb-2`, `mb-3`, `mb-4`, `me-1`, `me-2`, `gap-3`, `mt-3`) — do not introduce raw px values.

| Token | Value | Bootstrap class | Usage |
|-------|-------|-----------------|-------|
| xs | 4px | `*-1` | Icon-to-text gap (`me-1`) |
| sm | 8px | `*-2` | Compact element spacing (`mb-2`) |
| md | 16px | `*-3` | Default element/card-body spacing (`mb-3`, `gap-3`) |
| lg | 24px | `*-4` | Section / card block spacing (`mb-4`) |

Exceptions: none. Phase adds no novel spacing — it reuses the spacing already in the edited partials.

## Color

Inherit existing Bootstrap 5.3 semantic palette already in these partials. The 60/30/10 split is the
established app theme — Phase 359 introduces no new colors, only reuses semantic tokens for status/feedback.

| Role | Value | Bootstrap token | Usage |
|------|-------|-----------------|-------|
| Dominant (60%) | `#ffffff` | `bg-white` / card surface | Page background, `.card` surfaces |
| Secondary (30%) | `#f8f9fa` / `#6c757d` | `bg-light` / `text-secondary` | Card headers, muted text, "In Progress"/"No track" badges |
| Accent (10%) | `#198754` | `bg-success` / `text-success` | "Lulus/Selesai" pass badge + 100%-complete success alert + S1 skip-summary success alert |
| Destructive / block | `#dc3545` | `bg-danger` / `alert-danger` | S2 hard-block rejection messages (cross-year gate, graduation gate) |
| Warning (skip reasons) | `#ffc107` | `alert-warning` / `text-dark` | S1 skip portion of summary when Y > 0 (workers skipped) |

Accent (`bg-success`/green) reserved for: the "Lulus" / "Selesai" pass badge (replacing the level number) and
the all-deliverables-approved success state. **Never** apply green to generic interactive controls.

Removed color usage: `bi-award-fill text-warning` gold level icon (S3) and the chart line color (S4) are deleted.

## Copywriting Contract

All copy in **Bahasa Indonesia**. Match existing tone (formal-operasional, consistent with `TempData["Error"]`
messages already in `AssessmentAdminController.cs`).

| Element | Copy |
|---------|------|
| Primary CTA | (no new CTA — phase reuses existing "Buat Assessment" / "Mark Graduated" buttons) |
| S1 — Skip summary (all eligible) | `"{X} session berhasil dibuat."` (via `TempData["Success"]`) |
| S1 — Skip summary (partial) | `"{X} session dibuat, {Y} di-skip. Alasan: {N1} belum 100% deliverable, {N2} Tahun {T}-1 belum lulus."` (via `TempData["Warning"]` so both outcomes are visible) |
| S2 — Cross-year block (CreateAssessment) | `"Tidak bisa membuat exam Tahun {N}: Tahun {N-1} ({TrackType}) belum lulus untuk pekerja ini."` |
| S2 — Cross-year block (assign CoachMapping) | `"Tidak bisa assign Tahun {N}: Tahun {N-1} ({TrackType}) belum lulus untuk coachee ini."` |
| S2 — Graduation block | `"Tidak bisa menandai lulus (graduated): Tahun 3 belum lulus untuk pekerja ini."` |
| S3 — Pass badge (replaces "Level N") | `"Lulus"` — `<span class="badge bg-success"><i class="bi bi-award-fill me-1"></i>Lulus</span>` (no number) |
| S3 — Coachee dashboard stat (replaces numeric card) | Label `"Status Proton"`, value badge `"Lulus"` / `"Belum Lulus"` (no number, no gold "Level N") |
| S3 — Pending / not-yet state | `"Belum Lulus"` — `badge bg-secondary bg-opacity-25 text-secondary` (reuse existing "In Progress" style) |
| S3 — No-track state | `"No track"` — `badge bg-light text-muted` (UNCHANGED — keep existing) |
| Empty state (trend chart removed) | None — chart card is fully removed, leaving no empty placeholder. Do NOT add an "no data" panel. |
| Error state (server gate) | See S2 rows — rendered as `alert-danger` via `TempData["Error"]` in `_Layout.cshtml`. |
| Destructive confirmation | None — Phase 359 has no destructive UI action. Gate blocks are server-side rejections, not user-initiated deletes. |

**Wording rules:**
- Never display a numeric competency level anywhere (PCOMP-10 / A-M12). "Lulus"/"Selesai" only.
- Block messages must state BOTH the reason and the affected scope (Tahun N / pekerja), so admin knows what to fix.
- Skip-summary must surface even on partial success (use `Warning`, not silent `Success`) so admin sees the Y count.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Batch create with mixed eligibility | NOT all-or-nothing. Eligible workers get sessions; ineligible are skipped; summary shown (D-01). Page reloads to list with `TempData` banner. |
| Cross-year gate rejection | Hard-block, no override/warning-continue (D-05). Server rejects; user sees `alert-danger`; nothing is created. |
| Graduation gate rejection | Hard-block (D-10). "Mark graduated" returns rejection banner; `IsCompleted` not set. |
| Level number / trend chart | Removed entirely (S3/S4). No toggle, no hidden-but-present element — fields pruned from ViewModel. |
| Feedback rendering | Reuse `_Layout.cshtml:190-226` global `TempData` alert region (Success/Warning/Error/Info → `alert-*` dismissible). Do NOT build a new banner component. |
