# Phase 374 — UI Design Contract

> Visual and interaction contract for the Shuffle Toggle additions to the admin **ManagePackages** page. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope guard:** This phase adds ONLY, on `Views/Admin/ManagePackages.cshtml`, (1) one dedicated card "Pengacakan Soal & Jawaban" hosting two `form-switch` toggles (Acak Soal + Acak Pilihan Jawaban) + an explicit "Simpan Pengaturan" button, (2) a lock state (disabled switches + alert banner) when peserta have started, (3) a non-blocking size-mismatch warning alert (§9, live JS recompute), (4) a Pre/Post reminder alert (opsi Z) shown only on the Post page. Engine/read-logic = Phase 372/373. xUnit + Playwright = Phase 375. No new component library, no restyle. Reuse existing Bootstrap patterns and 372-UI-SPEC copy verbatim.

## Color

Project memakai palet Bootstrap default admin theme. Phase ini memakai warna semantik Bootstrap yang **sudah ada** untuk state alert; tidak memperkenalkan hex/warna baru.

| Role | Value | Usage di phase ini |
|------|-------|--------------------|
| Dominant (60%) | Bootstrap `--bs-body-bg` (putih) + `bg-light` card-header | Surface card Pengacakan + body form |
| Secondary (30%) | Bootstrap `text-muted` (#6c757d) | Help-text edukatif `text-muted` |
| Accent (10%) | Bootstrap `text-primary` (#0d6efd) | HANYA ikon `bi-shuffle` di card-header (meniru `bi-shield-lock text-primary` Token) + tombol "Simpan Pengaturan" `btn-primary`. Switch "checked" state pakai warna primary bawaan komponen `form-switch` Bootstrap — default komponen, bukan accent baru. |
| Semantic — Info (lock) | Bootstrap `alert-info` (biru muda) + `bi-lock-fill` | Lock banner — **identik** pola `SamePackage` lock `L29-44` (`alert-info` + `bi-lock-fill`) agar HC kenali sebagai "terkunci, bukan error". |
| Semantic — Warning (§9) | Bootstrap `alert-warning` (kuning) + `bi-exclamation-triangle` | Warning ukuran-paket-beda. Selaras pola warning existing di panel ringkasan (`text-warning` + `bi-exclamation-triangle` `L106-110`). Non-blocking → kuning, BUKAN merah/danger. |
| Semantic — Reminder Pre/Post (opsi Z) | Bootstrap `alert-warning` (kuning) + `bi-question-circle` / `bi-exclamation-triangle` | Reminder "Pre OFF, Post ON — sengaja?". Nada "konfirmasi niat", bukan error → kuning. |
| Destructive | not used | Tidak ada aksi destruktif di phase ini. JANGAN pakai `alert-danger`/`btn-danger` di card ini (TempData error PRG tetap pakai pola `L24-27` existing di luar card). |

**Accent reserved for:** (1) ikon `bi-shuffle.text-primary` di card-header, (2) tombol `btn-primary` "Simpan Pengaturan". Tidak ada elemen lain yang boleh memakai accent.

**Color rationale (untuk checker):**
- Lock = `alert-info` (bukan warning/danger): terkunci adalah state normal & informatif, bukan kesalahan. Konsisten dengan `SamePackage` lock yang juga `alert-info`.
- Warning §9 = `alert-warning` (bukan danger): non-blocking, HC mungkin sengaja (exam baris A/B). Kuning = "perhatikan", bukan "salah".
- Reminder Pre/Post = `alert-warning`: pertanyaan niat ("sengaja?"), bukan error.
- Disabled switch (lock UI) menggunakan tampilan disabled bawaan Bootstrap (opacity berkurang) — bukan warna baru.

## Interaction Contract

| Aspek | Kontrak |
|-------|---------|
| Save model (D-01) | **Explicit save**, BUKAN auto-save-on-flip. HC ubah satu/dua toggle → klik "Simpan Pengaturan" sekali → POST. Form POST + Redirect (PRG) ke ManagePackages + TempData (D-01a). Konsisten pola form existing halaman ini (`CreatePackage`/`DeletePackage`/`CopyPackagesFromPre`). BUKAN AJAX/Json. |
| Default state | Switch merefleksikan SAVED `ShuffleQuestions`/`ShuffleOptions` dari `ViewBag` (default ON untuk data lama/baru). Render `checked` dari nilai model, bukan default browser. |
| Independensi | Toggle 1 & Toggle 2 sepenuhnya independen — mengubah satu tidak mengubah/menonaktifkan yang lain. Tidak ada conditional show/hide antar keduanya. |
| Warning live recompute (D-03a) | JS membaca state checkbox Acak Soal **saat ini** (unsaved). Saat di-flip OFF (dan ≥2 paket-ber-soal + `hasMismatch`) → warning muncul; saat di-flip ON → warning hilang. Juga dievaluasi saat page load. Akurat sebelum klik Simpan. |
| Reminder saved-state (D-03b) | Reminder Pre/Post dibaca dari SAVED state Pre (lintas-halaman via `LinkedSessionId`), bukan live JS. Hanya tampil di Post page. Tidak berubah saat HC flip toggle di Post sebelum simpan (saved-state-driven by design). |
| Lock — disabled UI (D-04) | Saat lock-condition true (ada sibling `StartedAt != null` ATAU ada `UserPackageAssignment` grup): kedua `input.form-check-input` ber-`disabled`, tombol "Simpan Pengaturan" `disabled`, + lock alert banner `alert-info` ditampilkan di card menjelaskan alasan. |
| Lock — server guard (D-04a) | Disabled UI BUKAN satu-satunya guard (defense-in-depth). Endpoint `UpdateShuffleSettings` WAJIB tolak perubahan server-side saat lock-condition true (SHUF-11) → TempData error + PRG redirect tanpa menulis. |
| Hide ≠ Lock | Proton Tahun 3 / Manual entry → seluruh card **disembunyikan** (`HideShuffleToggle`), tidak dirender sama sekali. Started → card **tampil tapi disabled** (informasikan kenapa). Dua perilaku berbeda. |
| SamePackage independen | Toggle shuffle tetap **aktif/editable** walau `ViewBag.IsSamePackageLocked == true` (SamePackage lock hanya isi paket, bukan shuffle). Lock shuffle HANYA dari started/assignment condition, bukan dari SamePackage. |
| Keyboard / a11y | `input.form-check-input[type=checkbox]` + `label[for]` (pola `:506-507`) memberi label terasosiasi & focusable native. Tiap toggle WAJIB punya `id` unik + `label[for]` cocok. Tombol submit native focusable. Tidak perlu ARIA tambahan. Alert pakai `role="alert"` (pola `L31`). |
