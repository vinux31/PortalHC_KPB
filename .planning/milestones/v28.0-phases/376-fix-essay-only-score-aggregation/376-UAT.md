## Findings (non-defect — environment/runbook)

- **F1 [env, MEDIUM] App crash mid-e2e akibat SQL Named-Pipes NTLM-loopback flap (error 53).**
  Gejala: saat app lokal dijalankan dgn conn string `Server=lpc:localhost\SQLEXPRESS` ATAU named-pipe default, burst koneksi Integrated-Security saat grading/finalize memicu error 53 ("Named Pipes Provider: Could not open a connection") → app exit + sqlcmd ikut gagal. Ini meng-crash FLOW M M1 (false-fail chrome-error).
  Bukan bug produk Phase 376 (FLOW L 6/6 + FLOW M 6/6 hijau setelah fix env).
  **Fix/runbook:** jalankan app lokal dgn TRUE shared-memory pakai **nama mesin**, bukan `localhost`:
  `ConnectionStrings__DefaultConnection="Server=lpc:Lenovo\SQLEXPRESS;..."`.
  Catatan: `lpc:localhost\SQLEXPRESS` TIDAK resolve ke shared-memory (error 87 "remote") → fallback named-pipe → tetap flap. Wajib `lpc:<HOSTNAME>\SQLEXPRESS`. SQLEXPRESS service sempat butuh restart untuk clear stuck state. Pertimbangkan dokumentasikan di docs/DEV_WORKFLOW.md (memperluas memory reference_local_e2e_sql_env_fix).
