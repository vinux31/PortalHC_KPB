<!-- Key approval chain contracts extracted from codebase. -->

ApproveDeliverable (CDPController line 803):
- Role guard: isAtasanAccess = SrSupervisor OR SectionHead — returns Forbid() otherwise
- Status guard: only Status=="Submitted" can be approved (TempData["Error"] + redirect otherwise)
- Section guard: coachee must be in same section as approver (Admin exempt)
- Sets: Status="Approved", ApprovedAt=DateTime.UtcNow, ApprovedById=user.Id
- After save: calls CreateHCNotificationAsync if all deliverables in track approved

RejectDeliverable (CDPController line 885):
- Same role guard as ApproveDeliverable (SrSpv or SH)
- Validates rejectionReason not empty
- Sets: Status="Rejected", RejectionReason=rejectionReason, RejectedAt=DateTime.UtcNow

HCReviewDeliverable (CDPController line 974):
- Role guard: isHCAccess = userRole==UserRoles.HC — returns Forbid() otherwise
- Status guard: HCApprovalStatus must be "Pending"
- Sets: HCApprovalStatus="Reviewed", HCReviewedAt=DateTime.UtcNow, HCReviewedById=user.Id
- Redirects to CoachingProton (not Deliverable)

ApproveFromProgress / RejectFromProgress / HCReviewFromProgress (AJAX variants):
- Return Json({success, message}) — called from CoachingProton.cshtml inline buttons
- Same role guards as page-level variants

Deliverable view model fields:
- CanApprove: isAtasanAccess && Status=="Submitted"
- CanHCReview: isHC && HCApprovalStatus=="Pending"
- Note: no SrSpv-specific vs SH-specific distinction — both approve/reject the same Status field
