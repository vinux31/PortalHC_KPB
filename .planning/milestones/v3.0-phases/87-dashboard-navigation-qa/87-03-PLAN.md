# Plan 87-03: QA Login Flow, Navigation, Authorization, and AuditLog

**Goal:** Verify login flow works correctly, role-based navigation enforces visibility rules, section selectors function properly, and AuditLog page displays correctly.

## Context

Phase 87 requires verification of:
- **Login Flow**: Local auth login with inactive user block and return URL redirect
- **Role-based Navigation**: Kelola Data menu visible only to Admin/HC
- **Section Selectors**: KkjSectionSelect (KKJ Matrix) and CMP/Mapping (CPDP files)
- **AccessDenied Page**: Renders when unauthorized access attempted
- **AuditLog Page**: Displays assessment management audit trail

The verification approach:
1. **Code review**: Verify authorization logic, navigation visibility, login flow
2. **Browser verification**: Test login, navigation, section selectors with all roles
3. **Bug fix**: Fix any bugs found inline (<100 lines)

## Verification Criteria

1. Login flow works correctly for local auth:
   - Active users can login successfully
   - Inactive users are blocked with clear error message
   - Return URL redirect works securely
2. Role-based navigation enforces visibility:
   - Kelola Data visible only to Admin/HC
   - CMP and CDP visible to all authenticated users
3. Section selectors function correctly:
   - Admin/HC see all sections
   - L5-L6 users see only their section
   - Tab filtering works correctly
4. AccessDenied page renders for unauthorized access
5. AuditLog page displays correctly for Admin/HC
6. All authorization boundaries enforced
7. No security vulnerabilities (open redirect, CSRF, XSS)
