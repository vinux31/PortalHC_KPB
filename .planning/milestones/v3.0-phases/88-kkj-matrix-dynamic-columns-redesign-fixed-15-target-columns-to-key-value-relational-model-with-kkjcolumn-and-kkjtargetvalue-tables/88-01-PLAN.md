# Plan 89-01: Data Model & EF Core Migration

## Goal

Remove all 15 `Target_*` columns from `KkjMatrixItem`, remove all 15 `Label_*` columns from `KkjBagian`, add three new EF Core models (`KkjColumn`, `KkjTargetValue`, `PositionColumnMapping`), register them in `ApplicationDbContext`, and generate + apply the EF Core migration. This is the foundation all other plans depend on.

## must_haves

- All 15 `Target_*` properties removed from `KkjMatrixItem` model
- All 15 `Label_*` properties removed from `KkjBagian` model
- `KkjColumn` model exists with: `Id`, `BagianId` (FK to `KkjBagian`), `Name` (string), `DisplayOrder` (int), navigation properties
- `KkjTargetValue` model exists with: `Id`, `KkjMatrixItemId` (FK), `KkjColumnId` (FK), `Value` (string, default "-"), navigation properties
- `PositionColumnMapping` model exists with: `Id`, `Position` (string), `KkjColumnId` (FK to `KkjColumn`), navigation property
- Navigation collections added to `KkjMatrixItem` (`TargetValues`) and `KkjBagian` (`Columns`)
- `ApplicationDbContext` has DbSets: `KkjColumns`, `KkjTargetValues`, `PositionColumnMappings`
- EF Core fluent configuration in `OnModelCreating` for all three new tables (relationships, indexes, delete behaviors)
- Migration file generated via `dotnet ef migrations add AddKkjDynamicColumns`
- `dotnet ef database update` applied successfully
- `dotnet build` succeeds with 0 errors

## Context

Current state in `Models/KkjModels.cs`:
- `KkjMatrixItem` has 15 hardcoded `Target_*` string properties (Target_SectionHead through Target_HSE), all defaulting to "-"
- `KkjBagian` has 15 hardcoded `Label_*` string properties (Label_SectionHead through Label_HSE) with human-readable defaults
- No FK relationships, no navigation properties between these models

Target state per CONTEXT.md (locked decisions):
- `KkjColumn`: `Id`, `BagianId` (FK → KkjBagian), `Name` (string), `DisplayOrder` (int)
- `KkjTargetValue`: `Id`, `KkjMatrixItemId` (FK → KkjMatrixItem), `KkjColumnId` (FK → KkjColumn), `Value` (string)
- `PositionColumnMapping`: `Id`, `Position` (string), `KkjColumnId` (FK → KkjColumn)

4. After the `KkjBagian` configuration block, add the three new entity configurations:

```csharp
// KkjColumn: target column definitions per Bagian
builder.Entity<KkjColumn>(entity =>
{
    entity.ToTable("KkjColumns");

    entity.HasOne(c => c.Bagian)
        .WithMany(b => b.Columns)
        .HasForeignKey(c => c.BagianId)
        .OnDelete(DeleteBehavior.Cascade); // Deleting a Bagian cascades to its column definitions

    entity.HasIndex(c => new { c.BagianId, c.DisplayOrder });
    entity.HasIndex(c => new { c.BagianId, c.Name }).IsUnique(); // One column name per bagian
});

// KkjTargetValue: key-value store for target values per item+column cell
builder.Entity<KkjTargetValue>(entity =>
{
    entity.ToTable("KkjTargetValues");

    entity.HasOne(v => v.KkjMatrixItem)
        .WithMany(m => m.TargetValues)
        .HasForeignKey(v => v.KkjMatrixItemId)
        .OnDelete(DeleteBehavior.Cascade); // Deleting an item removes its target values

    entity.HasOne(v => v.KkjColumn)
        .WithMany(c => c.TargetValues)
        .HasForeignKey(v => v.KkjColumnId)
        .OnDelete(DeleteBehavior.Restrict); // Prevent deleting column if it has values

    entity.HasIndex(v => new { v.KkjMatrixItemId, v.KkjColumnId }).IsUnique(); // One value per cell
});

// PositionColumnMapping: maps user positions to KkjColumns
builder.Entity<PositionColumnMapping>(entity =>
{
    entity.ToTable("PositionColumnMappings");

    entity.HasOne(m => m.KkjColumn)
        .WithMany(c => c.PositionMappings)
        .HasForeignKey(m => m.KkjColumnId)
        .OnDelete(DeleteBehavior.Restrict); // Prevent deleting column if positions are mapped to it

    entity.HasIndex(m => m.Position);
    entity.HasIndex(m => new { m.Position, m.KkjColumnId }).IsUnique(); // No duplicate position-column pairs
});
```
