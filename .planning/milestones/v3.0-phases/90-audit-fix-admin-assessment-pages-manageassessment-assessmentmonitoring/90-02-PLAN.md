## Objective

View-level audit of ManageAssessment (3 tabs), AssessmentMonitoring, AssessmentMonitoringDetail, CreateAssessment, and EditAssessment. Fix rendering issues, missing navigation links, status badge gaps, and header button conditional logic bug.

## Context

A bug exists in ManageAssessment: The header buttons div is wrapped in a server-side `@if (activeTab == "assessment")` Razor condition. When the page loads with `?tab=training` or `?tab=history`, `header-assessment-btns` element is NOT rendered in the DOM, so the JS `shown.bs.tab` handler cannot find it to show/hide it on tab switches.

The AssessmentMonitoring group list may lack clickable links to AssessmentMonitoringDetail (the title column is a plain text cell â€” HC has no direct way to drill into a group from the monitoring list).

Read `Views/Admin/ManageAssessment.cshtml` lines 52-95 (page header and tab nav).

**Bug fix â€” header buttons conditional rendering:**

The current code conditionally renders the button div based on `activeTab`:
```razor
@if (activeTab == "assessment")
{
    <div class="d-flex gap-2" id="header-assessment-btns">
        ...buttons...
    </div>
}
else
{
    <div class="d-flex gap-2" id="header-other-btns" style="display:none!important">
    </div>
}
```

This means when `activeTab != "assessment"`, `header-assessment-btns` is absent from the DOM. The JS `shown.bs.tab` handler tries `document.getElementById('header-assessment-btns')` â€” gets null â€” tab switch no longer hides/shows buttons correctly.

**Fix:** Remove the conditional rendering. Always render `header-assessment-btns`, but set initial visibility via inline style:

```razor
<div class="d-flex gap-2" id="header-assessment-btns"
     style="@(activeTab != "assessment" ? "display:none" : "")">
    <a href="@Url.Action("CreateAssessment", "Admin")" class="btn btn-primary">
        <i class="bi bi-plus-circle me-1"></i> Buat Assessment
    </a>
    <a href="@Url.Action("AssessmentMonitoring", "Admin")" class="btn btn-outline-success">
        <i class="bi bi-binoculars me-1"></i> Monitoring
    </a>
    <a href="@Url.Action("AuditLog", "Admin")" class="btn btn-outline-secondary">
        <i class="bi bi-clock-history me-1"></i> Audit Log
    </a>
</div>
```

Remove the `else` branch with `header-other-btns` entirely.

**Also fix â€” Status badge in Assessment Groups table:**

Find the `statusBadge` switch expression in the Assessment Groups table (around line 193):
```csharp
var statusBadge = (string)group.Status switch
{
    "Open"      => "bg-success",
    "Completed" => "bg-secondary",
    "Upcoming"  => "bg-info",
    _           => "bg-secondary"
};
```

Add explicit handling for `Abandoned` and `InProgress`:
```csharp
var statusBadge = (string)group.Status switch
{
    "Open"       => "bg-success",
    "Completed"  => "bg-secondary",
    "Upcoming"   => "bg-info",
    "InProgress" => "bg-warning text-dark",
    "Abandoned"  => "bg-dark",
    _            => "bg-secondary"
};
```
