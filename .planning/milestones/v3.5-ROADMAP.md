## Phase Details

### Phase 105: User Guide Structure & Content

**Goal:** Users can access comprehensive user guide page with role-specific tabbed content covering all portal modules

**Depends on:** Nothing (first phase of milestone)

**Requirements:**
- GUIDE-NAV-01: User can access Guide page via "Panduan" link in navbar after login
- GUIDE-NAV-02: Guide page displays hero section with "Panduan Pengguna" title using gradient styling matching dashboard
- GUIDE-NAV-03: Guide page displays 5 tab navigation buttons (Dashboard, CMP, CDP, Account, Admin Panel)
- GUIDE-NAV-04: User can click tabs to switch between content sections without page refresh
- GUIDE-NAV-05: Admin Panel tab is hidden for non-Admin/HC users
- GUIDE-CONTENT-01: Each tab displays step-by-step instructions with numbered step cards
- GUIDE-CONTENT-02: Step cards include icon, title, and description
- GUIDE-CONTENT-03: Important information displayed in alert boxes (tips/catatan)
- GUIDE-CONTENT-04: Content organized using Bootstrap 5 accordion/collapse for sub-sections
- GUIDE-CONTENT-05: FAQ section displays at bottom of page with accordion behavior
- GUIDE-CONTENT-06: FAQ includes common questions: login, password reset, CMP vs CDP, assessments, evidence upload, approval flow, coaching progress
- GUIDE-ACCESS-01: Guide page requires authentication (non-logged users redirected to login)
- GUIDE-ACCESS-02: Role indicator badge displays user's current role at top of page
- GUIDE-ACCESS-03: Dashboard tab content available to all authenticated users
- GUIDE-ACCESS-04: CMP tab content available to all authenticated users
- GUIDE-ACCESS-05: CDP tab content available to all authenticated users
- GUIDE-ACCESS-06: Account tab content available to all authenticated users
- GUIDE-ACCESS-07: Admin Panel tab content visible only to Admin and HC users

**Success Criteria** (what must be TRUE):
1. Authenticated user can click "Panduan" link in navbar and navigate to Guide page without errors
2. Guide page displays hero section with gradient title and 5 tab buttons (or 4 tabs for non-Admin/HC users)
3. User can click tabs to switch between Dashboard, CMP, CDP, Account, and Admin Panel content sections without page reload
4. Each tab displays step-by-step instructions with numbered cards, icons, and collapsible sub-sections
5. FAQ section at bottom of page displays expandable accordion with 7 common questions
6. Non-Admin/HC users do not see Admin Panel tab or content
7. Unauthenticated users clicking navbar link are redirected to login page

### Phase 106: User Guide Styling & Polish

**Goal:** User guide page displays with premium visual styling matching existing design system and responsive mobile layout

**Depends on:** Phase 105 (requires structure and content to be complete)

**Requirements:**
- GUIDE-STYLE-01: Guide page uses CSS variables matching home.css (--gradient-primary, --shadow-*)
- GUIDE-STYLE-02: Page uses Inter font family
- GUIDE-STYLE-03: Cards use glassmorphism effect matching existing design system
- GUIDE-STYLE-04: Page content animates using AOS library
- GUIDE-STYLE-05: Step numbers display with gradient badges
- GUIDE-STYLE-06: Tab navigation styling matches existing design patterns
- GUIDE-STYLE-07: Page displays correctly on mobile devices (responsive breakpoints)
- GUIDE-STYLE-08: All content is in Indonesian language

**Success Criteria** (what must be TRUE):
1. Guide page displays with gradient colors matching home.css (primary gradient, shadow effects)
2. All text uses Inter font family (consistent with existing portal)
3. Step cards display with glassmorphism effect (semi-transparent background, blur, border)
4. Page elements animate on scroll using AOS library (fade-up, fade-in effects)
5. Step number badges display with gradient background matching portal design
6. Tab navigation buttons match existing navbar/tab styling (hover states, active state, transitions)
7. Page layout adapts correctly on mobile screens (tabs stack vertically, text remains readable, no horizontal scroll)
8. All text content is in Indonesian language (no English text visible except technical terms)

## Dependencies

```
Phase 105 (Structure & Content)
    ↓
Phase 106 (Styling & Polish)
```

**Rationale:**
- Phase 105 builds the functional foundation (controller, view, routing, content, role-based access)
- Phase 106 applies visual polish after structure is complete (allows for CSS refinement without touching business logic)
- Separation allows for iterative testing: verify functionality works in Phase 105, then apply styling in Phase 106

## Notes

- **Phase Split:** Structure (105) before Styling (106) follows established portal pattern — build functionality first, then apply visual polish
- **Role-Based Access:** Admin Panel tab gating (GUIDE-NAV-05, GUIDE-ACCESS-07) follows existing authorization pattern (User.IsInRole() checks in views)
- **Content Strategy:** Static Indonesian content in Phase 105 — no CMS or database storage needed (defer to v2+)
- **Styling Consistency:** Phase 106 leverages existing CSS variables from home.css — no new design system created
- **Mobile-First:** Phase 106 ensures responsive design works on mobile devices (Bootstrap 5 breakpoints)
