    Pendekatan DIREKOMENDASIKAN (paling minim risiko, tak butuh using/IUrlHelper di method static) = variabel lokal `string resolvedSrc` di dalam blok `@{ }`. `Url` (IUrlHelper) tersedia ambient di body Razor view (bukan di method static @functions), jadi resolusi sebagai variabel lokal aman:
    ```cshtml
    @{
        // ... existing imagePath/imageAlt/cap/ariaContext/ariaLabel ...
        // F-09 (PXF-01): jadikan path PathBase-aware. Path tersimpan selalu leading-slash "/uploads/..".
        // Url.Content("~/uploads/..") prepend PathBase ("/KPB-PortalHC" di Dev, "" di lokal).
        // Defensive (D-01a): prepend "/" bila path tak diawali "/"; bila sudah diawali "~" jangan double-resolve.
        string resolvedSrc = "";
        if (!string.IsNullOrWhiteSpace(imagePath))
        {
            var p = imagePath.Trim();
            if (p.StartsWith("~")) { resolvedSrc = Url.Content(p); }      // sudah app-relative
            else { if (!p.StartsWith("/")) p = "/" + p; resolvedSrc = Url.Content("~" + p); }  // "~" + "/uploads/.." = "~/uploads/.."
        }
    }
    ```
    Lalu di markup: ganti `src="@imagePath"` (L38) → `src="@resolvedSrc"` DAN `data-img-src="@imagePath"` (L45) → `data-img-src="@resolvedSrc"`. Pakai `resolvedSrc` yang SAMA untuk kedua atribut (src render + lightbox). JANGAN ubah `alt`, `data-img-alt`, class, style, onclick, aria-label.
