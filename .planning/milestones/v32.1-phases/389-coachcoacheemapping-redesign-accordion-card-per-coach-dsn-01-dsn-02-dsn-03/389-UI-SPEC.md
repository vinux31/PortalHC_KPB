## Spacing Scale

Expressed as **Bootstrap 5 spacing utilities** (Bootstrap base = `$spacer` 1rem/16px; steps 0‚Äì5 = 0, 4px, 8px, 16px, 24px, 48px ‚Äî all multiples of 4). No custom CSS spacing tokens are introduced.

| Token | Bootstrap util | Value | Usage in this phase |
|-------|---------------|-------|---------------------|
| xs | `gap-1` / `g-1` / `ms-1` / `me-1` | 4px | Icon‚Üítext gap inside buttons/badges, Aksi button-group gap (`d-flex gap-1`, preserved L302), chevron/icon inline gap |
| sm | `gap-2` / `g-2` / `p-2` | 8px | Toolbar inter-button gap (`d-flex gap-2`, preserved L48), card-header internal element gap, badge/avatar gap |
| md | `mb-3` / `p-3` / `g-3` | 16px | **Inter-card vertical rhythm** (`card ... mb-3` per coach ‚Äî D-01), card-header & card-body padding (Bootstrap default `1rem`), filter row `g-2`‚Üídata spacing |
| lg | `mb-4` / `py-4` | 24px | Section breaks (import-results card `mb-4` L132 preserved, filter form `mb-3` preserved) |
| xl | `py-5` | 48px | Empty-state vertical padding (`text-center py-5` L219 preserved) |

Exceptions:
- **Avatar circle = 36px** fixed (`style="width:36px;height:36px"`) ‚Äî reused verbatim from `ManageWorkers.cshtml:251-254` (D-03). 36px is a 4-multiple (9√ó4) and is kept identical to the established app idiom; do NOT re-derive.
- **Avatar font-size = 0.8rem** (12.8px) ‚Äî part of the reused idiom (D-03), kept verbatim.
- Card-header & card-body inherit Bootstrap default padding (`var(--bs-card-spacer-y/x)` = 1rem = 16px). No override.

Inherited from Bootstrap 5 + Inter (`_Layout.cshtml`). This phase introduces NO new font sizes or weights ‚Äî it reuses existing Bootstrap utility classes already present in the file. Sizes/weights below are the Bootstrap defaults the markup relies on.

| Role | Size | Weight (util) | Line Height | Where used |

## Component Inventory (Bootstrap 5) ‚Äî new structural markup for this phase

> Substitute for the shadcn "Registry" notion. These are the Bootstrap components the executor assembles. No external components, no new CSS files ‚Äî at most a minimal scoped `<style>`/`@section Styles` for the chevron rotation (D-05, Claude's discretion).

### 1. Coach card (D-01) ‚Äî replaces grouped `<table>` per coach
- Container: `<div class="card border-0 shadow-sm mb-3">` per `group` in `groupedCoaches` (idiom matches import-results card L132 + sibling Phase 388).
- **Card-header = clickable collapse toggle** (D-02, D-05, D-14):
  - `data-bs-toggle="collapse"` ¬∑ `data-bs-target="#collapse-@idx"` (id pair moved here from the old `<tr class="table-primary">` L250; `idx++` loop preserved ‚Äî D-14).
  - **NO `data-bs-parent`** ‚Üí cards are independent, multiple may be open simultaneously (D-05).
  - Composition left‚Üíright: avatar-initial circle (36px) ¬∑ coach name (`fw-semibold`) ¬∑ section (`text-muted` small, `‚Äî @group.CoachSection`, only if non-empty) ¬∑ spacer (`ms-auto` / `flex-grow-1`) ¬∑ **load badge** (right-aligned, D-04) ¬∑ **chevron icon** (`bi-chevron-down`, rotates on open ‚Äî D-05).
  - Layout: `d-flex align-items-center` with `gap-2`; `style="cursor:pointer"` preserved from L250.
- **Card-body = collapse container** (D-02): `<div class="collapse" id="collapse-@idx">` ‚Äî `collapse` WITHOUT `show` (default CLOSED ‚Äî D-05, deliberate change from current `collapse show` L270). Inside: `<div class="card-body p-0">` ‚Üí the mini coachee table.

### 2. Avatar-initial (D-03) ‚Äî reuse verbatim from `ManageWorkers.cshtml:251-254`
```
<div class="avatar-initial rounded-circle bg-primary text-white me-2 d-flex align-items-center justify-content-center fw-bold"
     style="width:36px;height:36px;font-size:0.8rem">
    @( ((string)group.CoachName).Length > 0 ? ((string)group.CoachName).Substring(0,1).ToUpper() : "?" )
</div>
```
Fallback `"?"` when name empty. `bg-primary` neutral (not load/section-tinted ‚Äî D-03). Decorative: `aria-hidden="true"` is acceptable since the coach name is adjacent visible text (a11y section).

### 3. Mini coachee table (D-06, D-07) ‚Äî inside `table-responsive`
- Wrapper: `<div class="table-responsive">` (horizontal scroll on narrow ‚Äî D-06, D-10 responsive).
- Table: `<table class="table align-middle mb-0">` with `thead` (header style at executor discretion; old `table-dark` thead may move inside or simplify ‚Äî keep readable).
- **9 columns (the old "Coachee Aktif" column is REMOVED ‚Äî D-07):** Nama ¬∑ NIP ¬∑ `@OrgLabels.GetLabel(0) Penugasan` ¬∑ `@OrgLabels.GetLabel(1) Penugasan` ¬∑ Jabatan ¬∑ Proton Track ¬∑ Status ¬∑ Mulai ¬∑ Aksi.
  - The removed "Coachee Aktif" header (old L241) + empty cell (old L300) are dropped; its value now lives in the card-header load badge (D-02/D-04/D-07).
- Each coachee row: `<tr class="@(coachee.IsActive ? "" : "table-light text-muted")" data-mapping-id="@coachee.Id">` ‚Äî **`data-mapping-id` MUST remain** (delete-row hook `submitDelete()` L973-974, D-08). Non-active rows `table-light text-muted` (D-08).
- Cell contents preserved verbatim: Nama, NIP, AssignmentSection/Unit (em-dash fallback), CoacheePosition, ProtonTrack badge, Status badge, StartDate (`dd MMM yyyy` id-ID culture).

### 4. Aksi column (D-09 ‚Äî PARITY-CRITICAL, verbatim block, order locked)
Preserve the entire Aksi block (old L301-341) inside `<div class="d-flex gap-1">`, button order and `if/else IsCompleted ‚Üí IsActive ‚Üí else` chain **unchanged** (Phase 356 D-06 lesson):
1. **Edit** ‚Äî `btn-sm btn-outline-secondary`, `onclick="openEditModal(@coachee.Id, '@coachee.CoachId', 0, '...startDate...', '...name...', '...section...', '...unit...')"` (7 args).
2. If `IsCompleted` ‚Üí **Graduated** badge `bg-info` + `bi-award` (checked FIRST).
3. Else if `IsActive` ‚Üí **Nonaktifkan** `btn-sm btn-outline-danger` (`confirmDeactivate`) + **Graduated** form (POST `MarkMappingCompleted` + `@Html.AntiForgeryToken()` + `confirm(...)`, `btn-sm btn-outline-info`).
4. Else ‚Üí **Aktifkan** `btn-sm btn-outline-success` (`reactivateMapping`) + **Hapus** `btn-sm btn-outline-danger` (`confirmDelete`).
All buttons `btn-sm`.

### 5. Toolbar (D-10, D-11)
- Container preserved: `<div class="d-flex gap-2">` ‚Üí add `flex-wrap` for narrow-screen graceful wrap (responsive).
- All buttons normalized to `btn-sm` (D-10).
- **Group the 3 Excel actions visually** ‚Äî Download Template / Import Excel / Export Excel ‚Äî via `btn-group` (or a tight segment) with consistent outline style (D-10, exact markup = Claude's discretion).
- **"Tambah Mapping" = solo `btn-primary` CTA on the right** (D-10).
- **Remove dead `onclick`** `onclick="document.getElementById('assignModal').querySelector('[data-bs-dismiss]') && null"` (D-11, L58) ‚Äî keep `data-bs-toggle="modal" data-bs-target="#assignModal"` (function unchanged). Import button keeps `data-bs-toggle="modal" data-bs-target="#importMappingModal"`.

### 6. Chevron state (D-05 ‚Äî Claude's discretion)
- Chevron reflects open/closed: e.g. `bi-chevron-down` rotated 180¬∞ (or `bi-chevron-up`) when expanded.
- Acceptable: minimal inline transition OR a scoped `<style>`/`@section Styles` block (no shared CSS file for one page ‚Äî D, code_context). Tie rotation to Bootstrap collapse state (`.collapsed`/`aria-expanded` or `.show`).

## Accessibility (Phase 354 lesson ‚Äî ASSERT via Playwright runtime, NOT assumption)

The card-header toggle is the only new interactive control; it MUST be keyboard- and screen-reader-accessible. Verify at runtime in Phase 390 (and during executor smoke), not by grep/build alone.

| Requirement | Contract |
|-------------|----------|
| Card-header role | `role="button"` on the clickable header div (OR render as a real `<button>`). |
| Expanded state | `aria-expanded="false"` default (matches CLOSED state ‚Äî D-05), toggled to `true` on open by Bootstrap collapse. |
| Controls relationship | `aria-controls="collapse-@idx"` pointing at the collapse body (id pair unique per `idx` ‚Äî D-14). |
| Keyboard | Focusable (`tabindex="0"` if not a native button) + activatable by Enter and Space. |
| Avatar | Decorative ‚Äî `aria-hidden="true"` acceptable (coach name is adjacent visible text ‚Äî D-03). |
| Load badge | Needs accessible context so the count is not an orphan number (e.g. visually-hidden label or `title`/`aria-label` such as "N coachee aktif"). Exact mechanism = executor discretion; presence verified at runtime. |
| Chevron icon | Decorative (`aria-hidden="true"`); state conveyed by `aria-expanded`, not the icon. |
