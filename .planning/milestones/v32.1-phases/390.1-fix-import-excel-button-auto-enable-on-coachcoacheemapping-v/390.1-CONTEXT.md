<decisions>
## Implementation Decisions

### Teknik fix
- **D-01:** Bungkus registrasi listener dalam `DOMContentLoaded` (BUKAN event delegation di `document`). Alasan: input `#importMappingFile` adalah markup statis server-rendered (bukan di-inject dinamis), jadi `DOMContentLoaded` dijamin elemen sudah ada → cukup, paling minimal, diff terkecil, paling dekat intent asli. Event delegation = overkill untuk elemen statis.
- **D-02:** Pertahankan handler body apa adanya: `document.getElementById('btnImportMapping').disabled = !this.files.length;` → parity disable-on-deselect (kosongkan file → tombol disable lagi) otomatis terjaga.
- **D-03:** Akar masalah (verified): `Views/Admin/CoachCoacheeMapping.cshtml:1038` register `getElementById('importMappingFile')?.addEventListener(...)` saat inline `<script>` dieksekusi (parse time), TAPI input ada di line 1059 — di dalam modal yang ditaruh SETELAH `</script>` (line 1041) dalam `@section Scripts` yang sama. Saat script jalan, elemen belum di-parse → `?.` no-op → listener tak nempel. Cuma **1 instance** anti-pattern ini di file (grep dikonfirmasi, tak ada sibling); pre-existing (identik di `7548c6d0~1`, bukan regresi 389).

### Test / verifikasi
- **D-04:** Extend spec existing `tests/e2e/coachcoacheemapping-389.spec.ts` (TAMBAH test case, BUKAN file spec baru). Satu suite parity, run `--workers=1`, `Authentication__UseActiveDirectory=false`.
- **D-05:** Assertion inti: buka modal Import Excel → pilih `tests/fixtures/import-mapping-390.xlsx` → `#btnImportMapping` jadi enabled. ROADMAP juga minta spec 389 existing **tetap green** (tak boleh regresi).

### Claude's Discretion
- Penempatan persis blok `DOMContentLoaded` di dalam `@section Scripts` (boleh bungkus baris 1038–1040 saja, atau gabung ke handler init lain bila ada — selama elemen dijamin sudah ada).
- Detail langkah Playwright (selector buka modal, cara set file input) sesuai pola spec 389 yang ada.

</decisions>
