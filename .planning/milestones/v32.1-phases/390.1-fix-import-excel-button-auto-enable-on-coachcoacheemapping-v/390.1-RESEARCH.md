## Architecture Patterns

### Pattern 1: Tunda binding event sampai DOM siap (DOMContentLoaded)
**What:** Untuk elemen statis server-rendered yang posisinya di bawah inline `<script>` dalam DOM, bungkus registrasi listener dalam `document.addEventListener('DOMContentLoaded', ‚Ä¶)`.
**When to use:** Saat inline `<script>` non-defer mereferensikan elemen yang ter-render belakangan dalam dokumen (kasus ini PERSIS). Listener akan terpasang setelah seluruh dokumen ter-parse.
**Why not event delegation (REJECTED per D-01):** Input `#importMappingFile` statis (bukan di-inject dinamis), jadi delegation di `document` overkill, diff lebih besar, dan menjauh dari intent kode asli. [CITED: CONTEXT D-01]

### Anti-Patterns to Avoid
- **Top-level `getElementById(...)` di inline non-defer script untuk elemen di bawahnya:** persis bug ini. `?.` menyembunyikan kegagalan (no-op senyap) sehingga tidak ada error console ‚Üí sulit terdeteksi via grep/build (Phase 354 lesson). [VERIFIED]
- **Mengubah handler body / menambah logic baru:** keluar scope; D-02 mengunci handler body identik.
- **Memindah seluruh isi `<script>` ke `DOMContentLoaded`:** tidak perlu ‚Äî sisa kode adalah function declarations; hanya 1038‚Äì1040 yang bermasalah.

## Common Pitfalls

### Pitfall 1: Listener tetap tak nempel walau sudah DOMContentLoaded (salah scope wrap)
**What goes wrong:** Membungkus baris yang salah atau menaruh `DOMContentLoaded` di luar `<script>`.
**Why it happens:** `@section Scripts` mengandung markup HTML setelah `</script>`; `DOMContentLoaded` HARUS di dalam `<script>` (1038‚Äì1040), bukan di markup.
**How to avoid:** Bungkus tepat baris 1038‚Äì1040 di dalam `<script>` yang ada (sebelum `</script>` baris 1041).
**Warning signs:** Test DSN-07 RED ‚Äî tombol tetap disabled setelah file dipilih.

### Pitfall 2: Mengandalkan grep/build untuk membuktikan fix
**What goes wrong:** `dotnet build` sukses & grep cocok, tapi listener tetap tak terpasang di runtime.
**Why it happens:** Razor view dinamis ‚Äî binding event hanya terbukti di browser. (Phase 354 lesson.) [CITED: CONTEXT code_context]
**How to avoid:** WAJIB jalankan Playwright test DSN-07 di localhost (`dotnet run` + `--workers=1`).
**Warning signs:** Skip menjalankan e2e dan langsung commit.

### Pitfall 3: Test jadi destruktif (klik "Upload & Proses")
**What goes wrong:** Menekan submit ‚Üí benar-benar meng-import data ke DB lokal ‚Üí kotori state.
**Why it happens:** Niat "test end-to-end".
**How to avoid:** DSN-07 hanya assert `toBeEnabled()`. JANGAN klik `#btnImportMapping`. (Selaras CLAUDE.md Seed Workflow ‚Äî jangan kotori DB lokal.)
**Warning signs:** Muncul data mapping baru / perlu RESTORE DB pasca-test.
