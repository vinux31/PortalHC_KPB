## Info

### IN-01: Tidak ada guard bila `#btnImportMapping` hilang (defensive, opsional)

**File:** `Views/Admin/CoachCoacheeMapping.cshtml:1039-1040`
**Issue:** Listener menggunakan optional chaining `?.` pada `#importMappingFile` (aman bila input tidak ada), namun di dalam handler `document.getElementById('btnImportMapping').disabled = ...` mengakses `#btnImportMapping` tanpa guard. Dalam markup saat ini kedua elemen selalu berada dalam modal yang sama, jadi secara praktis ini tidak akan pernah `null` ketika handler ter-trigger — sehingga ini bukan bug nyata, hanya asimetri gaya defensif dengan baris di atasnya. Tidak perlu diubah untuk fase ini.
**Fix:** Bila ingin konsisten secara defensif (opsional):
```javascript
document.addEventListener('DOMContentLoaded', function () {
    document.getElementById('importMappingFile')?.addEventListener('change', function() {
        var btn = document.getElementById('btnImportMapping');
        if (btn) btn.disabled = !this.files.length;
    });
});
```
