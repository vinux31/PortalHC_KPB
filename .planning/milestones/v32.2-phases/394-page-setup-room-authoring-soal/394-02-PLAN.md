## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → GET /Admin/CheckTitleAvailability | Read-only title query; reused existing endpoint |
| Browser form-state → (eventual) POST | Setup + selected UserIds captured client-side; validated server-side at commit (393/395) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-394-02-CSRF-readonly | Tampering | `CheckTitleAvailability` | accept | Endpoint is `[HttpGet]` read-only (no state change) → no antiforgery token needed (mirror existing). |
| T-394-02-IDOR | Information Disclosure / Tampering | worker picker UserIds | mitigate | Picker renders only existing active AspNetUsers; arbitrary UserId injected via crafted POST is rejected at service preflight (NIP resolution fails → reject-all, 393 D-03). Client picker = convenience; server is authoritative. |
| T-394-02-future-date | Tampering | CompletedAt backdate | mitigate | Client `max=today` + `validateStep(1)` reject future; server preflight `CompletedAt.Date > today` reject-all (InjectAssessmentService:~356, 393) is the real gate. |
| T-394-02-XSS | Tampering (Injection) | Cek-judul result + selected-participants render | mitigate | Render matched category/user strings via `.textContent` (not innerHTML); Razor auto-encodes the `@foreach` user rows. |
</threat_model>
