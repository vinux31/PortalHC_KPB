## Spacing Scale

Declared values (Bootstrap utility classes — all map to a 4px base grid):

| Token | Value | Bootstrap class | Usage in this phase |
|-------|-------|-----------------|---------------------|
| xs | 4px | `gap-1` / `me-1` / `mb-1` | gap between checkbox↔radio↔label in a unit row; badge right-margin (`me-1`) |
| sm | 8px | `gap-2` / `p-2` / `mb-2` | unit-row internal gap; checkbox-list container padding (`p-2`) |
| md | 16px | `g-3` (row gutter) / `mb-3` | form field rows (`row g-3`), field block bottom margin |
| lg | 24px | `mb-4` / `px-4` | card bottom margin, submit-button horizontal padding |
| xl | 32px | `p-4` / `py-5` | page/card body padding (existing layout) |

Exceptions:
- **Touch target:** each unit-row interactive control (checkbox, radio) must remain ≥ 24px hit area via Bootstrap's default `form-check-input` sizing; the clickable `<label>` widens the target. No custom px override.
- The checkbox-list container uses `border rounded p-2` (8px inner pad) to visually group it the same way a `form-select` reads as one control. Cap visible height with `max-height: 220px; overflow-y: auto;` only if a Bagian has many units (scroll, do not paginate).

## Typography

Inherit the app's existing Bootstrap type scale — this phase declares roles, not new sizes (no new fonts/sizes introduced). Two weights only.

| Role | Size | Weight | Line Height | Bootstrap class |
|------|------|--------|-------------|-----------------|
| Body / value | 16px (1rem base) | 400 (regular) | 1.5 | (default) |
| Label / small | 14px (`.small` ≈ 0.875rem) | 600 (semibold) for field labels | 1.5 | `form-label fw-semibold` (label) · `small` (unit-row label, table cell) |
| Heading (card header) | ~16px (`h6`) | 600 (semibold) | 1.2 | `h6 fw-semibold` |
| Display | not used in this phase | — | — | — |

Two weights total: **400** (body/value) and **600** (labels + card header). Card header uses `fw-semibold` (not `fw-bold`) to stay within the 2-weight cap while reading as a heading via the `h6` size step.

Notes:
- Unit-row labels use `class="form-check-label small"` (14px regular) — matches the compact density of `ManageWorkers.cshtml:262` `<small>` cells.
- Do NOT add inline `style="font-size:..."` magic numbers (v32.1 DSN-05 lesson) — use `.small` / `fw-semibold` utilities.

## Color

60/30/10 maps to the existing app theme; this phase only **adds** the primary-unit marker color. No new palette.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` / page `bg-light` | Page background, card bodies (`card border-0 shadow-sm`) |
| Secondary (30%) | Bootstrap `bg-primary bg-opacity-10` + neutral `text-muted` / `bg-secondary bg-opacity-25` | Secondary (non-primary) unit badges, card headers, table chrome |
| Accent (10%) | Bootstrap `success` (green) | **Reserved exclusively for the PRIMARY-unit marker** (badge + star) |
| Destructive | Bootstrap `danger` (red) | MU-07 hard-block error message (PROTON aktif); validation errors `text-danger` |

Accent reserved for: **the primary-unit marker only** — the `badge bg-success` (or `bg-success bg-opacity-10 text-success`) + `★` star on the one primary unit, and the radio that selects primary. Accent is NOT used for ordinary buttons, links, or secondary units. The existing app primary action color (`btn-primary` blue) is unchanged and is not "accent" in this contract.

**Contrast / decision (D-08, Claude's Discretion — defaults below):**
- **Primary unit badge (HTML surfaces):** `badge bg-success bg-opacity-10 text-success` + leading `<i class="bi bi-star-fill"></i>` star + a trailing `Utama` tag is OPTIONAL but recommended for screen-reader clarity (see Accessibility). Rationale: green = "this is the authoritative/primary unit"; tracks the existing semantic-badge idiom (`bg-*-10 text-*` at WorkerDetail:93) so it reads as one family with the Section badge, not a foreign style. `text-success` on `bg-opacity-10` clears WCAG AA.
- **Secondary units (HTML surfaces):** `badge bg-secondary bg-opacity-25 text-dark` (no star). Visually subordinate but legible.
- **Do NOT** rely on color alone to convey "primary" — pair the green with the `★` icon AND an `aria-label`/visible `Utama` text (accessibility requirement below).

## Component & Interaction Contract (phase-specific — the load-bearing section)

### A. Multi-select Unit widget (CreateWorker / EditWorker) — MU-01 / MU-02

**Replaces** the single `<select id="unitSelect">` at `CreateWorker.cshtml:119-124` / `EditWorker.cshtml:124-129`. Bagian picker stays a **single `<select id="sectionSelect">`** (unchanged). Rendered client-side by an extended `initSectionUnitMultiCascade(opts)` from `ViewBag.SectionUnitsJson` (no AJAX — D-01).

**DOM contract (idiom: Bootstrap 5 `form-check` rows inside a bordered group):**
```html
<label class="form-label fw-semibold">@OrgLabels.GetLabel(1) Penugasan</label>
<div id="unitMultiContainer" class="border rounded p-2" style="max-height:220px;overflow-y:auto;"
     role="group" aria-label="Daftar @OrgLabels.GetLabel(1) penugasan">
  <span class="text-muted small">Pilih @OrgLabels.GetLabel(0) dahulu untuk menampilkan daftar @OrgLabels.GetLabel(1).</span>
</div>
<div class="form-text">Centang setiap @OrgLabels.GetLabel(1) ... Pilih satu sebagai @OrgLabels.GetLabel(1) Utama.</div>
```
Each unit row (rendered by JS), grouped as a labelled radio-group for the primary axis:
```html
<div class="d-flex align-items-center gap-2 mb-1">
  <input type="checkbox" name="Units" value="{unit}" id="uu-chk-{i}" class="form-check-input uu-check mt-0">
  <input type="radio" name="PrimaryUnit" value="{unit}" id="uu-prim-{i}" class="form-check-input uu-primary mt-0" disabled>
  <label for="uu-chk-{i}" class="form-check-label small flex-grow-1">{unit}</label>
  <label for="uu-prim-{i}" class="form-check-label small text-success">Utama</label>
</div>
```

**State machine (verify each with Playwright — Phase 354 lesson):**

| State | Visual / behavior |
|-------|-------------------|
| No Bagian selected | Container shows muted placeholder copy; no rows; no checkboxes/radios in DOM |
| Bagian selected | Container re-renders one row per child unit of that Bagian (from dict). On Edit, pre-checks `selectedUnits`, pre-selects `primaryUnit` radio |
| Checkbox checked | Its `PrimaryUnit` radio becomes **enabled**; if no primary is currently set, this row's radio auto-selects (default primary = first checked, D-02) |
| Checkbox unchecked | Its radio is **cleared and disabled**; if it was the primary, primary auto-promotes to the first still-checked unit (or none if none remain) |
| Primary selected | The chosen row's radio is `checked`; its "Utama" label is `text-success`; exactly one radio across the group is checked at all times when ≥1 box is checked |
| 0 units checked | No primary; no radio enabled; valid state → server sets `ApplicationUser.Unit = null` (D-02). No error |
| Re-render on Bagian change | Checking/clearing resets to the new Bagian's units. Changing Bagian on Edit clears prior selections (units must belong to the new Bagian — Invariant #1) |
| Validation error (server) | Units chosen but `PrimaryUnit` ∉ checked set → re-render form, show inline `text-danger small` "Pilih salah satu {Unit} sebagai {Unit} Utama." under the container |

**Accessibility (required — checker Dimension 2):**
- Container is `role="group"` with `aria-label` naming the Unit tier.
- Every checkbox/radio has a matching `<label for="...">` (association, not wrapping) — keyboard `Tab` reaches each control, `Space` toggles checkbox, arrow keys move within the `PrimaryUnit` radio group.
- Disabled radios carry the native `disabled` attribute (announced as unavailable), not just visual greying.
- The primary marker is conveyed by **text** ("Utama") + color, never color alone.
- Focus order: checkbox → its primary radio → next row (DOM order matches reading order).

### B. Multi-unit DISPLAY (MU-03) — HTML surfaces vs print/Excel

**HTML surfaces** — Profile (`Profile.cshtml:80,86`), Settings (`Settings.cshtml:99,117`), WorkerDetail (`:101-104`), ManageWorkers table cell (`:262`), Home dashboard (`Index.cshtml`): render the unit set as inline badges, primary first, primary highlighted.

```razor
@if (Model.Units != null && Model.Units.Any())
{
    @foreach (var u in Model.Units.OrderByDescending(x => x == Model.PrimaryUnit).ThenBy(x => x))
    {
        var isPrim = u == Model.PrimaryUnit;
        if (isPrim)
        {
            <span class="badge bg-success bg-opacity-10 text-success me-1">
                <i class="bi bi-star-fill me-1" aria-hidden="true"></i>@u <span class="visually-hidden">(Utama)</span>Utama
            </span>
        }
        else
        {
            <span class="badge bg-secondary bg-opacity-25 text-dark me-1">@u</span>
        }
    }
}
else
{
    <span class="text-muted fst-italic">Belum diisi</span>  @* table cells: render "-" instead (D-09) *@
}
```
- **Primary badge:** `bg-success bg-opacity-10 text-success` + `bi-star-fill` + trailing `Utama` (visible) + `visually-hidden` reinforcement for SR.
- **Secondary badge:** `bg-secondary bg-opacity-25 text-dark`, no star, no "Utama".
- **Ordering:** primary first, then alphabetical (matches `_PSign`/Excel ordering for consistency).
- **Density:** in the ManageWorkers/Home **table cells**, wrap each badge but keep `small`; if a worker has many units the cell may wrap to 2 lines — acceptable. Empty cell renders `-` (not the italic "Belum diisi" panel) to match `:262`.

**Print / cert card** (`_PSign.cshtml:40-43`) and **Excel export** (`WorkerController.cs:186-187`): **plain text, primary-first, comma-joined** — badges do not print (D-08). `_PSign` shows ALL units (D-07, deliberate — do NOT revert to primary-only).
```razor
@* _PSign.cshtml — replace :40-43 *@
@if (Model.Units != null && Model.Units.Any())
{
    <div class="psign-label">@string.Join(", ", Model.Units.OrderByDescending(u => u == Model.PrimaryUnit).ThenBy(u => u))</div>
}
@* no row when empty — preserves existing behavior *@
```
```csharp
// Excel cell (replace WorkerController.cs:186-187): primary-first comma text in the SAME column (Cell 7 area)
string unitsText = (u.Units == null || !u.Units.Any())
    ? ""
    : string.Join(", ", u.Units.OrderByDescending(x => x == u.PrimaryUnit).ThenBy(x => x));
```
- Print/Excel do NOT mark the primary with a star or "Utama" — primary-first ordering is the only signal (plain text, D-08). This is intentional and accepted (D-07 tradeoff: cert attribution stays primary per D1=b while the printed card lists all units).

### D. MU-07 destructive confirmation — Edit Worker

Two asymmetric paths, both surfaced on the Edit form (server round-trip re-prompt, the leaned Claude's-Discretion mechanism — no new endpoint):
- **PROTON aktif → HARD-BLOCK (D-11):** render the red error (copy above) at the top of the form via `ModelState`; the form does not save. No confirmation dialog — the operator must resolve PROTON elsewhere.
- **Coach mapping aktif (no PROTON) → confirm-then-auto-deactivate (D-10):** on first POST the server detects impact and re-renders the form with a **Bootstrap modal** (idiom: existing confirm modals in the repo) listing impacted mappings + a hidden `ConfirmedDeactivate=true` field; operator confirms via `btn-danger` "Ya, Hapus & Nonaktifkan", resubmits, and the deactivate + unit removal happen in one transaction.
