## The One New Element — Contract

### Markup (Razor — `ManageOrganization.cshtml`)

Sisipkan sebagai `<li>` **terakhir** di dalam `<ul class="mb-2">` (`#cascadeConfirmModal`), tepat **setelah** baris `cascadeGuidance`:

```html
<li><strong id="cascadeUserUnits">0</strong> baris keanggotaan unit</li>
```

**Aturan kontrak (mirror byte-for-byte dari saudaranya):**

| Aspek | Nilai (WAJIB sama dgn baris existing) | Sumber baris-acuan |
|-------|----------------------------------------|--------------------|
| Elemen pembungkus | `<li>` di dalam `<ul class="mb-2">` existing | semua baris `cascade*` |
| Nilai hitung | `<strong id="cascadeUserUnits">0</strong>` (default literal `0`, bold via `<strong>`) | `cascadeUsers` (L222) |
| Element id | `cascadeUserUnits` (camelCase, prefix `cascade`, konsisten dgn `cascadeUsers`/`cascadeMappings`/`cascadeKompetensi`/`cascadeGuidance`) | konvensi id existing |
| Teks label | literal **" baris keanggotaan unit"** (plain text setelah `</strong>`, diawali 1 spasi) | pola label literal existing |
| Posisi | baris **terakhir** dalam `<ul>` (setelah `cascadeGuidance`) | urutan modal existing |
| Tipografi / spacing / warna | **0 deklarasi baru** — inherit dari `<ul>`/`<li>`/`<strong>` Bootstrap existing | seluruh modal |
| Ikon | **tidak ada** (baris existing tak pakai ikon di dalam `<li>`) | semua baris `cascade*` |

### Konvensi label — literal text, BUKAN `@OrgLabels.GetLabel()`

Baris-baris existing memakai **teks literal Indonesia** di dalam `<li>` (`user`, `mapping coach-coachee`, `kompetensi PROTON`, `file panduan`) — **bukan** helper `@OrgLabels.GetLabel(0/1)`. Baris baru **WAJIB mengikuti konvensi ini**: gunakan literal **"baris keanggotaan unit"** (label konsep D-03), **JANGAN** menyisipkan `@OrgLabels.GetLabel(1)`. Rasionalnya: "keanggotaan unit" merujuk pada baris junction `UserUnits` (konsep teknis-fungsional, bukan tier-name org yang configurable), sehingga konsisten dengan gaya label literal modal ini dan tidak menimbulkan drift bila label tier org diubah admin.

### JS populate (`orgTree.js`)

**(a)** Di `showCascadeConfirm(pv)` (L361-364), tambah baris **persis** mengikuti pola existing (gunakan guard `|| 0` yang sama):

```javascript
document.getElementById('cascadeUserUnits').textContent = pv.affectedUserUnitsCount || 0;
```

**(b)** Di `submitUnitModal()` (L426-427), tambahkan `affectedUserUnitsCount` ke penjumlahan `total` agar modal tetap muncul bila satu-satunya dampak adalah baris `UserUnits` (mis. rename unit yang hanya punya membership sekunder):

```javascript
const total = (pv.affectedUsersCount || 0) + (pv.affectedMappingsCount || 0)
            + (pv.affectedKompetensiCount || 0) + (pv.affectedGuidanceCount || 0)
            + (pv.affectedUserUnitsCount || 0);   // NEW 403 (D-03)
```

### Zero / empty handling — MATCH siblings

- Baris **selalu dirender** di markup dengan default literal `0` (persis seperti `cascadeUsers` dst.). Tidak ada cabang show/hide untuk baris individual.
- Modal `#cascadeConfirmModal` **secara keseluruhan** hanya muncul saat `total > 0` (logika existing `submitUnitModal` L428). Saat muncul, baris `cascadeUserUnits` ikut tampil dengan nilainya (bisa `0` bila dampak datang dari kategori lain) — **identik perilaku dengan 4 baris existing**. Tidak ada empty-state khusus; tidak ada penyembunyian baris ber-nilai-0 (siblings pun tidak menyembunyikan).
- Reparent tanpa rename → `affectedUserUnitsCount = 0` (D-01b) → baris tampil `0` bila modal toh muncul karena kategori lain. Tidak ada penanganan khusus.

### Parity (preview == actual) — bukan keputusan UI, tapi mengikat tampilan

Nilai yang ditampilkan baris ini = `affectedUserUnitsCount` dari `PreviewEditCascade`, yang WAJIB sama persis dengan jumlah baris `UserUnits` yang benar-benar di-rename oleh `EditOrganizationUnit` (D-03a, RESEARCH Pattern 4: keduanya hitung `uu.Unit == oldName` **tanpa** filter `IsActive`). UI hanya menampilkan angka server-authoritative; tidak menghitung apa pun di klien.

### Accessibility parity

Sama persis dengan 4 baris existing: teks dalam `<li>` di `<ul>` semantik, di dalam `modal-body` yang sudah ber-`aria-labelledby="cascadeConfirmLabel"` + tombol `btn-close aria-label="Tutup"` (existing). **Tidak ada** atribut ARIA tambahan, role, atau `aria-live` baru (siblings tak punya; baris baru harus paritas, bukan melebihi). Bila checker menuntut, paritas = cukup.
