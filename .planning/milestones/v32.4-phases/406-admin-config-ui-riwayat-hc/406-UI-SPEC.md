# Phase 406 — UI Design Contract — Admin Config UI + Riwayat HC

> Visual and interaction contract for Phase 406 (milestone v32.4 Ujian Ulang / Attempt-Retake).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Design system state: EXISTING / MATURE — REUSE.** This is a brownfield ASP.NET Core MVC
> app on Bootstrap 5.3.0 + Bootstrap Icons 1.10.0 + Razor. There is NO design-token system to
> invent. This contract documents the EXISTING conventions and specifies, prescriptively, how
> the two new surfaces (retake config card + riwayat HC modal) conform to them by mirroring
> already-shipped components. Do NOT introduce new colors, fonts, spacing scales, or libraries.

**Surfaces in scope (only these two):**
1. **Retake config card** — `Views/Admin/ManagePackages.cshtml` (mirror shuffle card `:84-132`), placed AFTER the shuffle card. Binding in `CreateAssessment.cshtml` (Step 3) + `EditAssessment.cshtml` (near PassPercentage).
2. **Riwayat HC modal** — `Views/Admin/AssessmentMonitoringDetail.cshtml` (per-peserta row → Bootstrap modal, mirror `#akhiriSemuaModal :500` / `#reshuffleResultModal :481`).

Out of scope (Phase 407): worker self-service UI in `Results.cshtml`/`Records.cshtml`, gating tiers, `RetakeExam` endpoint.

## Spacing Scale

Use Bootstrap 5.3 spacing utilities ONLY (already a 4px-based scale: `0=0, 1=4px, 2=8px, 3=16px, 4=24px, 5=48px`). All values are multiples of 4 by construction. Do NOT hand-roll `style="margin:..."`.

| Token (Bootstrap) | Pixels | Usage in this phase |
|-------------------|--------|---------------------|
| `*-1` | 4px | Icon-to-text gap inside labels/badges (`me-1`), inline `gap-1` between row action buttons |
| `*-2` | 8px | Toggle-to-helper gap (`mb-2`), `gap-2` button clusters, badge spacing |
| `*-3` | 16px | Helper-text bottom gap (`mb-3`), card body inner blocks, accordion item gaps |
| `*-4` | 24px | Card bottom margin (`card mb-4`), modal body section breaks |
| `py-3` / `py-4` | 16/24px | Card header / container vertical padding (match existing) |

Exceptions: none. Reuse the EXACT spacing utilities the shuffle card and existing modals already use (`card mb-4`, `card-header bg-light`, `card-body`, `form-check form-switch mb-2`, `form-text text-muted mb-3`).

## Typography

No custom typography. Reuse Bootstrap defaults + the heading classes already used by mirror components. Two weights only, matching the existing app.

| Role | Class / Element | Size (Bootstrap default) | Weight | Usage |
|------|-----------------|--------------------------|--------|-------|
| Card title | `<h5 class="mb-0">` (ManagePackages cards) / `<h6 class="mb-0">` (Create/Edit cards) | h5≈20px / h6≈16px | semibold (Bootstrap heading default ~600) | "Ujian Ulang" card header |
| Modal title | `<h5 class="modal-title">` | ≈20px | semibold | "Riwayat Percobaan — {nama}" |
| Body / table | default `<td>`, `<p>` | 16px (1rem) | regular (400) | accordion rows, per-soal table cells |
| Label | `<label class="form-check-label">` / `form-label fw-bold` | 16px | regular (toggle label) / bold (`fw-bold` field label, mirror Create Step-3) | toggle + number-input labels |
| Helper / muted | `<div class="form-text text-muted">`, `.small` | ≈14px (0.875em) | regular (400) | "0 = tanpa jeda", "jam", timestamps |

Line height: inherit Bootstrap defaults (body ~1.5, headings ~1.2). Do NOT set custom line-height.

**Weight rule:** only two weights appear — regular (400) for body/helper, semibold/bold for headings and the bold field labels (`fw-bold`/`fw-semibold`) that the existing Create/Edit forms already use. Do not introduce a third weight.

## Color

No custom palette. Use Bootstrap 5.3 semantic contextual classes EXACTLY as the mirror components do. The "10% accent" in this brownfield context = Bootstrap `primary` (used sparingly for the save CTA + card-header icon), and semantic `success`/`danger`/`warning` reserved for state meaning only.

| Role | Bootstrap class | Usage (this phase) |
|------|-----------------|--------------------|
| Dominant surface (60%) | page/card background (`bg-white`, default body) | ManagePackages background, modal background |
| Secondary (30%) | `card`, `card-header bg-light` (config card) / `modal-header` default | card chrome, accordion item headers |
| Accent (10%) — primary | `btn btn-primary`, `text-primary` (header icon) | **Save CTA** (`btn btn-primary` "Simpan Pengaturan"), card-header icon `bi bi-arrow-repeat text-primary` |
| Success | `text-bg-success` badge / `text-success` | Attempt result badge **"Lulus"**, per-soal correct ✓ (`text-success`) |
| Danger | `text-bg-danger` badge / `text-danger` | Attempt result badge **"Gagal"**, per-soal wrong ✗ (`text-danger`) |
| Warning | `alert alert-warning` | **Non-blocking MaxAttempts warning** (D-03), pending-essay marker tone |
| Info | `text-bg-info` / `badge bg-info` | **"Percobaan saat ini"** current-attempt marker badge |
| Muted | `text-muted` | timestamps, helper text, empty-state, pending-essay "Menunggu" |

**Accent reserved for (explicit list — never "all interactive elements"):**
- `btn btn-primary` — the single Save CTA in the config card (mirror shuffle's `btn btn-primary`).
- `text-primary` — the card-header icon only.
- All other buttons use `btn-secondary` / `btn-outline-secondary` (modal close, dropdown trigger) — NOT primary.

**Semantic-color discipline:** `success`/`danger`/`warning`/`info` are reserved for STATE meaning (pass/fail/warning/current-attempt), never decoration. Do not use them on neutral chrome.

### 1. Retake config card — `ManagePackages.cshtml` (after shuffle card, ~`:132`)

Mirror shuffle card structure (`:89-131`). Differences from shuffle: **no lock logic** (D-03), **progressive disclosure** of two number inputs (D-04), **non-blocking warning** instead of disabled state.

```
@if (ViewBag.HideRetakeToggle != true)        // RetakeRules.ShouldHideRetakeToggle → hide Pre-Test/Manual
{
    bool arChecked = ViewBag.AllowRetake == true;
    <div class="card mb-4">
        <div class="card-header bg-light">
            <h5 class="mb-0"><i class="bi bi-arrow-repeat text-primary me-1"></i>Ujian Ulang</h5>
        </div>
        <div class="card-body">
            <form method="post" asp-action="UpdateRetakeSettings" asp-controller="AssessmentAdmin">
                @Html.AntiForgeryToken()
                <input type="hidden" name="assessmentId" value="@ViewBag.AssessmentId" />

                <!-- Toggle (mirror form-check form-switch) -->
                <div class="form-check form-switch mb-2">
                    <input class="form-check-input" type="checkbox" name="allowRetake" id="allowRetake"
                           value="true" @(arChecked ? "checked" : "") />
                    <label class="form-check-label" for="allowRetake">Izinkan Ujian Ulang</label>

                <!-- Progressive disclosure: revealed when toggle ON (d-none default + JS) -->
                <div id="retakeFields" class="@(arChecked ? "" : "d-none")">
                    <div class="mb-3" style="max-width: 320px;">
                        <label for="maxAttempts" class="form-label fw-bold">Maksimal Percobaan</label>
                        <input type="number" class="form-control" name="maxAttempts" id="maxAttempts"
                               min="1" max="5" value="@(ViewBag.MaxAttempts ?? 2)" />
                        <div class="form-text text-muted">Berapa kali peserta boleh mencoba (termasuk percobaan pertama). Rentang 1–5.</div>
                        <!-- Non-blocking warning (D-03) -->
                        @if ((int)ViewBag.MaxAttempts < (int)ViewBag.RetakeMaxAttemptsUsedInGroup)
                        {
                            <div class="alert alert-warning d-flex align-items-start mt-2 mb-0" role="alert">
                                <i class="bi bi-exclamation-triangle me-2 mt-1"></i>
                                <div>Maksimal percobaan yang Anda set lebih kecil dari jumlah percobaan yang sudah dipakai sebagian peserta. Peserta tersebut tidak akan kehilangan percobaan yang sudah berjalan, tetapi tidak bisa menambah percobaan baru. Pengaturan tetap bisa disimpan.</div>
                            </div>
                        }
                    </div>
                    <div class="mb-3" style="max-width: 320px;">
                        <label for="retakeCooldownHours" class="form-label fw-bold">Jeda Ujian Ulang (jam)</label>
                        <input type="number" class="form-control" name="retakeCooldownHours" id="retakeCooldownHours"
                               min="0" max="168" value="@(ViewBag.RetakeCooldownHours ?? 24)" />
                        <div class="form-text text-muted">Jeda minimal sebelum boleh mencoba lagi. 0 = tanpa jeda. Satuan: jam (maksimal 168 = 7 hari).</div>
                    </div>
                </div>

                <button type="submit" class="btn btn-primary"><i class="bi bi-save me-1"></i>Simpan Pengaturan</button>
            </form>
        </div>
    </div>
}
```

**Progressive disclosure JS (inline `<script>` at end of view, mirror existing token-toggle pattern `CreateAssessment :521` `tokenSection`):** on `#allowRetake` `change`, toggle `#retakeFields` `.d-none`. Inputs stay in DOM (always submitted; server clamps regardless).

**Interaction rules:**
- **No lock.** Card is always editable even after exam started (D-03) — unlike shuffle. Do NOT add `disabled` attributes or a lock alert.
- Warning is **non-blocking**: it renders near MaxAttempts but never disables Save (D-03). Server `Math.Clamp` is the real guard; `min`/`max` on inputs are UX hints only.
- Hidden **entirely** (card not rendered) when `ViewBag.HideRetakeToggle == true` (Pre-Test / Manual Entry) — D-04, RTK-05.
- POST → existing `UpdateRetakeSettings:5564` (RBAC Admin/HC + AntiForgery + sibling propagation + PRG). No new endpoint.

## Accessibility

| Element | Requirement |
|---------|-------------|
| Modal | `tabindex="-1"`, `aria-labelledby="riwayatModalLabel"`, `aria-hidden="true"`; `btn-close` has `aria-label="Tutup"`. Bootstrap manages focus trap + Esc-to-close + return-focus to trigger (mirror existing modals). |
| Trigger button | `aria-label`/`title="Riwayat percobaan untuk {nama}"`; keyboard-activatable (`<button type="button">`, not `<div>`). |
| Accordion | `accordion-button` has `aria-expanded` + `aria-controls`; collapse panel `id` matches `aria-controls`. Bootstrap handles keyboard toggle (Enter/Space). |
| Status icons (✓/✗/—) | icon-only cells include a text equivalent (`title` + visually-hidden text e.g. `<span class="visually-hidden">Benar</span>`) so meaning is not color-only. Color (`text-success`/`text-danger`) is reinforced by distinct glyph shape. |
| Config toggle | `<label for="allowRetake">` correctly associated; `form-check form-switch` is keyboard-focusable; disclosure reveal does not trap focus. |
| Number inputs | `<label for>` associated; `min`/`max` advisory; `aria-describedby` may point to the helper `form-text` (optional). |
| Warning alert | `role="alert"` so it is announced when rendered. |
| Color independence | Lulus/Gagal use a text label ("Lulus"/"Gagal") inside the badge, not color alone. |
