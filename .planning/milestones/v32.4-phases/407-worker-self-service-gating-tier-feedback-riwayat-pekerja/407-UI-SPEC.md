# Phase 407 — UI Design Contract — Worker Self-Service + Gating Tier Feedback + Riwayat Pekerja

> Visual and interaction contract for Phase 407 (milestone v32.4 Ujian Ulang / Attempt-Retake).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Design system state: EXISTING / MATURE — REUSE.** Brownfield ASP.NET Core MVC app on
> Bootstrap 5.3.0 + Bootstrap Icons 1.10.0 + Razor. There is NO design-token system to invent.
> This contract documents the EXISTING conventions and specifies, prescriptively, how the new
> worker-side surfaces (retake CTA + cooldown countdown + lock + tier-gated feedback + worker
> riwayat) conform to them by mirroring already-shipped components — including the **sibling
> Phase 406 contract** (`406-UI-SPEC.md`) whose riwayat accordion and tri-state verdict
> conventions this phase reuses, GATED. Do NOT introduce new colors, fonts, spacing scales, or
> libraries.

**Surfaces in scope (worker-facing only):**
1. **Retake control block** — `Views/CMP/Results.cshtml` (action-buttons area `:420-431` + result summary card `:43-105`): "Ujian Ulang" button, "Percobaan ke-X dari N" counter, live cooldown countdown (disabled button), lock alert ("hubungi HC"). *(RTK-10)*
2. **Confirmation modal** — `Views/CMP/Results.cshtml` (new Bootstrap modal, mirror existing modal idiom): destructive-action confirm before POST `CMP/RetakeExam`. *(RTK-09 / D-02)*
3. **Tier-gated answer feedback** — `Views/CMP/Results.cshtml` (review block `:316` + else `:413`): expand the boolean `AllowAnswerReview` into a 3-state tier (`showFullReview` / `showWrongFlagsOnly` / `showScoreOnly`). *(RTK-11 / D-03)*
4. **Worker riwayat (own attempts)** — `Views/CMP/Results.cshtml` + `Views/CMP/Records.cshtml`: list of own attempts (score / pass-fail / date / current-marker) with per-soal drill-down that OBEYS the same tier gating as §3. *(RTK-12 / D-04)*

Out of scope (Phase 408): full Playwright lifecycle test + security audit (RTK-14). Out of scope generally: schema/migration (0), admin config UI (406), grading-engine changes.

## Color

No custom palette. Use Bootstrap 5.3 semantic contextual classes EXACTLY as the existing Results/Records surfaces and the Phase 406 riwayat accordion do. The "10% accent" in this brownfield context = Bootstrap `primary` (used sparingly for the primary CTA), and semantic `success`/`danger`/`warning`/`info` reserved for STATE meaning only.

| Role | Bootstrap class | Usage (this phase) |
|------|-----------------|--------------------|
| Dominant surface (60%) | page/card background (`bg-white`, default body) | Results/Records page background, modal background |
| Secondary (30%) | `card shadow-sm`, `card-header bg-light`, `accordion` chrome | result card, feedback card, riwayat card, accordion item headers |
| Accent (10%) — primary | `btn btn-primary` | **Retake CTA** "Ujian Ulang" (`btn btn-primary`), confirm-in-modal CTA "Ya, Ujian Ulang" — and the existing "Lihat Sertifikat" primary CTA (`:424`, unchanged) |
| Success | `bg-success` / `text-bg-success` / `text-success` | Pass status badge "Lulus", per-soal correct ✓ (`text-success`), correct-option highlight (`list-group-item-success`) — feedback shown ONLY when tier permits |
| Danger | `bg-danger` / `text-bg-danger` / `text-danger` | Fail status badge "Tidak Lulus", per-soal wrong ✗ (`text-danger`), wrong-selected-option highlight (`list-group-item-danger`) |
| Warning | `alert alert-warning` | **Lock state** ("Batas percobaan tercapai, hubungi HC") + cooldown-active notice tone (alternatively `alert-secondary` muted — see Components §1) |
| Info | `alert alert-info` / `badge bg-info` | "Percobaan saat ini" current-attempt marker badge (mirror 406), "Tinjauan jawaban tidak tersedia" notice (`:415`, unchanged), pending-grading banner (`:37`, unchanged) |
| Muted | `text-muted` | counter "Percobaan ke-X dari N", cooldown countdown, timestamps, empty-state, pending-essay "Menunggu", concealed-answer-key explanatory note |

**Accent reserved for (explicit list — never "all interactive elements"):**
- `btn btn-primary` — the **single retake CTA** "Ujian Ulang" (in result card / action row) and its in-modal confirm twin "Ya, Ujian Ulang", plus the pre-existing "Lihat Sertifikat" CTA (unchanged).
- All other buttons use `btn-outline-secondary` / `btn-secondary` (modal cancel "Batal", "Kembali" `:428`, accordion triggers) — NOT primary.

**Semantic-color discipline:** `success`/`danger`/`warning`/`info` are reserved for STATE meaning (pass/fail/lock/cooldown/current-attempt), never decoration. Do not use them on neutral chrome. The retake CTA is `primary` (an action), NOT `success` — passing is not what the button means.

**Leak-safe color rule (D-03):** while a retake remains possible (`showWrongFlagsOnly`), the answer-key colorings `list-group-item-success` (correct option) and the "(Jawaban Benar)" label MUST NOT render. Only the worker's own selection + the per-soal ✓/✗ verdict colors appear. Color is never the only signal (always paired with glyph + text — see Accessibility).

> These are PRESCRIPTIVE markup contracts. Executor implements them against the cited mirror lines. All classes are real and already in the codebase. **Tier and eligibility values come from the ViewModel** (`AssessmentResultsViewModel`) — the view renders; it does NOT compute eligibility (that is `RetakeRules`/`RetakeService` per CONTEXT, surfaced into the VM in `CMPController.Results`). ### 1. Retake control block — `Results.cshtml` action area (`:420-431`)

Place inside / adjacent to the existing `<div class="d-flex gap-2 mb-4">` action row, OR as a dedicated card section directly under the result summary card (`:105`). Mirror the existing button idiom (`btn ... me-1` + `bi` icon).

```
@* Retake control — rendered from VM flags; server is authoritative *@
@if (Model.IsCapReached)
{
    <div class="alert alert-warning mb-4" role="alert">
        <i class="bi bi-lock-fill me-2"></i>
        Batas percobaan tercapai (@Model.MaxAttempts dari @Model.MaxAttempts). Hubungi HC bila perlu kesempatan tambahan.
    </div>
}
else if (Model.CanRetake)
{
    <div class="d-flex align-items-center gap-2 mb-3 flex-wrap">
        @if (Model.CooldownUntilUtc.HasValue && Model.CooldownUntilUtc > DateTime.UtcNow)
        {
            <button type="button" class="btn btn-primary" id="btnRetake" disabled
                    data-cooldown-until="@Model.CooldownUntilUtc.Value.ToString("o")">
                <i class="bi bi-hourglass-split me-1"></i>Bisa ulang dalam <span id="retakeCountdown">--:--:--</span>
            </button>
        }
        else
        {
            <button type="button" class="btn btn-primary" id="btnRetake"
                    data-bs-toggle="modal" data-bs-target="#retakeConfirmModal">
                <i class="bi bi-arrow-repeat me-1"></i>Ujian Ulang
            </button>
        }
        <span class="text-muted small">Percobaan ke-@Model.CurrentAttempt dari @Model.MaxAttempts</span>
    </div>
}
```

**Interaction rules:**
- **Live countdown (D-01):** inline `<script>` reads `data-cooldown-until` (ISO-8601 UTC, round-trip `"o"`), ticks `#retakeCountdown` every 1s as `HH:MM:SS`. On reaching `<= 0`: clear the interval and either auto-enable+relabel the button to "Ujian Ulang" wired to the modal, OR (planner's choice) prompt reload. Mirror existing exam-timer `setInterval` idiom.
- **Server authoritative:** countdown is UX only; `CMP/RetakeExam` re-checks `CanRetakeAsync` on POST and rejects if cooldown/cap not actually satisfied (D-01).
- **Cap reached → `alert-warning` lock, NO button** (RTK-10).
- The CTA is `btn btn-primary` (action color), never `btn-success`.

### 3. Tier-gated feedback — `Results.cshtml` review block (`:316` / `:413`)

Expand the current boolean branch into three. **Do not delete the existing full-review markup** (`:316-411`) — it becomes the `ShowFullReview` branch. Add a `ShowWrongFlagsOnly` branch that renders the SAME question list but suppresses answer-key signals; keep the existing `else` (`:413-418`) as `ShowScoreOnly`.

```
@switch (Model.RetakeMode)   // ShowFullReview | ShowWrongFlagsOnly | ShowScoreOnly
{

        @* EXISTING markup :318-411 verbatim — full review with answer key *@

    case RetakeReviewMode.ShowWrongFlagsOnly:

- The essay branch (`:397-406`) MUST NOT render `@question.CorrectAnswer` (the rubric/key) in this tier — only `@question.UserAnswer`.
- Answer key reappears ONLY in `ShowFullReview` (passed OR exhausted). The tier value is computed server-side; the view trusts the VM.

    @if (Model.RiwayatAttempts == null || !Model.RiwayatAttempts.Any())
    {
        <div class="text-muted text-center py-4">Belum ada riwayat percobaan. Riwayat akan muncul setelah kamu mengulang ujian.</div>
    }
    else
    {
        <div class="accordion" id="riwayatPekerjaAccordion">
          @* for each attempt (descending AttemptNumber): mirror 406 accordion-item *@
          @* header: Percobaan ke-{N}: {skor}% — <badge Lulus/Tidak Lulus/Menunggu Penilaian> — {tanggal} [+ badge bg-info "Percobaan saat ini" if IsCurrentAttempt] *@
          @* body: table table-sm table-hover align-middle mb-0 — headers: No / Soal / Jawaban Saya / Status / Skor *@
          @* Status cell: tri-state ✓/✗/— with visually-hidden text (mirror 406) *@
          @* GATING: if this attempt's group still retake-eligible → drill-down shows verdict only (no key); if passed/exhausted → may show full (subject to AllowAnswerReview) *@
        </div>
    }
  </div>
</div>
```

**Rules:**
- **Reuse `RiwayatUnifier.Build(...)` + `RiwayatAttemptViewModel`** (CONTEXT: gating added at view/VM layer, NOT in the pure unifier).
- **Header badges:** `text-bg-success`="Lulus", `text-bg-danger`="Tidak Lulus", `bg-warning text-dark`="Menunggu Penilaian" (mirror 406 partial `:27-38`). Worker copy uses "Tidak Lulus" (consistent with Results status badge `:82` and Records `:222`), not "Gagal".
- **Current attempt:** `badge bg-info ms-2` "Percobaan saat ini" — driven by `IsCurrentAttempt` (RTK-12 adds this flag to `AllWorkersHistoryRow`).
- **Gating (D-04):** the per-soal table inside each attempt obeys the SAME tier logic as §3 for that attempt's assessment group. While retake remains possible → ✓/✗ + own answer, NO key. Passed/exhausted → key may appear (subject to `AllowAnswerReview`).
- **Records.cshtml:** the existing "Lihat Hasil" link (`:262`) already routes to Results where this section lives — sufficient by default. A per-row "Riwayat Percobaan" trigger is OPTIONAL (planner's choice, D-04 discretion); if added, place in the existing Aksi cell (`:260-283`) as `btn btn-sm btn-outline-secondary` with `bi bi-clock-history` + `title`/`aria-label`.
- **Placement vs modal:** inline accordion card on Results is the default; a modal variant is acceptable (CONTEXT discretion) but MUST reuse the same gated markup.

**XSS (D-02 / RTK-11 leak-safety):** `QuestionText`, `UserAnswer`/`AnswerText`, option text rendered with Razor `@` (auto HTML-encode). If any body is built client-side, use `.textContent`/`.setAttribute`, never `innerHTML` with raw user strings. NEVER `Html.Raw` for user content.
