# Phase 415 — UI Design Contract

> Visual and interaction contract for Phase 415 (Section Foundation + Import Excel Diperluas).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **SCOPE: admin-tool consistency contract, NOT a new brand surface.** Phase 415 adds a NEW
> Section management panel + a Section dropdown + Section-grouped question list + an import
> error list — all INSIDE the existing admin screen `Views/Admin/ManagePackageQuestions.cshtml`
> (plus the existing `Views/Admin/ImportPackageQuestions.cshtml`). The portal already runs an
> established ASP.NET MVC + Bootstrap 5 admin design system. This contract MUST MATCH that
> system. Do NOT invent new tokens, fonts, or a new visual language. All copy in Bahasa Indonesia.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (server-rendered ASP.NET MVC Razor — shadcn gate NOT applicable; stack is not React/Next/Vite) |
| Preset | not applicable |
| Component library | **Bootstrap 5** (existing global, loaded via `_Layout`) — cards, tables, badges, alerts, form-switch, nav-tabs, btn-outline-* |
| Icon library | **Bootstrap Icons** (`bi bi-*`) — already used across all admin views |
| Font | Inherit from `_Layout` / Bootstrap default system stack (`-apple-system, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif`). Do NOT introduce a new font. |

**Detected existing conventions (source of truth — reuse verbatim):**
- Layout: `container-fluid py-4`, two-column `row g-4` with `col-lg-7` (list) + `col-lg-5` (form).
- Card: `card shadow-sm` + `card-header bg-light` with `<span class="fw-semibold">` title + `card-body` / `card-body p-0` for tables.
- Table: `table table-hover table-sm mb-0 align-middle` + `thead class="table-light"`; numeric cells `text-center`; muted cells `text-muted`.
- Toggle precedent (ManagePackages "Acak Soal"): `form-check form-switch` > `input.form-check-input[type=checkbox]` + `label.form-check-label` + `div.form-text.text-muted` helper text.
- Buttons: `btn btn-outline-{success|primary|secondary|danger|info} btn-sm`; icon-only action buttons in tables use `btn-sm py-0 px-1` with a leading `<i class="bi …">` + `title="…"`.
- Badges: `badge` + contextual class (`QuestionTypeLabels.BadgeClass` precedent) + `small`.
- Alerts: `alert alert-{success|danger|warning} alert-dismissible fade show` + `btn-close` dismiss button.
- Empty state: centered `p-4 text-center text-muted` with a large `bi bi-inbox fs-2 d-block mb-2` icon + Bahasa Indonesia guidance line.
- Destructive confirm: inline `<form … onsubmit="return confirm('…?')">` (native confirm) on delete actions.

## Spacing Scale

Bootstrap 5 spacing utility scale (rem-based, multiples of 4px). Reuse the EXISTING utility classes — do NOT hand-roll inline pixel spacing except where the existing view already does (e.g. fixed table column widths).

| Token | Value | Bootstrap utility | Usage |
|-------|-------|-------------------|-------|
| xs | 4px | `*-1` (gap-1, py-1) | Icon gaps, compact icon-button padding |
| sm | 8px | `*-2` (gap-2, mb-2) | Spacing between toggles, button groups, form-check rows |
| md | 16px | `*-3` (mb-3, p-3) | Default form-field spacing, card body padding |
| lg | 24px | `*-4` (mb-4, py-4) | Page header bottom margin, container vertical padding, `row g-4` gutters |
| xl | 32px | `*-5` | Major block separation (rarely needed in this phase) |

Exceptions:
- Fixed table column widths via inline `style="width:Npx"` are an EXISTING convention (e.g. `width:40px` for `#`, `width:110px` for Tipe). The Section panel/list reuses the same pattern for `No. Section` (`width:90px`), toggle columns (`width:120px`), and `Aksi` (`width:120px`). These are layout sizing, not spacing tokens.
- No new spacing tokens introduced. No 44px touch-target override (desktop admin tool, mouse-first; Bootstrap `btn-sm` is acceptable and matches existing admin density).

## Typography

No custom type scale. Inherit Bootstrap defaults + the existing admin view's class usage. The four roles below map to classes ALREADY in use on this screen.

| Role | Size | Weight | Line Height | Implementation (existing class) |
|------|------|--------|-------------|---------------------------------|
| Page heading | ~24px (Bootstrap `h3` = 1.5rem) | 600 (`fw-semibold`) | 1.2 | `<h1 class="h3 fw-semibold mb-0">` |
| Card / section title | ~16px (1rem) | 600 (`fw-semibold`) | 1.5 | `<span class="fw-semibold">` in `card-header bg-light` |
| Body / table cell | 16px (1rem) base; table uses `table-sm` density | 400 | 1.5 | default `table` text; `text-muted` for secondary |
| Label / helper | ~14px (`.small` = 0.875rem) | 600 (`fw-semibold`) for labels / 400 for helper | 1.5 | `form-label fw-semibold small` (label) + `form-text text-muted` (helper) |

Weights in use: exactly **2** — **400 (regular)** for body/table/helper text + **600 (`fw-semibold`)** for headings, titles, and labels. Both are existing Bootstrap utilities already on this screen, not new declarations. Do NOT add medium (500/`fw-medium`), bold (700), or any custom weight.

## Color

No custom palette. Use Bootstrap 5 contextual/theme colors EXACTLY as the existing admin uses them. The 60/30/10 split below describes how the existing surface already allocates color — the new Section UI must follow it.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Bootstrap `body` background / white (`#fff`) | Page background, card bodies, table rows |
| Secondary (30%) | Bootstrap `bg-light` (`#f8f9fa`) | `card-header`, `thead.table-light`, image-thumb placeholder backgrounds |
| Accent (10%) | Bootstrap `primary` (`#0d6efd`) + `success` (`#198754`) | See reserved-for list below |
| Destructive | Bootstrap `danger` (`#dc3545`) | Delete Section / delete question buttons; D-13 mismatch error alert |

Accent reserved for:
- **`btn-primary` / `btn-outline-primary`** — primary submit on the Section create/edit form ("Simpan Section"), the Edit-question action, and the "Template Universal" download (matches existing precedent where Universal template is `btn-outline-primary`).
- **`btn-outline-success`** — Excel-related actions only (Import/Download template), matching the existing "Import Excel" precedent.
- **`text-info` / `border-info`** — the format-reference card (existing convention in ImportPackageQuestions).
- **`btn-outline-secondary`** — non-primary navigation/preview (e.g. "Kembali", preview).
- Do NOT apply accent color to plain text, table cells, or non-interactive section headers. Section group headers in the question list (SEC-05) use neutral `bg-light` + `fw-semibold`, NOT a colored band.

Semantic state colors (reuse existing, no new hues):
- Success feedback → `alert-success` (e.g. "Section berhasil disimpan.").
- Warning (non-blocking, e.g. backward-compat old-file note, empty-cell warn) → `alert-warning`.
- Error / hard-block (D-13 mismatch, import errors) → `alert-danger`.

**Interaction contracts (minimal JS, per D-415-02 "JS minim"):**
- Section reorder = edit the `No. Section` integer field then save; NO drag-drop, NO SortableJS.
- Assign question→Section = pick from the `Section` dropdown in the existing create/edit-question form; NO bulk-assign, NO drag.
- "Semua Section mulai halaman baru" quick button = one POST (or sets all StartNewPage toggles then saves); confirm not required (non-destructive, reversible).
- Section panel lives in the LEFT/top region of `ManagePackageQuestions.cshtml` as an additional `card shadow-sm` so HC sees Sections + questions together (D-415-01); reuse the existing two-column `row g-4` rhythm, do not restructure the page.
- Import error list and result alerts render on the EXISTING `ImportPackageQuestions.cshtml` post-back (server-rendered alert blocks), consistent with current TempData alert pattern — no new client-side preview is required for 415.
