## Summary

Perubahan 415.1 mengganti guard kepemilikan WR-02 di `AssessmentAdminController.SubmitEssayScore`
(`Controllers/AssessmentAdminController.cs:4213-4222`) dari ownership-by-package
(`q.AssessmentPackage.AssessmentSessionId == sessionId`, yang false-positive karena paket di-pool
lintas sesi-sibling by design) menjadi cek keanggotaan assignment
(`questionId ∈ UserPackageAssignment.GetShuffledQuestionIds()` milik `sessionId`).

**Verdict inti: BENAR & AMAN.** Predikat baru:
- Menerima soal cross-package-tapi-ter-assign (assignment ADA + memuat `questionId`) — tujuan fix tercapai.
- Menolak tamper asli (assignment ADA tapi TIDAK memuat `questionId`) — invarian anti-tamper utama tetap terjaga.
- Sumber-kebenaran IDENTIK dengan GET `EssayGrading` loader (`:4134-4142`) → konsistensi GET↔POST terjamin (tak ada soal yang bisa diketik tapi ditolak simpan, atau sebaliknya).
- `.Contains()` berjalan IN-MEMORY pada entitas yang sudah dimaterialisasi (`FirstOrDefaultAsync` dulu) — PITFALL-2 dipatuhi, bukan LINQ-to-SQL.
- Tidak ada null-deref (`assignment != null` short-circuit; helper tak pernah return null — catch `JsonException` → list kosong).
- Guard berantai lain UTUH dan terurut SEBELUM WR-02: status PendingGrading (`:4195`), question-null (`:4200`), range 0..ScoreValue (`:4202`), tipe-Essay WR-01 (`:4211`). RBAC `[Authorize(Roles="Admin, HC")]` + `[ValidateAntiForgeryToken]` tetap ada (`:4185-4186`).
- Test replica `RunSubmitEssayScoreLogicAsync` (`PostLisensorPolishTests.cs:145-148`) byte-identik dengan controller; 4 skenario (cross-package SUCCESS, genuine-tamper reject, null-allow, regresi tipe/status/range) hijau.

Tidak ada celah di mana HC bisa merusak skor **sesi lain**: upsert (`:4226-4243`) selalu mem-key pada
`(AssessmentSessionId == sessionId, PackageQuestionId == questionId)` — target tulis selalu `sessionId`

## Low

### LO-01: Cabang D-02 (assignment null) melonggarkan invarian kepemilikan per-soal pada data degenerate

**File:** `Controllers/AssessmentAdminController.cs:4219-4222`
**Issue:**
Saat `UserPackageAssignment` untuk `sessionId` TIDAK ditemukan (`assignment == null`), cek keanggotaan
di-skip total → `questionId` ber-tipe Essay APA PUN dapat dinilai terhadap sesi tersebut, termasuk soal
dari paket yang tak ada hubungannya dengan sesi itu. Guard lama (ownership-by-package) memblokir kasus
ini; predikat baru tidak.

Severity = **LOW** (bukan blocker) karena pagar berlapis membatasi dampak:
- Hanya dapat dipicu role tepercaya `Admin, HC` (RBAC `:4185`) — bukan worker / anonim.
- Tulisan tetap ter-scope ke `sessionId` saat ini (`:4232`) → **tidak bisa mengorupsi sesi lain**.
- Hanya soal tipe-Essay (WR-01 `:4211`), hanya saat PendingGrading (`:4195`), skor terbatas
  `0..question.ScoreValue` (`:4202`).
- Kasus null sungguh degenerate: peserta yang benar mengerjakan PASTI punya UPA
  (StartExam/eager/inject semua membuatnya).

Ini adalah **tradeoff D-02 yang disengaja & terdokumentasi** (CONTEXT.md D-02; komentar `:4217-4218`),
bukan bug. Dicatat untuk transparansi audit — invarian "essay-score hanya untuk soal yang ter-assign ke
sesi ini" melemah pada edge-case null, ditukar dengan tidak menciptakan dead-end baru bagi HC pada data
rusak.

**Fix (OPSIONAL — terima as-designed):** Bila ingin mengetatkan tanpa mengembalikan dead-end, ganti
allow-on-null menjadi fallback yang masih memeriksa kepemilikan via paket sesi, mis.:
```csharp
if (assignment != null)
{
    if (!assignment.GetShuffledQuestionIds().Contains(questionId))
        return Json(new { success = false, message = "Soal bukan milik sesi ini." });
}
else
{
    // Fallback degenerate: soal minimal harus tipe-Essay milik paket yang ter-link ke sesi ini.
    var belongs = await _context.PackageUserResponses
        .AnyAsync(r => r.AssessmentSessionId == sessionId && r.PackageQuestionId == questionId);
    if (!belongs)
        return Json(new { success = false, message = "Soal bukan milik sesi ini." });
}
```
Namun ini menambah kompleksitas demi kasus yang hampir tak terjadi; **rekomendasi: pertahankan D-02
apa adanya** dan cukup dokumentasikan (sudah dilakukan via komentar). Tak perlu re-ship.
