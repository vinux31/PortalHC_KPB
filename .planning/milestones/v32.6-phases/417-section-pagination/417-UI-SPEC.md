# Phase 417 — UI Design Contract (Section Pagination)

> Visual and interaction contract for the section-aware exam pagination render. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Brownfield .NET MVC (Razor) + Bootstrap 5 project — NOT a greenfield React/Tailwind app.** This phase generalizes the EXISTING flat 10-per-page pagination in `Views/CMP/StartExam.cshtml` to be section-aware. All contracts below REUSE the existing Bootstrap design system and the patterns already present in that view. No new design system, component library, icon set, or token scale is introduced.

---

## Scope of this contract

Surfaces touched by Phase 417 (PAG-01, PAG-02, PAG-03):

| Surface | File | Change |
|---------|------|--------|
| Exam render (peserta) | `Views/CMP/StartExam.cshtml` | Section header on section change + `(lanjutan)` on auto-split + page indicator with section name + resume toast |
| Question navigator/palette (desktop sidebar) | `Views/CMP/StartExam.cshtml` `#panelNumbers` | Group number badges per-Section with section label (flat for no-Section) |
| Question navigator (mobile drawer) | `Views/CMP/StartExam.cshtml` `#drawerNumbers` | Same per-Section grouping, synced from desktop |
| Admin quick-button | Kelola Section surface (Phase 415, `ManagePackageQuestions`/`ManagePackageSections`) | Wording + placement of "Semua section mulai halaman baru" button (sets `StartNewPage=true` for all sections) |

**Backward-compat invariant (D-416-04, carried):** assessment with NO sections (all `SectionId=null`) renders the IDENTICAL old flat pagination — no section header, flat `1..N` navigator, page indicator without section label. Section complexity appears ONLY when HC deliberately uses sections.

OUT (other phases): scoped shuffle (416 done), dynamic options A–F (418), export section labels (419/PAG-04).

## Spacing Scale

REUSE existing Bootstrap utility spacing already in `StartExam.cshtml`. Bootstrap's scale is a multiple-of-4 system (`spacer = 16px`; `1=4px, 2=8px, 3=16px, 4=24px, 5=48px`). All multiples of 4.

| Token | Value | Bootstrap util | Usage in this phase |
|-------|-------|----------------|---------------------|
| xs | 4px | `gap-1` / `p-1` | Navigator badge gap (existing `gap-2` = 8px kept) |
| sm | 8px | `mb-2` / `gap-2` / `me-2` | Section header bottom margin within group; navigator grid gap (existing `6px`/`8px`) |
| md | 16px | `mb-3` / `p-3` | Card spacing (existing `card mb-3`), page padding |
| lg | 24px | `mb-4` / `p-4` | Page-navigation block bottom margin (existing `mt-3 mb-4`) |
| xl | 48px | `py-5` | Empty/large block padding (existing) |

Exceptions:
- Mobile touch targets: `list-group-item min-height: 48px` and `#mobileFooter .btn min-height: 48px` (existing rule at `StartExam.cshtml` ~L1580). Section header must NOT shrink these.
- Navigator badge fixed box `36×32px` (existing `#panelNumbers .badge`). Section group label sits ABOVE each badge group, full width, not inside the grid.

## Typography

REUSE existing Bootstrap type utilities already in the view. No new sizes introduced. Three roles are relevant to this phase's new elements.

| Role | Element | Size | Weight | Bootstrap util |
|------|---------|------|--------|----------------|
| Question body | Question text + options | ~16px (1rem, body default) | regular 400 | (default), `fw-bold` only on number badge line |
| Section header | Section name above its question group | ~18px (`fs-6`→`h6`-scale, 1.1–1.15rem) | semibold 600 | `h6 fw-semibold` OR `fw-bold text-primary` (match existing `#examHeader h6 fw-bold text-primary`) |
| Section header continuation marker | `(lanjutan)` suffix | ~14px (0.875rem) | regular 400, muted | `text-muted small fw-normal` appended to header |
| Navigator group label | Section name above badge group | ~14px (0.875rem) | semibold 600 | `small text-muted fw-bold` (match existing `small text-muted fw-bold` "Daftar Soal") |
| Page indicator | `<Nama Section> — Halaman 2/5` | ~14px (0.875rem) | regular/semibold | `small` or `text-muted small` near page nav |

Line heights: Bootstrap defaults — body `line-height: 1.5`; headings `line-height: 1.2`. Do not override.

**Weight discipline:** exactly two weights in new elements — regular (400) for body/marker/page-count, semibold/bold (600/700) for section headers and group labels. Matches the existing view (it already uses only regular + `fw-bold`/`fw-semibold`).

## Color

REUSE existing Bootstrap theme colors already in `StartExam.cshtml`. No new palette. The 60/30/10 split maps to the existing exam page.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` / `bg-white` + page neutral background | Exam page surface, card bodies (`card border-0`) |
| Secondary (30%) | Bootstrap `secondary` (`#6c757d`) + light surfaces | Sticky header (`bg-white shadow-sm`), navigator sidebar card, unanswered nav badges (`bg-secondary`), muted text, section header text when using `text-muted` |
| Accent (10%) | Bootstrap `primary` (`#0d6efd`) | Section header text (`text-primary`, matching existing title), current/Next page button (`btn-primary`), question number badge (`badge bg-primary`) |
| Success (semantic) | Bootstrap `success` (`#198754`) | Answered nav badges (`bg-success`) — existing; Submit button (`btn-success`) — existing |
| Warning (semantic) | Bootstrap `warning` | Resume toast (`text-bg-warning` / `toast align-items-center text-bg-warning`) — REUSE existing `showResumeFailureToast` styling |
| Destructive | Bootstrap `danger` (`#dc3545`) | "Keluar Ujian" button (`btn-outline-danger`) — existing, NOT introduced by this phase |

Accent (`primary`) reserved for: section header text, the question-number badge, the active/Next page-navigation button, and the answered-state is `success` (not accent). Do NOT color the entire navigator grid or section headers in accent beyond text — keep the 10% discipline.

**Section header is text-only, not a colored band.** Use `text-primary fw-semibold` (or `fw-bold`) text — do NOT add a full-width colored background bar (would break the 60/30/10 balance and the calm exam surface). A thin `border-bottom` divider under the header is acceptable (neutral border, e.g. `border-bottom pb-1`).

**Resume toast color note:** the existing `showResumeFailureToast` uses `text-bg-warning` (failure semantics). The new D-417-06 resume toast is INFORMATIONAL ("Lanjut dari soal no. X"), not a failure. Reuse the same toast MECHANISM/markup pattern but it MAY use `text-bg-info` or `text-bg-primary` for informational tone. Planner/executor decides; do not invent a new toast container — reuse `#resumeToastContainer` + `bootstrap.Toast`.

## Interaction & Layout Contracts

These are the load-bearing interaction contracts the executor must honor (specific, not exploratory).

### IC-1 — Section header placement (D-417-01, §7.1)
- Header renders ABOVE the first question of each section, at the point the section changes (MAY appear mid-page in default flow mode — this is allowed by §7.1).
- Name only, no "Section N:" prefix. Text style `text-primary fw-semibold`, optional `border-bottom pb-1 mb-2`.
- For no-Section assessments: NO header rendered (flat behavior identical to today).

### IC-2 — Auto-split continuation (D-417-02, §7.1)
- When a section longer than `questionsPerPage` auto-splits across pages, REPEAT the section header on each continuation page with the `(lanjutan)` suffix (`text-muted small fw-normal`).
- The first page of the section shows the plain header (no `(lanjutan)`).

### IC-3 — Page-break before StartNewPage section (PAG-02, §7.1, §7.2)
- A section with `StartNewPage=true` begins a NEW `exam-page` div (page break BEFORE it). Other sections unaffected.
- Page number is computed at render time (controller iterates ordered questions; increment page when (a) `StartNewPage=true` for the question's section, or (b) current page already holds `questionsPerPage` questions). Page number is NEVER persisted per-question (D-11).
- "Lainnya" group (SectionId=null) has no toggle, never forces a page-break except when the page is full (§15.A).

### IC-4 — Navigator per-section grouping (D-417-03, §15.E)
- `#panelNumbers` (desktop) and `#drawerNumbers` (mobile): render a section label, then that section's number badges, then the next section label, etc.
- Each badge keeps existing styling: `badge rounded-pill`, `bg-success` (answered) / `bg-secondary` (unanswered), `36×32px` box, click → `jumpToQuestion(questionId, pageNumber)`.
- Group label is a full-width row (`grid-column: 1/-1` or a separate flex/block row) — it MUST NOT consume a badge grid cell or misalign the 7-column grid.
- DisplayNumber is GLOBAL `1..N`, NOT reset per section (§15.A). The grouping is visual only.
- No-Section assessment: flat grid exactly as today (no group labels).

### IC-5 — Page indicator (D-417-04)
- Shows active section name + page: `{Nama Section} — Halaman {n}/{total}`. No-Section: `Halaman {n}/{total}`.
- Uses section name only (consistent with IC-1), never "Section N:".
- Placement: near existing page navigation / mobile footer; `small`/`text-muted` style. Updates on `changePage()`.

### IC-6 — Resume toast (D-417-06)
- On resume to a computed page > 0, show informational toast `Lanjut dari soal no. {X}` (X = global DisplayNumber of first question on the resumed page).
- REUSE the existing `showResumeFailureToast` pattern/container (`#resumeToastContainer`, `bootstrap.Toast`, `position-fixed bottom-0 end-0 p-3`, `z-index:1100`, auto-hide 6000ms). Do NOT build a new toast mechanism.
- Page 0 resume (or fresh open) → no toast.

### IC-7 — Mobile parity (§15.E, CONTEXT mobile rule)
- Mobile = 5 questions/page (`ViewBag.QuestionsPerPage`, existing). Same section rules apply: header, `(lanjutan)`, `StartNewPage` break, navigator grouping in the offcanvas drawer.
- Existing mobile touch-target minimums (48px) preserved; section header must not push interactive controls below the fold awkwardly — keep header compact.

### IC-8 — Autosave flush on cross-section page change (§15.E)
- When changing page across a section boundary, pending SignalR autosaves (essay/MA) MUST flush — preserve the existing cross-page flush behavior (`assessment-hub.js`, guard in `changePage()`). This phase must not regress it.

### IC-9 — Admin quick-button (§15.D, Claude discretion)
- `Semua section mulai halaman baru` button on the Kelola Section surface sets `StartNewPage=true` for ALL sections.
- Standard Bootstrap button styling consistent with that surface (recommend `btn btn-outline-primary btn-sm` with `bi bi-file-earmark-break` or similar icon). Placement: near the section list header / toolbar. Confirm-on-click not required (non-destructive, reversible per-section toggle).
