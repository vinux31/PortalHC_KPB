**NAV-03 — Guide Pages:**
1. Review `Guide()` action:
   - Verify it passes UserRole to the view correctly.
   - Check if Guide.cshtml conditionally shows/hides module cards based on role.
2. Review `GuideDetail(string module)` action:
   - Valid modules: "cmp", "cdp", "account", "data", "admin".
   - Admin modules ("data", "admin") restricted to Admin/HC — verify redirect logic.
   - Check: What happens with null or empty module param? Should return redirect to Guide.
   - Check: Is module case-sensitive? If URL has "CMP" instead of "cmp", does it 404?
3. Review `Views/Home/Guide.cshtml`:
   - Verify module card links pass correct module parameter.
   - Verify role-conditional display matches GuideDetail role gating (so users don't see cards they can't access).
4. Review `Views/Home/GuideDetail.cshtml`:
   - Check for broken links or references to non-existent pages.
   - Verify content renders correctly for each module value.

**NAV-04 — Navigation Link Integrity:**
5. Audit ALL links in `_Layout.cshtml`:
   - `asp-controller="Home" asp-action="Index"` — brand link
   - `asp-controller="CMP" asp-action="Index"` — CMP hub
   - `asp-controller="CDP" asp-action="Index"` — CDP hub
   - `asp-controller="Home" asp-action="Guide"` — Panduan
   - `asp-controller="Admin" asp-action="Index"` — Kelola Data
   - `asp-controller="Account" asp-action="Profile"` — Profile dropdown
   - `asp-controller="Account" asp-action="Settings"` — Settings dropdown
   - `asp-controller="Account" asp-action="Logout"` — Logout form
   - Verify each controller has the corresponding action method.

6. Audit hub page links (CMP/Index, CDP/Index, Admin/Index):
   - Read each hub view and verify every card link points to an existing controller action.
   - Check for hardcoded URLs vs tag helpers (hardcoded may break under path changes).

7. Check for redirect loops:
   - Login redirects to Home/Index — verify no auth loop.
   - AccessDenied page — verify it doesn't itself require a role that triggers another redirect.

Fix any bugs found inline. Document all findings.

Use grep to find all asp-action references and cross-check with controller methods:
