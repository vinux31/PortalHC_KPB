## Objective

Fix bulk renew sehingga setiap AssessmentSession mendapat RenewsTrainingId (atau RenewsSessionId) yang cocok dengan record milik user masing-masing, bukan hanya record dari user pertama.

## Root Cause

1. GET `CreateAssessment` menerima `int? renewTrainingId` — single value. Saat bulk renew mengirim `?renewTrainingId=1&renewTrainingId=4&renewTrainingId=7`, hanya ID pertama (1) yang ter-bind
2. POST loop (baris 1292-1316) assign `model.RenewsTrainingId` (single value) ke semua sessions

## Approach

**Ubah GET handler** untuk menerima `List<int> renewTrainingId` (dan `List<int> renewSessionId`). Build mapping `{UserId → SourceId}` dari database, simpan di hidden fields via ViewBag.

**Ubah POST handler** loop untuk lookup FK per-user dari mapping, bukan dari single model value.

## Tasks

### Task 1: Fix GET handler dan POST loop untuk per-user FK mapping

**Files:** `Controllers/AdminController.cs`, `Views/Admin/CreateAssessment.cshtml`

**GET handler changes (around line 1021):**
1. Change parameter from `int? renewTrainingId` to `List<int>? renewTrainingId`
2. When `renewTrainingId` has multiple values:
   - Query all TrainingRecords by IDs: `_context.TrainingRecords.Where(t => renewTrainingId.Contains(t.Id))`
   - Build `Dictionary<string, int>` mapping `{UserId → TrainingRecord.Id}`
   - Store in `ViewBag.RenewalFkMap` as JSON string
   - Use first record for title/category prefill (same group = same judul)
   - Set `model.RenewsTrainingId = null` (akan di-resolve per-user di POST)
3. When `renewTrainingId` has single value: keep existing behavior (backward compat for single renew link)
4. Same pattern for `renewSessionId` (List<int>)

**View changes (CreateAssessment.cshtml):**
5. Add hidden input `<input type="hidden" name="RenewalFkMap" value="@ViewBag.RenewalFkMap" />` in the form
6. Keep existing hidden inputs for single renewal mode

**POST handler changes (around line 1292-1316):**
7. Add `string? RenewalFkMap` parameter to POST action (or read from Request.Form)
8. If `RenewalFkMap` is not empty, deserialize to `Dictionary<string, int>`
9. In the session creation loop, replace:
   ```csharp
   RenewsSessionId = model.RenewsSessionId,
   RenewsTrainingId = model.RenewsTrainingId
   ```
   With:
   ```csharp
   RenewsSessionId = fkMap != null && fkMap.ContainsKey(userId) && isSessionMap ? fkMap[userId] : model.RenewsSessionId,
   RenewsTrainingId = fkMap != null && fkMap.ContainsKey(userId) && !isSessionMap ? fkMap[userId] : model.RenewsTrainingId
   ```

**Verification:**
- Build must compile with 0 errors
- Bulk renew 3 users → each session gets correct per-user FK
- Single renew link still works (backward compat)
