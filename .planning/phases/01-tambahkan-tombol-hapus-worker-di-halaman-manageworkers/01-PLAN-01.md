# Plan 01: Tambahkan tombol hapus worker di ManageWorkers

<objective>
Menambahkan tombol hapus (delete) di tabel ManageWorkers untuk user yang sudah di-deactivate. Tombol memanggil endpoint DeleteWorker yang sudah ada di AdminController.
</objective>

## Tasks

<task id="1">
<title>Tambahkan tombol hapus dan fungsi confirmDelete di ManageWorkers.cshtml</title>
<read_first>
- Views/Admin/ManageWorkers.cshtml (current view — btn-group pattern di baris 266-292, confirmDeactivate JS di baris 359-362)
- Controllers/AdminController.cs baris 5059-5063 (DeleteWorker endpoint signature)
</read_first>
<action>
Di Views/Admin/ManageWorkers.cshtml, dalam block `else` (user non-aktif, baris 283-291), tambahkan form DeleteWorker di samping form ReactivateWorker di dalam btn-group yang sama.

Form baru (tambahkan SETELAH closing tag `</form>` dari ReactivateWorker, sebelum closing `</div>` btn-group):

```html
<form method="post" action="@Url.Action("DeleteWorker", "Admin")" style="display:inline;" onsubmit="return confirmDelete(@Json.Serialize(user.FullName));">
    @Html.AntiForgeryToken()
    <input type="hidden" name="id" value="@user.Id" />
    <button type="submit" class="btn btn-sm btn-outline-danger" title="Hapus Permanen">
        <i class="bi bi-trash"></i>
    </button>
</form>
```

Tambahkan fungsi JS `confirmDelete` di bagian script, setelah fungsi `confirmDeactivate` (baris 362):

```javascript
function confirmDelete(userName) {
    return confirm(`HAPUS PERMANEN user "${userName}"?\n\nSemua data terkait (assessment, coaching, training) akan dihapus dan TIDAK DAPAT dikembalikan.`);
}
```
</action>
<acceptance_criteria>
- ManageWorkers.cshtml contains `Url.Action("DeleteWorker", "Admin")`
- ManageWorkers.cshtml contains `function confirmDelete(`
- ManageWorkers.cshtml contains `btn-outline-danger`
- ManageWorkers.cshtml contains `bi-trash`
- Delete form appears inside the `else` block (non-active users only)
- Delete form includes `@Html.AntiForgeryToken()`
- Delete form includes hidden input with `name="id"`
</acceptance_criteria>
</task>
