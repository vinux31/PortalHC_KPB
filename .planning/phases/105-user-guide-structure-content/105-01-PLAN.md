## Requirements
- GUIDE-STYLE-01: Guide page uses CSS variables matching home.css

## Problem Summary
The existing implementation references CSS classes that don't exist in guide.css:
1. Classes `btn-cdp`, `btn-account`, `btn-data`, `btn-admin` used in GuideDetail.cshtml but undefined
2. These classes appear on accordion buttons but serve no functional purpose
3. Inconsistent class usage creates potential styling issues

## Solution
Add missing CSS class definitions to guide.css with appropriate styling matching the existing design system.

## Tasks

### Task 1.1: Define Missing Button Classes
**Files:**
- `wwwroot/css/guide.css`

**Action:**
Add CSS class definitions for `btn-cdp`, `btn-account`, `btn-data`, `btn-admin` in guide.css after the existing `.icon-*` classes (around line 233 in guide.css). Each class should match the gradient of its corresponding icon class:
- `btn-cdp` → uses `--gradient-success` (teal/green)
- `btn-account` → uses `--gradient-info` (blue)
- `btn-data` → uses `--gradient-orange` (orange)
- `btn-admin` → uses `--gradient-warning` (pink/red)

Classes should only apply to accordion buttons in guide detail pages. Minimal styling: can inherit most properties from `.guide-list-btn`. Add subtle accent color differentiation if needed.

**Code to Add:**

**Verify:**
