<name>Task 2: CSS Performance Optimization</name>
<files>wwwroot/css/guide.css</files>
<action>
1. Open wwwroot/css/guide.css and audit for redundant selectors:
   - Search for duplicate property declarations using grep or manual review
   - Look for repeated color values that could be CSS variables: `#667eea`, `#764ba2`, etc.

     * Check for duplicate `:hover` states on same selector

   - Avoid overly specific selectors like `div.guide-module-card div.header` - use `.guide-card-header` instead

   - Remove unnecessary nesting: `.guide-module-card .guide-card-header` can be `.guide-card-header` if unique
3. Consolidate responsive breakpoints:
   - Check if multiple `@media` queries can be merged: search for all `@media (max-width: 768px)` occurrences
   - Group all mobile styles into single `@media (max-width: 768px)` block at end of file
   - Ensure no duplicate breakpoint definitions: `@media (max-width: 768px)` appearing multiple times
4. Remove unused CSS:
   - Search HTML files for class usage: search Guide.cshtml and GuideDetail.cshtml for all class names
   - Identify unused classes in guide.css (classes defined but not used in HTML)
   - Remove any dead code or commented-out styles
   - Verify all gradient variables are actually used: search for `--gradient-primary`, `--gradient-success`, etc.
5. Add CSS compression hints:
   - Add comment `/* Critical CSS */` before above-fold content styles (hero, cards)
   - Consider if file is large enough to split critical vs non-critical CSS (currently 771 lines - acceptable)

6. Specific optimizations to implement:
   - Consolidate duplicate `transition` properties into single declaration

   - Group related selectors: `.guide-card-header, .guide-card-footer { common styles }`

<name>Task 3: AOS Animation Performance Tuning</name>
<files>Views/Home/Guide.cshtml, Views/Home/GuideDetail.cshtml</files>
<action>
1. Open Views/Home/Guide.cshtml and review AOS usage:
   - Search for all `data-aos` attributes to understand current usage
   - Note current animation types: fade-down, fade-up, fade-right
   - Note current delay values: 100, 150, 200, 250, 300ms
2. Optimize AOS initialization in _Layout.cshtml or locally:
   - Locate AOS.init() call (likely in _Layout.cshtml or at bottom of Guide.cshtml)
   - Update to: `AOS.init({ duration: 600, easing: 'ease-out-cubic', once: true, offset: 50, delay: 0 });`
   - Set `once: true` to animate only once (not on every scroll) - improves performance
   - Set `offset: 50` to trigger animations when element is 50px from viewport
3. Review AOS attributes throughout Guide.cshtml:
   - Verify `data-aos-delay` values are staggered appropriately (100, 150, 200, 250, 300ms)
   - Remove excessive delays that feel sluggish (delays > 300ms)
   - Ensure delay pattern is consistent and doesn't cause visible lag
4. Add AOS disable for reduced-motion preference:
   - Add JavaScript before AOS.init(): `if (window.matchMedia('(prefers-reduced-motion: reduce)').matches) { AOS.init({ disable: true }); } else { AOS.init({ duration: 600, easing: 'ease-out-cubic', once: true, offset: 50 }); }`
   - Ensure animations don't run for users who prefer reduced motion
5. Optimize heavily animated sections:
   - Consider using CSS-only animations for simple fades (instead of AOS) for better performance
   - Add `data-aos-anchor-placement="top-bottom"` for better trigger timing on elements
   - Reduce animation duration for faster feel: add `data-aos-duration="500"` for quicker animations
6. Add lazy loading for below-fold content:
   - Add `loading="lazy"` to images if any are added later
   - Consider lazy-loading FAQ section: add `data-aos-delay="300"` to FAQ section to delay animation
   - Ensure above-fold content (hero, cards) loads immediately
7. Apply same optimizations to GuideDetail.cshtml:
   - Review AOS usage in GuideDetail.cshtml
   - Apply same AOS.init() settings
   - Add prefers-reduced-motion check
</action>
<verify>
1. Scroll through Guide page and verify animations feel smooth (60fps)
2. Use Chrome DevTools Performance tab to check for jank: Record performance while scrolling
3. Verify animations only trigger once per element (once: true)
4. Test with prefers-reduced-motion enabled in DevTools - verify animations are disabled
5. Verify animation delays feel responsive, not sluggish
6. Check CPU usage during animations - should be minimal (< 30% on modern devices)
7. Test on low-end devices or throttled CPU mode in DevTools
8. Verify above-fold content animates immediately, below-fold has appropriate delays
</verify>

- [ ] Animations disabled when prefers-reduced-motion is set
