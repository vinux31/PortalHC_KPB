<name>Task 2: Performance Optimization Review</name>
<files>wwwroot/css/guide.css</files>
<action>
1. Review all animations to ensure they only use performant properties:
   - Search for all `transition:` declarations
   - Verify they only animate: transform, opacity, box-shadow, filter
   - Replace any layout property animations: avoid width, height, top, left, margin, padding
2. Add CSS containment for better performance:
   - Add `contain: layout style;` to card containers to isolate reflows
   - Add `contain: layout;` to FAQ items
   - Add `will-change: transform;` only on :hover state (not default state)
3. Optimize keyframe animations:
   - Review all `@keyframes` declarations
   - Ensure they only use transform and opacity
   - Add `transform: translateZ(0);` to trigger GPU acceleration if needed
   - Remove any unnecessary keyframe animations
4. Add reduce-motion support to all animations:
   - Wrap all animation-heavy styles in `@media (prefers-reduced-motion: no-preference)`
   - Add simplified versions for `@media (prefers-reduced-motion: reduce)`
   - Ensure all transitions are disabled or simplified when reduced motion is preferred
5. Remove any performance bottlenecks:
   - Check for large box-shadows that cause repaints
   - Simplify complex gradients if they cause performance issues
   - Remove unused @keyframes animations
6. Add performance hints:
   - Add `content-visibility: auto;` to below-fold sections if browser supports it
   - Use `transform: translate3d(0, 0, 0);` for GPU acceleration on animated elements
</action>
<verify>
1. Use Chrome DevTools Performance tab to record animation performance
2. Verify animations run at 60fps on modern devices
3. Check for layout thrashing in Performance timeline
4. Verify GPU acceleration is used (green layers in DevTools)
5. Test on low-end device or throttled CPU mode
6. Verify reduced motion simplifies or disables all animations
7. Check that no layout properties (width, height, top, left) are animated
</verify>
<done>
- [ ] All animations only use transform, opacity, box-shadow, filter
- [ ] CSS containment added to isolate reflows
- [ ] will-change only used on :hover state
- [ ] GPU acceleration used for keyframe animations (translate3d)
- [ ] All animations respect prefers-reduced-motion
- [ ] No layout property animations (width, height, top, left)
- [ ] Animations run at 60fps on modern devices
</done>
</task>
