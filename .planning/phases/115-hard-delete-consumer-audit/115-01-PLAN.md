**Backend — ProtonDataController.cs:**

Add two new endpoints (follow existing patterns like SilabusDelete):

1. `[HttpGet] GetKompetensiCascadeInfo(int id)` — Load Kompetensi with Include chain (SubKompetensiList -> Deliverables), then count ProtonDeliverableProgresses and CoachingSessions referencing those deliverables. Return JSON: `{ success, nama, subKompetensiCount, deliverableCount, progressCount, sessionCount }`.

2. `[HttpPost][ValidateAntiForgeryToken] DeleteKompetensi([FromBody] request)` — Accept `{ kompetensiId: int }`. Use `Database.BeginTransactionAsync()`. Delete bottom-up in this exact order:
   - CoachingSessions WHERE ProtonDeliverableProgressId IN (progressIds) — no FK constraint, must be explicit
   - ProtonDeliverableProgresses WHERE ProtonDeliverableId IN (delivIds)
   - Deliverables (from loaded tree)
   - SubKompetensi (from loaded tree)
   - Kompetensi itself
   - SaveChanges + CommitAsync. On error: RollbackAsync, return `{ success: false, message: "Gagal menghapus: " + ex.Message }`.

   Note: ActionItems under CoachingSessions have `DeleteBehavior.Cascade` so they auto-delete when sessions are removed.

**Frontend — Views/ProtonData/Index.cshtml:**

1. **Delete button:** In the view-mode Kompetensi row action buttons (around line 427-437 where Nonaktifkan/Aktifkan buttons are built), add after the existing toggle button:
   ```
   <button class="btn btn-sm btn-outline-danger btn-delete-kompetensi ms-1" title="Hapus" data-id="{kompId}" data-name="{kompetensiName}"><i class="bi bi-trash me-1"></i>Hapus</button>
   ```
   Button appears on ALL Kompetensi rows (active and inactive), only in view mode.

2. **Confirmation modal:** Add `kompetensiDeleteModal` following the same pattern as existing `silabusDeleteModal` (around line 157-175). Modal body has:
   - Dynamic text: "Hapus Kompetensi [nama]?"
   - Cascade info: "Ini akan menghapus X Sub Kompetensi, Y Deliverable, Z Progress Coaching, W Sesi Coaching."
   - All counts displayed equally, no special coloring
   - Alert-danger div (hidden by default) for error messages
   - Confirm button (btn-danger) and Cancel button

3. **AJAX handlers:**
   - On `.btn-delete-kompetensi` click: GET `/ProtonData/GetKompetensiCascadeInfo?id={id}`, populate modal body with counts, show modal
   - On confirm button click: POST `/ProtonData/DeleteKompetensi` with `{ kompetensiId }` + antiforgery token, on success hide modal and call `loadSilabusData()`, on error show alert-danger inside modal with error message
