<task type="auto">
  <name>Task 2: Fix CreateAssessment success modal JSON injection bug</name>
  <files>Views/Admin/CreateAssessment.cshtml</files>
  <action>
Read .planning/debug/create-assessment-success-modal.md for full diagnosis.

The bug: @Html.Raw() injects JSON with double quotes into a JS string literal, breaking parsing and preventing the modal from showing.

Fix: Use @Html.Raw(Json.Serialize(...)) or encode the JSON properly for JS consumption. Options:
1. Use a data attribute on a hidden element and read with getAttribute (safest)
2. Use @Json.Serialize() which handles escaping correctly
3. Assign to a JS variable directly: `var data = @Html.Raw(Json.Serialize(ViewBag.SuccessData));`

Choose whichever approach is cleanest. Ensure the success modal displays with correct assessment details after form submission.
