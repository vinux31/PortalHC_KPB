**DEF-01 — Defensive query guard (around line 1420 in CDPController.cs, CoachingProton action):**

Find the query that fetches active ProtonTrackAssignment IDs for the progress join. Modify it to:

1. Group active assignments by (CoacheeId, ProtonTrackId).
2. From each group, select only the one with the maximum Id (or latest AssignedAt).
3. Use those deduplicated assignment IDs for the subsequent ProtonDeliverableProgress query.

Pattern:
```csharp
var assignmentIds = await _context.ProtonTrackAssignments
    .Where(a => a.IsActive && /* existing filters */)
    .GroupBy(a => new { a.CoacheeId, a.ProtonTrackId })
    .Select(g => g.OrderByDescending(a => a.Id).First().Id)
    .ToListAsync();
```

Then use `assignmentIds` in the progress query instead of the unguarded list. This ensures even if duplicates somehow survive, the UI only shows one set of deliverable rows per coachee+track.
  </action>
