<task type="auto">
  <name>Task 1: Add controller actions for delete preview and delete</name>
  <files>Controllers/AdminController.cs</files>
  <action>
Add two new actions to AdminController, placed after the existing CoachCoacheeMappingReactivate action:

1. **CoachCoacheeMappingDeletePreview** (GET, Authorize Roles="Admin, HC"):
   - Takes `int id` parameter
   - Finds the CoachCoacheeMapping by id, returns 404 if not found
   - Returns 400 if mapping is still active (EndDate == null) — deletion only allowed for deactivated mappings
   - Counts ProtonTrackAssignments where CoacheeId == mapping.CoacheeId (both active and inactive for this mapping's coachee)
   - Counts ProtonDeliverableProgress rows linked to those assignments
   - Loads coach name and coachee name via Include or separate query
   - Returns Json with: coacheeName, coachName, assignmentCount, progressCount

2. **CoachCoacheeMappingDelete** (POST, Authorize Roles="Admin, HC"):
   - Takes `int id` parameter
   - Finds the CoachCoacheeMapping by id, returns 404 if not found
   - Returns 400 Json if mapping is still active (EndDate == null) with message "Hanya mapping nonaktif yang dapat dihapus"
   - Loads all ProtonTrackAssignments for the mapping's CoacheeId where the assignments were created during or associated with this mapping
   - Actually: since a coachee has one active mapping at a time, and this mapping is deactivated, delete ALL ProtonTrackAssignments for this coachee (active + inactive) — they all belong to this mapping context
   - For each assignment, delete all ProtonDeliverableProgress rows (RemoveRange)
   - Delete all the ProtonTrackAssignments (RemoveRange)
   - Delete the CoachCoacheeMapping itself (Remove)
   - Call SaveChangesAsync (single implicit transaction)
   - Log via _auditLog.LogAsync with actionType "DeleteMapping", description: "Hapus mapping: Coach {coachName} -> Coachee {coacheeName}, {N} track assignments, {M} progress records deleted", targetId: id, targetType: "CoachCoacheeMapping"
   - Return Json with success: true, message: "Mapping berhasil dihapus"
  </action>
