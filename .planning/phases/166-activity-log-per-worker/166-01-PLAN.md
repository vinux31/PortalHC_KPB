   - Properties: `int Id`, `int SessionId` (FK to AssessmentSession), `string EventType` (enum codes: "started", "page_nav", "disconnected", "reconnected", "submitted"), `string? Detail` (e.g. "Halaman 3"), `DateTime Timestamp`

   - Configure cascade delete in OnModelCreating so logs delete when session is deleted

2. Register `DbSet<ExamActivityLog> ExamActivityLogs` in `Data/ApplicationDbContext.cs`

3. Add EF migration: `dotnet ef migrations add AddExamActivityLog`

4. Update `Hubs/AssessmentHub.cs`:
   - Inject `ApplicationDbContext` via constructor (Hub supports DI)
   - Add `LogPageNav(int sessionId, int pageNumber)` method: creates ExamActivityLog with EventType="page_nav", Detail="Halaman {pageNumber}", Timestamp=DateTime.UtcNow. Fire-and-forget pattern: `_ = Task.Run(async () => { ... SaveChangesAsync ... })` using a NEW scope from IServiceProvider to avoid DbContext threading issues. Actually, better: inject IServiceScopeFactory, create scope inside Task.Run, resolve DbContext from that scope.
   - Override `OnDisconnectedAsync(Exception? exception)`: resolve session from Context.User claim, log "disconnected" event. Use IServiceScopeFactory same pattern. Must call `base.OnDisconnectedAsync(exception)`.
   - Override `OnConnectedAsync()`: log "reconnected" event only if the user has an active (InProgress) session. Use IServiceScopeFactory. Must call `base.OnConnectedAsync()`.
   - For OnDisconnectedAsync/OnConnectedAsync: find the user's active session via `Context.UserIdentifier` (which is the UserId from cookie auth). Query `AssessmentSessions.Where(s => s.UserId == userId && s.Status == "InProgress").Select(s => s.Id).FirstOrDefaultAsync()`. If no active session, skip logging.
