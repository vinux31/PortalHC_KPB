  <action>
Read each controller file completely and audit for these bug categories:

**Category A — Null reference risks:**
- FirstOrDefault/SingleOrDefault results used without null check before accessing properties
- Navigation properties accessed without Include() or null guard
- User claims accessed without null check

**Category B — Silent exception swallowing:**
- catch blocks that are empty or only log without re-throwing or returning an error
- try/catch that returns success even on failure

**Category C — Query logic errors:**
- Wrong LINQ predicates (e.g., comparing wrong fields)
- Missing .Include() causing lazy-load failures (EF Core has no lazy loading by default)
- Queries that could return wrong results due to missing filters (e.g., not filtering by current user)

**Category D — Flow/redirect errors:**
- RedirectToAction pointing to non-existent actions
- Missing return statements after redirects (code continues executing)
- Wrong HTTP method attributes (e.g., GET for state-changing operations)

**Category E — Validation gaps:**
- POST actions that don't validate ModelState
- Missing bounds checks on numeric inputs
- File operations without existence checks

For each bug found:
1. If fix is safe and obvious — fix it immediately
2. If fix has risk of behavioral change — document and defer with justification
3. Record every finding (fixed or deferred) for the SUMMARY

Note: Quick task 26 already fixed critical bugs (open redirect, null Excel crash, silent catches). Focus on remaining issues not covered by that fix.
