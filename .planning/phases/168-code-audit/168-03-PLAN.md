1. Get the list of all .cs files excluding obj/, bin/, and Migrations/ directories.
2. For each .cs file:
   a. Read the file content
   b. Identify all `using` statements at the top
   c. For each using statement, check if ANY type from that namespace is actually used in the file body
   d. Remove using statements where no type from that namespace appears in the file
3. Common patterns to check:
   - `using System.Linq` — needed if .Where, .Select, .FirstOrDefault, .ToList, .Any, .Count, .OrderBy etc. are used
   - `using System.Threading.Tasks` — needed if Task, async are used
   - `using Microsoft.AspNetCore.Mvc` — needed in controllers
   - `using Microsoft.EntityFrameworkCore` — needed if .Include, .ThenInclude, DbContext operations used
   - `using System.ComponentModel.DataAnnotations` — needed if [Required], [StringLength] etc. used
4. Be careful with extension methods — `using System.Linq` is needed even if "Linq" doesn't appear as a type name, because LINQ methods are extension methods.
5. After cleanup, run `dotnet build` to verify nothing breaks. If build fails, restore the removed using that caused it.
6. Count total removals per file for the summary.

This task depends on plans 01 and 02 completing first so we clean imports on the final state of files (not on code that will be deleted or modified).
