**5. POST /ProtonData/ImportSilabus** — accepts `IFormFile? excelFile`, `string? bagian`, `string? unit`, `int? trackId` from form fields. Validates file (non-null, .xlsx/.xls only, max 10MB — same as ImportWorkers). On validation failure, sets TempData["Error"] and returns View().

Parse logic:
- Read worksheet rows, skip row 1 (header). Skip rows where all 6 content columns are blank.
- Columns: col1=Bagian(ignored, use form bagian param), col2=Unit(ignored), col3=Track(ignored), col4=Kompetensi, col5=SubKompetensi, col6=Deliverable, col7=Target.
- Validate: Kompetensi, SubKompetensi, Deliverable must not be empty. Bagian and unit (from form params) must not be empty. trackId must have value. If invalid, add ImportSilabusResult with Status="Error" and skip.
- Upsert logic:
  1. Find or create ProtonKompetensi by (Bagian, Unit, ProtonTrackId, NamaKompetensi). If creating: set Urutan = (max existing Urutan for same Bagian/Unit/Track) + 1, IsActive = true.
  2. Find or create ProtonSubKompetensi by (ProtonKompetensiId, NamaSubKompetensi). If creating: set Urutan = (max existing Urutan for same ProtonKompetensiId) + 1.
  3. Find or create ProtonDeliverable by (ProtonSubKompetensiId, NamaDeliverable). If creating: set Urutan = (max existing Urutan for same ProtonSubKompetensiId) + 1. On both create and find: update Target to the value from the row.
  4. Status = "Created" if deliverable was new, "Updated" if deliverable existed.
- Call _context.SaveChangesAsync() after processing all rows (one batch save).
- Store results in ViewBag.ImportResults. Return View() to show results on same page.
- If bagian/unit/trackId provided and all rows succeeded (no errors), redirect: `return RedirectToAction("Index", new { bagian, unit, trackId, tab = "silabus" })`.
- If any errors exist, return View() with results.

Wrap file processing in try/catch. On exception: set TempData["Error"] = $"Gagal memproses file: {ex.Message}" and return View().
