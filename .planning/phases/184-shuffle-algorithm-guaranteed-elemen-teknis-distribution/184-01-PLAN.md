
3. **Identify distinct ET groups** (non-null ElemenTeknis values across all packages):
   ```csharp
   var etGroups = allQuestions
       .Where(x => !string.IsNullOrWhiteSpace(x.Question.ElemenTeknis))
       .Select(x => x.Question.ElemenTeknis!)
       .Distinct()
       .ToList();
   ```

4. **Phase 1 — Guarantee one question per ET group (best-effort):**
   - For each ET group, collect all candidate questions (from any package) that have that ElemenTeknis value
   - Shuffle candidates, pick the first one
   - Add to `selectedIds` HashSet (to avoid duplicates)
   - Track package usage counts for balanced distribution later
   - If selectedIds.Count >= K after this phase, stop (more ET groups than quota — best-effort)
   - Cap phase 1 selections at K

5. **Phase 2 — Fill remaining quota with balanced package distribution:**
   - Remaining = K - selectedIds.Count
   - Build pool of unselected questions from all packages
   - Use the existing slot-list logic to distribute remaining slots evenly across packages, but only from unselected questions
   - For each slot, pick next unselected question from that package (ordered by q.Order)
   - If a package runs out of unselected questions, redistribute its remaining slots to other packages

6. **Phase 3 — Shuffle the final list:**
   - Combine phase 1 + phase 2 question IDs into a single list
   - Fisher-Yates shuffle the combined list
   - Return shuffled list

7. **Fallback:** If `etGroups` is empty (no questions have ElemenTeknis), fall back to existing slot-list algorithm exactly as before (copy current L1702-1740 logic).

**Edge cases per user decisions:**
- NULL ElemenTeknis questions participate in Phase 2 (fill remaining) but NOT Phase 1 (ET guarantee)

- If K < etGroups.Count, cap Phase 1 at K selections
