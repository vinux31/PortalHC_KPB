## Interaction Contract

### ManageCategories Page Interactions

| Interaction | Trigger | Response |
|-------------|---------|----------|
| Add category | Click "Tambah Kategori" button | Navigate to AddCategory form (or inline modal — at executor discretion, matching ManageWorkers inline-vs-page approach) |
| Edit category | Click "Edit" button in row | Navigate to EditCategory form |
| Delete category | Click "Hapus" button in row | Bootstrap modal confirmation appears; on confirm POST to DeleteCategory |
| Toggle active | Click "Nonaktifkan" / "Aktifkan" | POST to ToggleCategoryActive; page reloads with success alert |
| Success alert | After any POST redirect | `alert alert-success` with `TempData["Success"]` message; auto-dismissible via `btn-close` |
| Error alert | After failed POST redirect | `alert alert-danger` with `TempData["Error"]` message; auto-dismissible |

### Category Dropdown Interaction (CreateAssessment + EditAssessment)

| Interaction | Trigger | Response |
|-------------|---------|----------|
| Auto-fill pass % | User selects category from dropdown | JS reads `data-pass-percentage` attribute from selected `<option>`; if `!passPercentageManuallySet`, writes value to PassPercentage input |
| Manual override | User types in PassPercentage input | Sets `passPercentageManuallySet = true`; subsequent category changes no longer auto-fill |
| Category not in list (EditAssessment orphan) | Model.Category not found in DB categories | Render as disabled `<option>` at bottom with original value so form submits correctly |

### Form Validation States

| State | Visual | Bootstrap class |
|-------|--------|-----------------|
| Valid field | Green border | `:valid` (browser-native with Bootstrap validation) |
| Invalid field | Red border + message | `is-invalid` + `div.invalid-feedback` |
| Required empty | Red border on submit | `required` attribute + Bootstrap validation |
| Duplicate name server error | `ModelState` error displayed inline | `asp-validation-for` tag helper |
