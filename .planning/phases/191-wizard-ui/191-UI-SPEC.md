## Step Indicator Contract

This section covers the visual progress bar — the primary new UI element in this phase.

### Pill States

| State | Visual | Bootstrap Classes |
|-------|--------|-------------------|
| Completed (selesai) | Green pill with checkmark icon | `nav-link bg-success text-white` + `bi-check-circle-fill` icon |
| Active (aktif) | Blue pill with filled circle | `nav-link bg-primary text-white active` |
| Pending (belum) | Grey outlined pill | `nav-link text-muted border` (disabled, not clickable) |
| Visited-but-not-active | Outline blue, clickable | `nav-link text-primary border border-primary` |

### Structure

```html
<nav class="mb-4">
  <ul class="nav nav-pills nav-fill gap-2">
    <li class="nav-item">
      <!-- Step 1: completed or active or pending -->
    </li>
    <!-- x4 steps -->
  </ul>
</nav>
```

- 4 pills, horizontal, full-width (`nav-fill`)
- Each pill: step number icon + step label text
- Step labels (Indonesian): "1. Kategori", "2. Users", "3. Settings", "4. Konfirmasi"
- Completed step icon: `bi-check-circle-fill me-1`
- Active step icon: `bi-circle-fill me-1` (filled dot)
- Pending step icon: `bi-circle me-1` (outline dot)
- Clickable only for steps with state = visited-but-not-active or completed

## Step Panel Contract

### General Rules
- All 4 step panels exist in the DOM simultaneously inside a single `<form>`
- Non-active panels hidden via `d-none` Bootstrap class
- Transition: instant show/hide (no CSS animation — consistent with existing page patterns; Claude's discretion per CONTEXT.md)
- Each panel wrapped in `<div id="step-{n}" class="step-panel">` — show/hide toggled by JS

### Step 1 — Kategori
Fields (in order):
1. Category — `form-select` — required — triggers Proton conditional display
2. Title — `form-control` — required — `<input type="text">`
3. Track dropdown — `form-select` — conditional: visible only when Assessment Proton selected; hidden otherwise

Layout: single column, `row g-3`, each field `col-12` or `col-md-8` max width.

### Step 2 — Users
Layout:
- Filter bar: Section filter (`form-select`, inline) + Search (`form-control`, inline) — `row g-2 mb-3`
- User list: scrollable checkbox list in a `border rounded` container, max-height 320px, `overflow-y: auto`
- Each user row: `form-check` with `form-check-input` + `form-check-label`
- Minimum 1 selection required to proceed (validated on "Selanjutnya" click)
- Proton mode: same layout but coachee list replaces normal user list; section filter hidden

### Step 3 — Settings
Field layout (two-column grid `row g-3`):

| Column A (`col-md-6`) | Column B (`col-md-6`) |
|-----------------------|-----------------------|
| Schedule date (`input[type=date]`) | Schedule time (`input[type=time]`) |
| Duration (minutes) (`input[type=number]`) | Status (`form-select`) |
| PassPercentage (`input[type=number]`) | Token toggle (`form-check form-switch`) |
| Shuffle questions toggle (`form-check form-switch`) | Shuffle options toggle (`form-check form-switch`) |
| ExamWindowCloseDate (`input[type=date]`, optional, `col-md-6`) | ValidUntil (`input[type=date]`, optional, min=today, `col-md-6`) |

ValidUntil label: "Tanggal Expired Sertifikat (opsional)"
ValidUntil helper text: `<div class="form-text">Kosongkan jika sertifikat tidak memiliki batas waktu.</div>`

### Step 4 — Konfirmasi
Three Bootstrap cards stacked vertically, `mb-3` between each:

**Card 1 — Kategori & Judul**
- Card header: "Kategori & Judul" + "Edit" link (right-aligned, `btn btn-sm btn-outline-secondary ms-auto`)
- Card body: read-only display of Category value + Title value

**Card 2 — Peserta**
- Card header: "Peserta" + "Edit" link
- Card body: comma-separated names, first 5 shown
- If >5 selected: show "...dan N lainnya" as a `<button type="button" class="btn btn-link btn-sm p-0">` that expands remaining names inline
- Count badge: `<span class="badge bg-secondary ms-2">{N} peserta</span>` in card header

**Card 3 — Settings**
- Card header: "Settings" + "Edit" link
- Card body: key-value pairs in `<dl class="row mb-0">` — Schedule, Duration, Status, Token, PassPercentage, ValidUntil (show "Tidak ada" if empty)

**Submit button:**
```html
<button type="submit" class="btn btn-success btn-lg w-100 mt-3">
  <i class="bi bi-check-circle-fill me-2"></i>Buat Assessment
</button>
```

**"Kembali ke Konfirmasi" button** (shown when editing from Step 4):
```html
<button type="button" id="btnBackToConfirm" class="btn btn-outline-primary mt-2 d-none">
  <i class="bi bi-arrow-left me-1"></i>Kembali ke Konfirmasi
</button>
```

## Validation Contract

### Inline Validation Trigger
- Triggered on "Selanjutnya" button click, not on blur
- Invalid fields: add `is-invalid` class to `form-control` / `form-select`
- Error message: `<div class="invalid-feedback">` immediately after the input

### Per-Step Required Fields

| Step | Required | Validation Message |
|------|----------|--------------------|
| Step 1 | Category, Title | "Pilih kategori assessment." / "Judul assessment wajib diisi." |
| Step 2 | At least 1 user | "Pilih minimal 1 peserta untuk melanjutkan." — shown as `alert alert-danger` above list, not inline |
| Step 3 | Schedule date, Schedule time, Duration, Status | "Tanggal jadwal wajib diisi." / "Waktu jadwal wajib diisi." / "Durasi wajib diisi." / "Status wajib dipilih." |
| Step 4 | None — read-only | n/a |

"Selanjutnya" button: disabled state (`disabled` attribute) set by JS while current step has invalid required fields. Enabled when all required fields in current step are filled.
