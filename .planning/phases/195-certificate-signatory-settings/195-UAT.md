## Tests

### 1. Create Parent & Sub-Category
expected: Go to Admin > Kelola Data > Manajemen Kategori. Add a new parent category (e.g. "Mandatory HSSE Training"). Then add a sub-category under it by selecting the parent from the "Kategori Induk" dropdown. The table shows the sub-category indented with an arrow icon under its parent, and the parent shows a "Parent" badge.
result: pass

### 2. Signatory Dropdown & P-Sign Preview
expected: In ManageCategories, edit a category. The signatory dropdown is searchable (tom-select). Select a user as signatory — a P-Sign preview box appears immediately showing the Pertamina logo, position, and name.
result: pass

### 3. Delete Category With Children Blocked
expected: Try to delete a parent category that has sub-categories. The delete button should be disabled with a tooltip saying "Hapus sub-kategori terlebih dahulu".
result: pass

### 4. CreateAssessment Optgroup Dropdown
expected: Go to CMP > create a new assessment. The category dropdown groups sub-categories under their parent using optgroup headers. Parent categories without children also appear.
result: pass

### 5. EditAssessment Pre-Selection
expected: Edit an existing assessment that belongs to a sub-category. The category dropdown shows optgroup grouping and the current category is pre-selected correctly.
result: pass

### 6. Certificate Shows Dynamic Signatory
expected: View a certificate for an assessment whose category has an assigned signatory. The certificate header shows the Pertamina logo + "HC PORTAL KPB". The footer shows a P-Sign badge with the signatory's position and name (no QR code).
result: pass

### 7. Certificate Signatory Fallback
expected: View a certificate for an assessment whose category has NO assigned signatory. The footer P-Sign falls back to showing "HC Manager" as the position.
result: pass

### 8. PDF Certificate Download
expected: Download the PDF version of a certificate. It shows the same Pertamina logo header and P-Sign footer with the correct signatory (or HC Manager fallback). No QR code present.
result: pass
