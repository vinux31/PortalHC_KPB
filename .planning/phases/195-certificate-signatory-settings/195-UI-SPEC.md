## Spacing Scale

Declared values (Bootstrap 5 utility scale — project-wide convention):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (gap-1) | Icon gaps, tight inline spacing |
| sm | 8px (gap-2 / p-2) | Compact element spacing, btn-group gaps |
| md | 16px (p-3) | Default cell padding, form field spacing |
| lg | 24px (mb-4) | Section breaks between card and header |
| xl | 32px (py-4) | Page-level container top/bottom padding |
| 2xl | 48px (py-5) | Empty state vertical centering |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Certificate footer margin-top: 60px (certificate layout — matches existing cert-preview-A2.html)
- Certificate P-Sign margin-right: 32px (nearest multiple of 4 — was 30px in mockup, adjusted to grid)
- Certificate score badge: positioned 40px from bottom-right (matches existing badge-score)
- Certificate logo-section gap: 16px (nearest multiple of 4 — was 15px in mockup, adjusted to grid)
- P-Sign on certificate: logo height 48px, psign-label font 0.85rem, psign-name font 0.9rem (per CONTEXT.md cert Design A2)

## Typography

### Admin UI (ManageCategories, forms)

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / table cell | 14px (Bootstrap default small) | 400 | 1.5 |
| Label / form-label | 14px + fw-bold | 700 | 1.5 |
| Section heading (h2) | 20px | 700 (fw-bold) | 1.2 |
| Badge / small hint | 12px | 400 | 1.3 |

Two weights only: 400 (body, badge) and 700 (labels, headings). fw-semibold (600) is not used.

### Certificate (Certificate.cshtml)

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Recipient name / Certificate title | 42px | 700 italic (Playfair) | 1.0 |
| Course title / assessment name | 28px | 700 | 1.0 |
| Body text (subtitle, achievement text) / Logo text "HC PORTAL KPB" | 18px | 400 / 700 | 1.6 / 1.3 |
| Footer labels (date, NIP, serial) / Logo subtitle | 12px | 400 | 1.3 |

Four sizes only: 42px, 28px, 18px, 12px. The certificate title ("Certificate of Completion") uses 42px — same as recipient name. Logo text "HC PORTAL KPB" uses 18px (collapsed from 22px). Footer date labels and logo subtitle both use 12px (collapsed from 14px footer labels).

Certificate fonts: Playfair Display for headings and recipient name; Lato for all body, labels, and P-Sign text.

## Component Inventory

### New Components (this phase)

#### 1. ManageCategories — Indented Tree Table

Parent rows render normally. Child rows render with visual indent:
- `ps-4` (16px left padding) on the Nama Kategori cell for child rows
- Prefix child row name cell with `<i class="bi bi-arrow-return-right text-muted me-2 small"></i>` (no extra column)
- Parent row Nama Kategori cell: add `<span class="badge bg-light text-secondary border ms-2 small">Parent</span>` when the category has children
- Grandchild rows (depth 2): `ps-5` (32px) left padding + same arrow icon

#### 2. ManageCategories — Add/Edit Form: Parent Category Dropdown

- Rendered as `<select class="form-select" name="parentId">` with a blank first option ("— Tidak ada (Kategori Utama) —")
- Shows only categories at depth 0 or 1 (never shows grandchildren as selectable parents)
- If category already has children, it appears in the dropdown as a potential parent
- Hint text below: `<div class="form-text">Kosongkan untuk membuat kategori utama.</div>`

#### 3. ManageCategories — Add/Edit Form: Signatory Dropdown

- Rendered as `<select class="form-select" id="signatorySelect" name="signatoryUserId">` with blank first option ("— Tidak ada —")
- All users listed (no role filter), format: "Nama Lengkap (NIP)" per option
- Searchable via tom-select library (lightweight, no CDN external call — load from wwwroot/lib or CDN with SRI)
- On change: JavaScript updates the P-Sign preview block below the dropdown

#### 4. ManageCategories — P-Sign Preview (inline, below signatory dropdown)

Visible only when a signatory is selected. Hidden (`d-none`) when blank.

```html
<div id="psignPreview" class="mt-3 d-none">
  <div class="text-muted small mb-1">Preview P-Sign pada sertifikat:</div>
  <div class="psign-badge-cert" style="display:inline-block; text-align:center; background:transparent;">
    <img id="psignPreviewLogo" src="/images/psign-pertamina.png" style="height:48px; margin-bottom:6px;" />
    <div id="psignPreviewPosition" style="font-size:0.85rem; color:#333; line-height:1.3;"></div>
    <div id="psignPreviewName" style="font-size:0.9rem; font-weight:700; color:#000; line-height:1.3; margin-top:4px;"></div>
  </div>
</div>
```

JavaScript reads `data-position` and `data-fullname` attributes from each `<option>` element in the signatory dropdown and populates the preview on `change`.

#### 7. Wizard Category Dropdown — optgroup

In CreateAssessment.cshtml and EditAssessment.cshtml, the flat `<select>` for categories becomes:

```html
<select name="categoryId" class="form-select" required>
  <option value="">-- Pilih Kategori --</option>
  @foreach (var parent in ViewBag.ParentCategories) {
    <optgroup label="@parent.Name">
      <option value="@parent.Id">@parent.Name (Induk)</option>
      @foreach (var child in parent.Children) {
        <option value="@child.Id">&nbsp;&nbsp;@child.Name</option>
      }
    </optgroup>
    @* Categories with no children render as direct option outside optgroup *@
  }
</select>
```

Both parent and child options are selectable. Categories with no children render as a standalone `<option>` (not inside `<optgroup>`).

## Interaction States

### Delete Category — blocked state

When a category has children, the delete button is replaced with a disabled button + tooltip:

```html
<button type="button" class="btn btn-sm btn-danger" disabled
        aria-label="Hapus kategori"
        title="Hapus sub-kategori terlebih dahulu sebelum menghapus kategori ini.">
  <i class="bi bi-trash"></i>
</button>
```

No modal shown for blocked delete. Bootstrap `title` attribute provides the reason.

### Delete Category — normal state (no children)

Existing modal pattern unchanged. Modal body updated to:

> Apakah Anda yakin ingin menghapus kategori "**{name}**"?
> Tindakan ini tidak dapat dibatalkan.

(Remove existing footnote about assessment data being unaffected — it was misleading; category data IS used in session display.)

Active delete button also carries `aria-label="Hapus kategori"` for accessibility.

### Signatory Preview — no signatory selected

`#psignPreview` div has class `d-none`. No preview shown.

### Signatory Preview — signatory selected

`d-none` removed. Preview populates from `data-position` and `data-fullname` on the selected `<option>`.

## Registry Safety

| Registry | Blocks Used | Safety Gate |
|----------|-------------|-------------|
| shadcn official | none | not applicable |
| tom-select (searchable dropdown) | tom-select v2 (CDN or local) | No external registry — standard npm/CDN library. No vetting gate required. Verify SRI hash if loaded from CDN. |

tom-select is a well-established MIT-licensed library (formerly Selectize successor). Load from CDN with integrity attribute, or download to `wwwroot/lib/tom-select/`. No third-party shadcn registry involved.

If tom-select adds complexity: fallback is native `<select>` with `size` attribute for visibility. Native is acceptable — CONTEXT.md marks searchable dropdown as Claude's discretion.
