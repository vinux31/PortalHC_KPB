**Method implementation rules (CRITICAL — merge superset from both controllers):**

1. **GetUnifiedRecords(string userId)**: Use CMP version (lines 1037-1087) as base because it includes `AssessmentSessionId = a.Id` and `GenerateCertificate = a.GenerateCertificate` in the assessment mapping that Admin version lacks. Logic is otherwise identical.

2. **GetAllWorkersHistory()**: Use CMP version (lines 1090-1173) as base because it includes `WorkerId = h.UserId` and `WorkerId = a.UserId` fields that Admin version omits. Logic is otherwise identical.

3. **GetWorkersInSection(...)**: Use Admin version (lines 5667-5760) as base because it includes `.Where(u => u.IsActive)` filter on the users query (CMP version at line 1179 is MISSING this filter). All other logic is identical.

4. **NotifyIfGroupCompleted(AssessmentSession completedSession)**: Use Admin version (lines 2722-2752) which checks `s.Status == "Completed" || s.Status == "Cancelled"` (line 2731). The CMP version (line 2874) only checks `s.Status == "Completed"` — this is the INCORRECT version per user decision. The Admin version is correct because cancelled workers should not block group completion notification.

Each method body should be copied verbatim from the specified source version, only changing `_context`, `_userManager`, `_notificationService`, `_logger` references (which are the same field names).
