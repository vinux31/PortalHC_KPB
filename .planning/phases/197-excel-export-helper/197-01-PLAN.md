Create `Helpers/ExcelExportHelper.cs` as a static class with two methods:

1. `public static IXLWorksheet CreateSheet(XLWorkbook workbook, string sheetName, string[] headers)`:
   - Calls `workbook.Worksheets.Add(sheetName)`
   - Writes each header string to row 1 cells (1-based: Cell(1, i+1))
   - Sets `Style.Font.Bold = true` on the header range (Range(1, 1, 1, headers.Length))
   - Returns the worksheet for the caller to populate data starting at row 2

2. `public static FileContentResult ToFileResult(XLWorkbook workbook, string fileName, ControllerBase controller)`:
   - Iterates all worksheets: `foreach (var ws in workbook.Worksheets) ws.Columns().AdjustToContents();`
   - Creates MemoryStream, calls `workbook.SaveAs(stream)`, resets position
   - Returns `controller.File(stream.ToArray(), "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet", fileName)`

Namespace: `HcPortal.Helpers`
Using statements: `ClosedXML.Excel`, `Microsoft.AspNetCore.Mvc`, `System.IO`

Note: Do NOT add background color in CreateSheet — most exports use bold-only headers. The few exports that need colored headers (ExportAssessmentResults, ProtonData ExportSilabus) will set color themselves after calling CreateSheet.

**Pattern for standard exports (most actions):**

BEFORE:
```csharp
using var workbook = new XLWorkbook();
var ws = workbook.Worksheets.Add("SheetName");
ws.Cell(1,1).Value = "Header1"; ws.Cell(1,2).Value = "Header2"; ...
ws.Range(1,1,1,N).Style.Font.Bold = true;
// ... data population ...
ws.Columns().AdjustToContents();
using var stream = new MemoryStream();
workbook.SaveAs(stream);
return File(stream.ToArray(), "application/vnd...", fileName);
```

AFTER:
```csharp
using var workbook = new XLWorkbook();
var ws = ExcelExportHelper.CreateSheet(workbook, "SheetName", new[] { "Header1", "Header2", ... });
// ... data population (unchanged) ...
return ExcelExportHelper.ToFileResult(workbook, fileName, this);
```

**Special cases to handle:**

1. **ExportAssessmentResults (AdminController ~2808):** Has metadata rows BEFORE headers. Keep metadata rows inline, then call CreateSheet for the data worksheet OR keep header setup inline since it starts at a non-standard row. If headers start at row > 1, keep the header setup inline and only use ToFileResult for the save/return part.

2. **Exports with colored headers (ProtonData ExportSilabus ~619, ExportAssessmentResults):** Call CreateSheet first, then add `headerRange.Style.Fill.BackgroundColor = XLColor.LightBlue` after.

3. **CMP ExportRecords (~523):** Has TWO worksheets (Assessment + Training). Call CreateSheet twice — once per sheet. Use ToFileResult once at end.

4. **CMP DownloadTrainingImportTemplate (~720):** Template with example rows and colored headers (#16A34A green). Keep the special styling (green bg, white font, example rows) inline. Only use ToFileResult for save/return.

5. **Exports with per-cell bold (not range-based):** If any export sets bold per-cell in a loop instead of via range, CreateSheet handles this — just pass the headers array.

Do NOT change: data population logic, file naming, column content, row ordering, any conditional logic, or import actions (XLWorkbook from stream).
