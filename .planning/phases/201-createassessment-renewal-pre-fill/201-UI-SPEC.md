## Renewal Banner — Spesifikasi Detail

### Layout

```
[ i  Mode Renewal: {Title} — {NamaPeserta}        [Batalkan Renewal] ]
```

- Seluruh banner: `alert alert-info d-flex align-items-center justify-content-between mb-3` dengan `role="alert"`
- Sisi kiri: ikon `bi bi-arrow-repeat me-2` + teks `<strong>Mode Renewal:</strong> {Title} — {NamaPeserta}`
- Sisi kanan: tombol `<a href="@Url.Action('CreateAssessment', 'Admin')" class="btn btn-sm btn-outline-secondary ms-3">Batalkan Renewal</a>`
- Posisi: langsung setelah blok TempData alert (success/error/warning) yang sudah ada, sebelum `<nav>` wizard nav-pills

### Variasi Konten Teks Banner

| Kondisi | Teks banner |
|---------|-------------|
| renewSessionId valid | "Mode Renewal: {session.Title} — {session.User.FullName}" |
| renewTrainingId valid (ada user) | "Mode Renewal: {training.Judul} — {training.User.FullName}" |
| renewTrainingId valid (user null) | "Mode Renewal: {training.Judul}" (tanpa nama peserta) |

### Tidak Ada Close Button

Banner renewal tidak memiliki `btn-close` dismiss. Satu-satunya cara menutup adalah tombol "Batalkan Renewal" yang mengarah ke `/Admin/CreateAssessment` tanpa query param.

## Interaction Contract

### Pre-fill Behavior (sumber: CONTEXT.md)

| Field | Sumber renewSessionId | Sumber renewTrainingId |
|-------|-----------------------|------------------------|
| Title | `AssessmentSession.Title` | `TrainingRecord.Judul` |
| Category | `AssessmentSession.Category` | Tidak di-pre-fill (pilih manual) |
| SubCategory | Trigger JS `change` event pada Category select setelah page load | Tidak di-pre-fill |
| Peserta | `AssessmentSession.UserId` via `ViewBag.SelectedUserIds` | Tidak di-pre-fill (pilih manual) |
| GenerateCertificate | Auto-checked (true) | Auto-checked (true) |
| ValidUntil | `AssessmentSession.ValidUntil + 1 tahun` (jika ada) | `TrainingRecord.ValidUntil + 1 tahun` (jika ada) |

Semua field pre-fill editable — HC dapat mengubah sebelum submit.

### SubCategory Pre-fill (Claude's Discretion)

Jika JavaScript wizard sudah memiliki handler `change` pada Category select untuk populate SubCategory dropdown, tambahkan `document.getElementById('Category').dispatchEvent(new Event('change'))` dalam inline script kondisional `@if (ViewBag.IsRenewalMode == true)` setelah DOM ready. Ini memastikan SubCategory ter-populate dari Category yang sudah ter-pre-fill.

### ValidUntil Null Handling (Claude's Discretion)

Jika `ValidUntil` sertifikat asal null: field ValidUntil dikosongkan (tidak ada pre-fill). Di renewal mode, field tetap required (tanda *) dan HC harus isi manual. Tidak ada pesan khusus — tanda * sudah cukup komunikatif.

### Category Inactive Handling

Jika Category dari sertifikat asal sudah dinonaktifkan: `model.Category` tetap di-set ke string nama category tersebut. Di view, Razor `selected` binding akan coba match ke option di dropdown. Jika tidak ada match (karena filtered `IsActive == true`), dropdown tampil sebagai "-- Pilih Kategori --". HC memilih category baru secara manual. Tidak ada pesan error khusus — behavior default dropdown Bootstrap sudah cukup.

### Inactive User Handling

Jika pemilik sertifikat asal sudah tidak aktif: `ViewBag.SelectedUserIds` berisi userId tersebut, namun user tidak ada di list karena query `Where(u => u.IsActive)`. Checkbox user tidak ter-render. HC memilih peserta manual. Tidak ada notifikasi khusus.

### Query Param Invalid

Redirect ke `/Admin/CreateAssessment` (tanpa param) + `TempData["Warning"]` dengan teks: "Sertifikat asal tidak ditemukan."

Banner TempData warning sudah ada di view (`alert alert-warning`) — tidak perlu elemen baru.

## State Inventory

| State | Trigger | Tampilan |
|-------|---------|----------|
| Normal mode | GET tanpa query param | Form standar, tidak ada banner, ValidUntil opsional |
| Renewal mode — valid | GET dengan renewSessionId/renewTrainingId valid | Banner info muncul, field ter-pre-fill, ValidUntil required |
| Renewal mode — invalid param | GET dengan ID tidak ditemukan di DB | Redirect ke normal mode + TempData warning |
| POST validation error — renewal | Submit tanpa ValidUntil di renewal mode | Halaman di-return dengan alert-warning ModelState, banner renewal tetap tampil, ValidUntil masih required |
| POST validation error — normal | Submit tanpa field required di normal mode | Perilaku existing tidak berubah |
