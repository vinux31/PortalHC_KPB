## Layout Contract

### Modal Shell (ada satu per halaman host)

```
[Modal Header]
  Icon bi-clock-history + "Riwayat Sertifikat — {NamaWorker}"  | [X close]

[Modal Body]  ← max-height: 70vh; overflow-y: auto
  [Loading state]  OR  [Error state]  OR  [Empty state]  OR  [Table content]

[No modal footer]  ← tidak ada tombol di footer; close via X saja
```

### Tabel Content (di dalam Modal Body)

```
[thead: table-light]
  Judul | Kategori | Sub Kategori | Valid | Status | Sumber | (Aksi — hanya mode renewal)

[tbody]
  [grouping header row: table-secondary, colspan="6" atau "7"]
    bi-link-45deg  {ChainTitle}  badge({N} sertifikat)

  [data rows]
    {Judul} | {Kategori} | {SubKategori} | {dd MMM yyyy / "Permanent"} | badge status | badge sumber | [btn Renew] (jika eligible)

  [grouping header row untuk chain berikutnya]
    ...
```

### Kolom Aksi

- **Mode renewal:** 7 kolom (+ kolom Aksi di kanan). Tombol Renew muncul HANYA pada baris dengan Status Expired atau Akan Expired DAN `IsRenewed == false`.
- **Mode readonly:** 6 kolom (tanpa kolom Aksi). Tidak ada tombol apapun.

### ChainTitle (judul grouping header)

Urutan fallback: `SubKategori` (jika tidak null/kosong) → `Kategori` (jika tidak null/kosong) → `Judul` sertifikat tertua di chain. Sumber: RESEARCH.md Open Questions #2.

## Interaction Contract

### Trigger Flow

**Dari Renewal Certificate page:**
1. Klik `btn btn-sm btn-outline-secondary` dengan icon `bi-clock-history` di kolom Aksi per baris
2. `data-worker-id` dan `data-worker-name` pada tombol memberikan konteks ke JS handler
3. `data-mode="renewal"` menentukan mode
4. JS: tampilkan modal → inject spinner → fetch `/Admin/CertificateHistory?workerId=...&mode=renewal` → inject HTML result

**Dari CDP CertificationManagement page:**
1. Klik nama pekerja yang sudah dijadikan `<a href="#">` (hanya jika `RoleLevel <= 4`)
2. `data-worker-id`, `data-worker-name`, `data-mode="readonly"` pada elemen link
3. JS: tampilkan modal → inject spinner → fetch `/Admin/CertificateHistory?workerId=...&mode=readonly` → inject HTML result

### Loading State

Tampil segera saat modal dibuka, sebelum AJAX selesai:
```html
<div class="text-center py-4">
  <div class="spinner-border text-primary" role="status">
    <span class="visually-hidden">Memuat...</span>
  </div>
  <p class="mt-2 text-muted">Memuat riwayat sertifikat...</p>
</div>
```

### Error State

Tampil jika fetch gagal (network error atau HTTP error):
```html
<div class="alert alert-danger mx-3 mt-3">
  <i class="bi bi-exclamation-triangle-fill me-2"></i>
  Gagal memuat riwayat sertifikat. Periksa koneksi lalu coba lagi.
</div>
```

### Empty State

Tampil jika pekerja tidak memiliki sertifikat apapun:
```html
<div class="text-center py-5">
  <i class="bi bi-journal-x fs-1 text-muted d-block mb-3"></i>
  <span class="fw-semibold">Belum ada sertifikat</span>
  <p class="text-muted mt-1 mb-0">Pekerja ini belum memiliki sertifikat yang tercatat.</p>
</div>
```

### Tombol Renew di Modal

- Klik Renew → `window.location.href` langsung ke `/Admin/CreateAssessment?renew{SessionId|TrainingId}={SourceId}`
- Tidak ada `modal.hide()` sebelum redirect (sumber: CONTEXT.md — "tanpa tutup modal dulu")
- Bootstrap modal akan hilang otomatis saat navigasi

### Event Delegation

JS menggunakan event delegation pada `document` (bukan direct binding) karena partial tabel bisa di-reload via AJAX:
- Selector: `.btn-history` untuk trigger di Renewal page
- Selector: `a.btn-history` atau `a[data-mode]` untuk trigger di CDP CertificationManagement
