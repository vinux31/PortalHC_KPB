## Component Inventory

### 1. Group Accordion Card

Struktur per sertifikat:

```
.card.border-0.shadow-sm.mb-3
  └── .card-header.d-flex.align-items-center.justify-content-between [clickable]
        ├── .d-flex.align-items-center.gap-2
        │     ├── i.bi.bi-chevron-right / bi-chevron-down [16px, toggle state]
        │     ├── span.fw-semibold [judul sertifikat — 16px weight 600]
        │     └── .d-flex.gap-1
        │           ├── span.badge [total orang — bg-secondary]
        │           ├── span.badge [N Expired — bg-danger] (hanya tampil jika > 0)
        │           └── span.badge [N Akan Expired — bg-warning.text-dark] (hanya tampil jika > 0)
        └── small.text-muted [Kategori / Sub Kategori — 13px]
  └── .collapse (id unik per group)
        └── .card-body.p-0
              └── [tabel anggota group]
              └── [pagination per group, jika TotalPages > 1]
```

State rules:
- Default: **semua collapsed** saat halaman pertama dimuat (sumber: CONTEXT.md Decisions)
- Chevron: `bi-chevron-right` = collapsed, `bi-chevron-down` = expanded
- Transisi: CSS `transition: all 0.2s ease` pada `.collapse` element (animasi halus, bukan instant)

### 2. Tabel Per Group

Kolom yang ditampilkan (sumber: CONTEXT.md — kolom yang dihilangkan: Judul Sertifikat, No):

| Kolom | Width | Keterangan |
|-------|-------|-----------|
| Checkbox | 40px | `form-check-input cb-select` per baris |
| Nama | auto | Nama pekerja |
| Kategori | d-none d-md-table-cell | Tersembunyi di mobile |
| Sub Kategori | d-none d-md-table-cell | Tersembunyi di mobile |
| Valid Until | 110px | Format `dd MMM yyyy` |
| Status | 120px | Badge Expired / Akan Expired |
| Aksi | 90px | Tombol riwayat + Renew |

Tabel styling: `table table-striped table-hover mb-0` (sama dengan existing).

### 3. Pagination Per Group

- Muncul hanya jika `group.TotalPages > 1`
- Sama dengan pagination existing (`pagination justify-content-center`)
- Data attribute `data-group-id="{groupKey}"` untuk target AJAX per group
- Page size default: **10 baris per group** (Claude's Discretion — cukup untuk sebagian besar sertifikat)
- Text info: "Menampilkan X - Y dari Z pekerja" (small text-muted, centered)

### 4. Badge Count di Group Header

Format: badge terpisah, tidak digabung dalam satu string.

- Total: `<span class="badge bg-secondary">{N} orang</span>`
- Expired: `<span class="badge bg-danger">{N} Expired</span>` — hanya tampil jika N > 0
- Akan Expired: `<span class="badge bg-warning text-dark">{N} Akan Expired</span>` — hanya tampil jika N > 0

Sumber: CONTEXT.md Decisions — format "8 orang — 3 Expired — 5 Akan Expired" diimplementasikan sebagai badge terpisah agar konsisten dengan badge existing di tabel.

### 5. Empty State (seluruh halaman)

Ketika tidak ada group sama sekali (0 sertifikat perlu renewal):

```html
<div class="text-center py-5">
  <i class="bi bi-patch-check-fill fs-1 text-success d-block mb-3"></i>
  <strong>Tidak ada sertifikat yang perlu di-renew</strong>
  <p class="text-muted mt-1 mb-3">Semua sertifikat aktif atau sudah di-renew.</p>
  <a ... class="btn btn-outline-secondary">
    <i class="bi bi-arrow-left me-1"></i>Kembali ke Kelola Data
  </a>
</div>
```

Sumber: pre-populated dari empty state existing di `_RenewalCertificateTablePartial.cshtml` baris 28-37.

## Interaction Contract

### Collapse/Expand Group

- Trigger: klik pada `.card-header` (seluruh area header, bukan hanya chevron)
- Mechanism: Bootstrap 5 `data-bs-toggle="collapse"` pada card-header, `data-bs-target="#{groupId}-body"`
- Chevron icon swap: via JavaScript saat event `show.bs.collapse` / `hide.bs.collapse`
- Animasi: Bootstrap collapse transition default (CSS height transition ~200ms)
- Keyboard: Enter / Space pada header mengaktifkan toggle (Bootstrap accessibility bawaan)

### Sorting Antar Group

- Diurutkan di backend (ViewModel grouping) — bukan client-side sort
- Kriteria: group dengan `MinValidUntil` terkecil (paling expired/dekat expired) tampil di atas
- Tidak ada UI untuk mengubah sort order di fase ini

### Sorting Dalam Group

- Expired dulu (Status = Expired), kemudian urut `ValidUntil` ascending
- Diimplementasikan di ViewModel grouping layer, bukan client-side

### Pagination Per Group

- Trigger: klik nomor halaman di pagination per group
- Mechanism: AJAX call ke `FilterRenewalCertificate` (atau endpoint baru per group) dengan parameter `groupKey` + `page`
- Response: HTML partial untuk isi tabel + pagination satu group
- Replace target: `#group-{groupKey}-body` innerHTML
- Loading state: spinner kecil (`spinner-border spinner-border-sm`) muncul di dalam group body selama fetch

### Checkbox

- Di fase ini: checkbox per baris tetap ada (untuk kompatibilitas struktur HTML)
- Logika select-all per group dan tombol "Renew N Pekerja" adalah Phase 209
- Checkbox ID format: `cb-{workerId}-{sourceId}` untuk uniqueness antar group

## Accessibility

- Group header: `role="button"`, `aria-expanded="true/false"`, `aria-controls="{groupId}-body"`
- Collapse panel: `id="{groupId}-body"`, `aria-labelledby="{groupId}-header"`
- Tabel per group: `<caption class="visually-hidden">Daftar pekerja untuk {JudulSertifikat}</caption>`
- Badge count: `aria-label="{N} orang, {N} expired, {N} akan expired"` pada container badge
- Focus order: header → chevron area → badge → tabel rows saat expanded
