## Interaction Contract

### Cascade Filter SubCategory

1. Saat halaman load: `#subCategoryFilter` dalam kondisi `disabled`, nilai `""`.
2. Saat `#categoryFilter` berubah ke nilai non-empty: populate `#subCategoryFilter` dari `subCategoryMapJson[selectedCategory]`, enable dropdown.
3. Saat `#categoryFilter` di-reset ke `""`: reset `#subCategoryFilter` ke `""`, disable kembali.
4. Saat `clearFilters()` dipanggil: reset semua filter termasuk `#subCategoryFilter` ke `""` dan `disabled = true`.

### Filter Logic Client-Side

Semua filter menggunakan `&&` (AND). Setiap baris ditampilkan hanya jika semua kondisi terpenuhi:

```
matchTitle = !search || row.dataset.title.includes(search.toLowerCase())
matchCategory = !category || row.dataset.category === category.toLowerCase()
matchSubCategory = !subCategory || row.dataset.subcategory === subCategory.toLowerCase()
matchYear = !year || row.dataset.year === year
matchType = !type || row.dataset.type === type.toLowerCase()
```

Catatan SubCategory: Assessment rows punya `data-subcategory=""`. Jika SubCategory filter aktif, semua Assessment rows akan tersembunyi — ini expected behavior karena Assessment tidak memiliki SubKategori.

### Modal Training Detail

- Event: `show.bs.modal` pada `#trainingDetailModal`
- Source: `event.relatedTarget` (tombol Detail yang diklik)
- Action: populate `#modalTrainingTitle` dan semua field `<dd>` dari `data-*` attributes
