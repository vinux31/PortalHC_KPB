**A. Mixed-Type Bulk Validation (per D-11, EDGE-01):**

Bulk renew flow: user memilih beberapa sertifikat dari grouped view → klik Renew → diarahkan ke CreateAssessment atau AddTraining. Penentuan tipe terjadi di JS client-side (modal memilih "Assessment" atau "Training"). Server-side, bulk renew ke AddTraining menggunakan `fkMapType` ("training" atau "session"). Bulk renew ke CreateAssessment menggunakan `RenewalFkMapType` ("session" atau "training").

Masalah: jika user entah bagaimana mengirim batch yang mencampur sourceId dari Training dan Assessment ke satu endpoint.

1. Di `AddTraining POST` (setelah fkMap parsing baris 5544 dan sebelum FK XOR guard yang ditambahkan Plan 01), tambahkan:
   ```csharp
   // Mixed-type bulk validation (per D-11)
   if (fkMap != null && bulkUserIds != null && bulkUserIds.Count > 1)
   {
       // fkMapType harus ada dan valid untuk bulk renewal

       {
           ModelState.AddModelError("", "Bulk renewal tidak dapat mencampur tipe Assessment dan Training. Renew per tipe secara terpisah.");
       }
   }
   ```
