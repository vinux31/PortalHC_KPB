Di file Views/Admin/CoachCoacheeMapping.cshtml, cari AJAX fetch handler untuk CoachCoacheeMappingAssign (sekitar line 663-676). Saat ini handler hanya punya:

```javascript
.then(data => {
    if (data.success) location.reload();
    else alert(data.message);
})
```

Ubah `.then(data => { ... })` callback menjadi:

```javascript
.then(data => {
    if (data.success) {
        location.reload();
    } else if (data.warning) {
        if (confirm(data.message)) {
            payload.ConfirmProgressionWarning = true;
            fetch('/Admin/CoachCoacheeMappingAssign', {
                method: 'POST',
                headers: {
                    'Content-Type': 'application/json',
                    'RequestVerificationToken': document.querySelector('[name=__RequestVerificationToken]').value
                },
                body: JSON.stringify(payload)
            })
            .then(r => r.json())
            .then(d => {
                if (d.success) location.reload();
                else alert(d.message);
            })
            .catch(err => alert('Terjadi kesalahan: ' + err.message));
        }
    } else {
        alert(data.message);
    }
})
```

Backend sudah return `{ success: false, warning: true, message: "X coachee belum menyelesaikan...", incompleteCount: N }` saat ada progression warning. Handler baru ini menampilkan confirm() dialog dengan pesan backend, dan jika user klik OK, mengirim ulang payload yang sama dengan tambahan `ConfirmProgressionWarning: true`.
