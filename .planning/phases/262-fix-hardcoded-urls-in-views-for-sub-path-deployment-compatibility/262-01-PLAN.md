<task type="auto">
  <name>Task 2: Inject global basePath variable dan appUrl helper di _Layout.cshtml</name>
  <files>Views/Shared/_Layout.cshtml</files>
  <read_first>Views/Shared/_Layout.cshtml</read_first>
  <action>
Di `Views/Shared/_Layout.cshtml`, tambahkan script block SEBELUM closing `</head>` tag (atau di awal `<body>` sebelum content lain, yang penting sebelum script manapun yang butuh basePath):

```html
<script>
    var basePath = '@Url.Content("~/")'.replace(/\/$/, '');
    function appUrl(path) { return basePath + (path.startsWith('/') ? path : '/' + path); }
</script>
```

Per D-04: `basePath` dihasilkan dari Razor `Url.Content("~/")` yang otomatis PathBase-aware. Trailing slash di-trim agar concatenation `basePath + '/Admin/...'` tidak menghasilkan double slash.

`appUrl` adalah helper function (Claude's discretion) untuk konsistensi URL building di JS — menghindari manual concatenation yang rawan error.
