**Bug yang diprediksi berdasarkan RESEARCH.md:**

**Kemungkinan bug EDGE-01: pendingAnswers tidak di-flush setelah koneksi HTTP pulih**
- Root cause: `flushPendingAnswers()` hanya dipanggil dari `connection.onreconnected()` di assessment-hub.js. Jika Playwright hanya block fetch (bukan WebSocket), SignalR tetap connected → onreconnected tidak fired.
- Fix di `Views/CMP/StartExam.cshtml`: Di dalam `saveAnswerAsync()`, setelah fetch berhasil (response.ok), tambahkan check: jika `pendingAnswers.size > 0`, panggil `flushPendingAnswers()`. Ini memastikan pending answers di-flush saat fetch pertama berhasil kembali setelah offline.
- Lokasi: fungsi `saveAnswerAsync()` sekitar line 387-430
- Kode fix:
```javascript
// Di dalam saveAnswerAsync, setelah response.ok block:
if (response.ok) {
    // ... existing code ...
    // Flush pending answers jika ada (koneksi pulih)
    if (pendingAnswers.size > 0) {
        flushPendingAnswers();
    }
}
```

**Kemungkinan bug: UpdateSessionProgress tidak dipanggil saat beforeunload**
- Root cause: Tidak ada `window.addEventListener('beforeunload', ...)` yang memanggil `saveSessionProgress()`
- Fix di `Views/CMP/StartExam.cshtml`: Tambahkan `navigator.sendBeacon()` call di beforeunload handler yang sudah ada (yang saat ini hanya return warning message). Kirim POST ke UpdateSessionProgress.
- Lokasi: `window.onbeforeunload` handler
- Kode fix:
```javascript
// Di window.onbeforeunload handler, sebelum return message:
const progressData = new FormData();
progressData.append('sessionId', SESSION_ID);
progressData.append('elapsedSeconds', computeElapsed());
progressData.append('currentPage', currentPage);
progressData.append('__RequestVerificationToken', document.querySelector('input[name="__RequestVerificationToken"]')?.value || '');
navigator.sendBeacon('@Url.Action("UpdateSessionProgress", "CMP")', progressData);
```
