**Kemungkinan bug: EXAM_EXPIRED check hanya dari ElapsedSeconds (stale)**
- Root cause per RESEARCH Pitfall 3: ElapsedSeconds diupdate setiap 30 detik. Jika timer habis tapi UpdateSessionProgress belum dipanggil, server mungkin menganggap waktu belum habis.
- Fix di `Controllers/CMPController.cs` (StartExam GET, sekitar resume detection): Tambahkan fallback check menggunakan `DateTime.UtcNow - session.StartedAt` selain `session.ElapsedSeconds`.
- Kode fix:
```csharp
// Di StartExam GET, setelah compute remainingSeconds dari ElapsedSeconds:
// Fallback: juga cek wall-clock time
var wallClockElapsed = (DateTime.UtcNow - session.StartedAt.Value).TotalSeconds;
var effectiveElapsed = Math.Max(session.ElapsedSeconds, (int)wallClockElapsed);
var remainingSeconds = durationSeconds - effectiveElapsed;
```

**Kemungkinan bug: auto-submit gagal karena form validation atau missing token**
- Jika modal muncul tapi submit gagal, periksa apakah `examForm.submit()` mengirim anti-forgery token dengan benar.
