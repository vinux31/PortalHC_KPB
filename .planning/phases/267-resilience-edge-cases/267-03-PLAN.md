  <action>
Di StartExam.cshtml, tambahkan document.visibilitychange listener yang me-re-anchor wall clock timer saat tab kembali visible. Ini BUKAN pause — timer tetap berjalan (by design, server-timed anti-cheat). Yang diperbaiki adalah DISPLAY accuracy saat tab kembali aktif.

**Kenapa perlu:** Browser throttle setInterval saat tab hidden. Saat tab kembali visible, ada gap antara wall clock elapsed dan interval ticks. Wall clock anchor (Date.now()) sebenarnya sudah handle ini karena updateTimer() menghitung dari Date.now() - timerStartWallClock. Tapi ada edge case: jika tab hidden lama, saat visible lagi interval bisa "catch up" dengan burst ticks. Re-anchor memastikan display langsung akurat.

**Lokasi:** Setelah baris 382 (setelah `updateTimer();` pertama kali), tambahkan:

```javascript
// -------- Visibility change: re-anchor timer on tab resume --------
// Timer TIDAK di-pause saat tab hidden (by design — server-timed exam).
// Re-anchor memastikan display langsung akurat saat tab kembali aktif,
// karena browser throttle setInterval di background tabs.
document.addEventListener('visibilitychange', function() {
    if (document.visibilityState === 'visible' && !submitted) {
        // Hitung berapa detik sudah berlalu sejak timer dimulai (wall clock)
        var wallElapsed = Math.floor((Date.now() - timerStartWallClock) / 1000);
        var remaining = Math.max(0, timerStartRemaining - wallElapsed);
        if (remaining <= 0) {
            // Timer sudah habis saat tab hidden — trigger time-up flow
            updateTimer();
        } else {
            // Langsung update display (jangan tunggu interval tick berikutnya)
            updateTimer();
        }
    }
});
```

Ini cukup sederhana karena updateTimer() sudah menggunakan Date.now() wall clock — jadi panggil updateTimer() langsung saat tab visible sudah cukup untuk menampilkan waktu yang benar. Tidak perlu mengubah logic timer itu sendiri.
