**2. Route Attribute Audit (VER-01, per D-03):**
Grep `[Route` di semua 9 controller files. Verifikasi bahwa semua domain controller memiliki `[Route("Admin/[action]")]` di class level. Kemudian verifikasi daftar action berikut ada di controller yang tepat:

- CoachMappingController: CoachCoacheeMapping, AssignCoach, EditMapping, DeleteMapping, ImportMapping, ExportMapping, DeactivateMapping, ReactivateMapping, MarkCompleted, GetEligibleCoachees
- DocumentAdminController: KkjMatrix, KkjUpload, KkjFileDownload, KkjFileDelete, KkjFileHistory, KkjBagianAdd, DeleteBagian, CpdpFiles, CpdpUpload, CpdpFileDownload, CpdpFileArchive, CpdpFileHistory
- TrainingAdminController: AddTraining, EditTraining, DeleteTraining, ImportTraining, DownloadImportTrainingTemplate
- RenewalController: RenewalCertificate, FilterRenewalCertificate, FilterRenewalCertificateGroup, CertificateHistory
- OrganizationController: ManageOrganization, AddOrganization, EditOrganization, ToggleOrganization, DeleteOrganization, ReorderOrganization
- AdminController: Index, Maintenance (saja — per D-06, D-07)

Grep setiap action name di controller-nya. Jika ada yang missing, catat sebagai finding.

**3. Authorization Audit (VER-02):**
Untuk setiap controller selain AdminController dan AdminBaseController, grep semua public method yang return IActionResult atau Task&lt;IActionResult&gt;. Untuk setiap method tersebut, verifikasi ada `[Authorize(Roles = "Admin, HC")]` attribute. Catat jika ada action yang missing authorize attribute.

Juga verifikasi AdminBaseController memiliki class-level `[Authorize]`.

**4. Duplicate Action Check:**
Pastikan tidak ada action name yang muncul di lebih dari satu controller (mencegah AmbiguousMatchException). Grep semua action names dan cek duplikasi.

**5. View Reference Audit:**
Grep `asp-controller="Admin"` di semua .cshtml files. Untuk setiap match, periksa apakah action yang direferensikan masih ada di AdminController. Jika action sudah pindah ke controller lain tapi view masih merujuk "Admin", itu adalah bug yang harus dilaporkan (tapi JANGAN fix — hanya catat, karena phase ini verification only per D-01).

Tulis hasil semua audit ke console output sebagai laporan terstruktur.
