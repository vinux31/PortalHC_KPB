## Interaction Contract

### Expand / Collapse Per Node

- **Trigger:** Klik di mana saja pada `.tree-row` (area baris node)
- **Animasi:** `max-height` transition dari `0` ke `auto`, durasi `200ms`, easing `ease-in-out`
- **Chevron:** `bi-chevron-right` rotate `90deg` saat expanded (`transition: transform 200ms ease`)
- **State:** Disimpan di dataset `data-expanded="true/false"` pada `.tree-node`
- **Default expand state:** Level 0 (Bagian) dan Level 1 (Unit) rendered expanded; Level 2+ collapsed — sumber: CONTEXT.md Decision #2

### Expand All / Collapse All

- **Trigger:** Klik tombol `#btn-expand-all`
- **Label awal:** "Expand All" (karena Level 2+ collapsed pada load)
- **Toggle logic:** Jika ada satu node collapsed → expand semua, label jadi "Collapse All". Jika semua expanded → collapse semua, label jadi "Expand All"
- **Sumber:** CONTEXT.md Decision #3

### Loading State

- **Trigger:** Saat `fetch('/Organization/GetOrganizationTree')` dimulai
- **Visual:** Spinner `spinner-border text-primary` di tengah `#org-tree-container` + teks "Memuat struktur organisasi..."
- **Durasi:** Spinner hilang saat `then()` atau `catch()` selesai

### Tree Lines (CSS)

- Implementasi via CSS `border-left: 1px solid #dee2e6` pada `.tree-children`
- Horizontal connector via `::before` pseudo-element dengan `border-top: 1px solid #dee2e6`, lebar 12px
- Sumber: Claude's Discretion — dipilih border approach (lebih compatible cross-browser vs SVG)

### Icon Per Level

- Level 0 (Bagian): `bi-building` — merepresentasikan entitas besar
- Level 1 (Unit): `bi-diagram-3` — merepresentasikan sub-unit kerja
- Level 2+ (Sub-unit): `bi-dot` (atau `bi-circle` 8px) — merepresentasikan leaf node
- Sumber: Claude's Discretion
