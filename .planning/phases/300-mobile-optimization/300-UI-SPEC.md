## Interaction Contract

### Sticky Footer Mobile (< 992px / breakpoint lg)

Struktur: `position: fixed; bottom: 0; left: 0; right: 0; z-index: 1019`

Layout tombol (kiri ke kanan):
1. `btn btn-outline-secondary` — "← Sebelumnya" (disabled jika halaman pertama)
2. `btn btn-outline-primary` — "≡" (trigger offcanvas drawer)
3. `btn btn-primary` — "Berikutnya →" ATAU `btn btn-success` — "Selesai & Submit" jika halaman terakhir

Height sticky footer: 64px (3xl). Konten exam area mendapat `padding-bottom: 72px` di mobile untuk mencegah footer menutupi soal terakhir.

Tombol inline Prev/Next yang sudah ada di dalam halaman: `d-none d-lg-flex` (disembunyikan di mobile, digantikan sticky footer).

### Offcanvas Drawer (MOB-03)

- Trigger: tombol [≡] di tengah sticky footer
- Posisi: `offcanvas-end` (muncul dari kanan) — Bootstrap 5 offcanvas component
- Backdrop: `data-bs-backdrop="true"` — klik luar menutup drawer
- Animasi: Bootstrap default slide-in (300ms ease) — tidak dikustomisasi
- Konten: grid nomor soal identik dengan sidebar desktop (`#panelNumbers`)
- Grid gap nomor soal di offcanvas: `8px` (token sm)
- Klik nomor soal: panggil `changePage(targetPage)` lalu `bootstrap.Offcanvas.getInstance(...).hide()`
- Judul offcanvas: "Navigasi Soal"

### Sidebar Desktop (>= 992px)

Tidak berubah — `col-lg-3` tetap ada dengan `#questionPanel` scrollable. (source: CONTEXT.md D-03)

### Landscape Mode (orientation: landscape AND max-width: 991.98px)

Media query: `@media (orientation: landscape) and (max-width: 991.98px)`
- Sidebar `col-lg-3` ditampilkan kembali dengan `width: 200px; flex: 0 0 200px`
- Offcanvas trigger di sticky footer disembunyikan
- Sticky footer tetap ada (Prev/Next tetap di bawah)
- (source: CONTEXT.md D-13, D-14)

### Swipe Gesture

TIDAK diimplementasikan. Navigasi hanya via tombol Prev/Next di sticky footer. (source: CONTEXT.md D-10)

### Page Size Mobile

Mobile (< 992px): 5 soal per halaman. Desktop: 10 soal per halaman (tidak berubah). (source: CONTEXT.md D-15)

Implementasi: client-side JS — variabel `questionsPerPage` di-override via conditional JS berdasarkan `window.innerWidth < 992` sebelum render halaman pertama. Konstanta Razor `const int questionsPerPage = 10` tetap dipakai server-side untuk pagination; JS menentukan tampilan. (source: CONTEXT.md D-16 — Claude's discretion)

### Anti-copy Compatibility

Anti-copy (Phase 280) tidak dimodifikasi. Event listeners yang diblokir (`copy`, `cut`, `paste`, `contextmenu`, `selectstart`, `dragstart`) tidak konflik dengan layout responsive. (source: CONTEXT.md D-11)

### Essay Textarea

Tidak ada handling khusus virtual keyboard. Browser default behavior. (source: CONTEXT.md D-12)

## Accessibility Notes (untuk dikerjakan lebih lengkap di Phase 302)

Minimum yang harus dipastikan di fase ini (tidak lengkap WCAG, hanya agar tidak mundur):
- Sticky footer buttons: `aria-label` eksplisit untuk tombol ≡ ("Buka navigasi soal")
- Offcanvas: `aria-controls`, `aria-expanded` pada trigger button (Bootstrap 5 handle otomatis)
- Touch targets memenuhi 48dp minimum (MOB-01)
