| Role | Value | Usage |

**Color coding p-value (difficulty index) — discretion Claude:**
| Kategori | Label | Bootstrap Class |
|----------|-------|-----------------|
| Mudah (p > 0.70) | badge "Mudah" | `bg-success` (#198754) |
| Sedang (0.30 ≤ p ≤ 0.70) | badge "Sedang" | `bg-warning text-dark` (#ffc107) |
| Sulit (p < 0.30) | badge "Sulit" | `bg-danger` (#dc3545) |

**Warning N < 30 (D-04):** Inline badge `bg-warning text-dark` bertuliskan "Data belum cukup (N<30)" — nilai discrimination index tetap ditampilkan dalam `text-muted`.

Sumber: pre-populated dari `AnalyticsDashboard.cshtml` heatmap pattern (`heatmap-high`, `heatmap-mid`, `heatmap-low`).

## Visual Focal Point

**Focal point utama layar Item Analysis** adalah tabel soal dengan kolom difficulty index (p-value) dan badge kategori warna (Mudah/Sedang/Sulit) — elemen ini menjadi visual anchor karena kombinasi warna badge `bg-success / bg-warning / bg-danger` menarik perhatian langsung ke distribusi kualitas soal di antara semua elemen halaman.

**Focal point utama layar Gain Score Report** adalah tabel per pekerja dengan kolom "Gain Score" yang ditampilkan dengan `fw-semibold` — kolom ini merupakan output akhir yang paling dicari pengguna dan menjadi anchor baca pertama di tab tersebut.

## Interaction Contract

### Tab Switching
- Tab "Item Analysis" dan "Gain Score Report" ditambahkan ke Bootstrap `nav-tabs` yang sudah ada di `AnalyticsDashboard.cshtml`.
- Switching tab via JavaScript tanpa page reload (pola existing).
- Saat tab aktif berubah, data di-fetch ulang jika filter sudah diterapkan.

### Filter Bar
- Dropdown "Pilih Assessment" ditambahkan ke filter bar existing, muncul setelah filter Kategori/SubKategori.
- Dropdown assessment hanya aktif saat tab Item Analysis atau Gain Score aktif.
- Dropdown assessment di-populate via AJAX berdasarkan filter Bagian/Unit/Kategori yang dipilih.
- Tombol "Terapkan Filter" men-trigger fetch ke endpoint baru (GetItemAnalysisData atau GetGainScoreData sesuai tab aktif).

### Loading State
- Gunakan class `.dashboard-loading` existing pada container panel saat AJAX in-flight.
- Spinner CSS-only (tidak ada library tambahan).

### Gain Score Report — Sub-view Toggle
- Dua sub-view dalam tab: "Per Pekerja" dan "Per Elemen Kompetensi" (D-07).
- Toggle via dua tombol radio-style (`btn-group btn-group-sm`) di card header tab.
- Switching sub-view tidak trigger AJAX ulang — data sudah di-fetch, render di-toggle via JavaScript show/hide.

### Distractor Analysis — Highlight Jawaban Benar
- Baris jawaban benar di tabel distractor di-highlight dengan `table-success`.
- Badge `bg-success` bertuliskan "Jawaban Benar" di kolom Keterangan.

### Export
- Klik "Export Item Analysis" atau "Export Gain Score" melakukan navigasi langsung ke endpoint download (`window.location.href`).
- Filter saat ini dikirim sebagai query string.
- Tidak ada modal konfirmasi — tombol langsung trigger download.
