**Snippet JS persis** (~12 baris, vanilla JS, no jQuery — per D-09; pattern dari 304-PATTERNS.md):
```html
    <script>
        (function () {
            var btn = document.getElementById('togglePwd');
            var input = document.getElementById('inputPassword');
            var icon = document.getElementById('togglePwdIcon');
            if (!btn || !input || !icon) return;
            btn.addEventListener('click', function () {
                var isHidden = input.type === 'password';
                input.type = isHidden ? 'text' : 'password';
                icon.classList.toggle('bi-eye', !isHidden);
                icon.classList.toggle('bi-eye-slash', isHidden);
                btn.setAttribute('aria-pressed', isHidden ? 'true' : 'false');
                btn.setAttribute('aria-label', isHidden ? 'Sembunyikan kata sandi' : 'Tampilkan kata sandi');
            });
        })();
    </script>
```

**Posisi tepat:** Setelah `</div>` (line 207, penutup main wrapper), setelah `</div>` line 208 (penutup outer container) — yaitu sebelum `</body>` line 209.

**Penjelasan logic untuk verifier (sesuai D-04 D-06):**
- IIFE `(function () { ... })()` — hindari global pollution
- Guard clause `if (!btn || !input || !icon) return;` — defensive jika DOM tidak match
- `isHidden = input.type === 'password'` — true berarti SAAT INI tersembunyi (akan di-show)
- Setelah swap: jika sekarang `text` (visible), set icon `bi-eye-slash` + aria "Sembunyikan kata sandi" + aria-pressed="true"
- Jika sekarang `password` (hidden), set icon `bi-eye` + aria "Tampilkan kata sandi" + aria-pressed="false"

**Yang TIDAK boleh dilakukan:**
- Tidak extract ke `wwwroot/js/password-toggle.js` (D-09 — YAGNI, hanya 1 form login)
- Tidak pakai jQuery — Login.cshtml self-contained, vanilla JS sudah cukup
- Tidak tambah dependency baru
- Tidak modifikasi handler form submit
- Tidak ubah lock icon kiri (`bi-lock-fill`)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser DOM ↔ User keyboard/mouse | User input untuk toggle visibility — pure client-side, tidak melewati server |
| Browser ↔ Server (POST /Account/Login) | Submit form — TIDAK terpengaruh eye-icon (tombol type=button) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-304-01 | Tampering | `button#togglePwd` jika misclassified sebagai submit button | mitigate | `type="button"` eksplisit (D-03) — `<button>` default `type="submit"` di dalam form, jadi explicit attribute mencegah accidental submit. Verified via grep di acceptance_criteria Task 1. |
| T-304-02 | Information Disclosure | Password plaintext visible di DOM saat user toggle ke "show" | accept | UX intentional. User yang trigger explicit. Browser DevTools sudah bisa baca password dari `<input type=password>` sebelum patch ini (tidak menambah attack surface). Tidak ada PII tambahan terekspos. Per `<planning_context>` instructions: "password visibility adalah UX optional yang tidak menambah attack surface." |
| T-304-03 | Spoofing | Eye-icon mengkonfusi user → user mengira mereka login di phishing site | accept | Pattern eye-icon adalah standar industri (Google, Microsoft, GitHub, semua major SaaS). User familiarity tinggi. Banner AD "Login menggunakan akun Pertamina" tetap visible sebagai trust indicator. |
| T-304-04 | Denial of Service | JS error di toggle handler crash halaman Login | mitigate | Guard clause `if (!btn \|\| !input \|\| !icon) return;` (Task 2). IIFE wrap mencegah global pollution. Vanilla JS no jQuery → no dependency loading risk. Tidak ada throw / network call di handler. |
| T-304-05 | Elevation of Privilege | Bypass auth via DOM manipulation toggle | accept | Toggle hanya mengubah `input.type` di client. Server validates kredensial seperti biasa. Tidak ada server-side bypass; tidak ada path auth baru. |
| T-304-06 | Repudiation | Tidak applicable | n/a | Eye-icon adalah pure UI tweak, tidak ada audit log impact. |

**ASVS L1 alignment:**
- V2.1.1 (password masking): Tetap default masked (initial state `type="password"`); user explicit action untuk reveal. Compliant.
- V11.1.1 (no auth logic in client): Toggle hanya mengubah display, kredensial tetap di-validate server-side. Compliant.

**No high-risk threat — security_enforcement gate: PASS.**
