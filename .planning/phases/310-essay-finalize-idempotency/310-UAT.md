## Prerequisite Setup

1. Login Admin (atau HC) ke aplikasi `http://localhost:5277/Account/Login`
2. Pastikan ada minimum 3 session ber-essay di DB:
   - Session A: Status=`PendingGrading` (essay sudah di-grade semua, belum di-finalize). Pakai existing flow: Buat assessment → Worker submit → HC grade essay sampai `EssayPendingCount=0`.
   - Session B: Status=`Completed` (sudah ter-finalize sebelumnya). Lanjutan dari Session A setelah klik Finalize.
   - Session C: Status=`Open` (worker belum mulai). Buat assessment baru tanpa worker submit.

## Step 1 — SC #1 (D-03 friendly no-op): Klik Finalize 2x sequential

1. Login Admin di tab #1
2. Navigate ke `/Admin/AssessmentMonitoringDetail/{sessionId_A}` (session Status=PendingGrading)
3. Klik tombol **Selesaikan Penilaian**
4. Confirm dialog → klik OK
5. Expected: page reload, status sekarang Completed, tombol disabled
6. Buka DevTools Console, run:
   ```js
   const token = document.querySelector('input[name="__RequestVerificationToken"]').value;
   const sessionId = {sessionId_A};
   fetch('/Admin/FinalizeEssayGrading', {
       method: 'POST',
       headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
       body: `sessionId=${sessionId}&__RequestVerificationToken=${encodeURIComponent(token)}`
   }).then(r => r.json()).then(console.log);
   ```
7. **Expected console output:**
   ```json
   {
     "success": true,
     "alreadyFinalized": true,
     "message": "Penilaian sudah diselesaikan sebelumnya pada {tanggal} WIB",
     "score": <int>,
     "isPassed": <bool>,
     "nomorSertifikat": "<string|null>"
   }
   ```
8. **Result:** [ ] PASS / [ ] FAIL — _isi setelah test_

## Step 2 — SC #2 (D-02 UI gate): Tombol disabled + tooltip BI

## Step 3 — SC #3 (notif dedup): UserNotifications distinct per recipient

1. Trigger Step 1 (klik Finalize 2x sequential)
2. Buka SQL Server Management Studio atau Azure Data Studio
3. Execute query:
   ```sql
   SELECT UserId, COUNT(*) AS NotifCount
   FROM UserNotifications
   WHERE Type = 'ASMT_ALL_COMPLETED'
     AND Title = 'Assessment Selesai'
     AND Message LIKE '%' + (SELECT Title FROM AssessmentSessions WHERE Id = {sessionId_A}) + '%'
     AND CreatedAt >= (SELECT CAST(Schedule AS DATE) FROM AssessmentSessions WHERE Id = {sessionId_A})
   GROUP BY UserId;
   ```
4. **Expected:** Setiap UserId (HC + Admin recipients) memiliki `NotifCount = 1` (NO duplicate per recipient)
5. **Result:** [ ] PASS / [ ] FAIL

## Step 5 — SC #5 (concurrent parallel finalize): Dual-tab simultaneous click

**Setup baru:** Buat session E baru dengan Status=PendingGrading (essay all graded).

1. Login Admin di **2 browser tab** (atau 2 browser window berbeda)
2. Kedua tab navigate ke `/Admin/AssessmentMonitoringDetail/{sessionId_E}`
3. Pre-stage: Posisikan kursor di tombol **Selesaikan Penilaian** di kedua tab
4. Click as simultan as possible (target window ≤ 1 detik)
5. Confirm dialog → OK di kedua tab
6. **Expected UI:** kedua tab tampak success (NO error toast merah)
   - Tab #1: page reload (winner thread, normal success path)
   - Tab #2: render `alert-info` biru "Penilaian sudah diselesaikan sebelumnya pada..." (loser thread, alreadyFinalized response)
7. Execute SQL queries:
   ```sql
   -- TrainingRecord — distinct
   SELECT COUNT(*) FROM TrainingRecords
   WHERE UserId = (SELECT UserId FROM AssessmentSessions WHERE Id = {sessionId_E})
     AND Judul = 'Assessment: ' + (SELECT Title FROM AssessmentSessions WHERE Id = {sessionId_E})
     AND Tanggal = (SELECT Schedule FROM AssessmentSessions WHERE Id = {sessionId_E});
   -- Expected: 1

   -- NomorSertifikat — distinct value
   SELECT COUNT(DISTINCT NomorSertifikat) FROM AssessmentSessions WHERE Id = {sessionId_E};
   -- Expected: 1 (single value, not null IF GenerateCertificate=true && isPassed)

   -- AuditLog — distinct
   SELECT COUNT(*) FROM AuditLogs WHERE Action = 'FinalizeEssayGrading' AND TargetId = {sessionId_E};
   -- Expected: 1

   -- UserNotifications — distinct per recipient
   SELECT UserId, COUNT(*) FROM UserNotifications
   WHERE Type = 'ASMT_ALL_COMPLETED' AND Message LIKE '%' + (SELECT Title FROM AssessmentSessions WHERE Id = {sessionId_E}) + '%'
   GROUP BY UserId;
   -- Expected: setiap UserId punya count = 1
   ```
8. **Result:** [ ] PASS / [ ] FAIL
