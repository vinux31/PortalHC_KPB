## Spacing Scale

Bootstrap 5 spacing utility (`p-*`, `m-*`, `gap-*`) menggunakan basis 0.25rem (= 4px) — kompatibel dengan 8-point scale. Phase 312 declared values:

| Token Bootstrap | Pixel | Usage di Phase 312 |
|-----------------|-------|---------------------|
| `*-1` | 4px | Icon-to-text gap di button (`bi-trash3 me-1`), inline padding |
| `*-2` | 8px | Icon-to-text gap default (`bi-arrow-right me-2`), kompak antar list item |
| `*-3` | 16px | Modal `.modal-body` default padding (Bootstrap), gap antar `<ul>` item dampak penghapusan, `mb-3` antar `<p>` warning |
| `*-4` | 24px | Section break dalam modal body (PrePost breakdown sub-section), `py-3` loading state padding |
| `*-5` | 48px | Tidak dipakai di Phase 312 |

**Modal sizing:**
- `modal-dialog` default = `modal-md` (max-width 500px) — cukup untuk single + group delete (5 baris dampak)
- `modal-lg` (max-width 800px) untuk PrePost variant dengan breakdown 2 sesi (Pre + Post)
- `modal-dialog-scrollable` aktif untuk fallback scroll jika cascade enumeration list panjang

**Exceptions:** None. Semua spacing Phase 312 ikut Bootstrap utility default (`me-1`, `me-2`, `mb-2`, `mb-3`, `py-3`).

3. Tombol "Hapus Permanen" di Step 2 footer
4. Warning alert background (`alert alert-danger`)
5. Dropdown trigger "Hapus..." (`dropdown-item text-danger`)

- `btn-danger` (#dc3545) + `text-white` → 4.53:1 ✓ AA pass

## Conditional Render Rules (Role × Status × ResponseCount)

**Decision per Q1 research:** Pilih opsi **B** — UI hide pakai Status only; backend authoritative untuk responseCount. Trade-off: HC + Open + has-response → button shown briefly → modal opened → backend reject + audit blocked. Defense-in-depth via backend guard. **Rationale:** preserve Phase 311 perf wins (no precompute responseCount per row), backend tetap proteksi (server-side guard di body method).

| Role | Status | responseCount | Button Visible | Click Behavior |
|------|--------|---------------|----------------|-----------------|
| Admin | * (any) | * (any) | ✓ tampil | Modal show → impact preview → confirm → DELETE OK |
| HC | Open / Upcoming / InProgress | (UI tidak cek) | ✓ tampil | Modal show → impact preview → confirm → backend cek server-side |
| HC | Completed | * (any) | ✗ HIDE entirely | (button tidak di-render) |
| HC + click di Open + responseCount > 0 | Open + has-response | >0 (server) | ✓ tampil briefly | Modal opens → user confirm → backend reject + AuditLog blocked + TempData error redirect |

**Razor pattern (single + group, applied di `_AssessmentGroupsTab.cshtml` line 248-274):**

```razor
@{
    bool isAdmin = User.IsInRole("Admin");
    string groupStatus = (string)group.Status;
    bool canHcDelete = groupStatus != "Completed";  // simplified — ResponseCount cek di server
}
@if (isAdmin || canHcDelete)
{
    <li>
        <button type="button" class="dropdown-item text-danger"
                data-bs-toggle="modal" data-bs-target="#deleteAssessmentModal"
                data-delete-type="@(group.IsPrePostGroup ? "prepost" : "group")"
                data-delete-id="@(group.IsPrePostGroup ? group.LinkedGroupId : group.RepresentativeId)"
                data-delete-title="@group.Title">
            <i class="bi bi-trash3 me-2"></i>@(group.IsPrePostGroup ? "Hapus Grup Pre-Post" : "Hapus Grup")
        </button>
    </li>
}
```

> Plan-phase decide apakah row-level individual `DeleteAssessment` button ada di partial atau hanya di lokasi lain. Verifikasi saat implementasi.

## Interaction & States

### Modal Flow State Machine

```
[Click button.dropdown-item.text-danger]
    ↓ (Bootstrap auto: data-bs-toggle="modal" data-bs-target="#deleteAssessmentModal")
[show.bs.modal event fires]
    ↓
[STATE: LOADING]
  - Step 1 panel visible, content hidden, spinner visible
  - Step 1 footer visible: [Batal] [Lanjutkan →] (Lanjutkan DISABLED)
  - fetch /Admin/GetDeleteImpact?type=...&id=...
    ↓ (success)
[STATE: IMPACT_LOADED]
  - Step 1 panel visible, spinner hidden, content visible (counts populated)
  - Lanjutkan button ENABLED
  - For prepost: #dam-prepost-breakdown visible
    ↓ (user clicks Lanjutkan)
[STATE: CONFIRM]
  - Step 1 panel hidden, Step 2 panel visible (warning + cascade list)
  - Step 1 footer hidden, Step 2 footer visible: [← Kembali] [🗑 Hapus Permanen]
    ↓ (user clicks Kembali → back to IMPACT_LOADED)
    ↓ (user clicks Hapus Permanen)
[STATE: SUBMITTING]
  - Form POST submit (carry @Html.AntiForgeryToken() hidden field)
  - Browser navigate (full reload, RedirectToAction pattern)
    ↓
[STATE: RESULT]
  - GET /Admin/ManageAssessment renders shell + flash TempData banner
  - Banner: alert-success (sukses) atau alert-danger (HC blocked)

[Alternative paths:]
  - Esc / backdrop click / btn-close → modal hide (any state)
  - AJAX fetch fail → loading element innerHTML replaced with error text
    "Gagal memuat data dampak. Tutup dan coba lagi." (text-danger)
    Lanjutkan tetap DISABLED (no proceed jika data gagal)
```

### State Transitions Detail Table

| State | Visible Elements | Active Buttons | Loading? | Trigger to Next |
|-------|------------------|----------------|----------|------------------|
| LOADING | Step 1 + spinner + loading text | Batal only (Lanjutkan disabled) | ✓ spinner-border | AJAX response 200 → IMPACT_LOADED; AJAX fail → ERROR |
| IMPACT_LOADED | Step 1 + content (5 list items + optional prepost breakdown) | Batal + Lanjutkan | ✗ | Click Lanjutkan → CONFIRM |
| CONFIRM | Step 2 (warning alert + cascade list + microcopy) | Kembali + Hapus Permanen | ✗ | Click Kembali → IMPACT_LOADED; Click Hapus Permanen → SUBMITTING |
| SUBMITTING | Step 2 (last visible state before page nav) | (form submit, browser nav) | Browser default | Page reload → RESULT |
| ERROR | Step 1 + error text (replaces spinner) | Batal only | ✗ | User must close + retry (no inline retry button by design — keeps modal simple) |
| RESET | (internal) — same as LOADING but with cleared state | — | — | Modal show event re-triggers fetch |
