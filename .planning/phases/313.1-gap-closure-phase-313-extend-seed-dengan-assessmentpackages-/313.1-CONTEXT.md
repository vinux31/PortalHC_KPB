## Implementation Decisions

### A. Seed Scope per Fixture
- **D-01:** **1 AssessmentPackage** per fixture (PackageNumber=1, PackageName="Phase 313 Test Package"). Cukup untuk trigger `packages.Any() == true` di line 921; multi-package cross-shuffle tidak diuji di 313.
- **D-02:** **3 PackageQuestions** per package (Order 1-3, QuestionType="MultipleChoice", ScoreValue=10, MaxCharacters=0, Rubrik=NULL, ElemenTeknis=NULL). Total 21 questions × 7 fixtures = ringan secara DB footprint.
- **D-03:** **4 PackageOptions** per question (Index 0-3, OptionText "Pilihan A/B/C/D", IsCorrect=true HANYA pada index 0). Predictable correct-answer untuk Playwright submit logic.

### B. Question Content Source
- **D-04:** **Synthetic hardcoded content** — bukan clone dari real production package. Format: `QuestionText="Phase 313 Test Q{1..3}: Pilih jawaban yang benar."`, `OptionText="Pilihan A/B/C/D"`. Self-contained, deterministic, tidak fragile terhadap data drift.
- **D-05:** Identik untuk semua 7 fixtures (tidak per-fixture variation) — mengurangi seed complexity. Jika test butuh per-fixture differentiation, gunakan fixture Title (sudah unik).

### C. Identity Preservation Strategy (SQL)
- **D-06:** Capture session IDs via `OUTPUT INSERTED.Id INTO @SessionIds` table-variable saat INSERT `AssessmentSessions` (existing seed sudah pakai bulk INSERT). Lalu loop atau cross-join `@SessionIds` dengan generated package data untuk INSERT `AssessmentPackages`. Capture package IDs via OUTPUT clause juga, lalu cross-join dengan question template untuk INSERT `PackageQuestions`. Repeat untuk `PackageOptions`.
- **D-07:** Gunakan **single transaction** (`BEGIN TRAN ... COMMIT`) bungkus semua INSERT — kalau gagal di tengah, rollback ke state sebelum seed (lebih aman dari `SET XACT_ABORT ON` saja).

### D. Idempotent Cleanup Ordering
- **D-08:** Cleanup ordering FK-respecting **sebelum** INSERT existing AssessmentSessions DELETE:
  1. `DELETE PackageOptions WHERE PackageQuestionId IN (SELECT Id FROM PackageQuestions WHERE AssessmentPackageId IN (SELECT Id FROM AssessmentPackages WHERE AssessmentSessionId IN (SELECT Id FROM AssessmentSessions WHERE Title LIKE 'Phase 313 Timer Fixture%')))`
  2. `DELETE PackageQuestions WHERE AssessmentPackageId IN (SELECT Id FROM AssessmentPackages WHERE AssessmentSessionId IN (SELECT Id FROM AssessmentSessions WHERE Title LIKE 'Phase 313 Timer Fixture%'))`
  3. `DELETE AssessmentPackages WHERE AssessmentSessionId IN (SELECT Id FROM AssessmentSessions WHERE Title LIKE 'Phase 313 Timer Fixture%')`
  4. `DELETE AssessmentSessions WHERE Title LIKE 'Phase 313 Timer Fixture%'` (existing, line 57-58)
- **D-09:** Tambah `DELETE Answers, UserPackageAssignments, PackageUserResponse` kalau ada FK ke fixture session — verify via Migrations snapshot. (Planner: cek schema relasi di `Migrations/ApplicationDbContextModelSnapshot.cs` saat plan, kalau tidak ada FK ke session fixture lewatkan.)
