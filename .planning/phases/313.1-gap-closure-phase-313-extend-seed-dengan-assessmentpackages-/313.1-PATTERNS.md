**Anti-patterns to avoid (per RESEARCH.md "Anti-Patterns" line 382-390 + Common Pitfalls line 711-790):**
- Tidak boleh `SCOPE_IDENTITY()` — pakai `OUTPUT INTO`.

### `OUTPUT INSERTED.<col> INTO @table-var` identity capture
**Source:** Microsoft T-SQL idiom (RESEARCH.md "Pattern 1" line 236-269) + Phase 309 anti-pattern lesson.
**Apply to:** Semua INSERT batch di Plan 01 (Sessions → Packages → Questions).
```sql

```
**Rationale:** `SCOPE_IDENTITY()` hanya return last row's ID (fragile untuk batch). `OUTPUT INTO` set-based, transactional, idiomatic.

### Cross-join template untuk hierarchical INSERT
**Source:** RESEARCH.md "Pattern 2" line 273-308.
**Apply to:** Plan 01 PackageQuestions + PackageOptions INSERT.
```sql
DECLARE @QuestionTemplate TABLE (...);
INSERT INTO @QuestionTemplate VALUES (...), (...), (...);

    INTO @QuestionIds(Id, AssessmentPackageId, [Order])
SELECT p.Id, t.[Order], t.QuestionText, ... FROM @PackageIds p CROSS JOIN @QuestionTemplate t;
```
**Rationale:** Cursor loop = T-SQL code smell; cross-join = single statement, transactional, faster.
