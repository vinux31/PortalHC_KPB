### Step-by-step UAT (11 step)

#### Golden Path

- [ ] **Step 1: Tab Assessment Groups initial load — no double filter**

  Tab Assessment Groups active by default. Visual check.

  **Expected:**
  - Breadcrumb + header + 3 tab visible.
  - Exactly **1 filter row** di Tab 1 (search input col-md-6 + Cari button + Kategori dropdown col-md-3 + Status dropdown col-md-3).
  - **Tidak ada filter row kedua** di bawah (shell shared form dihapus).
  - Header buttons (Buat Assessment | Monitoring | Audit Log) visible.
  - Stats badge "N grup assessment" visible.

  **Verdict:** PASS / FAIL — [catatan]

- [ ] **Step 2: Filter Tab 1 trigger HTMX (Network verify)**

  Buka DevTools Network → filter `Kategori = OJT` via dropdown.

  **Expected:**
  - 1 XHR request ke `/Admin/ManageAssessmentTab_Assessment?category=OJT&...` (200 OK).
  - Response = HTML partial (BUKAN full page).
  - Table refresh dengan hanya OJT visible. Badge count update.
  - URL browser tetap `/Admin/ManageAssessment` (HTMX swap, no navigation).

  **Verdict:** PASS / FAIL

- [ ] **Step 3: Pagination preserve filter state (bonus fix)**

  Pre-condition: filter `Kategori=OJT` active dan ada multiple pages. Klik page 2.

  **Expected:**
  - XHR request `?page=2&category=OJT&...` (kategori IKUT serta).
  - Table show OJT page 2.
  - Pagination UI update (active state pindah ke page 2).

  **Verdict:** PASS / FAIL

- [ ] **Step 4: Tab 2 Input Records — filter granular 5-field native**

  Klik tab Input Records.

  **Expected:**
  - Filter row visible: Bagian + Kategori Training + Unit + Status + Cari Nama/Nopeg (5 field).
  - Filter Tab 1 (`kategori=OJT`) **tidak muncul** di Tab 2 — Tab 2 punya kategori sendiri (Kategori Training), default empty `-- Pilih Kategori --`.
  - Header buttons **hidden** (hanya Tab 1).
  - Table "Input Records" render dengan list workers OR empty state.

  **Verdict:** PASS / FAIL

- [ ] **Step 5: Cascade Bagian → Unit**

  Tab 2 active. Filter `Bagian = Operation` (atau bagian apapun yang ada di data).

  **Expected:**
  - Filter Unit value cleared (kalau sebelumnya ada value) — onchange inline execute pre-HTMX.
  - XHR request `?section=Operation&...` (200 OK).
  - Unit dropdown populate dengan unit-unit dari Operation (cascade ViewBag.TrainingUnits).
  - Table refresh.

  **Verdict:** PASS / FAIL

- [ ] **Step 6: Tab 3 History — 2 sub-tab structure + Riwayat Assessment filter existing**

  Klik tab History.

  **Expected:**
  - 2 sub-tab visible: Riwayat Assessment | Riwayat Training (badge count masing-masing).
  - Sub-tab Riwayat Assessment active by default.
  - Filter row Riwayat Assessment visible (Cari nama/NIP + dropdown Title).
  - Filter Tab 1/2 tidak ikut.

  **Verdict:** PASS / FAIL

- [ ] **Step 7: Sub-tab Riwayat Training filter NEW**

  Klik sub-tab Riwayat Training.

  **Expected:**
  - Filter row (Cari nama pekerja / Nopeg) visible — **NEW Phase 322**.
  - Table training history render.
  - DOM inspect: `<table id="trainingHistoryTable">` + `<tr class="training-history-row" data-worker="...">` exists.

  **Verdict:** PASS / FAIL

- [ ] **Step 8: Filter sub-tab Training client-side**

  Sub-tab Training active. Ketik di input filter "andi" (atau nama yang ada di data).

  **Expected:**
  - Rows non-match hidden via `display: none` (DOM inspect).
  - Rows match tetap visible.
  - **Tidak ada XHR request** (client-side filter, no network call).
  - Console no JS error.

  **Verdict:** PASS / FAIL

#### Edge Cases

- [ ] **Step 9: Reset filter Tab 1**

  Tab 1 active dengan filter aktif (`Kategori=OJT` + page 2). Klik Reset Filter button.

  **Expected:**
  - Search input clear, kategori dropdown reset ke "Semua Kategori", status reset ke "Aktif (Open/Upcoming)", page reset ke 1.
  - XHR request ke `ManageAssessmentTab_Assessment` tanpa filter params.
  - Table reload default.

  **Verdict:** PASS / FAIL

- [ ] **Step 10: Error state (network drop simulation)**

  DevTools Network → throttle "Offline". Trigger filter di Tab 1 (ketik di search).

  **Expected:**
  - Error template visible: "Gagal memuat data" + Coba Lagi button (existing Phase 311 D-07 preserved).
  - Klik Coba Lagi dengan network back online → wrapper refetch + restore filter state (Phase 311 Plan 04 BUG-5A retry handler preserved).

  **Verdict:** PASS / FAIL

- [ ] **Step 11: Tab switch lazy `once` modifier preserved**

  Tab 1 active dengan filter `kategori=OJT`. Switch ke Tab 2, back ke Tab 1.

  **Expected:**
  - Tab 1 DOM masih sama (kategori=OJT, page state, scroll position).
  - **Tidak ada XHR request** waktu switch back ke Tab 1 (HTMX `once` modifier preserved).
  - Filter values di-preserve di DOM partial (tidak re-fetch).

  **Verdict:** PASS / FAIL
