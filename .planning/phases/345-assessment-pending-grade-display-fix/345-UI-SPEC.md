## Color

> **Inti kontrak phase ini.** Satu warna baru ditambahkan: **AMBER** untuk state "Menunggu Penilaian"
> (sesi `Status="Completed"` + `IsPassed==null`). Harus konsisten di 4 surface dan **jelas berbeda** dari
> Passed (hijau), Failed (merah), dan kosong (abu).

### Peta semantik status badge (3-way + empty) — WEB (Bootstrap)

| State (`IsPassed`) | Makna | Web class | Warna | Catatan |
|--------------------|-------|-----------|-------|---------|
| `true` | Lulus | `badge bg-success` | hijau | EXISTING — JANGAN diubah (rename "Passed"→"Lulus" = POL-01/Phase 347, bukan 345) |
| `false` | Tidak lulus | `badge bg-danger` | merah | EXISTING — JANGAN diubah |
| `null` (Completed + belum dinilai) | **Menunggu Penilaian** | **`badge bg-warning text-dark`** | **amber** | **BARU (D-01)** — teks = konstanta PendingGrading |
| (tidak ada nilai / non-assessment kosong) | — | `text-muted` (`—`) | abu | EXISTING — tetap untuk empty cell, BEDA dari pending |

### Peta semantik status — PDF (QuestPDF `Colors.*`)

| State (`IsPassed`) | Teks | QuestPDF color | Catatan |
|--------------------|------|----------------|---------|
| `true` | "Lulus" | `QuestPDF.Helpers.Colors.Green.Darken2` | EXISTING (`:4620-4621`) |
| `false` | "Tidak Lulus" | `QuestPDF.Helpers.Colors.Red.Darken2` | EXISTING |
| `null` | **"Menunggu Penilaian"** (konstanta) | **`QuestPDF.Helpers.Colors.Orange.Darken2`** | **BARU (D-01)** — tier `Darken2` match Green/Red existing → kontras seimbang |

### Distinctness (wajib lolos checker)

Amber pending **harus terbaca beda** dari 3 state lain:
- vs **Passed (hijau `bg-success`)** — beda hue (amber/kuning vs hijau). OK.
- vs **Failed (merah `bg-danger`)** — beda hue (amber vs merah). Ini perbaikan utama: sebelumnya pending salah render merah "Failed". OK.
- vs **empty (`text-muted` abu, `—`)** — pending = badge berisi; empty = teks abu tanpa badge. Beda bentuk + beda warna. OK.
- PDF: `Orange.Darken2` beda dari `Green.Darken2` & `Red.Darken2` pada tier kecerahan yang sama. OK.

### A11y (kontras)

`bg-warning` (amber Bootstrap, ~`#ffc107`) butuh teks gelap untuk kontras WCAG AA → **WAJIB `text-dark`**
(sudah dispesifikasikan di class). JANGAN `bg-warning` polos (teks putih default gagal kontras di atas amber).
Untuk PDF, `Orange.Darken2` cukup gelap sebagai foreground di atas background putih dokumen (sejajar pola
Green/Red.Darken2 existing yang sudah lolos secara visual).

### Tabel template (formalitas)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | inherit existing | Surface halaman — tidak disentuh |
| Secondary (30%) | inherit existing | Kartu / tabel — tidak disentuh |
| Accent (10%) | `bg-warning` amber (web) / `Colors.Orange.Darken2` (PDF) | **HANYA** badge/teks status "Menunggu Penilaian" |
| Destructive | inherit `bg-danger` / `Colors.Red.Darken2` | State Failed (existing) — tidak diubah |

Accent reserved for: **badge/teks status "Menunggu Penilaian" saja** (sesi Completed + `IsPassed==null`).
Tidak dipakai untuk elemen lain di phase ini.
