## Component Contracts (phase-specific reuse contracts)

> These are the load-bearing visual/interaction contracts. The 6 dimensions above are reuse
> declarations; this section is what the executor actually builds.

### C1 — Visible counter + filtered-empty-state (SF-03, Worker Detail)

- **Where:** `Views/CMP/RecordsWorkerDetail.cshtml`, `filterTable()` (`:336-358`) + a new counter
  element near the table card header (Claude's discretion on exact placement per D-01).
- **Template to mirror:** `Views/CMP/Records.cshtml:336-380` (`visibleCount` loop + empty-state
  inject/hide). Copy near-verbatim; rename id `myRecordsEmptyState` → `workerDetailEmptyState`.
- **Counter markup contract:**
  - Element rendered once (e.g. a `<span>`/`<div>` with `class="small text-muted"`).
  - Text content `Menampilkan X dari Y`; updated on every `filterTable()` run.
  - Container carries `aria-live="polite"` so screen readers announce the new count after each
    keystroke/filter change (a11y requirement — NON-NEGOTIABLE per D-01).
- **Empty-state row contract (verbatim shape):**
  ```html
  <tr id="workerDetailEmptyState">
    <td colspan="7" class="text-center p-5 text-muted">
      <i class="bi bi-inbox fs-1 d-block mb-2"></i>Tidak ada hasil untuk filter ini.
    </td>
  </tr>
  ```
  - `colspan="7"` — table has exactly 7 columns (`Tanggal, Nama Kegiatan, Tipe, Kategori, Sub
    Kategori, Status, Action`). Verified `RecordsWorkerDetail.cshtml:190-197`.
  - Injected only when `visibleCount === 0 && totalRows > 0`; hidden (`display:none`) otherwise.
- **No new CSS:** the row reuses Bootstrap `text-center p-5 text-muted` + `records.css`
  `.training-row` fadeIn is NOT applied to the empty-state row (it is not a `.training-row`).

### C2 — Kategori dropdown options from actual records (SF-04, Worker Detail)

- **Visual delta:** NONE. The Kategori `<select id="categoryFilter">` markup
  (`RecordsWorkerDetail.cshtml:137-148`) is UNCHANGED. Only the data source of its `<option>` list
  changes (master `AssessmentCategories` → distinct non-empty `unifiedRecords.Kategori`). This is a
  backend/data-source change (controller ViewBag or view-side LINQ per D-02 discretion).
- **Out of visual contract scope:** option-source mechanism (controller vs view LINQ), exact-equals
  compare preserved at `:352`. Documented here only to assert "no new visual element."

### C3 — Filter parity: My Records gains Kategori + Tipe (SF-05)

- **Where:** `Views/CMP/Records.cshtml` filter bar (`:54-93`).
- **Layout contract:** add two `<select>` cells matching Worker Detail's column sizing EXACTLY.
  Reuse the Worker Detail cell template verbatim (`RecordsWorkerDetail.cshtml:136-173`):
  - Cell wrapper: `col-12 col-sm-6 col-md-2`
  - Label: `<label for="..." class="form-label small text-muted mb-1">`
  - Control: `<select class="form-select">`
  - **a11y (NON-NEGOTIABLE):** every new `<select>` MUST keep its `<label for="...">` association
    (matches existing Worker Detail dropdowns `:137`, `:151`, `:157`, `:167`). No floating /
    placeholder-only selects.
- **Kategori select (My Records):** `id="categoryFilter"`, first option `Semua Kategori`, options =
  distinct-actual from My Records `unifiedRecords.Kategori` (consistent with D-02 source rule).
- **Tipe select (My Records):** `id="typeFilter"`, first option `Semua Tipe`, then two options.
  - **VALUE-MAPPING TRAP (load-bearing):** Worker Detail's Tipe `<option>` values are the full
    record-type strings `"Assessment Online"` / `"Training Manual"` displayed as `Assessment` /
    `Training` (`RecordsWorkerDetail.cshtml:170-171`), and its `filterTable()` lowercases BOTH the
    option value and `data-type` before compare (`:341`, `:354`). My Records rows carry
    `data-type` = `assessment` / `training` (lowercased record type). The executor MUST ensure the
    My Records Tipe `<option value>` matches whatever `data-type` the My Records rows actually
    carry (verify in `Records.cshtml` rows) — do NOT blindly copy `"Assessment Online"` if My
    Records rows use `assessment`. Match values to the existing `data-type` on THIS surface.
- **Row data contract:** My Records rows currently lack `data-category`; add
  `data-category="@(item.Kategori?.ToLower() ?? "")"` mirroring Worker Detail `:216` so the new
  Kategori filter can compare client-side.
- **Preserve:** the existing My Records year quick-button group (`:62-77`) UNCHANGED. SubKategori is
  NOT added to My Records (deferred per D-03).
- **Grid note:** My Records filter bar currently uses `col-md-6` halves. Adding Kategori + Tipe may
  require re-flowing to `col-md-...` cells consistent with the Worker Detail `col-md-2` sextet; the
  goal is visual parity of the filter SET, not pixel-identical layout. Keep all controls on the
  existing filter card (`card border-0 shadow-sm`) — no new card.

### C4 — Back-nav state preservation (SF-07)

- **Visual delta:** NONE — behavior only (sessionStorage-primary per D-04). The "Back to Team View"
