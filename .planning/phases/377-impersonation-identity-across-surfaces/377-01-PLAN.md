Pure decision function (BARU di Plan 02, di ImpersonationService.cs, public static untuk testability):
```csharp
// Given state impersonasi → keputusan identitas efektif (sentinel).
// Return values:
//   EffectiveUserDecision.UseRealUser   → caller pakai GetUserAsync(User) (SC4 non-impersonate)
//   EffectiveUserDecision.RoleModeEmpty → mode role: effective user = null → surface kosong + hint (D-03)
//   EffectiveUserDecision.TargetUser    → pakai TargetUserId (mode=user, X.Id) (SC2)
public enum EffectiveUserDecision { UseRealUser, RoleModeEmpty, TargetUser }

public static EffectiveUserDecision ResolveEffectiveUserDecision(
    bool isImpersonating, bool isExpired, string? mode, string? targetUserId);
```
Aturan keputusan (load-bearing — INI yang diuji):
- `(isImpersonating=false, *, *, *)` → `UseRealUser`  (SC4 short-circuit)
- `(true, isExpired=true, *, *)` → `UseRealUser`  (sesi kadaluarsa = treat as not impersonating)
- `(true, false, "role", *)` → `RoleModeEmpty`  (D-03)
- `(true, false, "user", null)` → `RoleModeEmpty`  (D-04 trigger; middleware sudah handle redirect — resolver fail-closed, JANGAN fallback admin)
- `(true, false, "user", "X")` → `TargetUser`  (SC2)
