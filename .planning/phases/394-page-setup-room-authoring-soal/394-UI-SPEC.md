## Color

Bootstrap 5 theme tokens (CSS custom properties). Page body is `bg-light` (dominant neutral); cards are white surfaces; primary blue is the single accent reserved for active/forward actions. Declared:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f8f9fa` (`bg-light`, `--bs-light`) + white card surfaces (`#ffffff`) | Page background, card bodies (`card shadow-sm border-0`), card-header `bg-light` |
| Secondary (30%) | `#6c757d` (`text-muted`, `--bs-secondary`) + `#e9ecef` borders | Helper text (`form-text text-muted`), section card headers, neutral badges (`bg-secondary`), `btn-outline-secondary` (Prev / Cek / Batal) |
| Accent (10%) | `#0d6efd` (`--bs-primary`) | See reserved list below |
| Success (semantic) | `#198754` (`--bs-success`) | Final commit/submit CTA (`btn-success`), completed wizard pills (`bg-success`), Section-C badge "C", success alerts |
| Destructive | `#dc3545` (`--bs-danger`) | Required-field asterisk (`text-danger *`), validation errors (`is-invalid` / `text-danger small` / `alert-danger`), "Hapus soal" buttons (`btn-outline-danger`) |
| Warning (semantic) | `#ffc107` (`--bs-warning`) | Duplicate-title-kembar notice, "MA: centang â‰Ą2 benar" info, backdate/dedup advisory alerts (`alert-warning`) |
| Info (semantic) | `#0dcaf0` (`--bs-info`) | "Mode Inject" page banner + Step-5 placeholder notice (`alert-info`), type badge |

**Accent (`#0d6efd` primary) reserved for:**
1. The **active** wizard pill (`bg-primary text-white active`) and forward-nav primary buttons (`btn-primary btn-next` "Selanjutnya").
2. Step-heading icons (`bi ... text-primary`) and section-card header icons.
3. The **active mode** indicator on the certificate radio group and the "Tambah Soal" primary button in authoring.
4. Selected-count badge (`badge bg-primary` "N terpilih" / "N peserta") in the worker picker.

Accent is NOT applied to all interactive elements: secondary navigation (Sebelumnya), the title "Cek" button, "Batalkan Semua", and Edit-from-confirm buttons use `btn-outline-secondary`. The final inject/commit action uses **success green**, not primary blue (mirrors CreateAssessment `btn-success btn-lg` "Buat Assessment").

## Interaction Contracts (runtime â€” Playwright-verified)

1. **RBAC gate:** Admin & HC see Section-C card â†’ `/Admin/InjectAssessment` opens; any other role â†’ 403 / AccessDenied (`[Authorize(Roles="Admin,HC")]`). *(INJ-03, SC1)*
2. **Wizard navigation:** 6 pills; forward only via `Selanjutnya` after step validation passes; pill state reflects active/completed/pending; `Sebelumnya` returns; edit-from-confirm jumps back then returns. *(D-01/D-02)*
3. **Title check:** debounced `Cek Judul` calls `GET /Admin/CheckTitleAvailability`, renders available/duplicate inline result. *(D-03)*
4. **Backdate guard:** date input `max=today`; client rejects future date with `is-invalid`. *(D-03, carry 393 D-06)*
5. **Worker picker:** search filters by name/email; section filter narrows; select/deselect all; selected count + live panel update; â‰Ą1 required to advance. *(INJ-06, SC4 â€” only existing AspNetUsers shown â‡’ unknown NIP impossible)*

7. **Certificate radio:** Auto shows format preview + ValidUntil + permanen; Manual shows NomorSertifikat + ValidUntil + permanen; Tanpa hides all. *(INJ-07, SC5)*
8. **Step 5 placeholder:** present as navigable panel with info notice; designed so Phase 395 can fill it without refactor. *(D-07)*
9. **No DB write:** all data held in form/hidden/JS state until final commit (which lands in Phase 395). *(D-07, 0-migration)*
