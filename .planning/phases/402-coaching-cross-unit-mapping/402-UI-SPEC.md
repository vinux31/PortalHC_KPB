## Interaction Contract — Assign Modal (the core of this phase)

> Ini adalah jantung Phase 402. Empat sub-kontrak interaksi, semua dalam `#assignModal` existing. Tiap kontrak menyatakan **state UI yang harus TRUE**, bukan implementasi.

### IC-1 — Coach-first auto-scope (CXU-01 / D-03)

**Trigger:** `#assignCoachSelect` (`:418`) `onchange` (coach option carries `data-section` `:422`).

| Saat | State UI yang HARUS TRUE |
|------|---------------------------|
| Coach BELUM dipilih (`value=""`) | Checklist coachee **kosong / disabled** (atau tampil prompt "Pilih coach terlebih dahulu"); Filter Seksi Coachee + Bagian Penugasan netral. |
| Coach dipilih | (1) Checklist `#coacheeChecklist` **hanya menampilkan** coachee yang `data-section == coach.dataset.section` (sembunyikan sisanya via `style.display`, idiom existing `filterCoacheesBySection` `:703-713`). (2) Dropdown **"Filter Seksi Coachee"** (`:427-436`) **di-disable + di-sembunyikan** (redundan — Bagian sudah dikunci coach). (3) Dropdown **"Bagian Penugasan"** (`:467-475`) **auto-set = coach.Section + disabled** (read-only; Invariant #1: 1 batch = 1 Bagian = Bagian coach). |
| Ganti coach ke Bagian berbeda | Reset: uncheck semua coachee, re-filter checklist ke Bagian coach baru, re-set Bagian Penugasan, hapus semua select unit per-baris yang ter-render. |

**Catatan kontrak:** Client-filter ini **murni UX**; server tetap enforce CXU-02 (lihat Error State E-1). DOM manipulasi tak boleh jadi boundary keamanan.

### IC-2 — Per-coachee unit dropdown KONDISIONAL (CXU-03 / D-01 / D-02)

**Sumber data:** `data-units` JSON per coachee-item (array nama unit aktif, **primary-first** = D-02 default), menggantikan `data-unit` scalar (`:442`). Disuplai server via dict `coacheeId→units` (RESEARCH Pattern 2 — `ApplicationUser` TAK punya nav `UserUnits`, Pitfall 1).

| Kondisi coachee | Elemen UI |
|-----------------|-----------|
| `units.length > 1` (multi-unit) | Render `<select class="form-select form-select-sm coachee-unit-select" data-coachee-id="…">` **inline di baris coachee** (di bawah/di samping `form-check-label` `:445-455`), opsi = `units`, **default selected = units[0]** (primary, D-02), bisa diubah operator. Muncul **hanya saat baris terlihat & dicentang** (atau saat terlihat — discretion executor; default: render saat coach dipilih untuk coachee yang lolos filter). |
| `units.length == 1` (single-unit, MAYORITAS) | **TIDAK ada dropdown.** Unit auto-dipakai dari `units[0]` (simpan di data-attr / dibaca saat submit). Minim UI churn (D-01 rasional). |
| `units.length == 0` (anomali) | Baris tetap tampil; submit untuk coachee ini **ditolak server** ("Unit penugasan wajib…"). Tidak ada dropdown; tidak crash. |

**Visual contract dropdown per-baris:** ukuran `form-select-sm` (lebih kecil dari select batch lama — konteks inline baris); margin atas `mt-1` (4px, sejajar skala Bootstrap); lebar `w-auto`/`d-inline-block` agar tidak full-width memenuhi baris. Tidak ada warna baru — `form-select` default Bootstrap. Tidak ada ikon.

### IC-3 — Relax lock "1 batch = 1 unit" → level Bagian (CXU-04 / D-01b)

| Aspek | Lama (DIHAPUS) | Baru (kontrak) |
|-------|----------------|----------------|
| Lock saat centang | `updateAssignmentDefaults` `:729-738` kunci `lockedUnit` dari centang pertama, **disable** coachee unit berbeda + `text-muted` | **Hapus** unit-lock. Coachee multi-unit dalam **Bagian yang sama** boleh dicentang bersama (multi-unit dalam 1 batch SAH). Satu-satunya pembatas tampilan = filter by Bagian coach (IC-1). |
| Auto-fill unit batch | `:751-763` auto-set `assignAssignmentUnit` saat `units.size==1` | **Hapus** — tidak ada unit batch tunggal lagi; unit per-baris. Bagian Penugasan di-set dari coach.Section (IC-1), bukan dari unit centang. |
| Backstop submit | `submitAssign` `:777-784` alert + abort bila `selectedUnits.size > 1` | **Hapus.** Ganti backstop (opsional, defense-in-depth) = semua coachee tercentang harus `data-section == coach.Section`; bila tidak → blok dengan pesan E-2. **JANGAN** tinggalkan sisa cek `selectedUnits.size > 1`. |
| Hint teks (`:463-465`) | "Satu batch assign hanya untuk satu unit. Coachee dari unit lain dinonaktifkan…" | **Ganti** dengan teks baru (lihat Copywriting H-1). |

### IC-4 — Payload map per-coachee (CXU-03 / D-05 — plumbing, bukan visual)

`submitAssign` (`:788-795`) membangun payload **map** `coacheeId→unit`: untuk tiap `.coachee-checkbox:checked`, ambil unit dari `.coachee-unit-select[data-coachee-id]` (multi-unit) **atau** satu-satunya unit dari `data-units` (single-unit). `AssignmentSection` tetap tunggal = coach.Section. Header `RequestVerificationToken` (`:801`) + `appUrl('/Admin/CoachCoacheeMappingAssign')` **dipertahankan**. Cabang `data.warning` lama (`:809+`) **DEAD untuk assign** (Phase 401 hard-block) — boleh disisakan tapi tak diandalkan. **Bukan kontrak visual**; dicantumkan agar planner tahu submit reshape.

## Accessibility Parity

Sama persis dengan modal existing (paritas, bukan melebihi):

- `#assignModal` sudah ber-`aria-labelledby="assignModalLabel"` + `btn-close aria-label="Close"` (existing) — **tak diubah**.
- `<select>` unit per-baris (IC-2): WAJIB punya label asosiatif yang dapat diakses. Karena inline di baris coachee, kontrak: tambahkan `aria-label="Unit penugasan @coachee.FullName"` (atau referensi `aria-labelledby` ke label coachee) agar screen-reader tahu select milik coachee mana. Ini paritas standar form-control, bukan ekstra.
- Dropdown Bagian Penugasan yang di-`disabled` (IC-1): gunakan atribut `disabled` Bootstrap standar (bukan `readonly` pada `<select>`); nilai tetap ter-submit lewat logika JS (bukan dari select disabled). Tidak ada `aria-live` baru.
- Hint `form-text` (H-1/H-2): teks statis dalam alur baca normal; tidak perlu `aria-live` (paritas dengan hint existing `:463`).
