# 415.1-01 SUMMARY — Guard WR-02 SubmitEssayScore assignment-based

## Yang dibangun

Predikat ownership WR-02 di `SubmitEssayScore` (`Controllers/AssessmentAdminController.cs`)
diganti dari ownership-by-package (false-positive) ke assignment-based:

```csharp
// LAMA (false-positive — paket di-pool lintas sesi-sibling):
var ownsQuestion = await _context.PackageQuestions
    .AnyAsync(q => q.Id == questionId && q.AssessmentPackage.AssessmentSessionId == sessionId);

// BARU (sumber-kebenaran sama dengan GET EssayGrading loader):
var assignment = await _context.UserPackageAssignments
    .FirstOrDefaultAsync(a => a.AssessmentSessionId == sessionId);
if (assignment != null && !assignment.GetShuffledQuestionIds().Contains(questionId))
    return Json(new { success = false, message = "Soal bukan milik sesi ini." });
```

- **Cross-package** (assignment ADA + memuat questionId) → LOLOS (lanjut upsert).
- **Genuine tamper** (assignment ADA tapi TIDAK memuat questionId) → reject "Soal bukan milik sesi ini."
- **Null assignment** (D-02, degenerate/non-paket) → IZINKAN (andalkan guard tipe-Essay + status + range).
- PITFALL-2 dipatuhi: load UPA dulu, `.Contains()` evaluasi IN-MEMORY (bukan LINQ-to-SQL).

Guard lain UTUH: WR-01 tipe-Essay, status PendingGrading, range 0..ScoreValue, upsert essay-kosong (386 D-08),
RBAC `[Authorize(Roles="Admin, HC")]` + `[ValidateAntiForgeryToken]`.
