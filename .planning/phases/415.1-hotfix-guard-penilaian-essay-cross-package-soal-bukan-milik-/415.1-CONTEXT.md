<domain>
## Phase Boundary

Memperbaiki **satu bug**: tombol "Simpan Tinjauan" pada penilaian essay menolak simpan dengan alert **"Soal bukan milik sesi ini."** untuk hampir semua peserta. Endpoint terdampak: `AssessmentAdminController.SubmitEssayScore` (`Controllers/AssessmentAdminController.cs:4184-4251`), guard WR-02 di `:4210-4215`.

**Akar masalah (sudah dikonfirmasi via pembacaan kode):** guard WR-02 (diperkenalkan v31.0 Fase 387, commit `387-01`, SUDAH di Dev) mensyaratkan paket-soal dimiliki sesi yang dinilai:
```csharp
var ownsQuestion = await _context.PackageQuestions
    .AnyAsync(q => q.Id == questionId && q.AssessmentPackage.AssessmentSessionId == sessionId);
if (!ownsQuestion) return Json(new { success = false, message = "Soal bukan milik sesi ini." });
```
Padahal SEMUA jalur bangun assignment mem-POOL paket lintas sesi-sibling via `siblingSessionIds.Contains(p.AssessmentSessionId)` — `CreateEagerAssignmentsAsync` (`:2546-2551`), StartExam/Reshuffle-single (`:6018-6023`), Reshuffle-all (`:6145-6148`). `ShuffleEngine.BuildQuestionAssignment` membagikan worker N paket via `workerIndex % jumlahPaket` → worker rutin dapat soal dari paket milik **sesi sibling lain** (umumnya semua paket dibuat di 1 sesi induk, worker lain meminjam). → guard gagal untuk hampir semua worker. GET `EssayGrading` loader (`:4131-4143`) memuat soal via `UserPackageAssignment.GetShuffledQuestionIds()` (longgar) sehingga soal MUNCUL & skor bisa diketik, tapi POST ditolak.

**Fix arah (terkunci):** ganti predikat ownership → cek `questionId ∈ UserPackageAssignment.GetShuffledQuestionIds()` milik `sessionId` (tamper-proof, sumber-kebenaran SAMA dengan GET loader). Guard lain DIPERTAHANKAN: WR-01 tipe-Essay (`:4208`), status-guard PendingGrading (`:4192`), range-guard 0..ScoreValue (`:4199`).

**OUT OF SCOPE:** mengubah desain pooling paket lintas-sesi (itu by-design & benar), refactor ShuffleEngine, mengubah GET loader (sudah benar), menyentuh fitur Section v32.6, perubahan skema/migration.

### Keamanan / Anti-Tamper (fallback saat assignment hilang)
- **D-02:** Jalur utama = cek kuat `questionId ∈ assignment.GetShuffledQuestionIds()`. Kalau `UserPackageAssignment` untuk `sessionId` **TIDAK ditemukan** (kasus langka/degenerate) → **IZINKAN** simpan (jangan blokir HC), andalkan guard tersisa: WR-01 tipe-Essay + status PendingGrading + range skor. Alasan: peserta yang benar mengerjakan PASTI punya assignment (StartExam/eager/inject semua bikin UPA); kasus null hanya muncul pada data rusak/non-paket di mana memblokir = dead-end baru bagi HC. HC adalah role tepercaya (Admin/HC), jadi relaksasi di kasus null = risiko kecil.
  - Pesan reject "Soal bukan milik sesi ini." TETAP dipakai HANYA untuk kasus genuine: assignment ADA tapi `questionId` BUKAN bagiannya (tampering nyata).
