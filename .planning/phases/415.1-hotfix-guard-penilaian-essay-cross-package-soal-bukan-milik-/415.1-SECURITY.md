# 415.1 SECURITY — Guard Penilaian Essay Cross-Package (assignment-based WR-02)

## Ringkasan

Hotfix 415.1 MERELAKSASI guard ownership WR-02 di `SubmitEssayScore`: dari
ownership-by-package (`q.AssessmentPackage.AssessmentSessionId == sessionId`, false-positive)
ke assignment-membership (`questionId ∈ UserPackageAssignment.GetShuffledQuestionIds()`).

**Pertanyaan keamanan inti:** Apakah relaksasi guard per-sesi ke cek keanggotaan-assignment
MASIH MEMBLOKIR genuine tampering (HC menilai soal yang TIDAK ada di assignment peserta)?
**Jawaban: YA — TERVERIFIKASI.** Predikat baru menolak ketika assignment ADA tetapi tidak
memuat `questionId` (genuine tamper), sumber-kebenaran IDENTIK dengan GET EssayGrading loader
(soal yang tak pernah dilihat peserta tak bisa dinilai). Cabang null-assignment (D-02) di-ACCEPT
secara sadar (role tepercaya Admin/HC + guard sisa tetap berlaku).

Semua 10 threat **CLOSED** (0 open). Verifikasi mengutip file:line implementasi nyata,
ditambah bukti unit (13/13), suite penuh (633/633), e2e controller-asli real-browser (2/2),
dan Dev UAT live (2026-06-23).

## Catatan Disposition

- **T-415.1-02 (accept):** Satu-satunya threat ber-disposition `accept`. Risiko = soal di-skor
  saat UPA hilang (data degenerate/non-paket). Diterima karena (1) role tepercaya Admin/HC via
  RBAC, (2) guard tipe-Essay + status PendingGrading + range tetap aktif, (3) peserta yang benar
  mengerjakan PASTI punya UPA → null hanya muncul pada data tak ada essay gradable realistis,
  (4) memblokir = dead-end baru bagi HC (justru tujuan hotfix). Genuine tamper TIDAK terbuka
  oleh relaksasi ini (T-415.1-01 mengunci). Rasional terdokumentasi di komentar controller
  L4217-4218 + CONTEXT D-02 + accepted-risk log di bawah.
- **T-415.1-07/08/09 (process/deploy/test):** local-workflow + IT-handoff. CLOSED-by-process —
  SEED_JOURNAL `cleaned` + migration=FALSE + branch hotfix terisolasi + e2e dialog-guard semua
  ter-tempat. Bukan kode-runtime app, tak ada celah implementasi.
