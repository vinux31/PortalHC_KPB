# Phase 421 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **SCOPE NOTE (read first):** Ini fase hardening backend-berat. Permukaan UI **kecil & terikat** pada affordance konfirmasi/peringatan yang dilapiskan ke halaman admin (HC) **yang sudah ada** (`Views/Admin/*`, `Views/CMP/Results.cshtml`). **TIDAK** ada halaman baru, design system baru, atau bahasa visual baru. Stack = **ASP.NET MVC server-rendered + Bootstrap 5 + Razor (.cshtml)** — **bukan SPA, tanpa React/Tailwind/shadcn.** Kontrak ini **mengunci pola** (toast/modal/confirm) agar konsisten dengan idiom existing. Semua teks UI hadap-pengguna **WAJIB Bahasa Indonesia** (aturan proyek `CLAUDE.md`). Token spacing/typography/color di bawah = **dokumentasi token Bootstrap 5 default yang sudah berlaku**, bukan sistem baru yang harus dibangun.

## Spacing Scale

Bootstrap 5 spacer utilities (`--bs-spacer = 1rem = 16px`). Gunakan kelas utilitas existing (`mb-*`, `me-*`, `mt-*`, `gap-*`, `p-*`), JANGAN inline-style px baru. Skala token (kelipatan 4) selaras langsung dengan tingkat Bootstrap:

| Token | Value | Bootstrap | Usage |
|-------|-------|-----------|-------|
| xs | 4px | `*-1` (0.25rem) | Jarak ikon↔teks dalam tombol (`me-1`), inline padding |
| sm | 8px | `*-2` (0.5rem) | `gap-2` antar tombol, jarak ikon alert (`me-2`) |
| md | 16px | `*-3` / `*-4` (1rem) | `mb-3`/`mb-4` antar blok form, margin alert/card |
| lg | 24px | `*-5` (1.5rem) | Pemisah seksi besar |
| xl | 32px | n/a (pakai card+`mb-4`) | Jarak antar-card |

Exceptions: **none** — tidak ada nilai spacing kustom baru untuk fase ini. Reuse utilitas Bootstrap yang sudah ada di `ManagePackages.cshtml` retake card (`mb-4`, `mb-3`, `mb-2`, `me-1`, `mt-2`). Touch target tombol konfirmasi tetap pakai ukuran default `.btn` Bootstrap (≥38px tinggi — sudah memenuhi target sentuh).

## Typography

Bootstrap 5 default (JANGAN deklarasi font-size kustom). Surface fase ini hanya konsumsi teks pada heading card, label form, body alert/modal, dan helper text — semua memakai role bawaan:

| Role | Size | Weight | Line Height | Bootstrap |
|------|------|--------|-------------|-----------|
| Body / alert / modal-body | 16px (1rem) | 400 (regular) | 1.5 | `--bs-body-font-size` / `--bs-body-line-height` |
| Label form (`fw-bold`) | 16px (1rem) | 600 (semibold) | 1.5 | `.form-label.fw-bold` |
| Helper text (`form-text`) | 14px (0.875rem) | 400 | 1.5 | `.form-text.text-muted` (`.small`) |
| Heading card (`h5`) | 20px (1.25rem) | 600 | 1.2 | `.card-header h5` |

Weights digunakan: **400 (regular)** + **600 (semibold)** — tepat 2, sesuai idiom existing. Tidak ada display/hero type di fase ini.

## Color

Bootstrap 5 semantic theme colors (JANGAN tambah hex kustom). Permukaan fase ini netral; warna **hanya** dipakai untuk membedakan tingkat pesan (warning/error/success/info) dan aksi destruktif. Konsisten dengan idiom existing (lihat `ManagePackages.cshtml:159`, `AssessmentMonitoringDetail.cshtml:38-48`, `Results.cshtml:492`).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#fff` / `--bs-body-bg` | Latar halaman & card body |
| Secondary (30%) | `--bs-light` (`#f8f9fa`) | `card-header bg-light`, latar netral |
| Accent (10%) | `--bs-primary` (`#0d6efd`) | Tombol primer konfirmasi (`btn-primary`: "Simpan Pengaturan", "Ya, …"), ikon judul card (`text-primary`) |
| Warning | `--bs-warning` (`#ffc107` / `alert-warning`) | Peringatan **non-blocking** (D-02 cooldown>window, D-05/D-07 MaxAttempts<terpakai, D-06 batal-hapus) |
| Danger / Destructive | `--bs-danger` (`#dc3545` / `alert-danger`, `text-danger`) | Pesan error server (`TempData["Error"]`), penanda aksi destruktif (cabut sertifikat D-04) |
| Success | `--bs-success` (`#198754` / `alert-success`) | `TempData["Success"]` konfirmasi aksi berhasil |

Accent reserved for: **tombol aksi primer pada confirm/save** (`btn-primary`) dan **ikon judul card** (`text-primary`). TIDAK untuk semua elemen interaktif — tombol "Batal" tetap `btn-secondary`, tombol destruktif memakai konteks `alert-danger`/`text-danger`, bukan accent.

**Aturan tingkat-pesan (WAJIB konsisten — kill-drift):**
- **Non-blocking warning** (setelan tetap tersimpan, hanya memperingatkan) → `alert alert-warning` + ikon `bi-exclamation-triangle`. Berlaku: D-02, D-05/D-07, D-06 (saat membatalkan hapus & minta konfirmasi ulang).
- **Error / blok** (aksi ditolak/dibatalkan server) → `TempData["Error"]` → `alert alert-danger` + ikon `bi-exclamation-triangle-fill`.
- **Sukses** → `TempData["Success"]` → `alert alert-success` + ikon `bi-check-circle-fill`.
- **Destruktif (cabut cert)** → konfirmasi sebelum aksi pakai warna danger di copy, eksekusi server-side.

## Interaction Contract (per-REQ)

Mengunci **alat yang tepat** untuk tiap titik aksi. Tiga pola tersedia, gunakan persis sesuai keputusan CONTEXT (D-01..D-07):

| Pola | Kapan dipakai | Idiom existing yang di-reuse |
|------|---------------|------------------------------|
| **A. Inline `alert alert-warning`** (non-blocking, server-rendered via ViewBag/TempData) | Peringatan yang TIDAK menghentikan aksi; tampil setelah/di samping kontrol | `ManagePackages.cshtml:159-162` (retake card) |
| **B. Server round-trip + hidden confirm flag** (server-authoritative; POST batal → warning → submit ulang dgn flag) | Konfirmasi destruktif yang HARUS server-otoritatif, BUKAN sekadar JS | guard hapus peserta `EditAssessment` POST (`AssessmentAdminController.cs:~1896-1934`) |
| **C. Konfirmasi pra-aksi** — Bootstrap modal ATAU native `onsubmit="return confirm(...)"` | Konfirmasi eksplisit sebelum aksi destruktif/retroaktif dari satu tombol | Modal: `Results.cshtml:534-553`. Confirm: `AssessmentMonitoringDetail.cshtml:335` (Reset) |

### RTH-01 / D-02 — Peringatan dini HC: cooldown bisa lewat window (Pola A)
- **Lokasi:** Card "Ujian Ulang" di `ManagePackages.cshtml` (sekitar `:157-163`, di bawah input `Jeda Ujian Ulang (jam)`), atau flash via `TempData["Warning"]` setelah POST `UpdateRetakeSettings`.
- **Pola:** Inline `alert alert-warning d-flex align-items-start` + `bi bi-exclamation-triangle` (mirror persis blok warning MaxAttempts existing `:159-162`). **Non-blocking** — setelan tetap tersimpan.
- **Pemicu:** `RetakeCooldownHours` bisa mendorong eligibility melewati sisa window sampai `ExamWindowCloseDate`.

### RTH-01 / D-01 — Tombol "Ujian Ulang" auto-hidden saat window tutup (eligibility-driven, BUKAN komponen baru)
- **Lokasi:** `Views/CMP/Results.cshtml` blok retake control (`:489-517`).
- **Pola:** TIDAK ada komponen/branch baru yang ditulis tangan untuk hide tombol — `RetakeRules.CanRetake` menerima `examWindowCloseDate` → `CanRetake=false` → cabang `else if (Model.CanRetake)` (`:497`) otomatis tidak render tombol. Tier-feedback (`ResolveReviewMode`) ikut konsisten.
- **Yang DI-spec hanya copy state tutup/kosong** (lihat Copywriting di bawah) — sebuah `alert alert-secondary` informasional bila pekerja membuka Hasil saat window sudah tutup & tak ada jalur retake. JANGAN buat tombol disabled bergaya baru; bila perlu pesan, pakai `alert alert-secondary`/`text-muted small` selaras cabang `IsCapReached`/cooldown existing.

### RTH-02 / D-04 — Konfirmasi cabut sertifikat sebelum HC reset sesi LULUS (Pola C)
- **Lokasi:** Aksi `ResetAssessment` di `AssessmentMonitoringDetail.cshtml` (`:334-341`, form dropdown-item) **dan** titik reset massal bila ada.
- **Pola:** Reuse `onsubmit="return confirm('…')"` existing (`:335`), TETAPI **copy diperkaya** untuk sesi ber-`NomorSertifikat` agar memperingatkan pencabutan sertifikat. **Conditional copy:** bila `session.IsPassed || session.NomorSertifikat != null` → confirm string memuat peringatan cabut cert; selain itu → confirm string reset biasa existing. Pencabutan/nol-kan tetap server-side (D-03); confirm = lapis UX.
- **Bila reset massal:** gunakan modal Bootstrap (Pola C, mirror `Results.cshtml:534`) yang menyebut jumlah sesi ber-sertifikat yang akan dicabut, karena `confirm()` tunggal tak cukup informatif untuk batch.

### RTH-04 / D-06 — Soft-confirm hapus peserta ber-riwayat (Pola B — server round-trip, WAJIB)
- **Lokasi:** `EditAssessment.cshtml` form hapus peserta (`:746-754` handler) + POST loop `removedUserIds` (`AssessmentAdminController.cs:~1896-1934`).
- **Pola:** **BUKAN modal/confirm murni client.** Alur: HC submit hapus → server deteksi sesi Abandoned / ber-`AttemptHistory`/`StartedAt` → **batalkan penghapusan** + render `TempData["Warning"]` ("peserta X punya riwayat ujian — konfirmasi untuk hapus") → HC submit ulang dengan **hidden field flag konfirmasi** (mis. `confirmRemoveWithHistory=true`) → server baru hapus + cascade arsip. Native `confirm()` existing (`:749`) boleh tetap sebagai first-line UX, tetapi keputusan akhir server-otoritatif (flag), bukan JS.
- **Affordance flag:** setelah warning tampil, render tombol "Tetap Hapus" (`btn btn-danger btn-sm`) yang men-submit ulang form dengan hidden flag = true. Warna danger karena destruktif.

### RTH-05 / D-07 — Konfirmasi pra-simpan saat MaxAttempts turun di bawah pemakaian (Pola C, non-blocking)
- **Lokasi:** Card "Ujian Ulang" `ManagePackages.cshtml` / POST `UpdateRetakeSettings`.
- **Pola:** **Modal konfirmasi pra-simpan** (Bootstrap, mirror `Results.cshtml:534-553`) yang muncul saat submit bila `maxAttempts < attemptsUsed` (hitung via helper snapshot-presence D-05). Body modal: "X peserta sudah memakai lebih dari batas baru, lanjutkan?". **Non-blocking** — bila dikonfirmasi, simpan apa adanya (perilaku retroaktif existing dipertahankan).
- **Catatan:** Blok inline `alert-warning` existing (`:159-162`) sudah menampilkan peringatan *setelah* tersimpan (state); D-07 menambah lapis konfirmasi *sebelum* simpan. Keduanya boleh ada — inline = info pasca-simpan, modal = gate pra-simpan. Konsistenkan copy (lihat Copywriting).

### RTH-03 — TIDAK ada permukaan UI baru
- Pure helper `CountEraRetakeArchives` (kill-drift counting) — backend. Satu-satunya efek UI: angka pada warning existing (`:157`) jadi konsisten. **Tidak ada elemen baru.**

Menghapusnya akan menghapus riwayat percobaan terkait. Klik 'Tetap Hapus' untuk melanjutkan."** (`alert alert-warning`, `TempData["Warning"]`) |
| Destructive confirmation — reset sesi LULUS / ber-cert (RTH-02 D-04) | **"Reset sesi ini? Sesi ini SUDAH LULUS dan memiliki nomor sertifikat. Mereset akan MENCABUT sertifikat (nomor dihapus) dan semua jawaban dihapus. Peserta dapat mengulang ujian. Lanjutkan?"** (conditional; bila tak ber-cert pakai copy reset existing `:335`) |
| Destructive confirmation — reset sesi biasa (existing, dipertahankan) | "Reset sesi ini? Semua jawaban akan dihapus dan peserta dapat mengulang ujian." |
| Error state (aksi ditolak server) | **"Aksi tidak dapat dilakukan: {alasan}. Periksa kembali lalu coba lagi."** (`TempData["Error"]` → `alert alert-danger`, `bi-exclamation-triangle-fill`) |

**Destructive actions (daftar + pendekatan konfirmasi):**
1. **Reset sesi LULUS/ber-cert** (D-04) → native `confirm()` conditional (massal → modal). Konsekuensi: cabut sertifikat + hapus jawaban.
2. **Hapus peserta ber-riwayat** (D-06) → server round-trip + hidden flag + tombol "Tetap Hapus" (`btn-danger`). Konsekuensi: hapus riwayat percobaan + cascade arsip.
3. **Turunkan MaxAttempts < terpakai** (D-07) → modal pra-simpan non-blocking. Konsekuensi: peserta tak bisa menambah percobaan baru (retroaktif).

## Consistency Rules (untuk executor & checker)

1. **JANGAN** buat halaman, partial, CSS, atau komponen JS baru di luar yang tercantum. Semua affordance dilapiskan ke view existing (`ManagePackages.cshtml`, `EditAssessment.cshtml`, `AssessmentMonitoringDetail.cshtml`, `Results.cshtml`).
2. **JANGAN** override font, warna kustom (hex), atau spacing px baru. Reuse utilitas & token Bootstrap 5 + Bootstrap Icons yang sudah dipakai.
3. **Tingkat pesan** harus memetakan warna sesuai tabel Color: non-blocking → `alert-warning`; error/ditolak → `alert-danger`; sukses → `alert-success`; info/tutup → `alert-secondary`/`text-muted`.
4. **Alat konfirmasi** harus sesuai Pola per-REQ: D-06 = **server round-trip** (BUKAN modal/JS murni); D-04 = native confirm (massal → modal); D-07 = modal pra-simpan.
5. **Semua teks UI Bahasa Indonesia.** Tombol primer aksi = `btn-primary`; batal = `btn-secondary`; destruktif = `btn-danger`/`text-danger`.
6. **Server tetap otoritatif** — semua peringatan/konfirmasi adalah lapis UX; eksekusi (gate window, cabut cert, hapus arsip) tetap server-side per CONTEXT.
