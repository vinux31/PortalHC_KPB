## Interaction Contracts (LOCKED тАФ from 422-CONTEXT.md D-01..D-07)

These are interaction decisions, already locked upstream. Restated here as the binding contract for the executor.

1. **SamePackage toggle = a `form-check form-switch` checkbox** (NOT a plain checkbox), to match the existing Shuffle/Retake cards on the same page. Posts to a controller action with antiforgery token.

2. **Toggle is destructive-ish тЖТ confirm-before is MANDATORY** (D-01). ON overwrites Post packages with a Pre clone; OFF unlocks. Use either the inline `d-none` confirm panel (`:46-68` idiom) or a Bootstrap modal (`:178-222` idiom). If a modal/inline-JS is used, it MUST be wrapped in `DOMContentLoaded` + guarded by `typeof bootstrap !== 'undefined'` (fase 390.1 / 421 lesson тАФ otherwise `bootstrap is not defined` ReferenceError aborts handler attach).

3. **Guard no-started тЖТ toggle DISABLED + clear message** (D-01). When ANY participant has StartedAt/InProgress/Completed in the group, the switch and its save button render `disabled`, with the disabled-state message shown inline. Server ALSO hard-rejects (server-authoritative); the disable is the friendly UX layer. Rejection surfaces as a NON-BLOCKING `TempData["Error"]` toast.

4. **Lock state (D-07) is server-authoritative; UI is the friendly layer.** When the Post is SamePackage-locked, "Kelola Soal" / package-edit / create-package / delete-package controls are hidden or disabled (extend the existing `ViewBag.IsSamePackageLocked != true` guards at `:312`, `:379`). The server hard-rejects locked POSTs with `TempData["Error"]` regardless. A disabled "Kelola Soal" must carry the lock tooltip explaining why.

5. **Shuffle warnings are NON-BLOCKING** (D-03, D-04). They appear as `alert-warning` banners; HC can always click Simpan. The K=min warning must appear on the ON path too (D-04), not only OFF.

6. **Mismatch indicator is single-source** (D-05). `hasMismatch` / `referenceCount` are computed ONCE in the controller via `PackageSizeAnalysis.Compute` (ViewBag/helper). The view-side duplicate computation at `:72-78` is DELETED; the view only renders the ViewBag value. Kill-drift.

7. **JS confirm strings use single quotes** (fase 421 lesson). For any inline `confirm()` or attribute-embedded message, avoid `Json.Serialize` inside double-quoted HTML attributes (quote-break bug). Mirror `confirmDeletePackage()` at `:405-412`.

8. **Backward-compat:** existing Pre-Post groups without the toggle, and Standard (non Pre-Post) assessments, render UNCHANGED. No new control appears on Standard assessments.
