<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser (worker) → POST /CMP/VerifyToken | Input token akses. `[HttpPost][ValidateAntiForgeryToken]` + authz owner/Admin/HC (TIDAK diubah). Token compare via AccessTokenMatches (TIDAK diubah). |
| browser (worker) → GET /CMP/StartExam | Gate masuk ujian. Sebelumnya baca TempData (client-round-trip, dapat dimanipulasi). Kini baca kolom DB server-authoritative. |
| controller/service → AssessmentSession.TokenVerifiedAt (DB) | Stamp (VerifyToken POST) + reset (RetakeService) — keduanya server-side write. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-427-01 | Tampering | Token-gate state (TempData round-trip dapat dimanipulasi/bocor lintas sesi) | mitigate | INTI fase: pindah state verifikasi dari TempData (client round-trip) ke kolom DB server-authoritative `TokenVerifiedAt`. Gate StartExam membaca DB, bukan nilai yang bisa di-tamper klien. Ini MENGURANGI tampering (FLOW-08 hardening). |
| T-427-02 | Spoofing/Elevation of Privilege | Akses ujian tanpa token | accept (no new surface) | Gate tetap: `IsTokenRequired && owner && StartedAt==null` → butuh TokenVerifiedAt!=null. VerifyToken tetap cek `AccessTokenMatches` + authz owner/Admin/HC + CSRF. Tidak ada endpoint/param/role baru. |
| T-427-03 | Denial of Service | Satu SaveChanges ekstra di VerifyToken POST | accept | Satu UPDATE per verifikasi token (POST, bukan loop). Minor; tidak ada amplifikasi. |
| T-427-04 | Tampering (lockout regresi) | Sesi InProgress lama (TokenVerifiedAt null) | mitigate | Guard `StartedAt == null` dipertahankan → sesi yang sudah dimulai (StartedAt!=null) melewati gate token tanpa cek TokenVerifiedAt → tidak terkunci pasca-deploy (SC#4). Migration nullable, no backfill → zero-downtime. |

**Ringkasan:** Fase = security hardening (FLOW-08). Memindahkan token-gate state ke kolom DB server-authoritative MENGURANGI tampering. Tidak ada surface authz/CSRF baru. Migration nullable+no-backfill+guard StartedAt==null = zero-downtime tanpa lockout. Tidak ada threat High.
