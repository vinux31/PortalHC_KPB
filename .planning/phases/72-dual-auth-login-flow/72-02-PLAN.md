<task type="auto">
  <name>Task 2: Adapt CreateWorker and EditWorker views for AD mode</name>
  <files>Views/Admin/CreateWorker.cshtml</files>
  <files>Views/Admin/EditWorker.cshtml</files>
  <action>
    Both views are typed Razor views (not standalone — they use _Layout). Apply the same IConfiguration injection pattern.

    ### CreateWorker.cshtml changes:

    1. At the top of CreateWorker.cshtml (after `@model` and `@{ }` block), add:
       ```razor
       @inject IConfiguration Config
       @{
           var isAdMode = Config.GetValue<bool>("Authentication:UseActiveDirectory", false);
       }
       ```

    2. In the "Informasi Pribadi" card, wrap the FullName and Email inputs to be read-only in AD mode:
       - For FullName input: Add `readonly="@(isAdMode ? "readonly" : null)"` attribute and a conditional hint:
         ```html
         <input asp-for="FullName" class="form-control @(isAdMode ? "bg-light" : "")"
                placeholder="Masukkan nama lengkap"
                readonly="@(isAdMode ? "readonly" : null)" />
         @if (isAdMode)
         {
             <div class="form-text text-info"><i class="bi bi-info-circle me-1"></i>Dikelola oleh AD — akan disinkronkan saat login</div>
         }
         ```
       - For Email input: Same pattern — add `readonly="@(isAdMode ? "readonly" : null)"` and the same hint.

    3. In the "Role & Kredensial" card, wrap the Password and ConfirmPassword col-md-6 divs in a conditional:
       ```html
       @if (!isAdMode)
       {
           <div class="col-md-6">
               ... existing Password field ...
           </div>
           <div class="col-md-6">
               ... existing ConfirmPassword field ...
           </div>
       }
       else
       {
           <div class="col-12">
               <div class="alert alert-info py-2 mb-0">
                   <i class="bi bi-info-circle me-2"></i>
                   <small>Password dikelola melalui portal Pertamina. Sistem akan membuat password acak untuk akun ini.</small>
               </div>
           </div>
       }
       ```

    ### EditWorker.cshtml changes:

    1. At the top, same IConfiguration injection:
       ```razor
       @inject IConfiguration Config
       @{
           var isAdMode = Config.GetValue<bool>("Authentication:UseActiveDirectory", false);
       }
       ```
       NOTE: EditWorker.cshtml already has `@{ ViewData["Title"] = "Edit Data Pekerja"; }` — add the inject BEFORE the @{ } block and add `isAdMode` variable INSIDE the existing @{ } block.

    2. FullName and Email inputs: same readonly pattern as CreateWorker.

    3. Password fields in "Role & Password" card — wrap both Password and ConfirmPassword col-md-6 divs with `@if (!isAdMode)`. In AD mode, show the same info alert about Pertamina portal.

    CRITICAL:
    - Local mode (isAdMode=false): both views completely unchanged — password fields visible, FullName/Email editable
    - Use `readonly="@(isAdMode ? "readonly" : null)"` NOT `disabled` — disabled inputs are excluded from form POST, readonly are included (prevents empty value overwriting)
    - Do NOT remove or rename any form fields — backend POST model binding must still work
    - Do NOT add JavaScript-based field hiding (pure Razor server-side conditional only)
  </action>
