Find the generated Up() method and add the SQL migration BEFORE the closing brace:
```csharp
protected override void Up(MigrationBuilder migrationBuilder)
{
    // Data migration: SectionHead users RoleLevel 4 → 3 (Phase 74 hierarchy restructuring)
    // SectionHead moved to level 3 (full access), same as management tier
    migrationBuilder.Sql(@"
        UPDATE Users
        SET RoleLevel = 3
        WHERE Id IN (
            SELECT u.Id FROM Users u
            INNER JOIN AspNetUserRoles ur ON u.Id = ur.UserId
            INNER JOIN AspNetRoles r ON ur.RoleId = r.Id
            WHERE r.Name = 'Section Head'
        )
    ");
}
```

NOTE: The table is "Users" (custom via ToTable in DbContext, verified in Phase 71). Identity system tables use "AspNetUserRoles" and "AspNetRoles" (standard Identity naming). Verify the actual table names by checking the existing migrations in the Migrations/ folder.
