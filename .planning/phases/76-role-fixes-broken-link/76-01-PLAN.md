**View change (`Views/ProtonData/Index.cshtml`):**
Replace the existing hash-based tab activation block (the `if (window.location.hash === '#override')` block in the Scripts section) with a server-side + JS approach:

Add a Razor variable at the top of the file (in the `@{ }` block):
```csharp
var activeTab = ViewBag.ActiveTab as string ?? "";
```

Replace the hash-based JS block with:
```javascript
// Query-param based tab activation (e.g. /ProtonData?tab=override)
var activeTabParam = '@Html.Raw(activeTab)';
if (activeTabParam === 'override') {
    var overrideTab = document.getElementById('override-tab');
    if (overrideTab) new bootstrap.Tab(overrideTab).show();
} else if (activeTabParam === 'guidance') {
    var guidanceTab = document.getElementById('guidance-tab');
    if (guidanceTab) new bootstrap.Tab(guidanceTab).show();
}
// Fallback: hash-based activation for any existing bookmarks
if (!activeTabParam && window.location.hash === '#override') {
    var overrideTabHash = document.getElementById('override-tab');
    if (overrideTabHash) new bootstrap.Tab(overrideTabHash).show();
} else if (!activeTabParam && window.location.hash === '#guidance') {
    var guidanceTabHash = document.getElementById('guidance-tab');
    if (guidanceTabHash) new bootstrap.Tab(guidanceTabHash).show();
}
```

This keeps backward compatibility with any existing hash-based bookmarks while the new query param approach works reliably on page load.
