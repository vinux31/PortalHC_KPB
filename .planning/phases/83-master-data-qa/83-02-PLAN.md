  <action>
Read AdminController.cs lines 384–475 (CpdpItems, CpdpItemsSave, CpdpItemDelete) and lines 1231–1288 (CpdpItemsExport). Read Views/Admin/CpdpItems.cshtml in full.

Review for these specific issues:

1. **CpdpItemsSave rename guard** — The guard checks `IdpItems.CountAsync(i => i.Kompetensi == existing.NamaKompetensi)`. Verify the IdpItems DbSet property name is correct in DbContext. If the property is named differently (e.g., `IdpPlanItems`, `UserIdpItems`), the query silently returns 0, bypassing the guard. Grep the DbContext for the correct property name and fix the query if needed.

2. **CpdpItemsSave null safety** — `row.NamaKompetensi ?? ""` — verify all required fields have the null-coalescing default. Check if `row.No` (a string) also has `?? ""` or could leave the DB with null in a non-nullable column.

3. **CpdpItemDelete — no reference guard** — Unlike KkjMatrixDelete, CpdpItemDelete does NOT check if IDP records reference this item before deletion. This can leave orphaned IDP records. Since the user decision is "fix localized bugs", add a reference check: before deleting, count `IdpItems.CountAsync(i => i.Kompetensi == item.NamaKompetensi)`; if > 0, return `Json(new { success = false, blocked = true, message = $"Tidak dapat dihapus — {refCount} IDP record masih mereferensi kompetensi ini." })`. This mirrors the KkjMatrixDelete guard pattern and is < 15 lines.

4. **CpdpItemsExport — section filter** — The export accepts `?section=` but the CpdpItems view may not pass the currently active section filter to the export URL. Check the view for the export button/link. If it does not include `?section=@ViewBag.CurrentSection` (or similar), the export always returns all rows even when the editor is filtered. Fix by preserving the section filter in the export link.

5. **CpdpItemsExport — response headers** — Verify the file download uses `Content-Disposition: attachment` (implicit when using `File(bytes, contentType, fileName)` in ASP.NET Core). This is standard — confirm no override is breaking it.

6. **View CSRF tokens** — All fetch POST calls in CpdpItems.cshtml (CpdpItemsSave, CpdpItemDelete) must include RequestVerificationToken header. Verify each call.

7. **CMP/Mapping cross-link** — After editing in Admin/CpdpItems, does a cross-link to CMP/Mapping exist in the view (breadcrumb, button, or info text)? If absent, add a simple anchor: `<a href="/CMP/Mapping" class="btn btn-sm btn-outline-secondary">Lihat di CMP/Mapping</a>`.

Fix all bugs found (< 100 lines aggregate). Run `dotnet build` to confirm no compilation errors.
