Read ProtonDataController.cs lines 336–502 (GuidanceList, GuidanceUpload, GuidanceDownload, GuidanceReplace, GuidanceDelete) in full. Read the Guidance tab section of Views/ProtonData/Index.cshtml.

Review for these specific issues:

1. **GuidanceReplace — old file deletion** — When replacing a file, GuidanceReplace must delete the old physical file from disk before saving the new one. Check if `File.Delete(oldFilePath)` is called. If missing:
   - Read the existing record's `StoragePath`
   - Build `oldFilePath = Path.Combine(_env.WebRootPath, existingRecord.StoragePath)`
   - Call `if (System.IO.File.Exists(oldFilePath)) System.IO.File.Delete(oldFilePath);`
   - Then save the new file and update the record
   This prevents wwwroot/guidance from accumulating orphaned files.

2. **GuidanceDelete — physical file deletion** — When deleting a record, GuidanceDelete must also delete the physical file. Check if `File.Delete(filePath)` is called. If missing, add it using the same pattern as above. Wrap in `if (File.Exists(...))` to avoid exceptions if file is already missing.

3. **GuidanceDownload — file not found handling** — If the file record exists in DB but the physical file is missing from disk (e.g., manually deleted), GuidanceDownload will throw. Add a check: after building `filePath`, if `!System.IO.File.Exists(filePath)`, return `NotFound()` with a user-friendly message instead of crashing.

4. **GuidanceUpload — wwwroot/guidance directory** — The upload saves to `wwwroot/guidance/`. Verify the directory creation: if `wwwroot/guidance` doesn't exist, `FileStream` creation will throw. Add `Directory.CreateDirectory(Path.GetDirectoryName(filePath)!)` before opening the FileStream if not already present.

5. **GuidanceUpload — file size limit** — Check if there's a `[RequestSizeLimit]` attribute or `maxRequestLength` config. If uploads silently fail for large files, the user will see a confusing error. Check if any size limit is configured; if none, note it in the summary (not a blocker, but worth documenting).

6. **GuidanceDownload — authorization** — The route is `/ProtonData/GuidanceDownload/{id}`. ProtonDataController has class-level `[Authorize(Roles = "Admin, HC")]`. This means only Admin/HC can download. Phase 86 (Plan IDP) will need coachees to also download guidance files. Check the current role attribute on GuidanceDownload specifically — if it restricts to Admin/HC only, add a note in the summary that Phase 86 will need to either: (a) remove/relax the role requirement on GuidanceDownload, or (b) create a separate download action in CDPController accessible to coachees. Do NOT change auth now — just document for Phase 86.

7. **View: file type filter in upload input** — The upload input `<input type="file">` should have `accept=".pdf,.doc,.docx,.xlsx"` to guide the user. Check if it's present. If absent, add it.
