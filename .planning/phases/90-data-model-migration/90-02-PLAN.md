# Plan 91-02: CpdpFile Model + EF Core Migration

## Goal

Add the `CpdpFile` entity to the codebase (mirroring `KkjFile` exactly), register it in
`ApplicationDbContext`, and apply a new EF Core migration named `AddCpdpFiles` that creates the
`CpdpFiles` table.  The `CpdpItem` table is NOT dropped here — that is Phase 93.

### Task 1 — Add CpdpFile class to Models/KkjModels.cs

**File:** `Models/KkjModels.cs`

Add the `CpdpFile` class immediately after the closing `}` of `KkjFile` (after line 28, before the
`GapAnalysisItem` comment on line 31).

```csharp
    // CPDP File — represents an uploaded PDF/Excel file for a given bagian (mirrors KkjFile)
    public class CpdpFile
    {
        public int Id { get; set; }
        public int BagianId { get; set; }
        public KkjBagian Bagian { get; set; } = null!;
        public string FileName { get; set; } = "";      // Original filename (display)
        public string FilePath { get; set; } = "";      // Relative path: /uploads/cpdp/{bagianId}/{safeName}
        public long FileSizeBytes { get; set; }
        public string FileType { get; set; } = "";      // "pdf", "xlsx", "xls"
        public string? Keterangan { get; set; }         // Optional description from upload form
        public DateTimeOffset UploadedAt { get; set; } = DateTimeOffset.UtcNow;
        public string UploaderName { get; set; } = "";
        public bool IsArchived { get; set; } = false;   // True = moved to history
    }
```

**Exact placement:** Insert between line 28 (`}` closing KkjFile) and line 30 (`// 2. CLASS UNTUK HALAMAN MAPPING`).

### Task 3 — Create and apply EF Core migration

Run these commands in sequence from the project root (`C:\Users\Administrator\Desktop\PortalHC_KPB`):

```bash
dotnet ef migrations add AddCpdpFiles
dotnet ef database update
```

**Expected outcome:**
- A new migration file `Migrations/{timestamp}_AddCpdpFiles.cs` is generated.
- The migration `Up()` method creates a `CpdpFiles` table with columns matching the `CpdpFile` entity.
- `dotnet ef database update` completes with no errors.
- The `CpdpItems` table still exists (NOT dropped — that is Phase 93).

**If `dotnet ef` is not on PATH**, use the full dotnet tool invocation:
```bash
dotnet tool run dotnet-ef migrations add AddCpdpFiles
dotnet tool run dotnet-ef database update
```
