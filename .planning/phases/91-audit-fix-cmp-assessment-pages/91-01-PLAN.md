<task type="auto">
  <name>Task 2: Fix single-package question shuffle and populate option shuffle</name>
  <files>Controllers/CMPController.cs</files>

Make two targeted edits to CMPController.cs:

**Fix 1 — Single-package shuffle (BuildCrossPackageAssignment, ~line 1135):**
The current single-package path returns questions in original DB order, making all workers see the same sequence. Fix: shuffle the questions using Fisher-Yates before returning.

Change:
```csharp
// Single package: return questions in original DB order (no shuffle per user decision)
if (packages.Count == 1)
{
    return packages[0].Questions.OrderBy(q => q.Order).Select(q => q.Id).ToList();
}
```
to:
```csharp
// Single package: shuffle question order so each worker sees a unique sequence
if (packages.Count == 1)
{
    var singlePackageIds = packages[0].Questions.OrderBy(q => q.Order).Select(q => q.Id).ToList();
    Shuffle(singlePackageIds, rng);
    return singlePackageIds;
}
```

**Fix 2 — Populate option shuffle (StartExam action, ~line 905–925):**
When creating a new `UserPackageAssignment`, build a `ShuffledOptionIdsPerQuestion` dictionary mapping each question ID to its shuffled option ID list, then serialize to JSON.

In the block that creates the `assignment` object, after `var shuffledIds = BuildCrossPackageAssignment(packages, rng);`, add:

```csharp
// Build option shuffle: per question, randomize A/B/C/D option order
// Stored as Dictionary<questionId, List<optionId>> serialized to JSON
var optionShuffleDict = new Dictionary<int, List<int>>();
var allPackageQuestions = packages.SelectMany(p => p.Questions).ToList();
foreach (var q in allPackageQuestions)
{
    var optionIds = q.Options.Select(o => o.Id).ToList();
    Shuffle(optionIds, rng);
    optionShuffleDict[q.Id] = optionIds;
}
```

Then replace:
```csharp
ShuffledOptionIdsPerQuestion = "{}"  // option shuffle removed per user decision
```
with:
```csharp
ShuffledOptionIdsPerQuestion = JsonSerializer.Serialize(optionShuffleDict)
```

Note: The `UserPackageAssignment.GetShuffledQuestionIds()` pattern already exists as a helper method. The option shuffle is stored but view rendering (using it to reorder displayed options) is handled in plan 91-02 (StartExam.cshtml). The scoring remains ID-based (IsCorrect on PackageOption) so shuffling display order does not affect scoring.
