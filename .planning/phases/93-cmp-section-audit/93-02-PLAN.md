### Task 2: Fix time-ago pluralization in CMP views

<files>
Views/CMP/*.cshtml
</files>

<action>
1. Search all CMP views for time-ago formatting ("menit", "jam", "hari", "bulan", "tahun lalu")
2. If found, add TimeAgo helper function to view following Phase 92 pattern:
```csharp
@functions {
    public static string TimeAgo(DateTime dateTime)
    {
        var span = DateTime.Now - dateTime;
        if (span.TotalDays > 365) return $"{(int)(span.TotalDays / 365)} tahun lalu";
        if (span.TotalDays > 30) return $"{(int)(span.TotalDays / 30)} bulan lalu";
        if (span.TotalDays > 0) return $"{(int)span.TotalDays} hari lalu";
        if (span.TotalHours > 0) return $"{(int)span.TotalHours} jam lalu";
        if (span.TotalMinutes > 0) return $"{(int)span.TotalMinutes} menit lalu";
        return "baru saja";
    }
}
```

3. Replace existing time-ago calls with `@TimeAgo(item.Date)`

Note: Indonesian pluralization is context-dependent. For time units:
- Tahun → tahun (singular and plural same)
- Bulan → bulan (singular and plural same)
- Hari → hari (singular and plural same)
- Jam → jam (singular and plural same)
- Menit → menit (singular and plural same)
</action>

<verify>
Run grep commands to verify time-ago handling:
```bash
# Find any remaining raw time-ago strings
grep -n -i "menit\|jam\|hari lalu\|bulan lalu\|tahun lalu" Views/CMP/*.cshtml | grep -v "TimeAgo"
```

Expected: No raw time-ago strings remain (all use TimeAgo helper or none exist).
</verify>

<done>
Time-ago pluralization handled correctly:
- TimeAgo helper function added to views that need it
- All time-ago strings use TimeAgo function
- Indonesian pluralization verified (singular/plural same form)
</done>
