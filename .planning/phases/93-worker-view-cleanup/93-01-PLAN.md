  <action>
Replace the existing Mapping action in CMPController with a new implementation that:

1. Queries all KkjBagians ordered by DisplayOrder.
2. Queries all CpdpFiles where IsArchived == false, grouped into a Dictionary&lt;int, List&lt;CpdpFile&gt;&gt; keyed by BagianId.
3. Applies role-based tab filtering:
   - Get the current user as ApplicationUser (using UserManager or _userManager.GetUserAsync(User)).
   - If user.RoleLevel >= 5 (L5 = Coach, L6 = Coachee) AND user.Section is not null/empty:
     - Filter bagians to only those whose Name == user.Section (case-insensitive comparison).
     - If that filter results in zero bagians (Section doesn't match any bagian), fall back to showing all bagians.
   - If user.RoleLevel <= 4 OR user is Admin/HC (roles 1-4): show all bagians.
4. Pass to ViewBag:
   - ViewBag.Bagians = filtered bagians list
   - ViewBag.FilesByBagian = filesByBagian dictionary
   - ViewBag.SelectedBagianId = first bagian's Id (or 0 if none)
5. Return View() with NO model (model is now via ViewBag, same as Admin/CpdpFiles pattern).

Remove the old `section` parameter and MappingSectionSelect redirect logic entirely.

Important: The Mapping action already exists in CMPController — find it and replace the body and signature. The route stays [HttpGet] Mapping() with no parameters.

CMPController uses `_context` (ApplicationDbContext) and `_userManager` (UserManager&lt;ApplicationUser&gt;) — check that both are already injected. If _userManager is missing, add it to the constructor.
