# Bug Inventory - PlanIdp Page (94-01)

## Bug #1: Missing Indonesian Culture in Date Formatting

**Category:** Localization
**Severity:** Medium
**File:** `Controllers/CDPController.cs`
**Line:** 167

**Issue:**
```csharp
UploadedAt = f.UploadedAt.ToString("dd MMM yyyy")
```

The date formatting in the coaching guidance JSON does not include the Indonesian culture parameter. This will cause dates to display in the server's default locale (likely English) instead of Indonesian.

**Expected Fix:**
```csharp
using System.Globalization;

UploadedAt = f.UploadedAt.ToString("dd MMM yyyy", CultureInfo.GetCultureInfo("id-ID"))
```

**Impact:**
- Dates in the Guidance tab will show in English (e.g., "Mar 05, 2026") instead of Indonesian (e.g., "05 Mar 2026")
- Affects user experience for Indonesian users

## Bug #2: Null Safety Issue on UploadedAt - RESOLVED (FALSE ALARM)

**Category:** Null Safety
**Severity:** Low
**File:** `Models/ProtonModels.cs`
**Line:** 167

**Issue:**

**Investigation Result:**
```csharp
public DateTime UploadedAt { get; set; } = DateTime.UtcNow;
```

The model shows that `UploadedAt`:
1. Is a non-nullable `DateTime` struct (not `DateTime?`)

**Conclusion:** NOT A BUG - The code is already safe. No fix needed.
