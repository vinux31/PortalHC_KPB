## Bug Details

### BUG-01: Missing Indonesian locale in SrSpvApprovedAt date formatting

**Location:** `Controllers/CDPController.cs:1355`
**Severity:** Medium
**Category:** Localization

**Code:**
```csharp
SrSpvApprovedAt = p.SrSpvApprovedAt.HasValue ? p.SrSpvApprovedAt.Value.ToLocalTime().ToString("dd/MM/yyyy HH:mm") : "",
```

**Issue:** Date formatted with "dd/MM/yyyy HH:mm" format string but not using Indonesian locale (`CultureInfo.GetCultureInfo("id-ID")`)

**Impact:** Dates display in generic format instead of Indonesian locale (e.g., "05/03/2026" instead of "5 Mar 2026")

**Fix:** Add `CultureInfo.GetCultureInfo("id-ID")` parameter
