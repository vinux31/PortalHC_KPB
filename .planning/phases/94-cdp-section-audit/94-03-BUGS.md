## Critical Issues

### 1. Missing DownloadEvidence Action

**Location:** `Controllers/CDPController.cs` - not present
**Severity:** CRITICAL
**Impact:** Evidence download links (line 102 in Deliverable.cshtml) point directly to file paths, bypassing authorization and validation

**Current Behavior:**
```csharp
// Line 102 in Deliverable.cshtml
<a href="@Model.Progress.EvidencePath" target="_blank" class="btn btn-sm btn-outline-primary">
    <i class="bi bi-download me-1"></i>Download
</a>
```

**Issue:**
- Direct file path exposure (`/uploads/evidence/{id}/{filename}`)
- No access control (anyone with URL can download)
- No validation of file existence
- No proper content-type headers
- No audit trail

**Expected Fix:**
Add DownloadEvidence action with:
- Role-based access validation (Coachee, Coach, SrSpv, SectionHead, HC)
- File path validation (prevent directory traversal)
- File existence check
- Proper content-type and Content-Disposition headers
- Audit logging
