### Task 4: Document findings and prioritize fixes

**Deliverables:**
1. Bug list with severity ratings (Critical/High/Medium/Low)
2. Group bugs by commit strategy:
   - ManageWorkers bugs → commit 1
   - CoachCoacheeMapping bugs → commit 2
   - Validation fixes → commit 3
   - Role gate fixes → commit 4
3. Identify which bugs need browser verification vs code-only fixes

**Priority definitions:**
- **Critical:** Crashes, null references, raw exceptions shown to users
- **High:** Broken flows, incorrect data displayed, navigation failures
- **Medium:** UX issues (unclear text, missing links, confusing UI)
- **Low:** Cosmetic issues, typos, minor inconsistencies

**Output:** Create `95-01-BUGS.md` with categorized bug list
