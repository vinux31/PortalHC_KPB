## Overview

This plan fixes bugs that span multiple Admin pages. Per CONTEXT.md, cross-cutting concerns get separate commits for clear tracking. Focus areas: validation error handling consistency and role gate verification.

## Tasks

### Task 1: Fix validation error handling inconsistencies (Commit 3)

**Common validation bugs to fix:**

**Missing Model State Validation:**
```csharp
// BEFORE (no validation check)
[HttpPost]
public IActionResult CreateWorker(ManageUserViewModel model)
{
    // Directly processes without validation
    var user = new ApplicationUser { ... };
    // ...
}

// AFTER (with validation)
[HttpPost]
public IActionResult CreateWorker(ManageUserViewModel model)
{
    if (!ModelState.IsValid)
    {
        TempData["Error"] = "Mohon periksa kembali input Anda.";
        return View(model);
    }
    // Process...
}
```

**Raw Exception Exposure:**
```csharp
// BEFORE (raw exception to user)
catch (Exception ex)
{
    TempData["Error"] = $"Gagal: {ex.Message}";  // Exposes internal details
    return View();
}

// AFTER (generic message with logging)
catch (Exception ex)
{
    _logger.LogError(ex, "CreateWorker failed for {Email}", model.Email);
    TempData["Error"] = "Gagal membuat worker. Silakan coba lagi.";
    return View();
}
```

**Missing TempData Error Messages:**
```csharp
// Ensure all POST actions use TempData["Error"] on failures
if (existingUser != null)
{
    TempData["Error"] = $"User dengan email {email} sudah ada.";
    return View(model);
}
```

**Files to review:**
- `Controllers/AdminController.cs` — all POST actions
- `Views/Admin/*.cshtml` — all form views for TempData error display

**Commit message format:**
```
fix(admin): Improve validation error handling across Admin forms

- Add ModelState.IsValid checks to POST actions missing validation
- Replace raw exception messages with generic error messages
- Ensure TempData["Error"] used consistently for validation failures
- Add error logging for troubleshooting

```

### Task 2: Fix role gate inconsistencies (Commit 4)

**Common role gate bugs to fix:**

**Missing Authorize Attribute:**
```csharp
// BEFORE (no authorization)
public IActionResult SomeAdminAction()
{
    // ...
}

// AFTER (with authorization)
[Authorize(Roles = "Admin, HC")]
public IActionResult SomeAdminAction()
{
    // ...
}
```

**Incorrect Role Restrictions:**
```csharp
// Ensure sensitive operations are Admin-only
[Authorize(Roles = "Admin")]  // NOT "Admin, HC"

```
