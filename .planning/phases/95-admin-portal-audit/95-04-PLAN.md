# Plan 95-04: Browser Verification — Smoke Test Admin Flows

**Goal:** Verify all bug fixes work correctly in browser using smoke tests — quick verification that pages load and obvious bugs are fixed

**Requirements:** ADMIN-01, ADMIN-02, ADMIN-03, ADMIN-04, ADMIN-05, ADMIN-06, ADMIN-07, ADMIN-08

## Overview

This plan performs browser smoke testing of all Admin flows to verify bug fixes from plans 95-02 and 95-03. Uses existing test data from Phases 83 and 85. Focus on verifying specific bugs that were fixed, not comprehensive testing.

## Tasks

### Task 1: Verify ManageWorkers flow (ADMIN-01)

**Test as Admin user:**
1. Navigate to /Admin/ManageWorkers
   - [ ] Page loads without errors
   - [ ] Worker list displays correctly
   - [ ] Filters work (search by name/email/NIP, section dropdown, role dropdown)
   - [ ] showInactive toggle works (shows/hides inactive workers)
   - [ ] Pagination works (click page 2, verify different users shown)

2. Test Create Worker:
   - [ ] Navigate to /Admin/CreateWorker
   - [ ] Fill form with invalid data (empty name, invalid email)
   - [ ] Submit → verify validation error shown via TempData["Error"]
   - [ ] Fill form with valid data
   - [ ] Submit → verify worker created successfully
   - [ ] Check date fields display in Indonesian format if present

3. Test Edit Worker:
   - [ ] Click Edit on existing worker
   - [ ] Verify form loads with existing data
   - [ ] Modify data and submit
   - [ ] Verify changes saved successfully

4. Test Deactivate/Reactivate:
   - [ ] Click Nonaktifkan on active worker
   - [ ] Verify worker marked inactive
   - [ ] Toggle showInactive → verify worker appears in list
   - [ ] Click Aktifkan Kembali → verify worker reactivated

**Expected results:**
- No crashes or null reference exceptions
- All filters work correctly
- Pagination doesn't break on edge cases (page > totalPages)
- Dates displayed in Indonesian format (if any)
- Validation errors show user-friendly messages

### Task 2: Verify CoachCoacheeMapping flow (ADMIN-05)

**Test as Admin user:**
1. Navigate to /Admin/CoachCoacheeMapping
   - [ ] Page loads without errors
   - [ ] Coach groups display correctly with coachee lists
   - [ ] Filters work (search by name/NIP, section dropdown)
   - [ ] Pagination works (click page 2, verify different coaches shown)

2. Test Assign Coach:
   - [ ] Click Tambah Mapping button
   - [ ] Select coach and coachees from dropdowns
   - [ ] Submit → verify assignment succeeds
   - [ ] Check dropdowns don't show inactive users

3. Test Edit Mapping:
   - [ ] Click Edit on existing mapping
   - [ ] Verify modal loads with current data
   - [ ] Modify and submit
   - [ ] Verify changes saved

4. Test Deactivate/Reactivate:
   - [ ] Click Nonaktifkan on active mapping
   - [ ] Verify mapping marked inactive
   - [ ] Toggle showAll → verify mapping appears
   - [ ] Click Aktifkan Kembali → verify mapping reactivated

5. Test Export:
   - [ ] Click Export Excel button
   - [ ] Verify file downloads correctly
   - [ ] Check Excel file contains mapping data

**Expected results:**
- No crashes or AJAX errors
- Assign/edit/deactivate/reactivate operations complete successfully
- CSRF validation works (no 403/400 errors)
- Dates (StartDate/EndDate) displayed in Indonesian format
- Dropdowns filter out inactive users

### Task 3: Verify validation error handling (ADMIN-07)

**Test invalid submissions across Admin forms:**
1. ManageWorkers CreateWorker:
   - [ ] Submit empty form → validation error
   - [ ] Submit duplicate email → specific error message

2. CoachCoacheeMapping Assign:
   - [ ] Assign coach as own coachee → error message
   - [ ] Assign coachee who already has coach → error message

3. Import Workers:
   - [ ] Navigate to /Admin/ImportWorkers
   - [ ] Submit without file → error message
   - [ ] Upload invalid file type (e.g., .pdf) → error message
   - [ ] Download template → verify Excel file with correct headers

**Expected results:**
- All validation errors show user-friendly TempData["Error"] messages
- No raw exception details exposed to users
- Error messages guide users to fix the issue
