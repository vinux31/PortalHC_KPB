# Plan 96-01: Audit Profile Page & Cross-Page Concerns

**Created:** 2026-03-05
**Status:** Pending
**Owner:** gsd-executor

## Goal

Verify Profile page displays correct user data with proper null safety and avatar initials, audit cross-page concerns (CSRF, auth checks, navigation), and identify bugs for fixing.

## Success Criteria (must_haves)

1. Profile page renders without errors for users with complete and incomplete profile data
2. Avatar initials display correctly for multi-word names, single-word names, and empty names
3. All optional profile fields show "-" when null/empty (no null reference exceptions)
4. CSRF tokens are present in both Settings forms and [ValidateAntiForgeryToken] on POST actions
5. Authentication checks redirect unauthenticated users to Login page
6. Navigation links work correctly (Profile ↔ Settings)
7. All identified bugs are documented with severity and fix priority

## Tasks

### Task 1: Code Review - Profile Page Null Safety and Avatar Initials

**Action:** Review Profile.cshtml for null safety patterns and avatar initials logic

**Steps:**
1. Read `Views/Account/Profile.cshtml` (101 lines)
2. Verify all optional fields use `@if (!string.IsNullOrEmpty(Model.XXX))` pattern:
   - FullName, NIP, Email, Telepon, Direktorat, Bagian, Unit, Jabatan
3. Verify avatar initials logic (lines 5-9) handles edge cases:
   - Empty/null name → "?"
   - Single-word name → first 2 characters (if length >= 2)
   - Multi-word name → first character of first 2 words
4. Verify role display shows "No Role" for users without role
5. Document any bugs found with severity (Critical/High/Medium/Low)

**Acceptance Criteria:**
- All fields verified for null safety
- Avatar initials logic verified for all edge cases
- Bug report created if issues found

**Files to Read:**
- `Views/Account/Profile.cshtml`

### Task 2: Code Review - CSRF Protection and Authentication Checks

**Action:** Verify CSRF tokens in Settings forms and authentication checks in AccountController

**Steps:**
1. Read `Views/Account/Settings.cshtml` (183 lines)
2. Verify CSRF token present in EditProfile form (line 35): `@Html.AntiForgeryToken()`
3. Verify CSRF token present in ChangePassword form (line 147): `@Html.AntiForgeryToken()`
4. Read `Controllers/AccountController.cs` (264 lines)
5. Verify `[ValidateAntiForgeryToken]` attribute on EditProfile POST (line 187)
6. Verify `[ValidateAntiForgeryToken]` attribute on ChangePassword POST (line 221)
7. Verify authentication checks in Profile action (lines 130-146):
   - `if (User.Identity?.IsAuthenticated != true) return RedirectToAction("Login");`
   - `if (user == null) return RedirectToAction("Login");`
8. Verify authentication checks in Settings action (same pattern)
9. Document any missing CSRF or auth issues

**Acceptance Criteria:**
- CSRF tokens verified in both forms
- ValidateAntiForgeryToken verified on both POST actions
- Authentication checks verified in Profile and Settings actions
- Bug report created if issues found

**Files to Read:**
- `Views/Account/Settings.cshtml`
- `Controllers/AccountController.cs`
