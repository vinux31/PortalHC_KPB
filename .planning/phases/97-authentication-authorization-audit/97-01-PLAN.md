# Plan 97-01: Authorization Matrix Audit

> Build comprehensive authorization matrix via exhaustive grep audit of all controllers and views

## Goal

Create exhaustive authorization matrix documenting every controller action's authentication requirements (public, authenticated, role-gated) and identify security gaps. Verify cookie security settings meet minimum baseline (httpOnly, secure, sameSite).

## Requirements Mapping

- **AUTH-01** (Login flow works correctly): Audit authorization gates for login/logout actions
- **AUTH-02** (Inactive users blocked): Verify inactive user check exists and is placed correctly before AD sync

## Success Criteria

1. Authorization matrix documents all controller actions with `[Authorize]` attributes across 6 controllers (Account, Admin, CMP, CDP, Home, ProtonData)
2. Matrix categorizes actions by access type: Public (Login, AccessDenied), Authenticated (no role requirement), Role-gated (Admin, HC, SrSpv, SectionHead, Coach, Coachee)
3. Security gaps identified: Actions missing role gates, inconsistent role names, manual checks that could use attributes
4. Cookie security settings verified in Program.cs: httpOnly, secure, sameSite attributes documented
5. Inactive user block verified at correct location (AccountController line 72-76, before AD sync)

### Task 97-01-03: Identify security gaps and recommendations

**Action:** Analyze matrix for security gaps and create fix recommendations

**Analysis checklist:**

1. **Missing role gates:**
   - Are all Admin/HC-only actions protected with `[Authorize(Roles = "Admin, HC")]`?
   - Are Admin-only delete actions protected with `[Authorize(Roles = "Admin")]`?
   - Any sensitive actions missing role requirements?

2. **Inconsistent role name formatting:**
   - Some attributes use "Admin, HC" (with space)
   - Some use "Admin,HC" (without space)
   - Both work but create inconsistency - document all instances

3. **Manual checks that could use attributes:**
   - AccountController Profile/Settings use manual auth checks (lines 132-134, 152-155)
   - These should use `[Authorize]` attribute instead for consistency

4. **Cookie security gaps:**
   - HttpOnly flag set? (prevents XSS cookie theft)
   - Secure flag set? (HTTPS-only, if SSL enabled)
   - SameSite flag set? (Strict or Lax for CSRF prevention)

5. **Inactive user block:**
   - Is check at correct location (before AD sync)?
   - Is error message user-friendly (Indonesian)?
   - Does it block both local and AD modes?

**Output:** Add "Security Gaps Identified" section to matrix document with:
- Critical gaps (must fix in plan 97-03)
- Medium gaps (should fix if time permits)
- Low gaps (nice to fix, defer to future)

**Verification:**
- All security gaps categorized by severity
- Recommendations documented for each gap
- Fix priority assigned (critical for plan 97-03, others for future)
