# Plan 97-02: Browser Verification - Critical Auth Flows

> Spot-check critical authentication and authorization flows via manual browser testing

## Goal

Create browser verification guide for critical auth flows (login, AccessDenied, navigation visibility, return URL security) and execute spot checks to identify bugs. Focus on high-risk areas identified in authorization matrix from plan 97-01.

## Requirements Mapping

- **AUTH-03** (AccessDenied page displays): Verify unauthorized access attempts show AccessDenied page
- **AUTH-04** (Role-based navigation visibility): Verify Kelola Data menu shows for Admin/HC only
- **AUTH-05** (Return URL redirect secure): Verify open redirect protection works correctly

## Success Criteria

1. Browser verification guide created with step-by-step instructions for 4 critical flows
2. Each flow has clear test steps, expected results, and bug reporting template
3. Guide includes test data requirements (1 user per role: Admin, HC, SrSpv, SectionHead, Coach, Coachee)
4. All critical flows tested and results documented (PASS/FAIL with evidence)
5. Any bugs found are documented with reproduction steps and severity level
