# Plan 98-01: IsActive Filter Consistency Audit

> Exhaustive grep audit of all IsActive filters across entities with spot-check verification

## Goal

Audit all IsActive filter usage across the portal to identify gaps where soft-deleted records (IsActive=false) may leak to user-facing queries. Document all entities with IsActive fields, verify consistent filtering in user-facing queries, and identify any missing filters.

## Requirements Mapping

- **DATA-01** (All IsActive filters applied consistently): Verify Workers, Silabus, Assessments, and all entities with IsActive fields filter correctly in queries

## Success Criteria

1. All entities with IsActive fields documented with model locations and line numbers
2. Exhaustive grep audit identifies every query that should filter by IsActive
3. Missing IsActive filters categorized by severity (critical = UI leaks, medium = internal queries, low = admin-only)
4. Spot-check verification of high-risk queries confirms no deleted records leak to UI
5. Fix recommendations documented for plan 98-03

### Task 98-01-03: Identify missing IsActive filters - High-risk queries

**Action:** Analyze high-risk user-facing queries for missing IsActive filters

**High-risk query categories:**

1. **User-facing list views:**
   - ManageWorkers table (AdminController)
   - CoachCoacheeMapping table (AdminController)
   - Assessment lists (AdminController, CMPController)
   - Silabus lists (ProtonDataController)

2. **Dropdown/populated selects:**
   - Worker dropdowns (ManageWorkers, CoachCoacheeMapping)
   - Coach dropdowns (CoachCoacheeMapping)
   - Silabus dropdowns (Plan IDP, Assessment create)

3. **Coaching Proton queries:**
   - Coachee list per coach
   - Coach list per coachee
   - Track assignments

**Analysis checklist:**

For each high-risk query category:

1. **Does the query filter by IsActive?**
   - Check grep results for `.Where(entity.IsActive)`
   - If yes → document as PASS
   - If no → document as GAP

2. **Does the query use showInactive toggle pattern?**
   - Check for `(showInactive || entity.IsActive)`
   - Verify default value (should be false - show only active)
   - If toggle exists → document as PASS with toggle
   - If no toggle and no filter → document as CRITICAL GAP

3. **Cross-entity consistency checks:**
   - Does query check parent.IsActive? (e.g., `.Where(m => m.Coach.IsActive)`)
   - Does query check child.IsActive? (e.g., `.Where(u => u.ProtonTrackAssignments.Any(a => a.IsActive))`)
