# Phase 999.17: Excel "zero-config" template — dropdown Data Validation + import skor per-soal - Context

**Gathered:** 2026-06-30
**Status:** Ready for planning

<domain>
## Phase Boundary

Bikin template Excel soal jadi "zero-config" (anti-salah, self-explanatory) untuk admin/HC:
1. **Dropdown Data Validation** pada kolom data-pasti `QuestionType` di template unduhan, supaya admin pilih dari daftar (MultipleChoice/MultipleAnswer/Essay) alih-alih ketik manual.
2. **Import skor (bobot) per-soal** lewat Excel: kolom `Skor` baru di template, ke-parse saat `ImportPackageQuestions` jadi `PackageQuestion.ScoreValue` (sekarang hardcode 10).

**OUT of scope:** skor HASIL peserta (sudah ada di Inject v32.2); paritas form authoring (SUDAH ADA — lihat code_context); tipe soal baru; breakdown skor per-section; dropdown pada kolom teks-bebas / dinamis.
</domain>

<decisions>
## Implementation Decisions

### Dropdown Data Validation (Fitur 1)
- **D-01:** Dropdown list HANYA pada kolom **`QuestionType`** (3 nilai tetap: `MultipleChoice`, `MultipleAnswer`, `Essay`). Satu-satunya kolom dengan set nilai tetap & terbatas.
- **D-02:** Dropdown diterapkan ke **semua template** (Universal 13/14-kolom + legacy MC/MA/Essay 9-kolom). DataValidation = validation-only, TAK mengubah struktur kolom → non-breaking untuk dual-format detection (`isNewFormat`).
- **D-03:** Cakupan baris **lebar: baris 2–1000** (range), bukan cuma baris contoh — admin dapat dropdown begitu mengetik di baris baru mana pun.
- **D-04:** Kolom **`Jawaban Benar` SKIP** (TIDAK dapat dropdown). Alasan: di Universal isinya campur — MC = 1 huruf, MA = comma-list (`A,C`), Essay = kosong; list-validation akan menolak `A,C` & memblokir Essay kosong → korup. (Opsi per-varian "A–D di template MC saja" DITOLAK user demi kesederhanaan.)
- **D-05:** Kolom lain (Pertanyaan, Opsi A–F, No. Section [angka dinamis], Nama Section, Elemen Teknis, Rubrik) = TANPA dropdown (teks bebas / nilai dinamis).

### Import Skor per-soal (Fitur 2)
- **D-06:** Makna "skor per-soal" = **bobot poin per soal** = field `PackageQuestion.ScoreValue` (int, default 10). BUKAN skor hasil peserta. Field & grading-weighted SUDAH ADA; gap tunggal = import.
- **D-07:** Tambah kolom **`Skor`** HANYA di template **Universal** (jadi 14-kolom). Legacy 9-kolom tetap apa adanya (bobot = default, kompatibel-mundur). Hindari otak-atik header-marker detection legacy.
- **D-08:** Default saat kolom Skor **kosong/absen = 10** (konsisten `ScoreValue` default existing). Tidak wajib diisi.
- **D-09:** Validasi server saat import: **integer positif 1–100**. Tolak ≤0, >100, desimal, non-angka (`ScoreValue` bertipe int).
- **D-10:** Kolom Skor juga diberi **Data Validation WholeNumber Between 1–100** di sel (range baris 2–1000) — anti salah-ketik di sumber, selaras tema zero-config. Server TETAP validasi ulang (jangan percaya client).
- **D-11:** Auto-normalisasi skor akhir ke **100 SUDAH ADA** — `finalPercentage = (int)((double)totalScore / maxScore * 100)` (`GradingService.cs:144`). Berapa pun total bobot mentah, skor akhir = persentase 0–100. TIDAK perlu kerjaan baru; cuma diverifikasi/dikunci dengan test. (Catatan: hasil di-truncate ke int — perilaku lama, di luar scope.)

### Claude's Discretion
- Nama header kolom persis ("Skor") + posisi kolom dalam header Universal (saran: setelah ScoreValue-relevan; planner tentukan agar dual-format detection tetap utuh).
- Cara wiring DataValidation ClosedXML (`ws.Range(...).CreateDataValidation().List(...)` untuk QuestionType; `.WholeNumber.Between(1,100)` untuk Skor) — detail API ke planner/researcher.
- Apakah baris contoh tetap ditampilkan di range yang sama dengan DataValidation.
</decisions>
