### `DownloadQuestionTemplate` (controller, file-I/O generate) — tambah DataValidation + kolom Skor

**Primary analog:** `Helpers/InjectExcelHelper.GenerateTemplate` (`Helpers/InjectExcelHelper.cs:44–135`) untuk bentuk "pure static `XLWorkbook` builder". **Secondary analog:** method `DownloadQuestionTemplate` itu sendiri (pola header/example/instruction yang harus dipertahankan).

**Header build pattern** (`AssessmentAdminController.cs:7279–7289`) — yang HARUS dimodifikasi untuk Universal jadi 14-kolom (append `"Skor"` setelah `"Rubrik"`; legacy 9-kolom JANGAN disentuh — D-07):
```csharp

    : new[] { "Pertanyaan", "Opsi A", "Opsi B", "Opsi C", "Opsi D", "Jawaban Benar", "Elemen Teknis", "QuestionType", "Rubrik" };
for (int i = 0; i < headers.Length; i++)
{

    ws.Cell(1, i + 1).Style.Font.Bold = true;
    ws.Cell(1, i + 1).Style.Fill.BackgroundColor = XLColor.FromHtml("#16A34A");
    ws.Cell(1, i + 1).Style.Font.FontColor = XLColor.White;
```

> Kolom map Universal: Pertanyaan=1 … Jawaban Benar=8 … ET=11, **QuestionType=12 (L)**, Rubrik=13 (M), **[BARU] Skor=14 (N)**. Legacy: **QuestionType=8 (H)**.

**Example-row pattern** (`AssessmentAdminController.cs:7293–7320`) — contoh QuestionType sudah memakai enum persis (`"MultipleChoice"`/`"MultipleAnswer"`/`"Essay"`), jadi nilai contoh valid terhadap dropdown (RESEARCH Pitfall 5). Jika kolom Skor di-append, contoh Universal `string[]` bertambah elemen ke-14 (mis. `"10"` atau dikosongkan — Claude's Discretion):
```csharp

// contoh Universal saat ini 13-elemen (7307): "...","MultipleChoice","" → jadi 14-elemen bila Skor di-append
```

**Stream-out pattern** (`AssessmentAdminController.cs:7346–7347`) — JANGAN ubah; `ToFileResult` auto-fit + save:

### `ImportPackageQuestions` POST (controller, file-I/O parse + CRUD) — parse Skor + atomic-reject

**Analog:** method itu sendiri. Tiga sub-pola yang diperluas + satu pola error yang DI-ADOPSI.

**1. Dual-format detection — JANGAN rusak (RESEARCH Pitfall 1)** (`AssessmentAdminController.cs:7459–7467`):
```csharp
string HeaderAt(int col) => System.Text.RegularExpressions.Regex
    .Replace((ws.Row(1).Cell(col).GetString() ?? "").Trim(), @"\s+", " ").ToLowerInvariant();
int colCount = ws.Row(1).LastCellUsed()?.Address.ColumnNumber ?? 0;
bool isNewFormat = colCount > 9
    && HeaderAt(6) == "opsi e"
    && HeaderAt(7) == "opsi f"
    && HeaderAt(9) == "no. section"
    && HeaderAt(10) == "nama section";
```
> Append Skor di kolom 14 TIDAK menggeser marker 6/7/9/10 → `isNewFormat` tetap utuh. File Universal 13-kolom lama (tanpa Skor) tetap terdeteksi NEW; `Cell(14)` kosong → default 10 (kompatibel-mundur).

**2. Row-tuple + isNewFormat read-branch** (`AssessmentAdminController.cs:7426`, `:7479–7516`) — tuple 13-field saat ini; tambah Score (parse `row.Cell(14)` hanya di branch `isNewFormat`; legacy → null/default):

```csharp
// Branch NEW (7479–7498) baca Cell(1..13); Rubrik dibaca di 7497:

// [BARU] setelah Rubrik: var rawScore = (row.Cell(14).GetString() ?? "").Trim();  // kolom N
// Branch OLD (7499–7516): tanpa Skor → score = null/default.
```
> Planner pilih: perlebar tuple jadi 14-field, ATAU bawa skor di list paralel. Tuple sudah 13-field besar — perlebar konsisten dgn pola existing (mengikuti cara Phase 415 menambah Opsi E/F + Section ke tuple yang sama).

**4. Parse + validasi Skor BARU (excerpt verified, RESEARCH §Code Examples)** — `System.Globalization` SUDAH di-import (`AssessmentAdminController.cs:12`):

**5. Error-surface pattern yang DI-ADOPSI — TOLAK-KERAS ATOMIC (D-12)** — salin pola `SectionMismatch` (`AssessmentAdminController.cs:7703–7708`):
```csharp
if (mismatchList.Any())
```

> Untuk Skor: kumpulkan `scoreErrors` SEBELUM build/persist, jika `.Any()` → `TempData["ScoreErrors"] = JsonSerializer.Serialize(scoreErrors)` + `RedirectToAction` + **0 write**. JANGAN pakai pola `errors.Add(...) ; continue;` (loop build `:7757/7767/7783/7796/7810`) — list `errors` itu TIDAK pernah disurface di success-path (silent swallow; RESEARCH Pitfall 2). Skor wajib pola tolak-keras yang TERSURFACE.
>
> ⚠️ Penempatan: validasi Skor harus jalan SEBELUM transaksi persist (`:7863–7882`) agar benar-benar 0-write. Idealnya satu pass validasi Skor atas semua `rows` mendahului build-loop, mirip cara guard SectionMismatch berjalan sebelum loop.

Planner bisa: (a) reuse key `SectionMismatch` (gabung pesan) ATAU (b) tambah blok kembar untuk `TempData["ScoreErrors"]` (salin blok 30–51 dengan judul "Skor tidak valid"):

```csharp
private static IFormFile BuildFile(string[] headers, IEnumerable<string[]> dataRows)
{
    using var wb = new XLWorkbook();
    var ws = wb.Worksheets.Add("Question Import");
    for (int i = 0; i < headers.Length; i++) ws.Cell(1, i + 1).Value = headers[i];
    int r = 2;
    foreach (var row in dataRows)
    {
        for (int c = 0; c < row.Length; c++)
            if (!string.IsNullOrEmpty(row[c])) ws.Cell(r, c + 1).Value = row[c];
        r++;
    }
    var ms = new MemoryStream(); wb.SaveAs(ms); ms.Position = 0;
    return new FormFile(ms, 0, ms.Length, "excelFile", "import.xlsx")
    { Headers = new HeaderDictionary(), ContentType = "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet" };
}
```

**Apply to:** parse Skor. `int.TryParse(rawScore, NumberStyles.None, CultureInfo.InvariantCulture, out v)` + range 1–100. Client DataValidation = hint anti-typo SAJA, server WAJIB re-validate.

**Apply to:** `DownloadQuestionTemplate` — JANGAN ubah; `XLWorkbook → AdjustToContents → SaveAs(MemoryStream) → controller.File(..., xlsx MIME, fileName)`.
