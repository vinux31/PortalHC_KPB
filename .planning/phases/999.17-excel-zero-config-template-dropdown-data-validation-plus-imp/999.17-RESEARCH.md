## Summary

Fase ini menambah dua kapabilitas kecil-tapi-presisi pada satu jalur kode yang sudah matang (`AssessmentAdminController.DownloadQuestionTemplate` + `ImportPackageQuestions`). Keduanya **murni di tier API/Backend** ŌĆö tidak menyentuh DB schema (no migration), tidak menyentuh grading, tidak menyentuh form authoring. Risiko terbesar BUKAN API ClosedXML (sudah diverifikasi penuh) melainkan **regresi senyap pada dual-format detection** (header-marker `isNewFormat` di `AAC:7463`) saat kolom `Skor` ditambahkan.

ClosedXML `DataValidation` adalah **greenfield di repo ini** (`git grep DataValidation` = 0), tapi library 0.105.0 mendukung penuh. API diverifikasi langsung dari `ClosedXML.dll` terpasang via reflection: `IXLRangeBase.CreateDataValidation()` ŌåÆ `IXLDataValidation` dengan `.List(string, bool)` (dropdown inline) dan `.WholeNumber.Between(int, int)`. Read-back untuk test tersedia via `IXLWorksheet.DataValidations` (enumerable `IXLDataValidation`) + `.GetAllInRange(...)`/`.TryGet(...)`.

Gap import = satu hardcode (`ScoreValue = 10` di `AAC:7832`). Kolom `Skor` paling aman **di-append sebagai kolom 14** (setelah `Rubrik`) ŌĆö tidak menggeser satupun marker deteksi (posisi 6/7/9/10) dan kompatibel-mundur dengan file Universal 13-kolom yang sudah beredar (Cell(14) kosong ŌåÆ default 10). Pola test sudah ada persis (`SectionImportTests.cs`): bangun `.xlsx` nyata via ClosedXML ŌåÆ IFormFile stub ŌåÆ panggil action asli ŌåÆ assert DB.

**Primary recommendation:** Ekstrak generator template ke helper pure static `QuestionTemplateBuilder.Build(string type) ŌåÆ XLWorkbook` (meniru `InjectExcelHelper`), pasang DataValidation di sana ŌåÆ unit-testable tanpa DB; tambah field Skor sebagai kolom 14 Universal + parse di branch isNewFormat; pakai pola "tolak-keras atomic" (seperti SectionMismatch) untuk Skor invalid (lihat Open Question Q1). Migration = FALSE.

### Pattern 1: Pure static template-builder (ekstraksi seam)
**What:** Pindahkan logika bangun-workbook dari controller ke `QuestionTemplateBuilder.Build(string type) ŌåÆ XLWorkbook`. DataValidation dipasang di helper.
**When to use:** Selalu untuk fase ini ŌĆö membuat DataValidation (greenfield) unit-testable TANPA membangun controller (yang butuh ~12 ctor arg) dan TANPA DB/fixture.
**Why:** Selaras pola repo (`InjectExcelHelper.GenerateTemplate`, `ShuffleEngine`, `SectionStructureComparer`, `CoerceCoachUnitScope`) ŌĆö "ekstrak seam pure" yang dipuji di STATE.md. Controller jadi tipis: `return ExcelExportHelper.ToFileResult(QuestionTemplateBuilder.Build(type), fileName, this);`

### Pattern 2: DataValidation pada range kolom (verified)
**What:** Pasang aturan ke seluruh kolom baris 2ŌĆō1000 sekali, bukan per-sel.
**Example:**
```csharp
// Source: ClosedXML 0.105.0 (verified DLL reflection) + wiki Data-Validation
// QuestionType dropdown (D-01/02/03) ŌĆö Universal kolom L=12, legacy kolom H=8
```

### Pattern 3: Tambah kolom tanpa merusak dual-format detection
**What:** Append `Skor` sebagai kolom 14 (N) ŌĆö setelah `Rubrik` (13/M). Header Universal jadi 14-kolom.
**Why aman:** `isNewFormat` (`AAC:7463`) hanya menguji `colCount > 9` + `HeaderAt(6)=="opsi e"` + `(7)=="opsi f"` + `(9)=="no. section"` + `(10)=="nama section"`. Append kolom 14 tidak menggeser satupun. File Universal 13-kolom lama tetap terdeteksi NEW; `Cell(14)` kosong ŌåÆ default 10 (kompatibel-mundur).

### Anti-Patterns to Avoid
- **Sisip Skor sebelum QuestionType / sebelum Section:** menggeser indeks parser (ET/Type/Rubrik) DAN berisiko menggeser marker 6/7/9/10 ŌåÆ korup dual-format. JANGAN.
- **Dropdown pada `Jawaban Benar`:** D-04 ŌĆö campur format MC/MA/Essay ŌåÆ list-validation menolak `A,C` & memblok Essay kosong. JANGAN.
- **Pasang DataValidation per-sel dalam loop 999 baris:** boros + ClosedXML konsolidasi range; pakai satu `Range("X2:X1000")`. 
- **Percaya validasi client (DV) sebagai otoritatif:** D-10 ŌĆö server WAJIB re-validate; DV hanya hint anti-typo.
- **Ubah `GradingService`:** D-11 ŌĆö normalisasi sudah ada; cuma kunci dengan test.
- **Hapus/ubah jalur legacy 9-kolom:** D-07 ŌĆö biarkan apa-adanya.

## Common Pitfalls

### Pitfall 1: Penambahan kolom Skor merusak `isNewFormat` (regresi senyap)
**What goes wrong:** Kolom Skor disisipkan di tengah ŌåÆ marker header (Opsi E@6, Opsi F@7, No. Section@9, Nama Section@10) bergeser ŌåÆ file salah-deteksi ŌåÆ peta kolom geser ŌåÆ korupsi senyap (jawaban/section nyasar).
**Why:** Detektor mengandalkan POSISI kolom 6/7/9/10 (`AAC:7463-7467`), bukan colCount saja (ini sengaja ŌĆö M2 fix Phase 415).
**How to avoid:** Append `Skor` sebagai kolom **14** (setelah Rubrik). Tidak menggeser marker. Tambahkan regression test yang mengimpor file 13-kolom lama (tanpa Skor) ŌåÆ tetap terdeteksi NEW + ScoreValue=10.
**Warning signs:** Test `SectionImportTests.LegacyFormat_*` atau `NewFormat_*` mulai gagal; section/correct-letter nyasar.

### Pitfall 2: Error Skor invalid TER-TELAN (silent skip) ŌĆö bukan "Tolak" sesuai D-09
**What goes wrong:** Pola per-baris existing memakai `errors.Add(...) ; continue;` ŌĆö TAPI list `errors` ini **tidak pernah disurface ke user di success-path** (hanya dipakai di early-return empty-worksheet, `AAC:7449`). Jadi baris ber-Skor invalid akan **drop senyap** (soal hilang sama sekali), bukan default 10, bukan pesan jelas. Bertentangan dengan tema zero-config & D-09 "Tolak".
**Why:** Loop build hanya `continue` pada baris invalid; final return cuma TempData Success/Warning + count. `errors` di-collect tapi tak ditampilkan.
**How to avoid:** Pakai pola **tolak-keras atomic** yang sudah terbukti (`SectionMismatch`, `AAC:7703-7708`): kumpulkan error Skor, jika ada ŌåÆ serialize ke `TempData["...Error"]` + `RedirectToAction` + **0 write**. View sudah punya scaffold render daftar (lihat `ImportPackageQuestions.cshtml:33-46`). Lihat Open Question Q1 untuk keputusan scope.
**Warning signs:** Admin impor 140 soal, satu Skor typo, hasil "139 added" tanpa tahu yang ke-140 hilang.

### Pitfall 3: Inline-list double-quote & limit 255 char
**What goes wrong:** Lupa membungkus nilai dengan literal double-quotes ŌåÆ ClosedXML/Excel menafsirkan string sebagai formula/range, dropdown tak muncul.
**Why:** Excel membedakan literal-list (`"a,b,c"`) vs formula via tanda kutip.
**How to avoid:** String harus `"\"MultipleChoice,MultipleAnswer,Essay\""` (37 char, < limit 255). `[CITED: ClosedXML wiki Data-Validation]`
**Warning signs:** Dropdown kosong/absen di Excel; atau error "list source must be delimited".

### Pitfall 4: `.xls` lama tak terbaca ClosedXML (pre-existing, di luar scope)
**What goes wrong:** Endpoint mengizinkan ekstensi `.xls` (`AAC:7375`) tapi ClosedXML hanya membaca OOXML `.xlsx` ŌåÆ file `.xls` biner asli melempar exception (sudah ditangkap ŌåÆ TempData Error generik).
**Why:** Pre-existing; bukan diintroduksi fase ini.
**How to avoid:** Tidak perlu diubah (di luar scope). Template DOWNLOAD selalu `.xlsx` ŌåÆ DataValidation persist sempurna. Cukup catat agar tak bingung saat test.

### Pitfall 5: Baris contoh vs DataValidation pada range yang sama
**What goes wrong:** Khawatir baris contoh (gray text, rows 2ŌĆō4) bentrok dengan DV pada L2:L1000 / N2:N1000.
**Why:** Tidak bentrok ŌĆö nilai contoh QuestionType ("MultipleChoice" dst) valid terhadap list; baris instruksi ada di kolom A saja (bukan L/N). DV hanya menegur saat EDIT, tidak menandai nilai existing.
**How to avoid:** Biarkan contoh tetap (Claude's Discretion). Pastikan nilai contoh QuestionType persis salah satu dari 3 enum (sudah benar di `AAC:7307-7320`).

## Open Questions

1. **Scope penolakan Skor invalid: skip-baris vs tolak-keras seluruh import?**
   - What we know: Pola per-baris existing = `errors.Add + continue` TAPI `errors` tak pernah disurface (silent). Pola "tolak-keras atomic" sudah ada & terbukti = `SectionMismatch` (0 write + daftar lengkap via TempData + view scaffold siap).
   - What's unclear: D-09 "Tolak" tidak eksplisit menyebut tolak-baris vs tolak-import.
   - Recommendation: **Tolak-keras atomic** (jika ada Ōēź1 Skor invalid ŌåÆ 0 write, surface daftar baris bermasalah, mirip SectionMismatch). Alasan: silent-skip akan men-drop soal tanpa jejak (lebih buruk dari default), bertentangan zero-config. Angkat ke `/gsd-discuss-phase` bila ragu.

2. **DataValidation pada template legacy: pasang juga numeric? (Tidak)**
   - What we know: D-07 ŌĆö kolom Skor HANYA Universal. Legacy 9-kolom tanpa Skor.
   - Recommendation: Legacy dapat **dropdown QuestionType saja** (D-02, kolom H=8). TANPA kolom/DV Skor. Sudah jelas dari decisions; dicatat agar planner tak keliru menambah Skor ke legacy.

3. **Range 1000 baris cukup?**
   - What we know: D-03 menetapkan baris 2ŌĆō1000.
   - Recommendation: Pakai 1000 sesuai D-03. Jika import >999 soal sekaligus, DV tak menjangkau baris >1000 (data tetap ter-import, hanya tanpa hint dropdown ŌĆö server validasi tetap jalan). Acceptable; jangan over-engineer.

## Security Domain

Endpoint import = attack surface (file upload). Fase ini menambah satu input-validation surface (Skor). Mayoritas kontrol sudah ada.

### Applicable ASVS Categories
| ASVS Category | Applies | Standard Control |
|---------------|---------|-----------------|
| V2 Authentication | yes (existing) | `[Authorize(Roles="Admin, HC")]` pada Download & Import `[VERIFIED: AAC:7266,7367]` |

| V5 Input Validation | **yes (BARU)** | Skor: `int.TryParse` + range 1ŌĆō100 server-side (D-09). Jangan percaya DV client (D-10) |
| V6 Cryptography | no | Tidak ada kripto baru |
| V12 File Upload | yes (existing) | Guard ekstensi `.xlsx/.xls` + size Ōēż5MB + try/catch parse `[VERIFIED: AAC:7373-7389]` |

### Known Threat Patterns for ClosedXML import
| Pattern | STRIDE | Standard Mitigation |
|---------|--------|---------------------|
| Skor injection (negatif/overflow/desimal) men-skew grading | Tampering | Server re-validate 1ŌĆō100, `NumberStyles.None`, tipe `int` (D-09) ŌĆö JANGAN andalkan DV client |
| Oversized/zip-bomb `.xlsx` | DoS | Size guard 5MB existing; ClosedXML stream read |
| CSV/formula injection di sel teks | Injection (downstream) | Out of scope (Skor=numeric; teks soal sudah ditangani existing rendering) |
| Bypass antiforgery / role | Spoofing/Elevation | `[ValidateAntiForgeryToken]` + `[Authorize]` existing ŌĆö pertahankan |
