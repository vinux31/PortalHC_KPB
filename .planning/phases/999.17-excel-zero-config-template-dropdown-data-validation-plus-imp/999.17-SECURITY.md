## Checklist-specific findings (8-point audit)

### 1. RBAC тАФ CLOSED
Kedua endpoint mempertahankan `[Authorize(Roles="Admin, HC")]`:
- `DownloadQuestionTemplate` тАФ AssessmentAdminController.cs:7266
- `ImportPackageQuestions` (GET :7278, POST :7294)

### 2. CSRF тАФ CLOSED
POST `ImportPackageQuestions` mempertahankan `[ValidateAntiForgeryToken]` (AssessmentAdminController.cs:7295). Kedua form view memanggil `@Html.AntiForgeryToken()` (ImportPackageQuestions.cshtml:133 file-tab, :154 paste-tab).

### 3. Server-authoritative validation тАФ CLOSED
Skor di-revalidasi server-side terlepas dari DataValidation client (D-10):
```
if (!int.TryParse(rawScore, NumberStyles.None, CultureInfo.InvariantCulture, out var sv)
    || sv < 1 || sv > 100)
    scoreErrors.Add($"Baris {i + 2}: Skor harus angka bulat 1-100. Ditemukan '{rawScore}'.");
```
(AssessmentAdminController.cs:7518-7520). `NumberStyles.None` menolak desimal/whitespace/tanda.
Crafted upload ditolak:
- `Skor=99999` тЖТ lolos TryParse tapi `sv>100` тЖТ reject
- `"abc"` тЖТ TryParse gagal тЖТ reject
- `"-5"` тЖТ `NumberStyles.None` (tanpa AllowLeadingSign) тЖТ TryParse gagal тЖТ reject
- `"5.5"` тЖТ `NumberStyles.None` (tanpa AllowDecimalPoint) тЖТ TryParse gagal тЖТ reject

Build-loop juga re-parse + clamp 1-100 dengan default aman 10 (:7782-7787) тАФ defense in depth.

### 4. File-upload guards тАФ CLOSED, tidak diregresi
Ekstensi whitelist `.xlsx/.xls` (:7302-7308) + ukuran тЙд5MB (:7312-7316) tetap mendahului parsing. Tidak disentuh diff 999.17.

### 5. Injection / XSS pada ScoreErrors тАФ CLOSED (highest-novelty risk)
`scoreErrors` adalah string yang MENYISIPKAN konten sel attacker-controlled: `Ditemukan '{rawScore}'.` (AssessmentAdminController.cs:7520). String di-serialize JSON ke `TempData["ScoreErrors"]` (:7524), lalu di-deserialize & render di view.

Render menggunakan Razor `@line` (auto-HTML-encode):
```
@foreach (var line in scoreErrors)
{
    <li>@line</li>
}
```
(ImportPackageQuestions.cshtml:67-69). Razor `@` meng-encode HTML тАФ payload sel mis. `<script>alert(1)</script>` keluar sebagai `&lt;script&gt;...`. **TIDAK ada `Html.Raw` / `@Html.Raw`** pada blok ScoreErrors maupun blok kembar SectionMismatch (grep nihil). XSS via konten sel ter-mitigasi.

Catatan kedalaman: JSON deserialize dibungkus try/catch тЖТ `null` (ImportPackageQuestions.cshtml:56-60); render hanya bila `Count>0`. Tidak ada jalur output mentah.

### 6. Atomic integrity тАФ CLOSED
Validasi skor (pass mandiri :7514-7526) mendahului build-loop (:7780+) dan persist transaction. Bila ada satu baris invalid тЖТ `return RedirectToAction(...)` SEBELUM `_context.PackageQuestions.AddRange` / `SaveChangesAsync` тЖТ 0 write (D-12). Persist sendiri atomik: `BeginTransactionAsync` + `CommitAsync`/`RollbackAsync` (:7828-7847). Tidak ada partial import.

### 7. DoS тАФ CLOSED
Dibatasi guard 5MB (:7312-7316). Loop parse skor iterasi `rawScores.Count` (lockstep jumlah baris terbaca), tanpa alokasi rekursif/tak-terbatas baru. Range DataValidation `N2:N1000` hanya metadata template (sisi download), bukan beban parse.

### 8. No new authz bypass тАФ CLOSED
Session-edit-lock guard (Post-Test terkunci) tetap di AWAL endpoint, SEBELUM parsing skor: `SessionEditLockRules.IsSessionEditLocked` (:7329-7333). Parsing/validasi skor tidak melompati guard ini maupun pengecekan `pkg == null` (:7324). Tidak ada jalur baru yang menulis ScoreValue di luar `ImportPackageQuestions` build-loop ber-authz.
