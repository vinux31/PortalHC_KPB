<objective>
Fix bug: EditWorker form menampilkan error validasi client-side "Password minimal 6 karakter" / "Password dan Konfirmasi tidak cocok" ketika admin mengedit data pekerja tanpa mengubah password. Password seharusnya opsional saat edit.

Root cause: Atribut `[StringLength(100, MinimumLength = 6)]` pada Password dan `[Compare("Password")]` pada ConfirmPassword di ManageUserViewModel menghasilkan client-side validation via jQuery unobtrusive validation. Server-side sudah handle dengan `ModelState.Remove()` tapi form tidak pernah sampai submit karena diblok client-side.

Fix: Tambahkan JS handler pada form submit yang menghapus validation rules dari password fields ketika keduanya kosong.
</objective>

Di bagian `<script>` pada EditWorker.cshtml, tambahkan event handler pada form `#editWorkerForm` submit yang:

1. Cek apakah kedua field `#passwordField` dan `#confirmPasswordField` kosong (trimmed)
2. Jika keduanya kosong:
   - Hapus atribut `data-val-*` dari kedua input (data-val-length, data-val-length-min, data-val-length-max, data-val-compare, data-val)
   - Atau lebih simpel: set `disabled` pada kedua field sebelum submit sehingga tidak dikirim ke server dan tidak divalidasi client-side. Tapi ini mencegah value dikirim — yang OK karena server sudah handle null password.

Pendekatan terbaik: Intercept form submit, jika kedua password kosong maka remove atribut `data-val` dari kedua input dan panggil `$.validator.unobtrusive.parse('#editWorkerForm')` untuk re-parse, lalu lanjutkan submit.

Implementasi konkret — tambahkan sebelum closing `</script>`:

```javascript
// Fix: bypass client-side validation untuk password saat edit (keduanya kosong = tidak ubah password)
document.getElementById('editWorkerForm').addEventListener('submit', function (e) {
    var pwd = document.getElementById('passwordField');
    var cpwd = document.getElementById('confirmPasswordField');

    // Hanya untuk mode non-AD (field ada)
    if (pwd && cpwd) {
        if (pwd.value.trim() === '' && cpwd.value.trim() === '') {
            // Hapus semua data-val attributes agar jQuery validation skip field ini
            [pwd, cpwd].forEach(function (el) {
                var attrs = [];
                for (var i = 0; i < el.attributes.length; i++) {
                    if (el.attributes[i].name.startsWith('data-val')) {
                        attrs.push(el.attributes[i].name);
                    }
                }
                attrs.forEach(function (attr) { el.removeAttribute(attr); });
            });
            // Re-parse form validation
            var form = $(this);
            form.removeData('validator').removeData('unobtrusiveValidation');
            $.validator.unobtrusive.parse(form);
        }
    }
});
```

PENTING: Event handler ini harus terdaftar SEBELUM jQuery validation handler. Karena jQuery unobtrusive validation bind pada `submit` juga, gunakan capture phase atau pastikan script ini dijalankan sebelum validation scripts di-parse. Alternatif lebih robust: override jQuery validate submitHandler.

Pendekatan paling robust — GUNAKAN INI:
Alih-alih event listener, override langsung setelah DOM ready:

```javascript
// Fix: password opsional saat edit
$(function () {
    var form = $('#editWorkerForm');
    var origSubmit = form.data('validator');
    if (origSubmit) {
        var origHandler = origSubmit.settings.submitHandler;
        origSubmit.settings.submitHandler = null; // let default handle
    }

    form.on('submit', function () {
        var pwd = $('#passwordField');
        var cpwd = $('#confirmPasswordField');
        if (pwd.length && cpwd.length && pwd.val().trim() === '' && cpwd.val().trim() === '') {
            // Remove validation rules for these fields
            pwd.rules('remove');
            cpwd.rules('remove');
            // Clear any existing errors
            pwd.removeClass('input-validation-error');
            cpwd.removeClass('input-validation-error');
            pwd.next('.field-validation-error, .text-danger').text('');
            cpwd.closest('.col-md-6').find('.text-danger').text('');
        }
    });
});
```

Pilih pendekatan yang paling clean. Yang paling sederhana dan pasti bekerja: hapus atribut `data-val` pada page load karena ini adalah halaman EDIT — password selalu opsional di sini.

SOLUSI FINAL (paling simpel, pasti bekerja):
Pada saat page load (di dalam block `<script>` yang ada), tambahkan kode berikut untuk menghapus semua atribut data-val dari password fields dan re-parse form:

```javascript
// Password opsional saat edit — hapus client-side validation
(function() {
    var pwd = document.getElementById('passwordField');
    var cpwd = document.getElementById('confirmPasswordField');
    if (pwd && cpwd) {
        [pwd, cpwd].forEach(function(el) {
            var toRemove = [];
            for (var i = 0; i < el.attributes.length; i++) {
                if (el.attributes[i].name.startsWith('data-val')) {
                    toRemove.push(el.attributes[i].name);
                }
            }
            toRemove.forEach(function(a) { el.removeAttribute(a); });
        });
        // Re-parse agar jQuery validation mengenali perubahan
        var $form = $('#editWorkerForm');
        $form.removeData('validator').removeData('unobtrusiveValidation');
        if ($.validator && $.validator.unobtrusive) {
            $.validator.unobtrusive.parse($form);
        }
    }
})();
```

Letakkan kode ini di akhir block `<script>` yang sudah ada, setelah fungsi `togglePassword`.
