## Critical Gap: Broken View References

### Root Cause

All 8 domain controllers use `[Route("Admin")]` + `[Route("Admin/[action]")]` for URL preservation. This means URLs like `/Admin/ManageWorkers` resolve correctly at the **routing level** (incoming requests work).

However, `Url.Action("ManageWorkers", "Admin")` in Razor views resolves by **controller name** (class name minus "Controller" suffix), NOT by route template. Since `ManageWorkers` is now in `WorkerController` (controller name = "Worker"), `Url.Action("ManageWorkers", "Admin")` returns **null**.

### Impact

- **99 total** `Url.Action("X", "Admin")` references across 26 views
- **28 valid** (reference `Index` or `Maintenance`, still in AdminController)
- **71 broken** (reference actions moved to domain controllers → return null)

### Why UAT Didn't Catch This

Phase 290's automated verification checked:
1. Build succeeds (C# compilation) — doesn't compile Razor views
2. Route attributes present — verifies incoming URL resolution, not outgoing URL generation
3. Authorization attributes — correct
4. No duplicate actions — correct

The gap is specifically in **outgoing URL generation** (`Url.Action` helper), which only fails at runtime when the view renders.
