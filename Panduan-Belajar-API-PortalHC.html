Apa itu API?

Konsep dasar yang wajib dipahami sebelum masuk ke teknis

Definisi Sederhana

API singkatan dari Application Programming Interface.

Dalam bahasa sehari-hari: API adalah "jembatan" atau "penghubung" antara dua sistem komputer agar bisa saling berbicara dan bertukar data.

🎯 Intinya: API memungkinkan satu aplikasi meminta data atau perintah ke aplikasi lain, tanpa perlu tahu "isi dapur" di dalamnya. Yang penting: minta → dapat jawaban.

📱 Contoh API di Kehidupan Nyata

Tanpa sadar, kamu sudah pakai API setiap hari:

  • Google Maps di Gojek — Gojek "minta" data peta dari Google Maps via API
  • Login dengan Google — Website minta Google konfirmasi identitas via API
  • Cuaca di HP — Aplikasi cuaca minta data dari server cuaca via API
  • Transfer bank via aplikasi — Aplikasi Anda minta server bank proses transfer via API
  • WhatsApp Web — Browser Anda minta pesan dari server WhatsApp via API

🎯 Kenapa API Itu Penting?

🚫 Tanpa API

  • Setiap aksi perlu reload halaman
  • Tidak bisa terintegrasi sistem lain
  • Tidak bisa dibuat aplikasi mobile
  • Data tidak bisa diakses dari luar
  • Sulit dibuat dashboard eksternal

✅ Dengan API

  • Data real-time tanpa reload
  • Integrasi ke SAP, Power BI, dll
  • Bisa dibuat aplikasi mobile
  • Data bisa diakses aman dari luar
  • Dashboard langsung dari data live

Cara Kerja API

Memahami alur request dan response secara detail

Alur Lengkap API

1

Client mengirim REQUEST

Browser atau aplikasi mengirim permintaan ke URL tertentu. Contoh: "Kasih saya data pegawai dengan ID 123"

2

Server menerima request

Server membaca permintaan, mengecek autentikasi (apakah user punya izin?), lalu meneruskan ke logika bisnis.

3

Server memproses data

Server mengambil data dari database, memproses kalau perlu, lalu menyiapkan jawaban dalam format JSON.

4

Server mengirim RESPONSE

Server mengirim balik data beserta kode status (200 = sukses, 404 = tidak ditemukan, 403 = tidak punya akses, dsb).

5

Client menerima & menampilkan

Browser/aplikasi menerima data JSON, lalu menampilkannya ke layar tanpa perlu reload halaman.

Jenis-jenis API

Tiga jenis yang relevan untuk PortalHC KPB

1. REST API — Yang Paling Umum

REST (Representational State Transfer) adalah jenis API yang paling banyak digunakan di dunia web. Prinsipnya sederhana: gunakan URL + HTTP Method untuk semua operasi data.

✅ Ini yang akan kita tambahkan ke PortalHC KPB. REST API cocok untuk semua fitur: ambil data pegawai, buat assessment, update training, dll.

Karakteristik REST API:

  • URL mewakili "sumber daya" (resource) — misal /api/pegawai
  • HTTP Method menentukan aksi (GET/POST/PUT/DELETE)
  • Data dikirim/diterima dalam format JSON
  • Stateless — setiap request berdiri sendiri, tidak bergantung request sebelumnya

2. WebSocket (SignalR) — Sudah Ada!

WebSocket adalah koneksi dua arah yang terus-menerus terbuka antara client dan server. Cocok untuk fitur real-time seperti chat, notifikasi langsung, atau monitoring live.

⭐ PortalHC KPB sudah menggunakan ini! Fitur monitoring ujian assessment menggunakan SignalR (implementasi WebSocket di ASP.NET). Peserta ujian mengirim jawaban real-time, dan HC bisa monitor secara live.

Perbedaan WebSocket vs REST:

REST API

  • Client minta → Server jawab → selesai
  • Koneksi tidak permanen
  • Cocok untuk CRUD biasa
  • Semua modul PortalHC (selain assessment)

WebSocket / SignalR

  • Koneksi terus terbuka dua arah
  • Server bisa kirim data kapan saja
  • Cocok untuk real-time
  • Assessment monitoring PortalHC ✅

3. Internal API (Controller Action)

Sebenarnya di PortalHC KPB saat ini sudah ada beberapa action yang mengembalikan data JSON — ini disebut internal API atau "quasi-API". Belum terstruktur sebagai REST API penuh, tapi sudah setengah jalan.

📌 Contoh yang sudah ada: SearchUsersApi, UnreadCount, GetOrganizationTree, RegenerateToken, dan beberapa lainnya sudah mengembalikan JSON.

Ini artinya tim developer sudah paham konsepnya — tinggal memperluas dan memformalkannya sebagai REST API yang terstruktur.

Potensi API per Modul

13 modul PortalHC KPB dan manfaat API di masing-masingnya

💡 Di bawah ini adalah analisis setiap modul PortalHC KPB: apa saja yang bisa dilakukan dengan API, dan integrasi apa yang bisa dibuka.

1. 🔐 Authentication (Login/Logout)

Kondisi saat ini: Login hanya bisa lewat form browser (tidak bisa dari aplikasi lain).

Dengan API:

  • Aplikasi mobile bisa login dengan email/password → dapat token (JWT)
  • Sistem lain bisa autentikasi ke PortalHC secara programatik
  • Single Sign-On (SSO) dengan sistem Pertamina lainnya

🎯 Use case nyata: Buat aplikasi mobile PortalHC — pegawai login dari HP, dapat notifikasi jadwal coaching, lihat progress CDP.

2. 👥 Pegawai (Worker Management)

Kondisi saat ini: Data pegawai hanya bisa diakses lewat halaman web.

Dengan API:

  • Power BI bisa pull data pegawai untuk dashboard visualisasi
  • Sinkronisasi otomatis dari sistem HR Pertamina (SAP, myHR, dsb)
  • Import/export data massal via sistem lain tanpa buka browser
  • Cek status aktif pegawai dari sistem lain

3. 📝 Assessment (Ujian Kompetensi)

Kondisi saat ini: Sudah punya beberapa endpoint JSON + SignalR real-time.

Dengan API REST penuh:

  • Jadwal assessment bisa diakses dari kalender eksternal (Google Calendar, Outlook)
  • Notifikasi otomatis ke WhatsApp/Teams saat assessment dijadwalkan
  • Hasil nilai bisa langsung masuk ke sistem pelaporan manajemen
  • Sertifikat bisa diunduh via API tanpa buka PortalHC

4. 🌱 CDP / Proton Track

Kondisi saat ini: Progress hanya bisa dilihat di PortalHC web.

Dengan API:

  • Dashboard progress CDP bisa tampil di Power BI untuk manajemen
  • Sinkronisasi deliverable dengan sistem Proton pusat Pertamina
  • Notifikasi otomatis ke coachee saat deliverable disetujui/ditolak
  • Laporan progress CDP per bagian untuk rapat manajemen

5. 📊 CMP (Competency Management Program)

Dengan API:

  • Data KKJ bisa diakses sistem lain untuk analisis gap kompetensi
  • Laporan alignment kompetensi masuk otomatis ke sistem pelaporan
  • Integrasi dengan LMS (Learning Management System) eksternal

6. 🤝 Coaching & Coach Mapping

Dengan API:

  • Jadwal coaching tampil di kalender Teams/Outlook otomatis
  • Coach bisa update log sesi dari HP (via mobile app)
  • Alert workload coach dikirim ke HC jika mendekati batas
  • Dashboard coaching per bagian untuk laporan manajemen

7. 🎓 Training Records

Dengan API:

  • Data training bisa disinkronkan dari sistem pelatihan eksternal
  • Laporan training masuk otomatis ke Power BI/Excel manajemen
  • Alert otomatis ke HC jika sertifikat hampir expired
  • Integrasi dengan sistem SAP HR Training

8. 🏢 Organisasi (Bagian/Unit)

Kondisi saat ini: Sudah ada GetOrganizationTree yang mengembalikan JSON.

Dengan API penuh:

  • Struktur org bisa ditarik sistem HR lain secara otomatis
  • Sinkronisasi perubahan struktur org dari sistem induk Pertamina

9. 📄 Dokumen (KKJ, CPDP)

Dengan API:

  • Dokumen KKJ bisa diakses dari sistem manajemen dokumen lain
  • Upload dokumen bisa dari aplikasi eksternal
  • Notifikasi otomatis saat dokumen baru diupload

10. 🔔 Notifikasi

Kondisi saat ini: Sudah ada UnreadCount sebagai API.

Dengan API penuh:

  • Notifikasi PortalHC bisa diteruskan ke WhatsApp, Teams, atau email
  • Sistem lain bisa kirim notifikasi ke pengguna PortalHC
  • Dashboard notifikasi di mobile app

11. 🏆 Sertifikat & Renewal

Dengan API:

  • Verifikasi sertifikat dari sistem eksternal (QR Code scan → API cek validitas)
  • Laporan sertifikat expired masuk ke email HC otomatis
  • Data sertifikat masuk sistem Pertamina pusat

12. ⚙️ Admin & Sistem

Dengan API:

  • Maintenance mode bisa diaktifkan dari sistem monitoring eksternal
  • Audit log bisa diintegrasikan ke SIEM (Security Information & Event Management)
  • Health check endpoint untuk monitoring uptime sistem

13. 📚 Proton Data (Kurikulum)

Dengan API:

  • Data kurikulum bisa disinkronkan dari sistem Proton pusat Pertamina
  • Update deliverable/silabus secara massal via import API
  • Laporan coverage kurikulum untuk manajemen

// Cara meminta data dari API (GET request)
async function ambilDaftarPegawai() {
  const response = await fetch('/api/pegawai', {
    method: 'GET',
    headers: {
      'Authorization': 'Bearer ' + userToken,
      'Content-Type': 'application/json'
    }
  });

  const hasil = await response.json();

  if (hasil.success) {
    // tampilkan data ke tabel HTML
    tampilkanTabel(hasil.data);
  } else {
    alert('Gagal: ' + hasil.message);
  }
}

API Controller di C# (ASP.NET Core)

Begini cara membuat endpoint API di PortalHC KPB menggunakan C#:

// File: Controllers/Api/PegawaiApiController.cs

[ApiController]
[Route("api/pegawai")]
[Authorize]  // harus login untuk akses
public class PegawaiApiController : ControllerBase
{
    private readonly ApplicationDbContext _db;

    public PegawaiApiController(ApplicationDbContext db)
    {
        _db = db;
    }

    // GET /api/pegawai — ambil semua pegawai aktif
    [HttpGet]
    public async Task<IActionResult> GetAll()
    {
        var pegawai = await _db.Users
            .Where(u => u.IsActive)
            .Select(u => new {
                u.Id,
                u.FullName,
                u.NIP,
                u.Position,
                u.Section,
                u.RoleLevel
            })
            .ToListAsync();

        return Ok(new { success = true, data = pegawai });
    }

    // GET /api/pegawai/{id} — detail 1 pegawai
    [HttpGet("{id}")]
    public async Task<IActionResult> GetById(string id)
    {
        var pegawai = await _db.Users.FindAsync(id);
        if (pegawai == null)
            return NotFound(new { success = false, message = "Pegawai tidak ditemukan" });

        return Ok(new { success = true, data = pegawai });
    }
}

Perbedaan MVC Controller vs API Controller

MVC Controller (yang sudah ada)

  • Return: View (halaman HTML)
  • Atribut: [Controller]
  • Base class: Controller
  • Dipakai browser biasa
  • Contoh: WorkerController

API Controller (yang akan ditambah)

  • Return: JSON data
  • Atribut: [ApiController]
  • Base class: ControllerBase
  • Dipakai aplikasi/sistem lain
  • Contoh: PegawaiApiController

✅ Keduanya bisa hidup berdampingan! Tidak perlu menghapus MVC controller yang sudah ada. Cukup tambah folder Controllers/Api/ yang baru.

🛡️ Keamanan API yang Harus Diperhatikan

Aspek KeamananPenjelasanImplementasi di PortalHC
AutentikasiHanya user yang sudah login boleh akses APIJWT Token atau session cookie yang sudah ada
OtorisasiRole Level 6 tidak boleh akses data adminCek RoleLevel di setiap endpoint (sudah ada pola ini)
Rate LimitingBatasi jumlah request per menit agar tidak disalahgunakanTambahkan middleware rate limiter
HTTPS OnlySemua komunikasi harus terenkripsiSudah diaktifkan di server Pertamina
Input ValidationValidasi semua data yang masuk untuk cegah SQL InjectionEntity Framework sudah parameterized query
Audit LogCatat semua aksi penting via APIGunakan AuditLogService yang sudah ada

📖 Istilah API yang Wajib Anda Tahu

IstilahArti Sederhana
EndpointURL spesifik tempat API bisa diakses. Contoh: /api/pegawai
RequestPermintaan yang dikirim ke API
ResponseJawaban yang dikirim balik oleh API
JSONFormat data yang dipakai API — seperti teks terstruktur
Token / JWT"Kartu tanda masuk" digital untuk akses API setelah login
RESTStandar desain API yang paling umum digunakan
SwaggerAlat untuk membuat dokumentasi API otomatis — bisa dicoba langsung di browser
WebSocket / SignalRJenis koneksi untuk data real-time (sudah ada di assessment)
CRUDCreate, Read, Update, Delete — 4 operasi dasar data
HeaderInformasi tambahan yang dikirim bersama request (contoh: token autentikasi)
Status CodeKode angka dalam response yang menunjukkan hasil (200=sukses, 404=tidak ada, dll)
Payload / BodyData utama yang dikirim dalam request POST/PUT
AuthenticationVerifikasi "siapa kamu?" — login
AuthorizationVerifikasi "boleh kamu akses ini?" — cek role/izin
Rate LimitingPembatasan jumlah request dalam waktu tertentu agar API tidak dibanjiri
