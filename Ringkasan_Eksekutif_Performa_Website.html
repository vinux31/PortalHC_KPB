Website Portal HC memiliki 7 modul utama: Assessment (Ujian Online), Coaching Proton, IDP (Rencana Pengembangan), Manajemen Sertifikat, Kelola Data Pekerja, Silabus & Panduan, serta Dashboard & Notifikasi.

Hasil Analisa Keseluruhan

BERFUNGSI BAIK — PERLU PERSIAPAN UNTUK SKALA BESAR

Website berjalan lancar untuk penggunaan harian. Namun ditemukan beberapa titik yang perlu diperkuat jika pengguna bertambah banyak secara bersamaan.

3. Temuan Utama

Hal yang Sudah Baik

Fondasi website sudah kuat
  • Sistem login & keamanan akses (otorisasi berbasis role) sudah benar
  • Proteksi agar ujian tidak bisa di-submit 2 kali sudah ada
  • Monitoring ujian secara real-time berfungsi baik
  • Pencatatan aktivitas admin (audit log) sudah aktif
  • 200 fitur berjalan tanpa bug fungsional

Hal yang Perlu Diperbaiki

# Temuan Penjelasan Singkat Risiko Perbaikan
1 Batas koneksi database terlalu rendah Batas standar 100 koneksi. Jika 100+ orang akses bersamaan, pengguna selanjutnya gagal masuk. Kritis Mudah
2 File upload bisa saling timpa Jika 2 orang upload file dengan nama sama di waktu bersamaan, file pertama bisa tertimpa dan hilang. Kritis Mudah
3 Halaman Coaching Proton berat Setiap kali dibuka, sistem membaca data dari 5 tabel bertingkat — membuat halaman lambat saat banyak pengguna. Kritis Sedang
4 Dashboard HC/Admin lambat Setiap buka dashboard, sistem mengecek semua sertifikat expired dan mengirim notifikasi satu per satu. Menengah Sedang
5 Pembacaan data belum dioptimasi 90% data hanya ditampilkan (tidak diedit), tapi sistem menyiapkan semuanya dalam mode "siap edit" — memakan memori 30–50% lebih banyak. Menengah Mudah
6 File belum ada backup Dokumen yang diupload hanya tersimpan di 1 lokasi (server). Belum ada mekanisme backup otomatis. Menengah Tinggi

Catatan Penting

Untuk penggunaan normal saat ini (10–20 orang bersamaan), website aman dan berfungsi baik. Perbaikan diperlukan jika perusahaan berencana menggunakan website untuk ujian massal atau jika jumlah pengguna aktif akan bertambah signifikan.
