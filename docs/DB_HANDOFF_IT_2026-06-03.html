4. Deploy Checklist (URUTAN WAJIB)

URUTAN WAJIB — Cegah State Drift

Step 1 backup DULU, kemudian Step 2 deploy code, baru Step 3 restart app pool. Jangan reverse order — restart sebelum deploy = app start dengan code lama, migration tidak ter-apply.

  1. Step 1 — BACKUP DB Dev (Section 1):

    Execute BACKUP DATABASE ke .bak file. Verifikasi file ter-create. Mandatory Cilacap SOP.

  2. Step 2 — Deploy code commit 43e94655: 
    cd /path/to/PortalHC_KPB
git fetch origin main
git checkout 43e94655   # atau git pull origin main kalau sudah ada commit terbaru
# (build step opsional kalau IT pakai published artifact pipeline)
dotnet publish -c Release -o /path/to/publish
  3. Step 3 — Restart IIS app pool (atau service equivalent): 
    iisreset    # Windows; atau equivalent restart untuk env IT

    Saat startup: Database.Migrate() apply migration 20260603012335_AddOrganizationLevelLabel → tabel OrganizationLevelLabels ter-create. Kemudian SeedData.InitializeAsync → SeedOrganizationLevelLabelsAsync auto-INSERT 3 baris default.

  4. Step 4 — Verifikasi migration applied: 
    SELECT TOP 3 MigrationId
FROM __EFMigrationsHistory
ORDER BY MigrationId DESC;
-- Expected paling atas: 20260603012335_AddOrganizationLevelLabel
  5. Step 5 — Verifikasi seed default (Section 3 query):

    Run SELECT * FROM OrganizationLevelLabels ORDER BY Level — expect 3 row Bagian / Unit / Sub-unit.

  6. Step 6 — Smoke test endpoint (Section 5):

    Navigate http://10.55.3.3/KPB-PortalHC/Admin/GetLevelLabels (sign in admin@pertamina.com first) → JSON dict { "0": "Bagian", "1": "Unit", "2": "Sub-unit" } returned.
