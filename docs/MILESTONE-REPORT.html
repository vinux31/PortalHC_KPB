5. Reusable Patterns Established (v16.0)

v16.0 mendirikan ~12 pola test automation reusable untuk milestone berikutnya (FLOW A-J refresh, multi-env, CI integration):

  • DOM-text matching post-shuffle (Phase 317) — qcard.filter({hasText: marker}) + label.list-group-item hasText option, untuk multi-question shuffled render.
  • Direct SignalR hub invoke (Phase 317) — page.evaluate(hub.invoke(...)) bypass UI debounce listener race.
  • verifyCertificatePdfDownload APIRequest pattern (Phase 318) — status 200 + content-type PDF + content-disposition Sertifikat_* + bytes > 1024.
  • verifyExcelDownload Excel MIME helper (Phase 319-01 staged → 319-03 consume) — adaptasi PDF pattern untuk xlsx/xls MIME.
  • interceptAnalyticsResponse<T> JSON intercept generic (Phase 319-01 → 319-03) — page.waitForResponse + JSON parse + cast T.
  • Strategy 1 direct POST untuk delete (Phase 319-02 U3, parallel Phase 319-01 T6) — DB query id + antiforgery token + direct page.request.post, avoid Bootstrap modal animation race.
  • maxRedirects: 0 untuk auth-gate test (Phase 319-03 V3) — preserve raw 302 status, jangan follow redirect chain ke login page 200 (false negative).
  • Timezone-aware API ≤ DB cross-check (Phase 319-03 W4) — controller filter periodeEnd = today WIB midnight vs DB query unbounded — assertion API ≤ DB instead of strict equality.
  • Multi-context HC+Worker SignalR broadcast (Phase 317 FLOW O) — browser.newContext() x 2 + defensive close di finally + waitForFunction observer.
  • AllowAnswerReview negative assertion pair (Phase 317 FLOW N) — positive .alert-info visible + negative .card "Tinjauan Jawaban" toHaveCount(0).
  • Pre-execute sanity check pattern (Phase 319) — verify route + selector + endpoint vs controller/view BEFORE eksekusi → patch PLAN.md inline untuk plan/implementasi sinkron.
  • DB backup/restore lifecycle per test run (Phase 315 matrix infra) — global.setup.ts BACKUP + seed; global.teardown.ts RESTORE + cleanup. Zero side-effect post-run.
