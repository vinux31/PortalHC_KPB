1. Pengenalan

Role HC, cara akses, login, dan pengaturan akun pribadi.

1.1 Role HC — Authority & Scope (L2)

HC (Human Capital) adalah role Level 2 dengan akses penuh ke pengelolaan SDM dan program coaching. Authority HC mencakup:

  • CMP: monitor seluruh assessment lintas Unit/Section, Analytics Dashboard, Records Team semua bagian, kelola Pre/Post Test, Budget Training, Certificate.
  • CDP: review semua deliverable, Coaching Proton Dashboard global, Bottleneck Report, Coach Workload, Final Reviewer di approval chain.
  • Kelola Data Proton: CRUD Silabus, upload Guidance Files, Override Data Pekerja.
  • Admin Panel: Kelola Pekerja/Units/Categories, Bank Soal + Create Assessment, Assessment Monitoring real-time, Coach-Coachee Mapping, Renewal Certificate, Training Record, KKJ/CPDP Files, Audit Log, Maintenance Mode, Impersonate.
💡 Beda dengan Admin (L1)
Admin punya semua hak akses HC + konfigurasi sistem yang lebih sensitif (perubahan role pekerja, hapus permanen, escalation override). HC tidak.

1.3 Login ke HC Portal

Browser → Halaman Login /Account/Login
1Buka URL Portal
Akses URL Dev atau Prod di browser modern (Chrome, Edge, Firefox versi terbaru).
2Masukkan Kredensial
Ketik email dan password yang sudah diaktifkan oleh Admin / Tim IT. Centang Remember Me kalau pakai komputer pribadi. Klik Login.
3Verifikasi Role Badge
Setelah login, cek navbar kanan atas. Avatar Anda akan tampil dengan badge HC. Kalau bukan HC, hubungi Admin untuk update role di Admin Panel → Kelola Pekerja.
4Logout
Klik avatar → Logout. Form Logout pakai metode POST — aman dari CSRF.
💡 Lupa password
Belum tersedia self-service reset di portal. Hubungi Admin / Tim IT untuk reset password manual.

1.4 Profile & Settings

Navbar → Avatar (kanan atas) → My Profile / Settings /Account/Profile /Account/Settings
1Lihat Profil (read-only)
Klik avatar Anda di pojok kanan atas navbar → pilih My Profile (/Account/Profile). Halaman menampilkan: nama, NIP, email, jabatan, unit kerja, dan role.
2Edit Profil
Klik avatar → Settings (/Account/Settings). Field yang bisa di-edit: Nama Lengkap, Jabatan, Nomor Telepon. Klik Simpan Profil di bawah form.
3Field yang Tidak Bisa Di-edit
NIP, Email, Role, Bagian, Direktorat, Unit hanya tampil read-only. Update field-field ini harus lewat Admin Panel — hubungi Admin / HC lain.
⚠️ Ganti Password
Belum ada fitur self-service ganti password di portal. Reset password harus via Admin / Tim IT. Kemungkinan password terhubung ke Active Directory perusahaan — reset lewat jalur IT standar.

2. CMP — Competency Management Platform

Pengelolaan kompetensi: KKJ, assessment, training records, analytics, dan sertifikat.

2.1 Library KKJ — Lihat Kebutuhan Kompetensi Jabatan

Navbar → CMP → (landing card) Dokumen KKJ /CMP/Index
1Buka Modul CMP
Klik menu CMP di navbar. Halaman landing CMP (/CMP/Index) menampilkan card-card modul.
2Klik Card Dokumen KKJ
Pilih card Dokumen KKJ & Alignment KKJ/IDP untuk navigate ke halaman lihat KKJ + CPDP.
💡 Library KKJ HC scope
Sebagai HC (L2), Anda bisa lihat KKJ untuk semua posisi/jabatan. Pekerja role lain hanya lihat KKJ posisi sendiri.

2.2 CPDP Mapping — Pemetaan Pengembangan Kompetensi

Navbar → CMP → Dokumen KKJ /CMP/DokumenKkj
1Buka Halaman Dokumen KKJ
Akses langsung /CMP/DokumenKkj atau via card di /CMP/Index.
2Pilih Tab
Pakai parameter ?tab= atau klik tab di halaman untuk switch antara Library KKJ, CPDP Mapping, dan dokumen alignment.
3Download File KKJ / CPDP
Klik tombol download di tiap dokumen. URL endpoint: /CMP/KkjFileDownload/{id} dan /CMP/CpdpFileDownload/{id}.
💡 Update file KKJ / CPDP
Upload file baru lewat Admin Panel → KKJ Matrix / CPDP Files. Lihat Bab 5.12 / 5.13.

2.3 Training Records — Riwayat Pelatihan Pribadi + Tim

Navbar → CMP → Records (Training Records) /CMP/Records
1Buka Halaman Records
Klik menu CMP → card / link Training Records. Tab default menampilkan riwayat assessment + training pribadi (unified records).
2Filter Data
Pakai filter di atas tabel: Section, Unit, Category, Status, atau search. Filter di-URL via query string (?section=&unit=&category=).
3Export Excel
Klik tombol Export. Endpoint: /CMP/ExportRecords untuk full export, atau /CMP/ExportRecordsTeamAssessment / /CMP/ExportRecordsTeamTraining untuk team-scoped export dengan filter aktif.
💡 Records vs Records Team
Halaman /CMP/Records sama untuk semua role; isi tab berubah sesuai role level. HC lihat tab Team View untuk seluruh tim.

2.4 Records Team — Monitoring Tim Lintas Section

Navbar → CMP → Records → (tab) Team View /CMP/Records /CMP/RecordsTeamPartial
1Buka Tab Team View
Di halaman Records, klik tab Team View. Tab ini hanya muncul untuk role level 1-4 (HC, Admin, Manager, Section Head, Sr Supervisor). HC bisa lihat semua section.
2Filter per Section / Unit / Kategori
Pakai dropdown Section, Unit, Kategori Assessment, Sub-Kategori, Status, atau search nama. Partial reload via AJAX ke /CMP/RecordsTeamPartial.
3Drill ke Detail Pekerja
Klik nama pekerja di tabel. Halaman detail /CMP/RecordsWorkerDetail?workerId={id} menampilkan unified history: assessment + training + sertifikat per worker.
4Export Tim
Klik Export Excel di tab Team View. Pilih Assessment atau Training — endpoint terpisah dengan filter aktif diteruskan sebagai query string.
💡 HC scope vs L4 scope
HC (L2) bisa lihat lintas section. Section Head / Sr Supervisor (L4) locked ke section mereka sendiri (auto-filter di controller).

2.5 Analytics Dashboard — Compliance, Fail Rate, Gain Score

Navbar → CMP → Analytics Dashboard /CMP/AnalyticsDashboard
1Buka Analytics Dashboard
Klik CMP → Analytics Dashboard. Dashboard load data via beberapa AJAX endpoint untuk komponen chart.
2Pilih Filter Cascade
Filter tersedia: Bagian, Unit (cascade dari Bagian via /CMP/GetAnalyticsCascadeUnits), Kategori, Sub-Kategori (cascade via /CMP/GetAnalyticsCascadeSubKategori), rentang tanggal.
3Review Komponen Chart
Komponen utama:
  • Summary — total assessment, pass/fail rate, distribusi (/CMP/GetAnalyticsSummary).
  • Fail Rate per kategori + drill-down (/CMP/GetFailRateData, /CMP/GetFailRateDrillDown).
  • Trend bulanan (/CMP/GetTrendData).
  • ET Breakdown per Excellent / Tinggi / dst (/CMP/GetEtBreakdownData).
  • Expiring Soon — sertifikat hampir kadaluarsa (/CMP/GetExpiringSoonData).
4Export Excel per Chart
Tiap komponen punya tombol Export sendiri: /CMP/ExportFailRateExcel, /CMP/ExportTrendExcel, /CMP/ExportEtBreakdownExcel.
💡 Item Analysis & Gain Score
Untuk Pre/Post Test analysis, klik card / link Pre-Post di Analytics. Endpoint khusus: /CMP/GetItemAnalysisData, /CMP/GetGainScoreData, plus export Excel terpisah.

2.6 Pre/Post Test — Setup & Monitor Gain Score

Navbar → CMP → Assessment (filter Pre/Post) /CMP/Assessment
1Buat Jadwal Pre-Test
Pre-Test dijadwalkan via Admin Panel → Create Assessment (lihat Bab 5.6). Pilih kategori Pre-Test + assign ke peserta + set jadwal sebelum tanggal training.
2Monitor Pre-Test Selesai
Buka CMP → Assessment (/CMP/Assessment), search "Pre-Test [topik]". Cek peserta yang sudah complete via tab Records atau Assessment Monitoring (Bab 5.8).
3Setelah Pelatihan, Buat Post-Test
Buat assessment Post-Test dengan paket soal yang sama persis. Sistem otomatis pair Pre + Post berdasarkan peserta + topik.
4Review Gain Score
Buka Analytics Dashboard → Gain Score. Sistem hitung selisih nilai Post vs Pre per peserta. Export ke Excel via /CMP/ExportGainScoreExcel?assessmentGroupId={id}.
💡 Item Analysis
Untuk lihat soal yang paling sulit / paling salah dijawab, buka Item Analysis di Analytics. Berguna untuk evaluasi kurikulum training.

2.7 Budget Training — Kelola Anggaran Pelatihan Tahunan

Navbar → CMP → Budget Training /CMP/BudgetTraining
1Lihat Daftar Budget
Buka /CMP/BudgetTraining. Daftar budget per tahun anggaran. Filter via dropdown tahun + search.
2Tambah Budget Item Baru
Klik Tambah Budget (/CMP/BudgetTrainingCreate). Isi nama training, anggaran, tahun, kategori, unit. Klik Simpan.
3Import Massal via Excel
Untuk input banyak, klik Import (/CMP/BudgetTrainingImport). Download template via Download Template (/CMP/DownloadBudgetTrainingTemplate), isi data, upload kembali. Wajib pilih Tahun Anggaran.
4Update Realisasi
Edit realisasi anggaran per item di tabel (quick edit inline via /CMP/BudgetTrainingQuickUpdate) atau bulk update (/CMP/BudgetTrainingBulkUpdateRealisasi).
5Export Laporan Tahunan
Klik Export (/CMP/BudgetTrainingExport?tahun={tahun}) untuk download Excel rekap budget vs realisasi per tahun.
⚠️ Bulk Delete
Tombol Bulk Delete akan menghapus item terpilih permanen. Backup / export dulu sebelum bulk delete.

2.8 Certificate — Download Sertifikat Assessment

Navbar → CMP → Results / Records → (per assessment) Certificate /CMP/Certificate/{id} /CMP/CertificatePdf/{id}
1Buka Halaman Results Assessment
Setelah peserta selesai assessment dan lulus passing grade, halaman Results (/CMP/Results/{id}) akan menampilkan link Sertifikat.
2Preview Sertifikat
Klik Lihat Sertifikat (/CMP/Certificate/{id}) — render preview di browser.
3Download PDF
Klik Download PDF (/CMP/CertificatePdf/{id}) untuk file resmi yang bisa di-cetak atau di-share.
💡 Sertifikat untuk renewal
Sertifikat dengan masa berlaku akan muncul di Renewal Certificate (Bab 5.10) saat mendekati expired. HC schedule renewal dari sana.

3. CDP — Competency Development Platform

Coaching Proton, IDP, deliverable, dan approval chain dengan HC sebagai final reviewer.

3.1 Plan IDP — Review Silabus Pengembangan

Navbar → CDP → Plan IDP /CDP/PlanIdp
1Buka Plan IDP
Klik CDP → Plan IDP. Halaman menampilkan silabus pengembangan kompetensi berdasarkan jabatan + level.
2Filter Bagian / Unit / Track
Pakai query string filter: ?bagian=&unit=&trackId= atau dropdown di halaman. Filter memungkinkan HC review silabus per scope tertentu.
3Download Guidance Document
Per item silabus yang punya guidance file, klik link download (/CDP/GuidanceDownload/{id}).
💡 Update silabus
Silabus dikelola via Kelola Data Proton (Bab 4.1). Edit / tambah / nonaktifkan dari sana, tidak dari Plan IDP.

3.2 Deliverable Monitoring — Semua Coachee

Navbar → CDP → Coaching Proton / Deliverable /CDP/CoachingProton /CDP/Deliverable/{id}
1Buka Halaman Coaching Proton
Klik CDP → Coaching Proton (/CDP/CoachingProton). HC lihat semua coachee lintas section dengan status deliverable.
2Filter & Search
Pakai filter Section, Unit, Track, periode, status (Not Started, In Progress, Pending Approval, Approved, Rejected). Search nama coachee untuk cari cepat.
3Buka Detail Deliverable
Klik judul deliverable. Halaman detail /CDP/Deliverable/{id} menampilkan deskripsi, target tanggal, evidence yang sudah ter-upload, status approval chain, dan history coaching session.
4Download Evidence
Untuk review evidence yang di-submit coachee, klik link download (/CDP/DownloadEvidence/{progressId} atau /CDP/DownloadEvidencePdf/{progressId} untuk format PDF).
💡 HC = Final Reviewer
HC tidak approve di tahap awal — tunggu Sr Supervisor + Section Head approve dulu. HC pakai endpoint HCReviewDeliverable di tahap terakhir (lihat 3.8).

3.3 Coaching Proton Dashboard — Monitoring Global

Navbar → CDP → Dashboard /CDP/Dashboard
1Buka Dashboard
Klik CDP → Dashboard. Dashboard menampilkan progress coaching global: total coachee aktif, distribusi status deliverable, progress per Section / Unit / Track.
2Filter Cascade
Filter via dropdown:
  • Section → cascade Unit (via /CDP/GetCascadeOptions?section={s})
  • Kategori → cascade Sub-Kategori (via /CDP/GetSubCategories?category={c})
  • Track — Panelman, Operator, dll
Filter trigger reload partial via /CDP/FilterCoachingProton.
3Drill ke Detail Coachee
Klik card coachee di dashboard untuk lihat deliverable + session history (loaded via /CDP/GetCoacheeDeliverables?coacheeId={id}).
💡 Tab Bottleneck
Bottleneck Report ada sebagai tab / section di Dashboard ini. Lihat Bab 3.5.

3.4 Histori Proton — Export Laporan per Periode

Navbar → CDP → Histori Proton /CDP/HistoriProton /CDP/HistoriProtonDetail
1Buka Histori Proton
Klik CDP → Histori Proton. Tampilan riwayat coaching per periode Proton yang sudah berjalan / selesai.
2Buka Detail per Coachee
Klik nama coachee. Halaman detail (/CDP/HistoriProtonDetail?userId={id}) menampilkan complete history: deliverable submitted, coaching session log, final assessment scores.
3Export Excel
Klik Export Excel (/CDP/ExportHistoriProton). Export bisa difilter via parameter periode + section + unit.
4Export Progress per Coachee
Untuk laporan per coachee individual:
  • Excel: /CDP/ExportProgressExcel?coacheeId={id}
  • PDF: /CDP/ExportProgressPdf?coacheeId={id}

3.5 Bottleneck Report — Identifikasi Deliverable Stuck

Navbar → CDP → Dashboard → (tab) Bottleneck /CDP/Dashboard /CDP/ExportBottleneckReport
1Akses Tab Bottleneck
Di halaman Dashboard (/CDP/Dashboard), pilih tab / section Bottleneck Report. Tabel menampilkan deliverable yang stuck (lama tidak progress) per coachee / unit.
2Filter Bottleneck
Filter umum:
  • Pending Approval > X hari — deliverable nunggu reviewer terlalu lama
  • Rejected — coachee belum re-upload setelah ditolak
  • Coach — coach tertentu yang banyak deliverable stuck
3Identifikasi Akar Bottleneck
Klik baris untuk drill detail. Akar umum: reviewer lambat (tagging Sr Supervisor / Section Head), coachee pasif (belum upload), scope unclear (deliverable bermasalah).
4Export untuk Eskalasi
Klik Export Bottleneck (/CDP/ExportBottleneckReport). Excel ini biasa dipakai HC untuk meeting eskalasi ke Section Head / Manager.
5Export Coaching Tracking
Untuk laporan coaching tracking lebih luas: /CDP/ExportCoachingTracking dengan parameter filter aktif.

3.6 Coach Workload — Beban Kerja Coach

Navbar → (Admin / CoachMapping) → Coach Workload /CoachMapping/CoachWorkload /CoachMapping/ExportCoachWorkload
1Buka Coach Workload
Akses /CoachMapping/CoachWorkload. Halaman menampilkan tiap coach + jumlah coachee aktif + deliverable pending review.
2Filter per Section
Pakai dropdown / query string ?section={s} untuk filter coach per section.
3Identifikasi Overload
Sort by jumlah coachee atau pending review. Coach dengan workload tinggi = kandidat untuk redistribusi mapping (lihat Bab 5.9).
4Export Workload Summary
Klik Export (/CoachMapping/ExportCoachWorkload) atau dari CDP: /CDP/ExportWorkloadSummary.
💡 Aksi setelah identifikasi overload
Buka Coach-Coachee Mapping (Bab 5.9), pindahkan coachee dari coach yang overloaded ke coach yang capacity-nya masih ada.

3.7 Certification Management — Sertifikat Coaching Proton

Navbar → CDP → Certification Management /CDP/CertificationManagement
1Buka Halaman
Klik CDP → Certification Management. Daftar sertifikat coaching: aktif, mendekati expired, expired.
2Filter Sertifikat
Pakai filter section, kategori, sub-kategori, rentang tanggal expired via /CDP/FilterCertificationManagement. Pagination via ?page={n}.
3Export Excel
Klik Export (/CDP/ExportSertifikatExcel) dengan parameter filter aktif.
4Schedule Renewal
Untuk sertifikat yang mendekati expired, lanjut ke Admin Panel → Renewal Certificate (Bab 5.10) untuk schedule assessment renewal.

3.8 Reviewer Chain — HC sebagai Final Reviewer

Workflow approval (multi-step, no single URL) (workflow)
Coachee Submit Evidence ↓ 1. Sr Supervisor Review ↓ 2. Section Head Review ↓ 3. HC Final Review ← Anda ↓ Approved / Rejected
1Notifikasi Evidence Siap Review
Setelah Sr Supervisor + Section Head approve, notifikasi muncul di bell icon HC. Klik untuk navigate ke deliverable (Bab 6.1).
2Buka Halaman Deliverable
Akses /CDP/Deliverable/{id}. Cek evidence yang ter-upload, coaching session log, dan history approval di bawah.
3Review & Approve / Reject
Klik HC Approve (POST /CDP/HCReviewDeliverable) untuk approve final. Atau Reject (POST /CDP/RejectDeliverable dengan rejectionReason) kalau evidence belum memenuhi standar.
4Batch HC Approve
Untuk approve banyak deliverable sekaligus, pakai Batch HC Approve (POST /CDP/BatchHCApprove dengan body JSON list progressId).
⚠️ Reject = Reset Chain
Kalau HC reject, seluruh chain (Sr Supervisor + Section Head + HC) ter-reset. Coachee harus upload evidence baru, di-review ulang dari Sr Supervisor lagi. Pastikan reject reason jelas + actionable.
⚠️ Jangan Skip Chain
HC tidak boleh approve langsung tanpa tunggu Sr Supervisor + Section Head approve dulu. Sistem enforce urutan via endpoint terpisah (Approve vs HCReview).

4. Kelola Data Proton

Silabus, Guidance Files, dan Override data sync Proton.

4.1 Silabus Proton — CRUD + Import/Export Excel

Navbar → Kelola Data → Silabus /ProtonData/Index
1Buka Halaman Silabus
Klik Kelola Data di navbar → halaman default /ProtonData/Index. Filter dropdown Bagian, Unit, Track. Toggle Show Inactive untuk lihat silabus yang sudah dinonaktifkan.
2Inline Edit Silabus
Edit langsung di tabel: ubah judul, urutan, target level. Klik Save untuk submit batch via /ProtonData/SilabusSave (POST JSON body).
3Deactivate / Reactivate
Untuk silabus obsolete, klik Deactivate (/ProtonData/SilabusDeactivate). Bisa di-reactivate kalau ternyata salah (/ProtonData/SilabusReactivate).
4Import Excel
Klik Import Silabus (/ProtonData/ImportSilabus):
  • Download template lewat /ProtonData/DownloadSilabusTemplate?bagian=&unit=&trackId=
  • Isi data sesuai format, pilih Bagian/Unit/Track di form
  • Upload Excel via tombol Upload
5Export Silabus
Klik Export (/ProtonData/ExportSilabus?bagian=&unit=&trackId=) untuk download Excel hasil silabus aktif.
⚠️ Delete Permanent
SilabusDelete menghapus permanen + cascade ke deliverable / coachee history. Pakai Deactivate dulu — delete hanya kalau silabus belum pernah dipakai. Endpoint preview cascade: /ProtonData/SilabusDeletePreview, /ProtonData/SubKompetensiDeletePreview, /ProtonData/KompetensiDeletePreview.

4.2 Guidance Files — Upload & Kelola Dokumen Rujukan

Kelola Data → Guidance List per Bagian/Unit/Track /ProtonData/GuidanceList
1Buka Daftar Guidance
Akses /ProtonData/GuidanceList?bagian={b}&unit={u}&trackId={t}. Daftar guidance file aktif untuk scope tersebut.
2Upload Guidance Baru
Klik Upload Guidance. Pilih Bagian/Unit/Track, lampirkan file (PDF rekomendasi). Submit ke /ProtonData/GuidanceUpload.
3Replace / Delete
Untuk update file: /ProtonData/GuidanceReplace/{id}. Untuk hapus: /ProtonData/GuidanceDelete (POST JSON body). File yang dihapus tidak bisa di-restore.
4Download
Coachee + Coach + HC bisa download guidance lewat /ProtonData/GuidanceDownload/{id} — link tampil di Plan IDP (Bab 3.1).
💡 Pakai PDF untuk standardisasi
Pakai PDF supaya format konsisten lintas device. Hindari DOCX/XLSX kecuali memang perlu di-edit oleh user.

4.3 Override Data Pekerja — Proton Sync Fallback

Navbar → Kelola Data → (tab) Override /ProtonData/Override /ProtonData/OverrideList
1Cek Status Sync
Buka /ProtonData/StatusData dulu untuk lihat status sync data dari source ke Proton. Identifikasi pekerja yang gagal mapping otomatis (mis: ID jabatan tidak ter-detect).
2Buka Override Page
Akses /ProtonData/Override untuk halaman utama, atau /ProtonData/OverrideList?bagian={b}&unit={u}&trackId={t}&statusFilter={s} untuk daftar dengan filter.
3Detail per Override
Klik baris untuk drill ke /ProtonData/OverrideDetail/{id}. Tampilan: data source vs data Proton yang diinginkan + field-field mapping.
4Save Override
Isi paksa value mapping (mis: mapping KKJ kalau Proton tidak detect ID Jabatan otomatis). Klik Save — submit ke /ProtonData/OverrideSave (POST JSON body). Data generated otomatis setelah override.
⚠️ Fitur Sensitif
Override mem-bypass auto-sync. Pakai hanya untuk fix data yang gagal sync otomatis — bukan untuk "fix" data source yang salah (data source harus diperbaiki di hulunya). Override masuk ke Audit Log.
