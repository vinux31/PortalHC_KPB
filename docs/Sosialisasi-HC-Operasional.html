BAGIAN 2 — CDP

CDP — Reviewer Chain

HC = Final Reviewer (Reviewer ke-3) di approval chain

SLIDE 9 / 23
Coachee Submit Upload evidence deliverable
→
1. Sr Supervisor Review pertama
→
2. Section Head Review kedua
→
3. HC Final Approve / Reject

⚠ Reject = Reset Chain

  • Reject di tahap mana pun = chain ulang dari awal
  • Coachee upload evidence baru
  • Sr Supervisor + Section Head review ulang
  • HC final review lagi

✅ Tidak Boleh Skip Chain

  • HC tunggu Sr Spv + Section Head approve
  • Sistem enforce urutan via endpoint
  • Tombol HC Approve baru aktif setelah Section Head approve
  • Batch HC Approve untuk efisiensi

BAGIAN 3 — KELOLA DATA

Override Data Pekerja

Fitur sensitif — pakai dengan hati-hati

SLIDE 14 / 23

🔍 Kapan Dipakai

  • Data Proton gagal sync otomatis dari source
  • Mapping KKJ tidak ter-detect (ID Jabatan invalid)
  • Mapping silabus per posisi error
  • Cek Status Sync dulu untuk identifikasi gap
  • Fallback manual setelah auto-sync gagal

⚠ Aturan Pakai

  • Bukan untuk fix data source yang salah — perbaiki di hulu (data source)
  • Override = bypass auto-sync, hanya untuk gap teknis
  • Setiap override ter-log di Audit Log
  • Bisa di-review oleh auditor / compliance
  • Konsultasi Admin / IT kalau ragu
💡 Best practice: Sebelum override, screenshot kondisi error + catatan reason di komunikasi internal. Audit Log cuma simpan action; reasoning ada di komunikasi paralel.

BAGIAN 4 — ADMIN PANEL

Pekerja + Bank Soal

Dua operasi yang paling sering HC lakukan

SLIDE 16 / 23

👥 Kelola Pekerja

  • CRUD — Tambah / Edit / Detail / Deactivate
  • Import Excel — massal pakai template, sekali upload ratusan worker
  • Export Excel — backup atau laporan struktur SDM
  • Update role — assign HC, Coach, dll
  • Search + filter Section/Unit/Role/Active

💾 Bank Soal

  • Manage Packages — paket soal per assessment
  • Import via Excel — download template MC/Essay, isi, upload
  • Preview sebelum publish — simulasi tampilan peserta
  • Multiple package per assessment untuk distribusi random
  • Copy paket Pre → Post untuk konsistensi
⚠ Pitfall import soal: Kunci jawaban di kolom Excel harus huruf kapital (A/B/C/D/E). Lowercase ditolak. Sheet validation di template auto-cek format.

BAGIAN 4 — ADMIN PANEL

Create & Monitor Assessment

Alur 4 langkah dari setup sampai eksekusi real-time

SLIDE 17 / 23
📝 1. Create Set title, kategori, jadwal, durasi, passing grade
→
👥 2. Assign Individu atau group/unit batch
→
👀 3. Monitor Real-time: status, progress, waktu
→
🛑 4. Force-Close Kalau freeze/disconnect/jadwal selesai

💰 Power Tools Monitor

  • Add Extra Time — tambah waktu kolektif
  • Reset Assessment — full retake per user
  • Regenerate Token — user salah token
  • Reshuffle Paket — randomize ulang

⚠ Force-Close Ireversibel

  • Auto-submit jawaban yang sudah diisi
  • User tidak bisa lanjutkan
  • Ter-log di Audit Log
  • Pakai untuk genuine case (freeze, disconnect)

BAGIAN 4 — ADMIN PANEL

Coach Mapping + Workload

Distribusi coachee per coach + monitoring beban kerja

SLIDE 18 / 23

🔗 Coach-Coachee Mapping

  • Assign coach ke coachee eligible per Proton track
  • Edit / Deactivate / Reactivate / Delete dengan preview cascade
  • Mark Completed di akhir periode Proton
  • Import Excel untuk mapping batch awal
  • Export mapping aktif untuk audit

📊 Coach Workload

  • Daftar coach + jumlah coachee aktif + pending review
  • Sort by workload — identifikasi overload
  • Set Threshold — warning otomatis kalau coach overload
  • Auto-suggest reassign — approve atau skip
  • Export untuk meeting redistribusi
💡 Daily routine HC: Kalau bell icon banyak notif evidence siap review, cek Coach Workload. Kemungkinan ada coach overload — redistribusi mapping untuk seimbangkan beban.

Detail: Panduan Operasional HC §5.9 & §3.6 — Mapping + Workload

BAGIAN 4 — ADMIN PANEL

Maintenance + Audit Log

Operational tools yang HC pakai saat investigasi atau update

SLIDE 19 / 23

🚧 Maintenance Mode

  • Scope All — freeze seluruh sistem (kecuali Admin)
  • Scope Specific — freeze modul tertentu (CMP, CDP, dll)
  • Isi pesan + estimasi waktu selesai
  • Komunikasi user dulu sebelum aktifkan
  • Action ter-log MaintenanceEnabled/Disabled

🔍 Audit Log Investigation

  • 1. Tentukan jendela waktu kasus
  • 2. Filter Audit Log per startDate/endDate
  • 3. Cari user + action terkait (cross-check timestamp)
  • 4. Drill aksi sensitif: ForceClose, Override, Delete, RoleChange
  • 5. Eskalasi ke Admin / IT kalau menemukan unauthorized action
💡 Impersonate (Admin only): HC tidak bisa pakai Impersonate. Kalau perlu debug user case, request ke Admin dengan detail: NIP user, menu/halaman, screenshot error, langkah reproduksi.

BAGIAN 5 — CLOSING

Tugas HC Cepat

Checklist harian, mingguan, bulanan

SLIDE 21 / 23

📅 Daily

  • Cek bell icon — evidence siap review
  • Buka Coaching Dashboard — tab Bottleneck
  • HC Approve evidence yang sudah lengkap
  • Monitor assessment berjalan
  • Resolve pending request user

📆 Weekly

  • Review Bottleneck Report mendalam
  • Cek Coach Workload + redistribusi kalau overload
  • Audit compliance per Section di Analytics
  • Follow-up renewal H-30
  • Tutup mapping coachee yang completed

🗓 Monthly

  • Schedule batch renewal H-60
  • Export Histori Proton + Bottleneck + Workload untuk laporan
  • Review Audit Log bulanan
  • Cek silabus aktif — ada yang perlu update?
  • Review Pre/Post Test Gain Score — insight kurikulum
💡 Tip: Kalau rutin ini dijalankan konsisten, 80% pekerjaan HC harian = approve evidence + monitor compliance + schedule renewal. Sisanya = resolve edge case.
