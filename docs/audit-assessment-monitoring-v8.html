4. Token Management (AMON-04)

4.1 copyToken()

should-improve FIXED D-14a: copyToken tidak punya fallback untuk clipboard permission denied

Issue: navigator.clipboard.writeText() tanpa .catch() — jika browser deny clipboard permission, fungsi diam-diam gagal tanpa feedback.

Fix: Tambahkan .catch() yang memanggil copyTokenFallback() menggunakan document.execCommand('copy'), dengan final fallback alert() yang menampilkan token untuk copy manual.

4.2 regenToken()

verified-ok D-14b: regenToken() berfungsi benar
  • AJAX POST ke /Admin/RegenerateToken/{id} dengan antiforgery token — benar
  • Token display (#token-display) diupdate setelah success — benar
  • Loading spinner dan disable button selama request — UX baik
  • Error handling: alert jika API return error — benar

4.3 RegenerateToken Endpoint

must-fix FIXED D-14c: RegenerateToken tidak memiliki audit log

Issue: Endpoint RegenerateToken melakukan operasi security-sensitive (regenerate access token untuk seluruh sibling group) tanpa audit trail.

Fix: Tambahkan _auditLog.LogAsync("RegenerateToken", ...) setelah SaveChangesAsync(), menyertakan: actor, assessment title, category, date, jumlah sibling yang diupdate.

Sibling invalidation: Sudah benar — update ALL siblings dengan Where(a => title && category && schedule.Date). Token lama otomatis invalid karena semua sibling mendapat token baru yang sama.

6. SignalR Architecture

6.1 Connection Lifecycle

  • onreconnecting: badge → "Reconnecting..." (warning) — ada
  • onreconnected: badge → "Live" (success) + rejoin monitor group — ada (dua handler, satu untuk badge, satu untuk JoinMonitor)
  • onclose: badge → "Disconnected" (danger) — ada

6.2 Initial Connection Badge

should-improve FIXED (partial) D-12c: setTimeout(2000) untuk set badge "Live" — fragile

Issue: Kode asli menggunakan setTimeout(2000) + state check untuk set badge "Live" dan invoke JoinMonitor. Ini fragile karena 2 detik bisa terlalu cepat (slow network) atau terlalu lambat (UX).

Fix: Ditambahkan promise-based approach via window.assessmentHubStartPromise. Jika promise exposed oleh assessment-hub.js, gunakan .then(). Jika tidak, fallback ke setTimeout yang existing.

Note: assessment-hub.js tidak expose assessmentHubStartPromise saat ini, jadi fallback path akan dijalankan. Fix ini adalah forward-compatible — jika assessment-hub.js diupdate untuk expose promise, badge langsung berfungsi optimal.
