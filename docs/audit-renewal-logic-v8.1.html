Section 3: Data Lama — Catatan dan Query Identifikasi

Keputusan D-03 (No Data Migration): Fix yang diterapkan di Phase 229 berlaku ke depan saja. Tidak ada data migration yang dilakukan terhadap record lama di database. Data lama yang FK-nya bermasalah (jika ada) tetap di database dan tidak diubah.

Data lama yang dibuat sebelum Phase 229 (sebelum 22 Maret 2026) mungkin memiliki FK renewal yang tidak terisi (NULL) meskipun record tersebut merupakan renewal dari record lain. Hal ini karena sebelum Phase 229, tidak ada server-side enforcement untuk FK renewal di AddTraining POST.

Untuk mengidentifikasi data lama yang berpotensi bermasalah, DBA dapat menjalankan query berikut di database PortalHC KPB:

Query 1: TrainingRecords tanpa FK renewal yang mungkin merupakan duplikasi

-- TrainingRecords tanpa FK yang seharusnya ada
-- (dibuat sebagai renewal tapi FK null — judul sama ada lebih dari 1 record per user)
SELECT
    tr1.Id,
    tr1.Judul,
    tr1.UserId,
    tr1.RenewsTrainingId,
    tr1.RenewsSessionId,
    tr1.CreatedAt
FROM TrainingRecords tr1
WHERE tr1.RenewsTrainingId IS NULL
  AND tr1.RenewsSessionId IS NULL
  AND tr1.Judul IN (
      SELECT Judul
      FROM TrainingRecords
      GROUP BY Judul, UserId
      HAVING COUNT(*) > 1
  )
ORDER BY tr1.Judul, tr1.UserId, tr1.CreatedAt;

Query 2: AssessmentSessions tanpa FK renewal yang mungkin merupakan duplikasi

-- AssessmentSessions tanpa FK yang seharusnya ada
-- (IsPassed=1 dan GenerateCertificate=1 — ada lebih dari 1 session dengan judul sama per user)
SELECT
    a1.Id,
    a1.Title,
    a1.UserId,
    a1.RenewsSessionId,
    a1.RenewsTrainingId,
    a1.Schedule,
    a1.ValidUntil
FROM AssessmentSessions a1
WHERE a1.RenewsSessionId IS NULL
  AND a1.RenewsTrainingId IS NULL
  AND a1.IsPassed = 1
  AND a1.GenerateCertificate = 1
  AND a1.Title IN (
      SELECT Title
      FROM AssessmentSessions
      WHERE IsPassed = 1 AND GenerateCertificate = 1
      GROUP BY Title, UserId
      HAVING COUNT(*) > 1
  )
ORDER BY a1.Title, a1.UserId, a1.Schedule;
Catatan interpretasi query: Hasil query di atas bukan berarti semua record yang muncul adalah data bermasalah. Record bisa muncul karena pekerja memang mengikuti sertifikasi yang sama lebih dari sekali secara legitimate (misalnya, sertifikasi 3 tahunan yang memang dibuat ulang). Investigasi manual diperlukan untuk menentukan mana yang merupakan renewal yang seharusnya memiliki FK.
