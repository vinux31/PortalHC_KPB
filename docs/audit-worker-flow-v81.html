Keputusan Arsitektur

[SignalR Worker Push] AdminController menggunakan Clients.User(userId) untuk per-worker signal (Reset/ForceClose) — user-targeting built-in di ASP.NET Core SignalR, tidak perlu JoinWorkerSession method tambahan di hub. Bulk close menggunakan Clients.Group($"batch-{batchKey}") yang sudah di-join via JoinBatch() di assessment-hub.js.
[assessmentHubStartPromise] Menggantikan setTimeout(2000) fallback yang flaky. async startHub() sudah return Promise secara native — expose ke window.assessmentHubStartPromise untuk digunakan consumer page. StartExam.cshtml menggunakan .then() untuk set badge Live setelah hub benar-benar connect.
[pendingAnswers queue] Queue di-implement di StartExam.cshtml scope (bukan di assessment-hub.js) karena saveAnswerAsync ada di scope yang sama. connection.onreconnected() di StartExam retry semua pending answers secara sequential.
[Timer wall clock] Timer exam di-refactor ke wall clock anchor: timerStartWallClock = Date.now() sebagai baseline. Setiap tick menghitung elapsed = (Date.now() - anchor) / 1000 — tidak ada drift setelah tab minimize atau laptop sleep. elapsedSeconds dihitung dari ELAPSED_SECONDS_FROM_DB + elapsed untuk akurasi progress save.
