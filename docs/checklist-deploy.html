Checklist Deploy ke Server

PortalHC KPB — HC Portal Human Capital Development

1. Pastikan Database Up-to-Date

Database lokal harus punya semua tabel yang dibutuhkan. Jalankan perintah ini untuk apply semua migrasi:

dotnet ef database update
  • Semua migrasi ter-apply tanpa error
  • Tabel-tabel utama ada: Users, AssessmentSessions, TrainingRecords, AssessmentCategories, CoachCoacheeMappings, dll

2. Testing Manual (UAT)

Flow Utama yang Harus Jalan

  • Login/logout semua role (Admin, HC, Worker)
  • Buat assessment → assign worker → worker ujian → lihat hasil → lihat sertifikat
  • Download sertifikat PDF (header Pertamina + P-Sign footer)
  • Coaching Proton (mapping → isi coaching → lihat histori)
  • Import data (workers, training, silabus Proton)
  • Export data (records, audit log, histori Proton)
  • Kelola Data admin — CRUD kategori, workers, unit kerja
  • Manage Categories — parent/sub-category hierarchy + signatory

Yang Sering Bermasalah

  • Akses halaman dengan role yang salah (harusnya ditolak / redirect ke AccessDenied)
  • Halaman kosong tanpa data (harus tampil pesan kosong, bukan error)
  • Upload file besar (timeout? error?)
  • Ujian dengan banyak soal (performa loading)
  • Concurrent users — dua orang ujian bersamaan

3. Cleanup Data Test

Hapus data testing yang tidak perlu sebelum deploy ke production:

  • User dummy / akun test
  • Assessment percobaan
  • Training record palsu
  • Coaching session test
Catatan: Juga jalankan cleanup untuk data double yang dibuat oleh bug ASSESS-08:
DELETE FROM TrainingRecords WHERE Judul LIKE 'Assessment: %';

4. Konfigurasi Server

Yang perlu disiapkan di server developer:

ItemPenjelasan
.NET RuntimeInstall .NET 9.0 Runtime (bukan SDK) di server
DatabasePilih: tetap SQLite (simple) atau pindah ke SQL Server (production-grade)?
HTTPS / SSLSertifikat SSL untuk keamanan koneksi
appsettings.Production.jsonConnection string, base URL, LDAP config untuk environment production
File storagePastikan folder wwwroot/uploads bisa ditulis oleh aplikasi
IIS / Reverse proxySetup IIS atau Nginx sebagai reverse proxy ke Kestrel

5. Security Check

  • Tidak ada password/secret hardcoded di source code
  • appsettings.json tidak mengandung credential sensitif (gunakan Environment Variables atau User Secrets)
  • LDAP connection string sudah benar untuk server production
  • File .env, appsettings.Development.json tidak ikut ter-deploy
  • Error page production tidak menampilkan stack trace (UseDeveloperExceptionPage hanya di Development)
Penting: Jangan deploy file HcPortal.db development ke production — database production harus fresh atau dari backup yang bersih.

6. Build & Publish

Jalankan perintah ini untuk build versi production:

dotnet publish -c Release -o ./publish
  • Build sukses tanpa error
  • Folder publish/ berisi semua file yang dibutuhkan
  • Copy isi folder publish/ ke server
  • Di server, jalankan: dotnet HcPortal.dll
  • Buka browser, akses aplikasi — pastikan halaman login muncul

7. Post-Deploy Verification

Setelah aplikasi jalan di server:

  • Login dengan akun Admin — dashboard muncul
  • Cek semua menu navigasi bisa diakses
  • Buat 1 assessment test → assign → ujian → lihat sertifikat
  • Import 1 file Excel kecil — pastikan berhasil
  • Cek LDAP login (jika pakai LDAP di production)
