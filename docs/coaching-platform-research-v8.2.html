Ringkasan 3 Platform Coaching

Tujuan dokumen ini: Memberikan baseline perbandingan konkret antara Portal KPB (sistem Proton coaching internal Pertamina) dengan tiga platform coaching enterprise. Dokumen ini menjadi lens utama bagi Phase 234–237 untuk memutuskan apa yang perlu diperbaiki, berdasarkan gap nyata — bukan asumsi.

360Learning

Jenis: LMS kolaboratif dengan fitur coaching peer/video

Target: L&D team yang ingin enable peer learning dan manager coaching

Pendekatan: Task/content-based — coaching via video assessment, scoring rubric, feedback kualitatif

  • Learning path hierarkis (setara track Proton)
  • Manager sebagai reviewer/coach terintegrasi langsung
  • Criteria guidelines untuk konsistensi antar coach
  • Certificate otomatis setelah semua modul selesai

BetterUp

Jenis: Enterprise coaching platform 1:1 dengan AI matching

Target: Perusahaan Fortune 500 — leadership development dan wellbeing

Pendekatan: Relationship-based — AI mencocokkan coachee dengan coach tersertifikasi eksternal

  • AI matching berdasarkan goals, preferensi, assessment
  • Workload visibility coach: jumlah coachee dan kapasitas visible di admin dashboard
  • Privacy-first: manajer tidak bisa lihat isi sesi coaching
  • ROI measurement: before/after assessment comparison

CoachHub

Jenis: Digital coaching B2B dengan marketplace coach tersertifikasi (3.500+ coach, 80 bahasa)

Target: Enterprise semua level karyawan, fokus measurement dan ROI

Pendekatan: Hybrid AI+human — AI companion (AIMY 2.0) dan pool coach global

  • CoachHub Insights: bottleneck visibility, identifikasi program stagnant
  • Bulk action dari monitoring view untuk high-volume approval
  • Action items di-track di dalam platform
  • Skill mastery report sebagai bukti completion

Posisi Portal KPB di antara Ketiga Platform

Portal KPB paling dekat dengan 360Learning dalam hal deliverable tracking hierarkis (Kompetensi → Sub-Kompetensi → Deliverable adalah hierarki 3-level yang tepat untuk konteks manufaktur/industri Pertamina). Namun gap signifikan ditemukan di area:

  • Onboarding coach — tidak ada visibility workload sebelum assign (gap vs BetterUp)
  • Batch action — tidak ada bulk approval HC Review (gap vs CoachHub)
  • Bottleneck visibility — tidak ada analytics mana yang stuck (gap vs CoachHub Insights)
  • Safety constraint — tidak ada guard concurrent approval, duplicate final assessment, hard-delete safety (gap vs semua platform)

Ketiga platform berbeda pendekatan: 360Learning task/content-based, BetterUp relationship-based, CoachHub hybrid AI+human. Portal KPB memiliki keunggulan konteks industrial yang tidak perlu digantikan — yang perlu diperbaiki adalah reliability, visibility, dan operational efficiency.

Area 1: Setup

Mencakup: konfigurasi silabus, track assignment, coach-coachee mapping, guidance file, import/export Excel.

Baseline As-Is Portal KPB

  • ProtonTrack: Track per bagian/unit (Tahun 1/2/3), dikonfigurasi Admin via ProtonDataController
  • Silabus hierarki 3-level: Kompetensi > Sub-Kompetensi > Deliverable; CRUD di ProtonDataController
  • Coaching Guidance: File upload (PDF/PPT) per track; ditampilkan via PlanIdp.cshtml tab kedua
  • Coach-Coachee Mapping: Admin assign coach ke coachee via AdminController; cascade deactivation ada tapi tanpa explicit DB transaction
  • Track Assignment: Admin assign coachee ke track (Tahun 1/2/3); seeding ProtonDeliverableProgress saat assign
  • Import/Export Excel: Ada untuk silabus dan mapping, dengan downloadable template; error handling belum diaudit end-to-end
  • Validasi: Tidak ada validasi duplikasi coach-coachee; tidak ada impact warning saat delete silabus yang aktif digunakan

360Learning

Berdasarkan dokumentasi resmi 360Learning, flow Setup berjalan sebagai berikut:

  1. Admin/author membuat learning path (setara track Proton) yang terdiri dari modul, kuis, dan video assessment exercise
  2. Criteria coaching dapat dikustomisasi per exercise — setara dengan silabus deliverable — dengan guidelines spesifik untuk memastikan konsistensi scoring antar reviewer
  3. Manager di-assign sebagai reviewer/coach per learner group, langsung di dalam platform tanpa step tambahan
  4. Konten learning path dapat di-versioning; jika ada progress aktif, soft-delete (arsip) digunakan alih-alih hard-delete
  5. Bulk import learner/mapping tersedia dengan validasi ketat per baris dan error report terstruktur

BetterUp

Berdasarkan dokumentasi resmi BetterUp, flow Setup adalah:

  1. HR/admin mendefinisikan program scope: goals program, populasi karyawan yang masuk, dan durasi program
  2. AI matching sistem secara otomatis mencocokkan coachee dengan coach berdasarkan goals, preferensi personal, dan hasil assessment awal
  3. Coachee mendapatkan shortlist beberapa coach yang direkomendasikan — coachee dapat memilih sendiri, bukan pure force-assign
  4. Tidak ada "track" hierarkis seperti KPB — pendekatan lebih fleksibel dan individualized per coachee
  5. Kapasitas coach (jumlah coachee aktif) tersedia di admin dashboard — HR bisa detect overload sebelum assign

CoachHub

Berdasarkan dokumentasi resmi CoachHub, flow Setup adalah:

  1. HR/admin mendefinisikan program scope dan populasi (siapa yang mendapat coaching, level/divisi apa)
  2. AI matching menyarankan coach terbaik dari pool 3.500+ coach tersertifikasi berdasarkan kebutuhan development
  3. Coachee dapat memilih coach dari shortlist yang AI rekomendasikan — memberikan agency kepada coachee
  4. Tidak ada "silabus" formal terstruktur — agenda sesi ditentukan bersama antara coach dan coachee di awal program
  5. Program dapat di-scale dari pilot (puluhan orang) ke enterprise-wide (ribuan orang) dengan konfigurasi yang sama

Area 2: Execution

Mencakup: submission evidence deliverable, approval chain multi-level, override admin, notifikasi, view silabus coachee.

Baseline As-Is Portal KPB

  • Evidence Submission: Coachee upload file bukti per deliverable; multi-file support tersedia
  • Approval Chain: Sr Supervisor → Section Head → HC Review (3 level); status field-based di ProtonDeliverableProgress
  • Override Admin: Admin bisa override status di luar chain normal; audit trail override belum diverifikasi
  • Notifikasi: Sistem dual (ProtonNotification + UserNotification); trigger di berbagai state transition, tapi coverage belum diaudit
  • PlanIdp View: Coachee lihat silabus dan guidance; role-scoped (coachee locked ke bagian sendiri)
  • State History: ProtonDeliverableHistory entity ada, tapi kelengkapan insert di semua trigger belum diverifikasi
  • Concurrent Approval: Tidak ada guard — dua approver bisa approve bersamaan, potensi corrupt state

360Learning

Berdasarkan dokumentasi resmi 360Learning, flow Execution untuk coaching assessment adalah:

  1. Learner submit video recording (pitch, skill demonstration) secara asynchronous — tidak perlu sinkronisasi jadwal dengan coach
  2. Coach/manager mereview submission melalui platform dengan scoring rubric yang sudah dikonfigurasi saat Setup, dilengkapi komentar kualitatif di forum 1:1
  3. Satu layer reviewer saja (tidak ada multi-level approval chain seperti KPB); ini memang berbeda konteks — KPB memiliki hierarki organisasi yang lebih formal
  4. Setelah review: learner mendapat notifikasi dan bisa revisi serta resubmit jika belum lolos
  5. Setiap state transition (submit, review, approve, reject) tercatat otomatis dengan timestamp dan aktor

BetterUp

Berdasarkan dokumentasi resmi BetterUp, flow Execution adalah:

  1. Sesi 1:1 via video call (Zoom/Teams/BetterUp native) — jadwal fleksibel, diatur sendiri oleh coachee dan coach berdasarkan kalender masing-masing
  2. Setelah sesi, coachee submit action items dan refleksi — tidak ada "evidence file upload" seperti KPB karena model coachingnya berbeda (conversation-based vs deliverable-based)
  3. Coach tidak approve/reject deliverable formal — peran coach adalah membimbing dan memberikan feedback, bukan evaluator output formal
  4. Progress diukur via assessment sebelum dan sesudah program — before/after comparison untuk mengukur perubahan
  5. Semua state transition (schedule, attend, submit action item, complete) tercatat dengan privacy guard: manajer tidak bisa akses detail isi sesi

CoachHub

Berdasarkan dokumentasi resmi CoachHub, flow Execution adalah:

  1. Sesi via video, call, atau chat — format fleksibel sesuai preferensi coachee dan coach, tanpa ada satu "media resmi" yang wajib
  2. CoachHub Companion (AI AIMY 2.0): support 24/7 antara sesi via Teams integration — coachee bisa latihan skill, tanya pertanyaan, atau drill konsep tanpa menunggu sesi berikutnya
  3. Action items di-track di dalam platform — coachee set komitmen, update progress, visible untuk coach
  4. Tidak ada "multi-level approval chain" — coach dan coachee menyepakati goal achievement bersama, dengan HR hanya melihat high-level metrics

Area 3: Monitoring

Mencakup: dashboard HC/coach, filter dan export, override admin, workload visibility, batch action, bottleneck analysis.

Baseline As-Is Portal KPB

  • CoachingProton page: Tabel deliverable dengan filter Bagian/Unit/Track/Tahun/Coachee; role-scoped
  • Dashboard: Chart.js untuk stats; role-scoped (HC/Admin lihat semua, coach lihat coachee-nya saja)
  • Export: Excel export progress data tersedia
  • Override Admin: Admin bisa override status; audit trail override belum diverifikasi
  • Pagination: Implemented di CoachingProton tracking table
  • Workload visibility: Tidak ada — HC tidak bisa lihat berapa coachee yang di-handle per coach
  • Batch action: Tidak ada — approval harus dilakukan satu per satu
  • Bottleneck analysis: Tidak ada — tidak ada visibility mana deliverable yang stuck paling lama

360Learning

Berdasarkan dokumentasi resmi 360Learning, fitur Monitoring mencakup:

  1. Manager dashboard menampilkan statistik per managee: enrollment rate, completion rate, average score, dan time-to-complete
  2. Weekly automated email report dikirim ke manager — ringkasan progress tim tanpa perlu cek dashboard manual
  3. Completion tracking per course, per cohort, dan per compliance group — dengan filter multi-dimensi
  4. Bulk completion approval: manager bisa approve multiple submissions sekaligus dari list view — mengurangi click per action
  5. Tidak ada "admin override" di luar chain — eskalasi dilakukan melalui peran admin group

BetterUp

Berdasarkan dokumentasi resmi BetterUp, fitur Monitoring mencakup:

  1. Admin dashboard: engagement metrics, session attendance rate, program ROI comparison across cohorts
  2. BetterUp Manage: modul khusus untuk manajer — visibility perkembangan tim tanpa mengekspos isi sesi (privacy-first design)
  3. Integrasi dengan Slack, Teams, Workday untuk notifikasi dan behavioral nudges langsung di tools sehari-hari
  4. ROI report: before-after assessment comparison, business impact metrics, goal achievement rate per program
  5. Coach capacity/workload visible di admin: berapa coachee aktif per coach, kapasitas tersisa — ini yang relevan untuk DIFF-01

CoachHub

Berdasarkan dokumentasi resmi CoachHub, fitur Monitoring mencakup:

  1. CoachHub Insights: real-time analytics tentang coaching impact company-wide — session frequency, goal progress, skill mastery metrics
  2. Dashboard HR: session attendance, goal progress per coachee, program engagement rate
  3. Identifikasi stagnant: Insights dapat menunjukkan program atau coachee yang tidak ada aktivitas dalam periode tertentu — ini yang relevan untuk DIFF-03
  4. Bulk action dari monitoring view: HR bisa melakukan action (reminder, eskalasi, approval) dari list view tanpa buka satu per satu — ini yang relevan untuk DIFF-02
  5. Coachee track personal growth sendiri di platform — self-monitoring yang reduce beban HR

Area 4: Completion

Mencakup: final assessment, competency level granting, coaching session linkage, histori, transisi 3-year journey.

Baseline As-Is Portal KPB

  • Final Assessment: Setelah semua deliverable approved, trigger competency level granting
  • Coaching Sessions: Entity terpisah (CoachingSession); linked ke deliverable progress; action items
  • HistoriProton: Timeline aktivitas coaching; coexist dengan legacy CoachingLog
  • 3-Year Journey: Tahun 1 → 2 → 3 lifecycle; assignment transition per tahun
  • Unique Constraint: Tidak ada DB unique constraint untuk final assessment — double-click atau retry bisa create duplicate
  • Session Linkage: CoachingSession entity ada, tapi linkage ke deliverable progress belum diverifikasi kelengkapannya
  • Progression Validation: Validasi agar tidak bisa skip Tahun 1 langsung ke Tahun 3 belum diaudit

360Learning

Berdasarkan dokumentasi resmi 360Learning, flow Completion adalah:

  1. Certificate of completion diterbitkan otomatis setelah semua modul dalam learning path selesai — tidak perlu trigger manual dari admin
  2. Approval workflow opsional tersedia: manager atau group admin dapat approve/reject sertifikat sebelum diterbitkan
  3. Download sertifikat sebagai PDF — otomatis tersedia di profil learner; admin dapat download completion report per cohort
  4. Recertification: validity period dapat di-set per certification, dengan auto re-enrollment saat expiry — completion cycle dapat berulang
  5. Session notes linked ke deliverable progress dengan action item tracking — timeline completion terhubung ke seluruh journey

BetterUp

Berdasarkan dokumentasi resmi BetterUp, flow Completion adalah:

  1. Tidak ada "final assessment" formal seperti KPB — completion didefinisikan sebagai: jumlah sesi terpenuhi + goals achieved sesuai target di awal program
  2. Coach menyusun summary perkembangan coachee untuk diberikan kepada HR — sebagai bukti program selesai
  3. Program bisa dilanjutkan (renewal dengan coach yang sama atau baru) atau dihentikan sesuai kebutuhan
  4. Tidak ada "3-year journey" eksplisit — tenure coaching lebih fleksibel sesuai kebutuhan organisasi
  5. Before/after assessment comparison sebagai evidence bahwa program memberikan dampak

CoachHub

Berdasarkan dokumentasi resmi CoachHub, flow Completion adalah:

  1. Program berakhir setelah durasi yang di-set saat Setup (misalnya 6 bulan); tidak ada trigger manual dari admin
  2. Summary perkembangan disusun oleh coach dan tersedia di platform untuk HR review
  3. Skill mastery report sebagai bukti completion — menggantikan certificate formal; berisi metrics konkret kemajuan coachee
  4. Tidak ada certificate formal — evidence adalah analytics data dari CoachHub Insights
  5. Siklus dapat berlanjut: setelah program selesai, coachee dapat enroll ke program baru dengan kesinambungan data

Implikasi untuk Implementasi

Ketiga differentiator ini direncanakan untuk Phase 237 (fase terakhir audit v8.2) setelah setup, execution, dan monitoring foundation diperbaiki di Phase 234–236. Urutan ini tepat karena:

  • DIFF-01 (workload indicator) membutuhkan data coach-coachee mapping yang clean — diperbaiki di Phase 234
  • DIFF-02 (batch approval) membutuhkan approval chain yang robust — diperbaiki di Phase 235
  • DIFF-03 (bottleneck analysis) membutuhkan state history yang lengkap — diperbaiki di Phase 235 dan 236
