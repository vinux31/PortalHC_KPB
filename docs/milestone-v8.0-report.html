1. Ringkasan Eksekutif

Milestone v8.0 "Assessment & Training System Audit" bertujuan mengaudit dan memperkuat fondasi sistem assessment dan training di Portal HC KPB. Dari 23 requirements yang didefinisikan, 12 terpenuhi, 2 di-defer per keputusan user, dan 10 orphaned karena phase-nya di-remove dari scope milestone.

Scope Aktual: 3 phases dieksekusi (223, 224, 227) dari 5 yang direncanakan. Phase 225 (Training Compliance) dan Phase 226 (Email Notification) di-remove dari scope untuk fokus pada legacy cleanup yang lebih kritikal.

Highlight:

  • Skor ElemenTeknis per session sekarang tersimpan di database (bukan hanya agregat)

  • Legacy question path sepenuhnya dimigrasi ke package format

  • Orphan tables (AssessmentCompetencyMaps, UserCompetencyLevels) dibersihkan

Detail Fix: CLEN-04 x AkhiriUjian (Integration Gap)

Masalah Ditemukan Saat Audit #1

Phase 227 memindahkan NomorSertifikat generation dari CreateAssessment ke SubmitExam. Namun, AkhiriUjian dan AkhiriSemuaUjian (HC force-close paths) memanggil GradeFromSavedAnswers dan set IsPassed/Status=Completed tanpa generate NomorSertifikat.

Impact: Jika HC force-close session dimana worker passed, NomorSertifikat stays null permanently — worker tidak bisa download sertifikat.

Fix Applied: Commit 478cca2

Cert generation block ditambahkan ke kedua path:

PathFileLinePattern
AkhiriUjian (single)AdminController.cs2616–2643if (session.GenerateCertificate && session.IsPassed == true) → CertNumberHelper.GetNextSeqAsync + Build
AkhiriSemuaUjian (bulk)AdminController.cs2710–2737foreach (s in sessionsToEnd.Where(...)) → same cert generation with retry loop

Ketiga path (SubmitExam, AkhiriUjian, AkhiriSemuaUjian) sekarang menggunakan pola identical: while-loop retry dengan CertNumberHelper.
