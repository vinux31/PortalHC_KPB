Pilihan Diagram Landscape Style — Discussion

3 opsi diagram landscape untuk §3.4 HC Portal. Tiap opsi: standar, contoh visual Sebelum vs Sesudah, pro/con. Pilih yang paling cocok untuk PCP submission.

🏛️ Opsi 1 — Layered Aktor (Purdue-Style Adaptation)

⭐ RECOMMENDED
Standar: Industry de-facto (Layered Architecture), turunan Purdue Model (ISA-95 / IEC 62443)
Origin: Adaptasi Purdue OT untuk web app HR
Familiar Untuk: Reviewer PCP yang sudah lihat slide OT

👍 Pro

  • Visual match slide PCP referensi (5 layer vertikal)
  • Reviewer langsung familiar (sejajar Purdue OT)
  • Sebelum vs Sesudah side-by-side natural
  • Mudah tempel marker A-F (issue) dan 1-7 (improvement)
  • Bisa render HTML/CSS atau redraw PowerPoint dengan mudah
  • Hub HC Portal di tengah = pesan visual kuat

👎 Con

  • "Layer aktor" bukan layer hirarkis murni (Purdue = network layer fisik)
  • Bisa terlihat dipaksakan kalau reviewer paham Purdue strict
🏆 Rekomendasi utama — visual paling match dengan slide PCP referensi, paling siap untuk submission.

👍 Pro

  • Standar formal C4 Model — citation-able
  • Visual "hub-and-spoke" sangat impactful (HC Portal di tengah)
  • Cocok untuk executive overview 1 page
  • Kontras Sebelum (scattered) vs Sesudah (centered) sangat jelas

👎 Con

  • Tidak match slide PCP (slide pakai 5 layer, C4 pakai hub)
  • Reviewer non-IT bisa asing dengan C4
  • Susah tempel marker A-F + 1-7 secara visual
  • Tidak menunjukkan tier/hirarki aktor
⚠️ Alternatif kuat tapi tidak match slide PCP referensi. Lebih cocok untuk dokumen IT internal.

Familiar Untuk: Enterprise architect (mostly tidak ada di PCP reviewer)

👍 Pro

  • Cocok tunjukkan arsitektur teknis + bisnis
  • 3 layer rapi (Business / App / Tech)
  • Bagus untuk audience IT architect

👎 Con

  • Tidak menunjukkan aktor (Pekerja/Coach/dll. hilang)
  • Sangat technical — reviewer PCP non-IT bingung
  • Tidak match slide PCP referensi visual
  • Fokus arsitektur, bukan workflow improvement
  • Hanya 3 layer (slide PCP punya 5)
⚠️ Bagus untuk dokumen arsitektur teknis, kurang cocok untuk improvement narrative PCP.
