Overview

Dokumen ini merupakan riset best practices untuk assessment/exam management dari sisi admin — bagaimana platform terkemuka mengelola pembuatan soal, penugasan assessment kepada peserta, serta pelaporan hasil.

Konteks Portal HC KPB

Portal KPB menggunakan model package-based assessment: soal dikelompokkan dalam paket (package) per kompetensi. Admin bisa membuat package, menambahkan soal ke package, lalu menugaskan package ke worker. Sistem ini sudah cukup mature, namun ada beberapa gap dalam hal UX filter/search, konfirmasi bulk assignment, dan preview assessment.


Riset ini membandingkan admin-side management flow dari tiga platform yang berbeda karakteristik: Moodle (LMS enterprise open-source), Google Forms Quiz (tool sederhana), dan Examly (platform assessment enterprise modern).

Fokus Riset

  • Flow admin membuat dan mengkonfigurasi assessment
  • Sistem question bank — bagaimana soal dikelola terpisah dari assignment
  • Mekanisme penugasan assessment ke peserta (individual dan bulk)
  • Fitur monitoring dan reporting pasca-exam
  • Gap antara Portal HC KPB dan best practices industri

Moodle Quiz

Model Platform

Moodle adalah open-source Learning Management System (LMS) yang digunakan oleh ribuan institusi pendidikan dan korporat. Quiz module adalah salah satu fitur intinya — Moodle 5.0 memperkenalkan "New Quiz" dengan pre-creating attempts. Confidence: HIGH (verified dari Moodle official documentation docs.moodle.org/501).

Admin UX Flow: Membuat Assessment

Step 1
Course → Add Activity/Resource → Quiz
→
Step 2
Konfigurasi Quiz: nama, tanggal open/close, time limit, grading method
→
Step 3
Question Bank: buat/import soal di bank terlebih dahulu, dikelompokkan per kategori
→
Step 4
Tambahkan soal ke Quiz: pilih manual atau "random dari bank per kategori"
→
Step 5
Preview Quiz sebagai student — validasi tampilan dan flow
→
Step 6
Reports: Quiz → Responses / Statistics (pasca exam)

Deskripsi UI

Moodle menggunakan antarmuka form yang terstruktur namun terasa lama dibanding platform modern:

  • Form wizard multi-step: setting quiz dibagi ke beberapa section (General, Timing, Grade, Layout, Question Behavior, Review Options, Appearance, Extra Restrictions, Overall Feedback)
  • Question bank sidebar tree: soal diorganisasi dalam tree hierarki per kategori/subkategori — admin drag-drop ke quiz atau klik "Add to Quiz"
  • Drag-drop question ordering: urutan soal bisa diatur dengan drag handle di sebelah kiri tiap soal
  • Grading method dropdown: Highest grade / Last attempt / First attempt / Average of all attempts
  • Review settings matrix: tabel checkbox kapan student bisa melihat: attempt (during attempt), immediately after attempt, later while quiz is still open, after quiz is closed

Fitur Unggulan Moodle untuk Referensi

Separate Question Bank

Soal tidak terikat ke satu quiz — soal yang sama bisa digunakan di banyak quiz berbeda. Bank dikelola secara terpusat dengan kategori dan tag.

Gap: Portal KPB soal terikat ke package, belum ada true question bank terpisah

Random Questions dari Bank

Tiap peserta mendapat set soal acak dari bank per kategori. Mengurangi kemungkinan contekan dan memastikan fairness.

Gap: Portal KPB belum ada randomisasi per peserta

Preview sebagai Student

Admin bisa mencoba quiz dengan mode "Preview" yang menampilkan tampilan persis seperti yang dilihat student — tanpa membuat attempt sungguhan.

Gap: Portal KPB belum ada mode preview admin

Review Settings Granular

Admin bisa set kapan student boleh lihat jawaban benar — saat pengerjaan, langsung setelah selesai, atau setelah semua peserta selesai.

Portal KPB punya conditional review — sudah mirip

Grade Override per Student

Instructor bisa override nilai individual jika ada permohonan keberatan atau faktor khusus.

Perlu dicek apakah Portal KPB memiliki ini

Attempt Tracking

Setiap attempt tercatat: status (In Progress / Completed / Abandoned), waktu mulai, waktu selesai, nilai, soal yang dijawab. Admin bisa lihat per student.

Portal KPB memiliki tracking serupa

Kelemahan Moodle

Kompleksitas Setup vs Simplisitas Portal KPB

Setup Moodle Quiz membutuhkan banyak langkah (course → activity → bank → questions → randomize → review settings). Untuk organisasi dengan assessment sederhana seperti Portal KPB, kompleksitas ini menjadi overhead. Portal KPB lebih streamlined untuk use case-nya.

Monitoring Tidak Real-Time Native

Moodle tidak menyediakan live monitoring dashboard tanpa plugin tambahan. Admin harus refresh halaman Reports secara manual untuk melihat progress terkini. Ini adalah kelemahan yang sudah diatasi Portal KPB dengan monitoring page-nya.

Google Forms Quiz

Model Platform

Google Forms Quiz adalah fitur quiz sederhana di atas Google Forms. Grading otomatis untuk soal pilihan ganda. Tidak ada fitur time limit native. Confidence: HIGH (verified dari Google Workspace Learning Center).

Relevansi untuk Portal KPB: Google Forms Quiz relevan sebagai negative reference — menunjukkan fitur apa yang penting dimiliki Portal KPB dan apa yang tidak boleh diabaikan. Portal KPB sudah jauh lebih advanced dari Google Forms Quiz.

Admin UX Flow: Membuat Quiz

Step 1
Buat form baru di Google Forms
→
Step 2
Settings → "Make this a quiz" toggle aktifkan
→
Step 3
Add questions → set answer key per soal → assign points
→
Step 4
Share via link atau embed di website
→
Step 5
Responses → View in Sheets untuk analisis lanjutan

Deskripsi UI

Google Forms menerapkan desain Material Design yang sangat bersih dan mudah digunakan:

  • WYSIWYG question editor: soal diedit langsung di tempat, termasuk preview opsi pilihan ganda
  • Inline answer key: setelah "Make this a quiz" aktif, muncul tombol "Answer Key" di setiap soal — klik langsung set jawaban benar dan poin
  • Drag-drop question reorder: handle drag di sisi kiri tiap soal
  • Response summary chart: setelah ada respons, tab "Responses" menampilkan grafik otomatis per pertanyaan
  • Individual response view: admin bisa klik per responden untuk melihat jawaban lengkap

Pola yang Relevan (Positif)

Simplisitas Pembuatan

Basic quiz bisa dibuat dalam kurang dari 5 menit. UX sangat streamlined untuk use case sederhana. Mengajarkan bahwa kompleksitas setup harus diminimalkan.

Auto-Grading Multiple Choice

Penilaian otomatis langsung setelah submit — tidak ada delay manual grading untuk soal objektif. Portal KPB sudah mengadopsi ini.

Portal KPB sudah match

Response Summary Visual

Chart otomatis menunjukkan distribusi jawaban per soal. Berguna untuk admin melihat soal mana yang paling banyak dijawab salah (analisis item dasar).

Gap: Portal KPB belum ada chart distribusi jawaban

Examly / iamneo.ai

Model Platform

Examly (kini rebranding sebagai iamneo.ai) adalah platform assessment enterprise yang fokus pada campus recruitment dan corporate assessment. Digunakan oleh perusahaan besar untuk pre-employment testing dan in-house certification. Confidence: MEDIUM (dari Tracxn profile + design portfolio case study).

Admin UX Flow: Membuat dan Mengelola Assessment

Step 1
Admin Dashboard → Create Test (wizard langkah per langkah)
→
Step 2
Add Questions: manual, import Excel, atau dari question bank
→
Step 3
Assign Candidates: individual (email invite) atau bulk upload via Excel
→
Step 4
Proctoring settings: webcam, screen sharing, browser lockdown level
→
Step 5
Schedule: set window start/end tanggal dan waktu
→
Step 6
Monitor live → lihat status real-time per kandidat
→
Step 7
Reports post-exam: scoring otomatis, ranking, per-question analytics

Deskripsi UI

Examly menggunakan tampilan dashboard modern dengan grid layout:

  • Admin dashboard: kartu statistik overview (total tests, active, completed, candidates invited)
  • Excel import wizard: drag-drop file → preview data → validasi error → konfirmasi import — multi-step dengan progress indicator
  • Candidate management grid: tabel kandidat dengan kolom status (Invited / Started / Submitted / Scored), sortable dan filterable
  • Question bank browser: grid soal dengan filter type (MC, essay, coding), tag, dan difficulty
  • Live monitoring view: tile tiap kandidat dengan foto (dari proctoring) + status badge real-time
  • Report dashboard: chart distribusi skor, daftar ranking, per-question difficulty analysis

Fitur Unggulan Examly untuk Referensi

Bulk Candidate Import via Excel

Admin upload file Excel dengan daftar kandidat, sistem parse otomatis, tampilkan preview sebelum konfirmasi. Wizard multi-step dengan validasi error per row.

Portal KPB sudah punya import Excel untuk workers — pola serupa

AI-Powered Proctoring

Eye tracking, face detection, tab monitoring — semua real-time. Flag otomatis jika deteksi pelanggaran. Portal KPB memiliki tab-switch detection (AINT-02/03) yang sedang dikembangkan.

Portal KPB dalam progress (v8.0)

Per-Question Analytics

Difficulty index per soal (berapa persen peserta menjawab benar), rata-rata waktu pengerjaan per soal. Membantu admin evaluasi kualitas soal.

Gap: Portal KPB belum ada item analytics

Automated Scoring dan Ranking

Nilai kalkulasi otomatis setelah submit, daftar ranking langsung tersedia. Portal KPB sudah punya ini untuk pass/fail.

Portal KPB sudah match untuk scoring dasar
