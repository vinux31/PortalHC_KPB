§2 Konteks Benchmark — 9 Platform

9 platform benchmark dibagi 2 grup: 5 HRIS+LMS Enterprise (best practice digital cert mainstream) + 4 platform Industri Migas (best practice sektor sama).

2.1 HRIS + LMS Enterprise (5 Platform)

Workday Learning HRIS Enterprise

Workday Learning terintegrasi langsung dengan modul HRIS, sehingga sertifikat, training, dan kompetensi pekerja terupdate otomatis saat pekerja pindah jabatan atau departemen. Punya skill cloud untuk mapping kompetensi ke jabatan + auto-assign remediation training saat ada gap. Compliance tracking real-time per departemen dengan reminder otomatis pre-expired.

Relevansi Portal HC: Pertamina punya ekosistem HR kompleks dengan KKJ per jabatan — pattern HRIS-Learning native Workday ideal untuk integrasi data pekerja.

Workday Learning Docs
SAP SuccessFactors Learning HRIS Enterprise

SAP SuccessFactors Learning fokus pada compliance enterprise dengan reporting drill-down by population/role/location. Joule AI agent (rilis 1H 2026) bertindak proaktif: identifikasi siapa at-risk, kirim reminder 4 minggu sebelum deadline, dan bundle pending approval jadi action queue siap-pakai untuk manager.

Relevansi Portal HC: SAP dipakai luas di BUMN Indonesia termasuk Pertamina — pattern compliance + audit reporting SuccessFactors familiar untuk tim HC.

SAP SuccessFactors 1H 2026
Cornerstone Learning LMS Dedicated

Cornerstone Learning dedicated LMS dengan fokus compliance regulated industry. Audit trail "defensible": setiap interaksi training tercatat dengan timestamp, IP address, dan score — siap untuk inspeksi regulator. Standar 21 CFR Part 11 (life sciences) didukung lewat konfigurasi e-signature + audit trail.

Relevansi Portal HC: Industri Migas/Energy = regulated sector. Pattern rigorous audit trail Cornerstone cocok untuk compliance audit Pertamina (BPK, internal audit, regulator).

Cornerstone Compliance
Docebo Cloud LMS Modern

Docebo cloud LMS modern dengan fokus AI + automation. AI otomatis tag skill ke konten saat upload, certification renewal bisa di-trigger user kapan saja (self-service), webhook event untuk skill changes terintegrasi sistem external. AgentHub 2026 satukan AI agent + skill intelligence dalam satu platform.

Relevansi Portal HC: Pattern AI auto-tag skill + self-service renewal bisa diadopsi bertahap ke modul CDP/CMP, terutama mapping otomatis KKJ ↔ sertifikat.

Docebo Inspire 2026
TalentLMS Mid-Market LMS

TalentLMS mid-market LMS dengan fokus simplicity + engagement. Gamification mendalam (point/badge/leaderboard/reward), branded certificate yang bisa di-customize logo + design, automated re-certification flow. UX mobile-first dengan onboarding cepat.

Relevansi Portal HC: Pattern gamification + branded certificate bisa naikkan engagement pekerja untuk mandatory training (K3, HSE), terutama kalau ada leaderboard per section.

TalentLMS Gamification

Sumber: vendor docs + community/blog posts, per riset 2026-05.

2.2 Industri Migas (4 Platform)

4 platform Industri Migas/Energy yang jadi acuan operational excellence + competency framework:

Shell Learning Academy Migas

Shell Learning Academy gabungkan praktik competence-based via hands-on roles + job tasks + training formal. Kolaborasi dengan PetroSkills Alliance (bersama BP) untuk training oil & gas industri-wide. Punya Graduate Programme 3-tahun dengan job rotation + coaching personal + leadership development.

Relevansi: Pola job rotation tracking + coaching pairing formal cocok untuk pengembangan pekerja Pertamina KPB.

Shell Learning Academy
Chevron OEMS Migas

Operational Excellence Management System (launched 2004, updated 2018) — framework manajemen operasional yang mengaitkan kompetensi dengan risiko + safeguards. Pelatihan operator wajib selesai dalam 3 tahun pertama kerja. Verifikasi external oleh Lloyd's Register selaras dengan standar ISO 14001 + ISO 45001.

Relevansi: Pola external auditor verification + time-bound mandatory training (rule 3 tahun) layak adopsi.

Chevron OEMS
ExxonMobil OIMS Migas

Operations Integrity Management System — framework 11 elemen, elemen ke-6 khusus "Personnel and Training". Mengaitkan kompetensi pekerja dengan pencegahan insiden lewat Process Safety Management. Independent verification oleh auditor korporasi internal. Matriks training terdokumentasi per asset (refinery, platform, terminal).

Relevansi: Pola linkage kompetensi ↔ risiko proses + matriks training per-asset cocok untuk konteks pengilangan KPB.

ExxonMobil Operations
PPSDM Migas (LSP) Domestik

Lembaga Sertifikasi Profesi (LSP) milik pemerintah Indonesia khusus migas — satu-satunya LSP yang sah untuk uji kompetensi migas. Ada 45 ruang lingkup sertifikasi; bidang hilir/pengilangan mencakup Crude Distilling Unit, Vacuum Distilling Unit, Operasi Boiler, Lab Pengujian. Mengacu SKKNI sesuai Permen ESDM (wajib).

Relevansi: Portal HC dapat berperan feeder data ke PPSDM Migas + tracker re-sertifikasi 3-tahun + repositori sertifikat LSP — kritis untuk compliance regulasi Indonesia.

PPSDM Migas ESDM

Detail framework OEMS/OIMS + Permen ESDM regulasi: lihat sertifikat-doc-gap-benchmark-v1.0 tag di git history (versi executive).

§4 Deep-Dive Top-5 Gap Kritis

5 gap paling kritis dengan deep-dive: kondisi saat ini, best practice external, rekomendasi spesifik, dan estimasi effort.

📍 Current State Portal HC: Renewal sertifikat hanya bisa di-trigger HC/Admin via menu Renewal Certificate di Admin Panel. Pekerja yang sertifikatnya akan expired hanya bisa nunggu notif + tindak lanjut HC. Tidak ada visibility "saya bisa renew sendiri kapan?".

🌐 Best Practice External: Docebo mengizinkan user trigger renewal kapan saja (selama status course Complete + cert config allow renewal). Pekerja proaktif, beban HC turun signifikan.

💡 Rekomendasi untuk Portal HC:

  • Tambah menu "Renew Sertifikat Saya" di CMP/CDP untuk peran Coachee
  • Allowlist tipe sertifikat yang boleh self-renew (non-Permanent + IsPassed)
  • Setelah self-renew → trigger workflow approval ke Coach/HC
  • Log audit per self-renew (siapa, kapan, sertifikat apa)
  • Notifikasi otomatis ke Coach saat ada self-renew request menunggu approval

⚙️ Effort Estimate: Medium (3-6 bulan)

📍 Current State Portal HC: Sertifikat PDF generate dengan nomor format KPB/SEQ/ROMAWI/YYYY tapi tidak ada cara external (rekruter, vendor, BPK) untuk verify keaslian sertifikat. Pihak eksternal harus phone/email HC manual untuk konfirmasi.

🌐 Best Practice External: TalentLMS branded certificate dengan public URL + verification. Recipient bisa share link, third-party verify status sertifikat real-time tanpa login.

💡 Rekomendasi untuk Portal HC:

  • Generate UUID unik per sertifikat saat issue (kolom baru di tabel)
  • Public endpoint /verify/{uuid} read-only, no authentication
  • Render halaman verify simple: nama pekerja, kompetensi, status, valid until (no detail PII sensitif)
  • Embed QR code di PDF sertifikat → arahkan ke URL verify
  • Rate limit endpoint untuk mencegah enumeration / scraping

⚙️ Effort Estimate: Quick Win (1-3 bulan)

📍 Current State Portal HC: Operasi sertifikat hanya per-record: HC harus klik satu-per-satu untuk renewal, re-issue, atau export individual. Dengan ribuan pekerja Pertamina KPB, ini jadi bottleneck operasional yang menghabiskan waktu HC.

🌐 Best Practice External: Cornerstone mendukung auto-reassign mass untuk recurring training + bulk export by population/role/location. Workday auto-enrollment rule menjalankan mass-action tanpa intervensi admin.

💡 Rekomendasi untuk Portal HC:

  • Tambah bulk-checkbox kolom di list CDP Certification Management
  • Bulk renewal action: 1 form input target → ratusan record diproses sekaligus
  • Bulk re-issue PDF (regenerate dengan template baru tanpa ubah data)
  • Bulk export Excel dengan filter advance (status, section, valid range, renewal chain depth)
  • Batch progress indicator + email summary ke HC saat batch selesai

⚙️ Effort Estimate: Medium (3-6 bulan)

📍 Current State Portal HC: Data pekerja (NIP, nama, section, jabatan) di-entry manual atau di-impor periodik via Excel. Saat pekerja pindah section/jabatan, status sertifikat tidak otomatis ter-update — perlu maintenance manual HC.

🌐 Best Practice External: Workday Learning HRIS-Learning native — saat pekerja promosi, training requirement otomatis update tanpa admin intervention. Single source of truth untuk data karyawan.

💡 Rekomendasi untuk Portal HC:

  • Integrasi API ke sistem HR Pertamina pusat (SAP HCM atau Oracle HCM)
  • Sync employee master nightly: NIP, jabatan, section, status active/inactive
  • Saat detect pindah section → auto-update RBAC scope + flag sertifikat affected
  • Webhook saat employee inactive → freeze sertifikat (stop generate notif)
  • Log full audit perubahan source-of-truth (apa yang berubah, kapan)

⚙️ Effort Estimate: Long-term (>9 bulan, butuh IT Pertamina pusat)

📍 Current State Portal HC: Aksi di Portal HC ter-log umum via file log aplikasi, tapi tidak ada audit trail per-record yang siap diserahkan ke regulator (BPK, audit internal Pertamina). Sertifikat tidak punya record "siapa edit, kapan, dari IP mana".

🌐 Best Practice External: Cornerstone Learning implement 21 CFR Part 11 — defensible audit trail dengan timestamp + IP + signature digital, exportable PDF untuk regulator. Setiap event training tercatat permanen.

💡 Rekomendasi untuk Portal HC:

  • Tabel SertifikatAuditLog dedicated: UserId, Action, SertifikatId, OldValue, NewValue, Timestamp, IP, UserAgent
  • Trigger insert log di tiap CRUD sertifikat + renewal + re-issue
  • Endpoint admin export audit trail per-sertifikat ke PDF (BPK-ready)
  • Retention policy ≥ 7 tahun (sesuai regulasi Pertamina)
  • Read-only — no delete/update ke audit table (immutable)

⚙️ Effort Estimate: Medium (3-6 bulan)

Top-5 dipilih berdasarkan impact operasional + frekuensi keluhan HC + benchmark gap. Per riset 2026-05.

§5 Roadmap Rekomendasi 3-Bucket

Rekomendasi roadmap pengembangan dibagi 3 bucket berdasarkan effort dan dependency:

Quick Win (1-3 bulan)

Config/UI tweak existing kode, no new arsitektur. Hasil cepat momentum.

  • QR verify URL sertifikat R-02 / §9.4 — endpoint public read-only + embed QR di PDF
  • DB Index pada ValidUntil R-01 / G-13 — quick wins query performance ekspirasi check
  • Auto-Email Reminder Expiry R-03 / G-02 — email template + dispatch baseline untuk Hangfire fase berikutnya
  • Rate-Limit Export endpoint R-04 / G-12 — throttle cegah OOM crash export besar
  • Cert Revocation List endpoint R-05 / N-07 — soft revoke + JSON list public
  • Permanent Case-Insensitive R-06 / G-24 — bug fix status derivation
  • Permanent + ValidUntil Validator R-07 / G-22 — bug fix validator data invalid
  • Renewal Cycle Detection (Union-Find) R-08 / G-18 — algoritma detect chain A→B→A
  • Email template polish + deep link §9.4 — link langsung ke renewal/detail page
  • Bulk export filter advance CDP §9.4 — extend Export Excel filter status/section/range
  • Analytics dashboard widget §9.4 — sum/group by status + section di homepage CDP
  • Branded certificate template improvement §9.4 — logo Pertamina KPB + watermark + signature
Medium (3-9 bulan)

New feature, 1-2 controller baru, kemungkinan migration DB.

  • Self-Service Renewal Portal pekerja G-01 + N-03 / §9.4 — beban HC turun drastis
  • Hangfire Pipeline + Expiry Automation R-09 / N-03 — daily query + dispatch + auto-create renewal assessment
  • Bulk action UI HC G-08 / §9.4 — mass renewal/re-issue + batch progress indicator
  • Skill graph foundation (KKJ ↔ sertifikat) N-04 / §9.4 — table mapping + visualisasi tree
  • Multi-channel notification (push/SMS) G-09 + §9.4 — Twilio/Firebase untuk SDM lapangan
  • Audit trail defensible (CFR-style) G-10 / §9.4 — tabel dedicated + trigger CRUD + export PDF BPK-ready
  • Soft Delete IsDeleted Flag G-14 — preserve renewal chain history
  • Cert PDF Async Generation G-16 — background queue untuk batch PDF (no thread block)
Long-term (>9 bulan)

Arsitektur baru atau integrasi external system.

  • HCM Integration (SAP HCM Pertamina pusat) §9.4 — single source of truth employee master sync
  • SSO/SAML enterprise (Pertamina AD) §9.4 — eliminasi password lokal + audit otomatis
  • Open Badges 1EdTech JSON-LD Issuer R-10 / N-01 — verifiable digital credential standar W3C VCDM 2.0
  • Blockchain Credential Verify N-02 — anchor hash ke blockchain public anti-fraud
  • AI skill recommendation + auto-tag §9.4 — Docebo-pattern ML pipeline atau external API
  • Object storage migration (Blob/S3) G-15 / §9.4 — scalability + DR cloud-native
  • External API webhook + LinkedIn import §9.4 — extensibility ekosistem external
  • LSP PPSDM Migas feeder integration Permen ESDM — compliance regulasi SKKNI nasional

Estimate awam — verify dengan tim IT sebelum execute. Sequencing dapat berubah berdasarkan resource availability + prioritas Pertamina KPB.
