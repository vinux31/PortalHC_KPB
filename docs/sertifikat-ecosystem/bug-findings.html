P01   🔴 HIGH   Path Traversal pada Upload Sertifikat Helpers/FileUploadHelper.cs:37
💡 Penjelasan Sederhana

Saat Admin/HC upload file sertifikat, sistem cuma cek extension nama file (boleh .pdf, .jpg, .png) — tapi tidak cek isi path di nama file. Penjahat yang tahu trick ini bisa kasih nama file aneh seperti ../../web.config.pdf dan file tersimpan keluar dari folder yang seharusnya — masuk ke folder rahasia server (config, kode aplikasi). Bahaya untuk security.

Detail Teknis

Function SaveFileAsync bangun safeFileName dengan template {timestamp}_{guid8}_{file.FileName}. Variable file.FileName diambil mentah dari client — tidak ada Path.GetFileName() untuk strip directory component (../, ..\, atau path absolute).

// Helpers/FileUploadHelper.cs:35-38 (kondisi sekarang)
var uniqueId = Guid.NewGuid().ToString("N")[..8];
var safeFileName = $"{DateTime.UtcNow:yyyyMMddHHmmssfff}_{uniqueId}_{file.FileName}";
//                                                                  ^^^^^^^^^^^^^
//                                                                  TIDAK DI-SANITASI
var filePath = Path.Combine(uploadDir, safeFileName);
// Path.Combine resolve ".." — keluar dari uploadDir
Cara Reproduksi
  1. Login sebagai Admin/HC, buka /Admin/AddTraining.
  2. Pilih file PDF normal. Rename via developer tools (intercept multipart) atau pakai tool seperti Postman/curl — set FileName ke ../../../wwwroot/dangerous.pdf (tetap berakhir .pdf supaya lolos validasi extension).
  3. Submit. Cek folder wwwroot/: file muncul di lokasi yang seharusnya tidak boleh.
Dampak
  • Security HIGH: File attacker bisa overwrite asset web atau muncul di lokasi serve-able (e.g., bypass auth jika diserve langsung).
  • Compliance: Audit OWASP Top 10 — A01 Broken Access Control / A04 Insecure Design.
  • Data Integrity: File hasil upload sertifikat tidak tracked benar di SertifikatUrl jika path traversal — link broken atau leak ke folder lain.
Cara Fix

Verifikasi: Run test upload dengan FileName ../../test.pdf — harus tersimpan sebagai {ts}_{guid}_test.pdf di folder uploads/certificates (tanpa ../).

P02   🟡 MED   Validasi MIME Type Hanya Extension-Only Helpers/FileUploadHelper.cs:12-24, TrainingAdminController.cs:206-215
💡 Penjelasan Sederhana

Sistem cuma cek nama file (".pdf", ".jpg") — bukan isi file sebenarnya. Mudah dibohongi: file virus virus.exe tinggal di-rename jadi virus.pdf dan lolos. File tersimpan di server (tidak langsung berbahaya kalau cuma nyimpen), tapi membuka pintu serangan kalau ada bagian sistem lain yang serve atau proses file ini.

Detail Teknis

Validasi pakai Path.GetExtension(file.FileName). Browser/sniffer tidak dipakai. Header Content-Type dari multipart juga tidak di-validate. Magic byte (PDF: %PDF, JPG: FF D8 FF, PNG: 89 50 4E 47) tidak di-check.

// Helpers/FileUploadHelper.cs:12-24
public static (bool IsValid, string? Error) ValidateCertificateFile(IFormFile? file)
{
    if (file == null || file.Length == 0) return (true, null);
    var ext = Path.GetExtension(file.FileName);  // hanya nama, bukan isi
    if (!AssessmentConstants.FileValidation.AllowedCertificateExtensions.Contains(ext))
        return (false, "Hanya file PDF, JPG, dan PNG yang diperbolehkan.");
    if (file.Length > AssessmentConstants.FileValidation.MaxCertificateFileSizeBytes)
        return (false, "Ukuran file maksimal 10MB.");
    return (true, null);
}
Cara Reproduksi
  1. Siapkan file executable kecil (test.exe) atau script HTML berbahaya.
  2. Rename ekstensi ke .pdf.
  3. Upload via /Admin/AddTraining. Validasi lolos.
  4. Cek file di folder wwwroot/uploads/certificates/ — file masih executable/script, hanya nama menipu.
Dampak
  • XSS: Jika web server auto-detect content-type berdasar magic byte, attacker bisa upload HTML berbahaya dengan nama .pdf → tampil sebagai HTML.
  • RCE chain: Jika ada feature lain yang process file (image resize, PDF parse), file fake bisa trigger vulnerability di parser.
  • Storage abuse: Selain itu, attacker bisa upload file legal-sized tapi bukan tipe yang dimaksud — abuse storage budget.
Cara Fix
// Tambah magic-byte check di ValidateCertificateFile:
public static (bool IsValid, string? Error) ValidateCertificateFile(IFormFile? file)
{
    if (file == null || file.Length == 0) return (true, null);

    var ext = Path.GetExtension(file.FileName).ToLowerInvariant();
    if (!AssessmentConstants.FileValidation.AllowedCertificateExtensions.Contains(ext))
        return (false, "Hanya file PDF, JPG, dan PNG yang diperbolehkan.");

    if (file.Length > AssessmentConstants.FileValidation.MaxCertificateFileSizeBytes)
        return (false, "Ukuran file maksimal 10MB.");

    // MAGIC BYTE CHECK
    using var stream = file.OpenReadStream();
    var header = new byte[8];
    stream.Read(header, 0, 8);
    stream.Position = 0; // reset untuk SaveFileAsync nanti

    bool magicOk = ext switch
    {
        ".pdf" => header[0] == 0x25 && header[1] == 0x50 && header[2] == 0x44 && header[3] == 0x46,  // %PDF
        ".jpg" or ".jpeg" => header[0] == 0xFF && header[1] == 0xD8 && header[2] == 0xFF,
        ".png" => header[0] == 0x89 && header[1] == 0x50 && header[2] == 0x4E && header[3] == 0x47,
        _ => false
    };

    if (!magicOk) return (false, "Isi file tidak cocok dengan ekstensi (magic byte mismatch).");
    return (true, null);
}

Alternatif: Pakai library MimeDetective via NuGet — lebih lengkap signature database.

  • Input form user kemungkinan WIB (browser local time)

// Models/CertificationManagementViewModel.cs:53-63
public static CertificateStatus DeriveCertificateStatus(DateTime? validUntil, string? certificateType)

    if (days < 0) return Expired;
    if (days <= 30) return AkanExpired;

P07   🟢 VERIFIED-OK   Cert Number Generation Race Condition Helpers/CertNumberHelper.cs, Services/GradingService.cs:302, :518
💡 Penjelasan Sederhana

Kalau 2 user submit ujian bersamaan, sistem perlu kasih NomorSertifikat unik (tidak boleh kembar). Awalnya kekhawatiran: kedua proses baca "nomor terakhir = 42" → keduanya simpen sebagai "43" → duplikat. Tapi sebenarnya aman: database punya unique index, dan kalau collision terjadi sistem otomatis retry sampai unik. Verified OK — sudah ditangani.

Detail Teknis (verifikasi)

Mekanisme race-safe:

  1. DB punya unique index IX_AssessmentSessions_NomorSertifikat (verified dari CertNumberHelper.IsDuplicateKeyException:39).
  2. GradingService.cs:302 dan :518 wrap cert generation dalam try-catch:
// Services/GradingService.cs:302+ (verified OK pattern)
int maxCertAttempts = 5;
int certAttempts = 0;
while (certAttempts < maxCertAttempts)
{
    try
    {
        var nextSeq = await CertNumberHelper.GetNextSeqAsync(_context, certYear);
        await _context.AssessmentSessions
            .Where(s => s.Id == sessionId && s.NomorSertifikat == null)
            .ExecuteUpdateAsync(s => s.SetProperty(r => r.NomorSertifikat,
                CertNumberHelper.Build(nextSeq, certNow)));
        break;  // success
    }
    catch (DbUpdateException ex) when (
        certAttempts < maxCertAttempts
        && CertNumberHelper.IsDuplicateKeyException(ex))
    {
        certAttempts++;  // retry with new SEQ
    }
}
Catatan

  • Format aktual: KPB/{SEQ:D3}/{ROMAN-MONTH}/{YEAR} — contoh KPB/043/V/2026.
  • 5 retry attempts. Probability collision > 5x dalam concurrent burst = sangat kecil (kombinasi dengan random GUID).

D01   🔴 HIGH   Dark Mode Toggle Merusak Semua Mermaid Diagram docs/sertifikat-ecosystem/index.html (theme toggle JS)
💡 Penjelasan Sederhana

Tombol ganti dark mode di file dokumentasi index.html rusak: setelah diklik, semua diagram berubah jadi gambar bom dengan tulisan "Syntax error in text". Toggle balik ke light mode juga tidak pulih. Hanya refresh halaman (F5) yang bisa kembalikan diagram. Bug di file dokumentasi sendiri, bukan di webapp Portal HC.

Detail Teknis

JS theme toggle re-render Mermaid dengan capture data-original dari el.textContent. Tapi capture dilakukan setelah Mermaid initial run sudah replace innerHTML dengan SVG. Akibatnya yang disimpan sebagai "original" adalah SVG markup, bukan syntax Mermaid asli. Re-render dengan SVG-as-source → parser fail.

// docs/sertifikat-ecosystem/index.html — JS yang buggy (sekitar line 1334)
themeToggle.addEventListener('click', () => {
  const current = html.getAttribute('data-bs-theme');
  const next = current === 'dark' ? 'light' : 'dark';
  html.setAttribute('data-bs-theme', next);

  document.querySelectorAll('.mermaid').forEach((el) => {
    const code = el.getAttribute('data-original') || el.textContent;
    //                                                ^^^^^^^^^^^^^^
    //  PERTAMA KALI: textContent = SVG markup (Mermaid sudah render)
    //  Yang disimpan sebagai data-original = SVG, bukan syntax asli
    if (!el.getAttribute('data-original')) el.setAttribute('data-original', code);
    el.innerHTML = code;  // ← Mermaid coba parse SVG sebagai syntax → error
    el.removeAttribute('data-processed');
  });
  mermaid.initialize({ ... theme: next === 'dark' ? 'dark' : 'default' ... });
  mermaid.run();
});
Cara Reproduksi
  1. Buka file://.../docs/sertifikat-ecosystem/index.html atau via HTTP server.
  2. Klik tombol bulan/dark mode di pojok kanan atas sidebar.
  3. Scroll ke section §3 atau §4 — semua Mermaid diagram tampil gambar bom merah "Syntax error in text — mermaid version 10.6.1".
  4. Konsol: Uncaught (in promise) #<Object> at fPt (mermaid.min.js:83:3761)
  5. Klik toggle balik ke light — tetap broken. Hanya F5 reload yang fix.
Dampak
  • UX: Pembaca yang pakai dark mode (developer preference) tidak bisa lihat diagram apapun.
  • Kepercayaan: Bug yang obvious mengurangi kredibilitas dokumen.
  • Distribusi: Tidak bisa share dokumen yang functional sampai fix.
Cara Fix
<script>
  // STEP 1: Capture original mermaid syntax SEBELUM Mermaid render
  document.querySelectorAll('.mermaid').forEach(el => {
    el.setAttribute('data-original', el.textContent.trim());
  });

  // STEP 2: Sekarang Mermaid initial render
  mermaid.initialize({ startOnLoad: true, theme: 'default', securityLevel: 'loose' });
  hljs.highlightAll();

  // STEP 3: Theme toggle pakai data-original yang sudah benar
  themeToggle.addEventListener('click', () => {
    const current = html.getAttribute('data-bs-theme');
    const next = current === 'dark' ? 'light' : 'dark';
    html.setAttribute('data-bs-theme', next);

    document.querySelectorAll('.mermaid').forEach((el) => {
      const code = el.getAttribute('data-original');  // sudah benar (syntax asli)
      el.innerHTML = code;
      el.removeAttribute('data-processed');
    });

    mermaid.initialize({ startOnLoad: false, theme: next === 'dark' ? 'dark' : 'default',
                         securityLevel: 'loose' });
    mermaid.run();
  });
</script>
