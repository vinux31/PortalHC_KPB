§1 Apa & Kenapa

Sertifikat di Portal HC KPB adalah bukti resmi bahwa seorang pekerja sudah punya kompetensi tertentu — entah dari lulus assessment online di sistem, atau dari training yang sudah dijalani di luar sistem dan didokumentasikan oleh HC.

Kenapa penting?

  • Compliance pengembangan SDM KPB — bukti kompetensi pekerja per jabatan
  • Evidence coaching & training — referensi sah untuk evaluasi atasan
  • Basis renewal cycle — kompetensi tidak selalu permanent, perlu refresh berkala agar pekerja tetap kompeten

2 Jalur Sumber Sertifikat
  • Otomatis (online) — Pekerja lulus assessment di Portal HC, sistem auto-generate nomor sertifikat format KPB/{NOMOR-URUT}/{ROMAWI-BULAN}/{TAHUN}. Contoh: KPB/15/V/2026.

Kapan Sertifikat TIDAK Terbit

Tim HC sering tanya: "Kenapa pekerja sudah selesai assessment tapi sertifikat tidak muncul?" Ini bukan bug — ada 3 alasan valid:

Gagal Lulus

Skor pekerja di bawah passing percentage yang ditetapkan. Status assessment: Failed. Sertifikat tidak terbit.

Lulus tapi Flag Mati

Assessment di-konfigurasi tanpa flag Generate Certificate (misal: latihan/simulasi internal). Pekerja lulus tapi tidak ada sertifikat terbit.

Nunggu Penilaian Essay

Pekerja lulus pilihan ganda, tapi ada soal essay yang belum dinilai HC. Status: Pending Grading. Sertifikat menyusul setelah HC selesai grading essay.

§4 Status & Kapan Berubah

Sertifikat punya 4 status yang dihitung sistem secara otomatis dan real-time:

Aktif

Kapan: Sertifikat valid, lebih dari 30 hari menuju tanggal expired.

Arti: Pekerja tetap kompeten, semua aman.

Aksi HC: Monitor saja, no action.

Akan Expired

Kapan: Kurang dari atau sama dengan 30 hari menuju tanggal expired.

Arti: Notifikasi otomatis sudah aktif ke pekerja + HC.

Aksi HC: Schedule training/assessment renewal.

Expired

Kapan: Tanggal kadaluarsa sudah lewat.

Arti: Kompetensi expired, pekerja perlu renewal segera.

Aksi HC: Trigger Renewal Certificate via Admin Panel.

Permanent

Kapan: Jenis sertifikat di-tag "Permanent" (training tertentu tanpa expiry).

Arti: Tidak pernah expired, tidak perlu renewal.

Aksi HC: No action.

Catatan teknis ringan: Status sertifikat bukan kolom database — sistem menghitungnya real-time setiap kali halaman dibuka, berdasarkan tanggal kadaluarsa + jenis sertifikat. Artinya: Anda tidak perlu update status manual; sistem otomatis menggeser sertifikat dari Aktif → Akan Expired → Expired sesuai berjalannya waktu.

§5 Renewal — Kenapa & Bagaimana

Kenapa renewal perlu?

Kompetensi tidak permanent. Sertifikat tertentu (misal K3, safety, sertifikasi profesi) wajib di-refresh periodik agar pekerja tetap kompeten dan compliant dengan regulasi.

Kapan trigger?
  • Otomatis — 30 hari sebelum tanggal kadaluarsa, sistem kirim notifikasi ke pekerja + HC.
  • Manual — HC/Admin proaktif via menu Renewal Certificate sebelum notif otomatis keluar.
Siapa boleh trigger?

Hanya Admin (L1) dan HC (L2). Peran lain (Manager, SectionHead, Coach, Coachee) tidak punya akses ke menu Renewal Certificate.

Alur Renewal
flowchart LR A[Sertifikat Lama
mau expired] --> B[Form Renewal

Baru dijalani pekerja] C --> D[Sertifikat Baru terbit] D --> E[Terhubung ke

Renewal Chain — Apa Itu?

Sertifikat baru hasil renewal "diingat" sistem terhubung ke sertifikat lama. Manfaatnya:

  • HC bisa lihat riwayat lengkap kompetensi pekerja: sertifikat ke-1 → renewal ke-1 → renewal ke-2, dst
  • Sertifikat lama yang sudah di-renew otomatis dianggap "sudah ditangani" — tidak masuk daftar outstanding meskipun statusnya Expired
  • Dashboard CDP cuma menampilkan sertifikat outstanding yang masih perlu di-renew, bukan sertifikat lama yang sudah ada penggantinya

§7 Peta Menu Aplikasi

Fitur sertifikat tersebar di 4 modul aplikasi Portal HC:

1. CMP — Competency Management Program

Menu utama: Records, Budget Training, Certificate, Submit Exam, Export Records

Di sini Anda bisa: melihat daftar sertifikat pekerja, mengelola anggaran training tahunan, melihat detail sertifikat per pekerja, ikut assessment online (Coachee), dan export Excel laporan.

Akses: Semua peran (dengan scope sesuai level)

2. CDP — Competency Development Program

Menu utama: Certification Management, Export Sertifikat Excel

Di sini Anda bisa: melihat dashboard utama tracking sertifikat (status, expired, renewal outstanding), export laporan sertifikat per kriteria untuk audit/review.

Akses: Semua peran (dengan scope sesuai level)

3. Admin Panel — Kelola Data

Menu utama: Manage Assessment, Renewal Certificate, Add/Edit Training, Finalize Essay Grading

Di sini Anda bisa: setup assessment baru (soal, passing score, flag generate cert), trigger renewal sertifikat, tambah/edit data training manual, nilai essay yang Pending Grading.

Akses: Admin + HC only

4. Notifikasi

Menu utama: List notifikasi user

Di sini Anda bisa: melihat alert pre-expired (30 hari sebelum kadaluarsa), notif sertifikat baru terbit, notif renewal sudah di-trigger.

Akses: Semua peran (notif personal per user)

§9 Gap & Best Practice External

Sistem sertifikat Portal HC KPB sudah jalan dan memenuhi kebutuhan dasar — tapi ada beberapa gap dibanding platform HRIS+LMS enterprise modern. Bagian ini bandingkan Portal HC dengan 5 platform terkemuka dunia (Workday, SAP SuccessFactors, Cornerstone, Docebo, TalentLMS) untuk identifikasi gap di 3 dimensi: Sistem, Flow, dan Fitur — plus roadmap rekomendasi yang actionable untuk tim HC.

Untuk versi developer-level gap analysis dengan ICE score (10 R-rec), lihat analisa-gap-benchmark.html.

9.3 Deep-Dive Top-5 Gap Kritis

5 gap paling kritis dengan deep-dive: kondisi saat ini di Portal HC, best practice external, rekomendasi spesifik, dan estimasi effort.

📍 Current State Portal HC: Renewal sertifikat hanya bisa di-trigger HC/Admin via menu Renewal Certificate di Admin Panel. Pekerja yang sertifikatnya akan expired hanya bisa nunggu notif + tindak lanjut HC. Tidak ada visibility "saya bisa renew sendiri kapan?".

🌐 Best Practice External: Docebo mengizinkan user trigger renewal kapan saja (selama status course Complete + cert config allow renewal). Pekerja proaktif, beban HC turun signifikan.

💡 Rekomendasi untuk Portal HC:

  • Tambah menu "Renew Sertifikat Saya" di CMP/CDP untuk peran Coachee
  • Allowlist tipe sertifikat yang boleh self-renew (non-Permanent + IsPassed)
  • Setelah self-renew → trigger workflow approval ke Coach/HC
  • Log audit per self-renew (siapa, kapan, sertifikat apa)
  • Notifikasi otomatis ke Coach saat ada self-renew request menunggu approval

⚙️ Effort Estimate: Medium (3-6 bulan)

📍 Current State Portal HC: Sertifikat PDF generate dengan nomor format KPB/SEQ/ROMAWI/YYYY tapi tidak ada cara external (rekruter, vendor, BPK) untuk verify keaslian sertifikat. Pihak eksternal harus phone/email HC manual untuk konfirmasi.

🌐 Best Practice External: TalentLMS branded certificate dengan public URL + verification. Recipient bisa share link, third-party verify status sertifikat real-time tanpa login.

💡 Rekomendasi untuk Portal HC:

  • Generate UUID unik per sertifikat saat issue (kolom baru di tabel)
  • Public endpoint /verify/{uuid} read-only, no authentication
  • Render halaman verify simple: nama pekerja, kompetensi, status, valid until (no detail PII sensitif)
  • Embed QR code di PDF sertifikat → arahkan ke URL verify
  • Rate limit endpoint untuk mencegah enumeration / scraping

⚙️ Effort Estimate: Quick Win (1-3 bulan)

📍 Current State Portal HC: Operasi sertifikat hanya per-record: HC harus klik satu-per-satu untuk renewal, re-issue, atau export individual. Dengan ribuan pekerja Pertamina KPB, ini jadi bottleneck operasional yang menghabiskan waktu HC.

🌐 Best Practice External: Cornerstone mendukung auto-reassign mass untuk recurring training + bulk export by population/role/location. Workday auto-enrollment rule menjalankan mass-action tanpa intervensi admin.

💡 Rekomendasi untuk Portal HC:

  • Tambah bulk-checkbox kolom di list CDP Certification Management
  • Bulk renewal action: 1 form input target → ratusan record diproses sekaligus
  • Bulk re-issue PDF (regenerate dengan template baru tanpa ubah data)
  • Bulk export Excel dengan filter advance (status, section, valid range, renewal chain depth)
  • Batch progress indicator + email summary ke HC saat batch selesai

⚙️ Effort Estimate: Medium (3-6 bulan)

📍 Current State Portal HC: Data pekerja (NIP, nama, section, jabatan) di-entry manual atau di-impor periodik via Excel. Saat pekerja pindah section/jabatan, status sertifikat tidak otomatis ter-update — perlu maintenance manual HC.

🌐 Best Practice External: Workday Learning HRIS-Learning native — saat pekerja promosi, training requirement otomatis update tanpa admin intervention. Single source of truth untuk data karyawan.

💡 Rekomendasi untuk Portal HC:

  • Integrasi API ke sistem HR Pertamina pusat (SAP HCM atau Oracle HCM)
  • Sync employee master nightly: NIP, jabatan, section, status active/inactive
  • Saat detect pindah section → auto-update RBAC scope + flag sertifikat affected
  • Webhook saat employee inactive → freeze sertifikat (stop generate notif)
  • Log full audit perubahan source-of-truth (apa yang berubah, kapan)

⚙️ Effort Estimate: Long-term (>9 bulan, butuh IT Pertamina pusat)

📍 Current State Portal HC: Aksi di Portal HC ter-log umum via file log aplikasi, tapi tidak ada audit trail per-record yang siap diserahkan ke regulator (BPK, audit internal Pertamina). Sertifikat tidak punya record "siapa edit, kapan, dari IP mana".

🌐 Best Practice External: Cornerstone Learning implement 21 CFR Part 11 — defensible audit trail dengan timestamp + IP + signature digital, exportable PDF untuk regulator. Setiap event training tercatat permanen.

💡 Rekomendasi untuk Portal HC:

  • Tabel SertifikatAuditLog dedicated: UserId, Action, SertifikatId, OldValue, NewValue, Timestamp, IP, UserAgent
  • Trigger insert log di tiap CRUD sertifikat + renewal + re-issue
  • Endpoint admin export audit trail per-sertifikat ke PDF (BPK-ready)
  • Retention policy ≥ 7 tahun (sesuai regulasi Pertamina)
  • Read-only — no delete/update ke audit table (immutable)

⚙️ Effort Estimate: Medium (3-6 bulan)

Top-5 dipilih berdasarkan impact operasional + frekuensi keluhan HC + benchmark gap. Per riset 2026-05.

9.4 Roadmap Rekomendasi

Rekomendasi roadmap pengembangan dibagi 3 bucket berdasarkan effort dan dependency:

Quick Win (1-3 bulan)

Config/UI tweak existing kode, no new arsitektur.

  • QR verify URL sertifikat — generate QR + endpoint public read-only, embed di PDF
  • Email template polish + deep link — link langsung ke renewal/detail page, bukan homepage
  • Bulk export filter advance CDP — extend Export Excel dengan filter status/section/range
  • Analytics dashboard widget — sum/group by status + section + tipe di homepage CDP
  • Branded certificate template improvement — logo Pertamina KPB + watermark + signature image
Medium (3-9 bulan)

New feature, 1-2 controller baru, kemungkinan migration DB.

  • Self-Service Renewal Portal pekerja — beban HC turun drastis
  • Bulk action UI HC — mass renewal/re-issue + batch progress indicator
  • Skill graph foundation (KKJ ↔ sertifikat) — table mapping + visualisasi tree
  • Multi-channel notification — push + SMS (Twilio/Firebase) untuk SDM lapangan
  • Audit trail defensible (CFR-style) — tabel dedicated + trigger CRUD + export PDF BPK-ready
Long-term (>9 bulan)

Arsitektur baru atau integrasi external system.

  • HCM integration (SAP HCM Pertamina) — single source of truth, koordinasi IT pusat
  • SSO/SAML enterprise (Pertamina AD) — eliminasi password lokal
  • AI skill recommendation + auto-tag content — Docebo-pattern, ML pipeline
  • Object storage migration (Blob/S3) — scalability + DR cloud-native
  • External API webhook + LinkedIn import — extensibility ke ekosistem external

Estimate awam — verify dengan tim IT sebelum execute. Sequencing dapat berubah berdasarkan resource availability + prioritas Pertamina KPB.
