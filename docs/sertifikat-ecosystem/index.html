§1 Executive Summary

Portal HC KPB mengelola sertifikat pekerja melalui 13 endpoint yang membaca dan menulis dari 9 tabel database. Sertifikat berasal dari 2 sumber:

  1. Assessment online — auto-generated saat user lulus exam + flag GenerateCertificate=true. Nomor sertifikat format KPB/{SEQ}/{ROMAN-MM}/{YYYY}.
  2. Training manual — Admin/HC upload bukti sertifikat external. SertifikatUrl menyimpan file path; NomorSertifikat di-input manual.

Status sertifikat di-derivasi runtime dari kombinasi ValidUntil + CertificateType ke 4 nilai: Aktif / AkanExpired / Expired / Permanent. Renewal chain ditrack via dual-FK (RenewsTrainingId, RenewsSessionId) yang melintasi kedua tabel sumber. Algoritma Union-Find dipakai untuk membedakan sertifikat yang sudah di-renew dari yang masih outstanding.

Modul-modul yang menyentuh sertifikat: CMP (Records, BudgetTraining, Certificate, SubmitExam, ExportRecords), CDP (CertificationManagement, ExportSertifikatExcel), Admin/KelolaData (ManageAssessment, RenewalCertificate, AddTraining/EditTraining), dan Notification.

§3 State Machine Sertifikat

Lifecycle sertifikat dari penciptaan sampai expiry/renewal. State Permanent, Renewed, Failed, dan NoCert bersifat terminal (sertifikat tidak transition lebih lanjut). PendingGrading hanya muncul untuk assessment dengan soal essay yang menunggu manual grading.

stateDiagram-v2 [*] --> Draft : create assessment Draft --> InProgress : start exam InProgress --> PendingGrading : submit + essay belum dinilai InProgress --> Failed : IsPassed=false InProgress --> NoCert : passed + GenerateCertificate=false InProgress --> Issued : passed + GenerateCertificate=true PendingGrading --> Failed : essay graded, total fail PendingGrading --> Issued : essay graded, pass + cert PendingGrading --> NoCert : pass + GenerateCertificate=false Issued --> Aktif : ValidUntil > today+30d Issued --> Permanent : CertificateType=Permanent Issued --> AkanExpired : ValidUntil within 30d Aktif --> AkanExpired : days <= 30 AkanExpired --> Expired : days < 0 Aktif --> Expired : data correction Expired --> Renewed : new TR/AS dgn Renews* FK AkanExpired --> Renewed : pre-emptive renewal Renewed --> [*] Permanent --> [*] Failed --> [*] NoCert --> [*]

Catatan: state Aktif/AkanExpired/Expired/Permanent di-derivasi runtime oleh SertifikatRow.DeriveCertificateStatus() — bukan kolom DB. Hanya AssessmentSession.Status ("Open", "InProgress", "Completed", "PendingGrading") yang persistent.

§7 Status Derivation Logic

Status sertifikat tidak disimpan di DB. Di-derivasi runtime dari kombinasi ValidUntil + CertificateType oleh function DeriveCertificateStatus().

7.1 Source Code
// Source: Models/CertificationManagementViewModel.cs:53-63
public static CertificateStatus DeriveCertificateStatus(DateTime? validUntil, string? certificateType)
{
    if (certificateType == "Permanent") return CertificateStatus.Permanent;
    if (validUntil == null) return CertificateStatus.Expired;  // non-permanent without expiry
    var days = (validUntil.Value - DateTime.UtcNow).Days;
    if (days < 0) return CertificateStatus.Expired;
    if (days <= 30) return CertificateStatus.AkanExpired;
    return CertificateStatus.Aktif;
}
7.2 Truth Table
CertificateTypeValidUntildaysStatus
"Permanent"any—Permanent
!= "Permanent"null—Expired
!= "Permanent"future > 30d> 30Aktif
!= "Permanent"0..30 days ahead0..30AkanExpired
!= "Permanent"past< 0Expired
7.3 Edge Case
  • Boundary inclusive (days == 30): kondisi days <= 30 inclusive → AkanExpired. Sertifikat tepat 30 hari mendatang sudah mulai di-flag.
  • days == 0 (hari ini): masuk AkanExpired (karena 0 >= 0 && <= 30). Tidak masuk Expired meski real-time mungkin sudah lewat jam expiry.
  • Permanent + ValidUntil terisi: ValidUntil diabaikan (return Permanent). Data invalid (kontradiktif) tidak dicegah saat input — lihat §9.
  • Timezone WIB vs UTC: pakai DateTime.UtcNow. ValidUntil kemungkinan disimpan local WIB (+7). Selisih 7 jam dapat misklasifikasi sertifikat yang expired tepat pada batas hari — lihat §9 #4.
  • Permanent semantics: "Permanent" assume case-sensitive string match. Variasi "permanent", "PERMANENT" tidak terdeteksi.

§8 Renewal Chain Algorithm

Model dual-FK lintas-tabel: TrainingRecord dan AssessmentSession keduanya bisa renew baik TR maupun AS lain via RenewsTrainingId dan RenewsSessionId (nullable, mutually exclusive per konvensi).

8.1 4 Batch Lookup (Union-Find)
// Source: CDPController.cs:3759-3794 (CertificationManagement)
var renewedByAsSessionIds = await db.AssessmentSessions
    .Where(a => a.RenewsSessionId != null && a.IsPassed == true)
    .Select(a => a.RenewsSessionId.Value).ToListAsync();

var renewedByTrSessionIds = await db.TrainingRecords
    .Where(t => t.RenewsSessionId != null)
    .Select(t => t.RenewsSessionId.Value).ToListAsync();

var renewedByAsTrainingIds = await db.AssessmentSessions
    .Where(a => a.RenewsTrainingId != null && a.IsPassed == true)
    .Select(a => a.RenewsTrainingId.Value).ToListAsync();

var renewedByTrTrainingIds = await db.TrainingRecords
    .Where(t => t.RenewsTrainingId != null)
    .Select(t => t.RenewsTrainingId.Value).ToListAsync();

// Union per type:
var renewedAssessmentSessionIds = renewedByAsSessionIds
    .Union(renewedByTrSessionIds).ToHashSet();
var renewedTrainingRecordIds = renewedByAsTrainingIds
    .Union(renewedByTrTrainingIds).ToHashSet();

// Per row:
row.IsRenewed = row.RecordType == "Assessment"
    ? renewedAssessmentSessionIds.Contains(row.SourceId)
    : renewedTrainingRecordIds.Contains(row.SourceId);
8.2 Contoh Chain
TR#1 (Pelatihan K3 Dasar, 2024)
    ↑ renewed by (AS#5.RenewsTrainingId = 1)
AS#5 (Assessment K3 Re-test, 2025, IsPassed=true)
    ↑ renewed by (TR#9.RenewsSessionId = 5)
TR#9 (Pelatihan K3 Renewal, 2026)

Pembacaan chain: TR#1 → AS#5 → TR#9 (chronological)
Display di UI: latest first (TR#9 active, TR#1+AS#5 jadi history)
8.3 Risiko (Audit Target §9)
  • Cycle Detection: Skenario A→B→A tidak dideteksi. Union-Find naive masukkan keduanya ke set "renewed" → keduanya hilang dari dashboard outstanding renewal.
  • Both FK Filled: Tidak ada CHECK constraint DB. Row dengan kedua RenewsTrainingId dan RenewsSessionId terisi = ambiguous semantic.
  • Orphan FK: Tidak ada FK constraint (kemungkinan disengaja untuk cross-table flexibility). Delete TR/AS yang di-renews → dangling reference.
  • AS.RenewsSessionId Filter IsPassed=true: renewal yang failed assessment tidak dihitung sebagai "menggantikan" — semantic correct, tapi bisa mengejutkan jika AS gagal padahal cert lama sudah expired.

§9 Bugs & Issues — Static Audit

Hasil scan 12 pattern dari Grep + Read controller per 2026-05-26. Severity: 🔴 high, 🟡 medium, 🟢 low/verified-ok. Status: Open / Verified / Mitigated.

🔴 HIGH #1 Path Traversal pada SertifikatUrl Upload Helpers/FileUploadHelper.cs:37

Problem: Nama file disimpan via safeFileName = $"{timestamp}_{uniqueId}_{file.FileName}". file.FileName tidak di-sanitasi. Attacker upload file dengan name ../../../etc/passwd bisa lolos karena Path.GetExtension di validator tetap return ".passwd" (gagal validasi) atau jika rename ke file.pdf dengan path component, hasil safeFileName = 20260526_abc12345_../../wwwroot/somewhere.pdf. Path.Combine(uploadDir, safeFileName) akan resolve .. dan keluar dari folder uploads/certificates.

Reproduksi: Upload IFormFile dengan FileName ../../web.config. Extension whitelist (.pdf/.jpg/.png) menolak, tapi nama x/../../../sertifikat.pdf lolos validasi extension.

Fix: Tambah Path.GetFileName(file.FileName) untuk strip directory components, atau ganti file.FileName di safeFileName dengan Path.GetExtension(file.FileName) saja (drop original name).

Status: Open

🟡 MED #2 MIME Type Validation Tidak Ada (Extension-only) Helpers/FileUploadHelper.cs:12-24, TrainingAdminController.cs:206-215

Problem: ValidateCertificateFile hanya cek extension via Path.GetExtension(file.FileName). Tidak ada validasi MIME type (file.ContentType) atau magic-byte check. Attacker bisa rename evil.exe ke evil.pdf dan upload — file tersimpan di server, tidak dieksekusi tapi membuka attack surface (XSS jika web server serve langsung dengan auto-detect content type, atau RCE jika diproses ulang).

Fix: Tambah magic-byte check (PDF: %PDF, JPG: FF D8 FF, PNG: 89 50 4E 47). Atau gunakan library seperti MimeDetective. Set explicit Content-Type: application/octet-stream + Content-Disposition: attachment di endpoint serve sertifikat.

Status: Open

🟡 MED #3 Renewal Cycle Detection Absent CDPController.cs:3759-3794, TrainingAdminController.cs:253-265

Problem: Validator di TrainingAdminController:253-265 hanya cek "target FK sudah pernah di-renew" (prevent double renewal). Tidak ada deteksi siklus: TR#A.RenewsTrainingId = B; TR#B.RenewsTrainingId = A (atau lebih panjang A→B→C→A). Union-Find di CertificationManagement akan masukkan keduanya ke set "renewed" → keduanya hilang dari dashboard outstanding.

Fix: Tambah graph walk saat insert renewal — traversal hingga visited atau hit cycle. Atau enforce DAG constraint: Tanggal renewal > Tanggal source.

Status: Open

🟡 MED #4 DateTime.Now vs DateTime.UtcNow Mix (Timezone Drift) Multi-file (34× Now, 92× UtcNow lintas Controllers)

Problem: Mixed usage di codebase: DateTime.UtcNow dipakai di DeriveCertificateStatus, tapi DateTime.Now sering muncul di CMPController (18×) dan CDPController (8×). Server tz WIB (+7). Sertifikat dengan ValidUntil = 2026-05-26 00:00 WIB jam 23:30 WIB hari sebelumnya: DateTime.UtcNow = 16:30 UTC 2026-05-25. Selisih (ValidUntil - UtcNow).Days bisa salah pembulatan.

Fix: Standardisasi ke DateTime.UtcNow di semua derivasi sertifikat. Konversi input user (WIB) ke UTC saat insert. Atau pakai DateOnly untuk ValidUntil (hilangkan time component).

Status: Open — perlu audit per-occurrence

🟡 MED #5 Hard Delete Cascade Orphan FK Renewal TrainingAdminController.cs:539, :744; AssessmentAdminController.cs:2136, :2295, :2447

Problem: Endpoint Delete pakai _context.TrainingRecords.Remove(record) dan _context.AssessmentSessions.Remove(session) langsung tanpa check ada record lain yang reference via RenewsTrainingId / RenewsSessionId. Hasil: dangling FK pointer.
