## 5. Phase 325 — Security Hardening

**Goal:** Tutup security gap upload file + perbaiki UX delete error 500.

### 5.1 P01 — Path Traversal Fix

**File:** `Helpers/FileUploadHelper.cs:37`

**Perubahan:**
```csharp
// SEBELUM (line 35-38):

var safeFileName = $"{DateTime.UtcNow:yyyyMMddHHmmssfff}_{uniqueId}_{file.FileName}";

// SESUDAH:
var uniqueId = Guid.NewGuid().ToString("N")[..8];

var safeFileName = $"{DateTime.UtcNow:yyyyMMddHHmmssfff}_{uniqueId}_{originalName}";
```

**Verifikasi:**
- Upload file via Postman dengan multipart `filename="../../test.pdf"` → tersimpan sebagai `{ts}_{guid}_test.pdf` di `uploads/certificates/`, **bukan** di parent folder.

### 5.2 P02 — Magic Byte Validation

**File:** `Helpers/FileUploadHelper.cs:12-24`

**Perubahan:** Tambah magic byte check di `ValidateCertificateFile`:

```csharp
public static (bool IsValid, string? Error) ValidateCertificateFile(IFormFile? file)
{
    if (file == null || file.Length == 0) return (true, null);

    var ext = Path.GetExtension(file.FileName).ToLowerInvariant();
    if (!AssessmentConstants.FileValidation.AllowedCertificateExtensions.Contains(ext))
        return (false, "Hanya file PDF, JPG, dan PNG yang diperbolehkan.");

    if (file.Length > AssessmentConstants.FileValidation.MaxCertificateFileSizeBytes)
        return (false, "Ukuran file maksimal 10MB.");

    // Magic byte check
    using var stream = file.OpenReadStream();
    var header = new byte[8];
    stream.Read(header, 0, 8);
    stream.Position = 0; // reset untuk SaveFileAsync nanti

    bool magicOk = ext switch
    {
        ".pdf"            => header[0] == 0x25 && header[1] == 0x50 && header[2] == 0x44 && header[3] == 0x46, // %PDF
        ".jpg" or ".jpeg" => header[0] == 0xFF && header[1] == 0xD8 && header[2] == 0xFF,
        ".png"            => header[0] == 0x89 && header[1] == 0x50 && header[2] == 0x4E && header[3] == 0x47,
        _ => false
    };

    if (!magicOk) return (false, "Isi file tidak cocok dengan ekstensi (magic byte mismatch).");
    return (true, null);
}
```

**Library alternatif:** NuGet `MimeDetective` (5000+ signature). Untuk PortalHC cukup 3 format, hardcode lebih simple + zero dependency.

**Verifikasi:**
- Upload `.exe` rename `.pdf` → ditolak dengan pesan "Isi file tidak cocok dengan ekstensi".
- Upload PDF/JPG/PNG asli → lolos.
- Upload zero-byte file → ditolak.

### 5.3 P05 — Hard Delete FK (Quick Patch)

**File:** `TrainingAdminController.cs:539, :744` + `AssessmentAdminController.cs:2136`

**Perubahan per endpoint:**

```csharp
try
{
    // Pre-check referencing rows
    var referencingTr = await _context.TrainingRecords
        .CountAsync(t => t.RenewsTrainingId == record.Id);
    var referencingAs = await _context.AssessmentSessions
        .CountAsync(a => a.RenewsTrainingId == record.Id);

    if (referencingTr + referencingAs > 0)
    {
        TempData["Error"] = $"Tidak bisa hapus: {referencingTr + referencingAs} sertifikat lain "
                          + "menggunakan record ini sebagai sumber renewal. "
                          + "Hapus atau update sertifikat pemakai terlebih dulu.";
        return RedirectToAction("Index");
    }

    _context.TrainingRecords.Remove(record);
    await _context.SaveChangesAsync();
    TempData["Success"] = "Training record dihapus.";
}
catch (DbUpdateException ex)
{
    _logger.LogWarning(ex, "Delete failed for TrainingRecord {Id}", record.Id);
    TempData["Error"] = "Gagal hapus: ada constraint database yang dilanggar.";
}
return RedirectToAction("Index");
```

**Note:** Soft delete (`IsDeleted` column + global query filter) **tidak masuk Phase 325** — defer ke v20.0 backlog sebagai phase tersendiri (arsitektur perubahan besar, ~1-2 hari sendiri).

**Verifikasi:**

- Setup: TR C tanpa referencing → delete → sukses, redirect dengan TempData success.

## 6. Phase 326 — Validator Hardening

**Goal:** Cegah data kontradiktif tersimpan via form Add/Edit Training.

### 6.1 P03 — DAG Enforcement (Cycle Prevention)

**File:** `TrainingAdminController.cs:253-265` + EditTraining handler

**Perubahan:** Tambah validator setelah `srcAlreadyRenewed` check:

```csharp
if (model.RenewsTrainingId.HasValue)
{
    var src = await _context.TrainingRecords.FindAsync(model.RenewsTrainingId.Value);
    if (src != null && src.Tanggal >= model.Tanggal)
    {
        ModelState.AddModelError("",
            "Tanggal renewal harus lebih besar dari tanggal sertifikat yang di-renew.");
    }
}
```

**Rasional:** Renewal chain semantik = sertifikat baru pasti lebih muda dari yang di-renew. Cycle (A↔B) otomatis ditolak karena salah satu tanggal pasti melanggar monotonic constraint.

**Note Edit case:** Sama validator, tapi exclude `model.Id` dari check (tidak boleh renew dirinya sendiri):
```csharp
if (model.RenewsTrainingId == model.Id)
{
    ModelState.AddModelError("RenewsTrainingId", "Sertifikat tidak boleh renewal dirinya sendiri.");
}
```

**Verifikasi:**
- Setup: TR A tanggal 2026-01-01. Try Add TR B dengan RenewsTrainingId=A + tanggal 2025-12-01 → form error display.
- Setup: TR A tanggal 2026-01-01. Add TR B RenewsTrainingId=A + tanggal 2026-06-01 → lolos.

### 6.2 P06 — Permanent + ValidUntil Reject

**File:** `TrainingAdminController.cs` Add + Edit POST handler

**Perubahan:**

```csharp
if (model.CertificateType == "Permanent" && model.ValidUntil != null)
{
    ModelState.AddModelError("ValidUntil",
```

**Verifikasi:**

### 6.3 Phase 326 Testing

## 7. Phase 327 — Timezone DateOnly Refactor

**Goal:** Eliminasi timezone drift permanen untuk `ValidUntil` dengan migrasi ke `DateOnly`.

### 7.1 Strategy Rationale

Bug-findings P04 propose 2 pilihan: (A) DateOnly migration, (B) UtcNow standardize. Kita pilih **(A) DateOnly**:

1. Cert validity semantik **harian** ("berlaku sampai 26 Mei 2026") — komponen jam tidak relevan.
2. Mati permanen — tidak perlu helper `WibToUtc()` rentan miss occurrence.
3. Scope sempit (1 field × 2 tabel) → migration low risk.
4. Bug-findings sendiri label A sebagai "pilihan paling simple" (line 495).

`DateTime.Now` di lokasi non-ValidUntil (logging, audit timestamp, CreatedAt) **tidak disentuh** — defer ke v20.0 polish.

### 7.2 Model Changes

**File:** `Models/TrainingRecord.cs`, `Models/AssessmentSession.cs`

```csharp
// SEBELUM:
public DateTime? ValidUntil { get; set; }

// SESUDAH:
public DateOnly? ValidUntil { get; set; }
```

### 7.3 ViewModel + Status Derivation

**File:** `Models/CertificationManagementViewModel.cs:53-63`

```csharp
public static CertificateStatus DeriveCertificateStatus(DateOnly? validUntil, string? certificateType)
{
    if (certificateType == "Permanent") return Permanent;
    if (validUntil == null) return Expired;
    var today = DateOnly.FromDateTime(DateTime.UtcNow);
    var days = validUntil.Value.DayNumber - today.DayNumber;
    if (days < 0) return Expired;
    if (days <= 30) return AkanExpired;
    return Aktif;
}
```

### 7.4 EF Migration

**Nama:** `ChangeValidUntilToDateOnly`

**Perubahan:**
- `TrainingRecords.ValidUntil`: `datetime2` → `date`
- `AssessmentSessions.ValidUntil`: `datetime2` → `date`
- Data existing: SQL Server otomatis `CAST(datetime2 AS date)` — truncate time component (lossy tapi semantic OK karena cert harian).

**Pre-migration check (lokal):**
```sql
-- Berapa row punya komponen jam non-zero?
SELECT COUNT(*) FROM TrainingRecords
WHERE ValidUntil IS NOT NULL AND CAST(ValidUntil AS TIME) <> '00:00:00';
SELECT COUNT(*) FROM AssessmentSessions
WHERE ValidUntil IS NOT NULL AND CAST(ValidUntil AS TIME) <> '00:00:00';
```

Probability nol untuk PortalHC (form input `<input type="date">` = midnight default), tapi confirm sebelum migrate.

**Rollback:** EF `Down()` migration revert `date` → `datetime2` (data tanggal tetap valid, jam = `00:00:00`).

### 7.5 Razor View Adjustments

**File:** Form input + display di Admin/CMP/CDP views

- `[DataType(DataType.Date)]` annotation tetap sama (kompatibel dengan `DateOnly?`).
- Format display: `{0:yyyy-MM-dd}` atau `.ToString("dd MMMM yyyy")` — output identik.
- ASP.NET Core 6+ binder support `DateOnly` di form binding native — minimal/no view change.

### 7.6 Phase 327 Testing

- **Unit:** `DeriveCertificateStatus` dengan DateOnly inputs:
  - Expired (validUntil < today)
  - AkanExpired (today + 1 ≤ validUntil ≤ today + 30)
  - Aktif (validUntil > today + 30)
  - Permanent (any validUntil, certificateType=Permanent)
  - Expired (validUntil null, non-Permanent)
- **Manual smoke:**
  - Add training Annual + ValidUntil today+1 → status display "AkanExpired"
  - Add training Annual + ValidUntil today-1 → status "Expired"
  - Display ValidUntil di Renewal dashboard, CMP report, CDP report, Worker view sertifikat — semua render tanggal benar
- **Regression halaman wajib:**
  - `/Admin/ManageAssessment` tab Training
  - `/Admin/RenewalCertificate`
  - `/CMP/Records`
  - `/CDP/CertificationManagement`
  - Worker dashboard sertifikat
  - PDF generation (`/CMP/CertificatePdf/{id}`) — format tanggal di PDF tetap correct
- **Playwright:** Cek `tests/` directory — kalau ada scenario sentuh ValidUntil, jalankan ulang.

### 7.7 Phase 327 Migration Workflow

Per `docs/SEED_WORKFLOW.md`:

1. Snapshot DB lokal sebelum apply migration.
2. Jalankan `dotnet ef migrations add ChangeValidUntilToDateOnly`.
3. Review generated migration file — pastikan `AlterColumn` benar (datetime2 → date).
4. Run pre-migration SQL check (komponen jam non-zero count).
5. `dotnet ef database update` di lokal.
6. Smoke test 5 halaman wajib.
7. Restore snapshot kalau ada drama, fix, retry.
8. Commit migration file + model changes + viewmodel changes bersama.

## 8. Data Flow Summary

```
Phase 325 (Security)
  ├─ Upload form → FileUploadHelper.ValidateCertificateFile (magic byte check) → SaveFileAsync (Path.GetFileName strip)
  └─ Delete endpoint → pre-check referencing → safe delete OR TempData error

Phase 326 (Validator)
  ├─ AddTraining POST → DAG validator (target.Tanggal < new.Tanggal) + Permanent+ValidUntil rejector
  └─ EditTraining POST → same validators (+ self-renewal check)

Phase 327 (Timezone)
  ├─ Model: ValidUntil DateOnly?
  ├─ Migration: datetime2 → date (truncate time component)
  ├─ DeriveCertificateStatus: DateOnly arithmetic
  └─ Razor: [DataType(Date)] tetap sama, format yyyy-MM-dd
```

## 9. Error Handling

- **Phase 325:**
  - Magic byte mismatch → ModelState error "Isi file tidak cocok dengan ekstensi (magic byte mismatch)".
  - Path traversal → silent strip via `Path.GetFileName()` (defensive, tidak ada error path baru).
  - FK delete violation → pre-check + TempData["Error"] user-friendly + log warning.
- **Phase 326:**
  - Cycle/monotonic violation → ModelState error "Tanggal renewal harus lebih besar dari tanggal sertifikat yang di-renew".
  - Self-renewal → ModelState error "Sertifikat tidak boleh renewal dirinya sendiri".
  - Permanent + ValidUntil → ModelState error "Sertifikat Permanent tidak boleh punya tanggal expired".
- **Phase 327:**
  - Migration failure → rollback via EF `Down()`.
  - DateOnly binding error di form → ASP.NET Core default error handling (kompatibel sejak .NET 6).

## 12. Risk Register

| Risk | Mitigasi |
|------|----------|
| Magic byte false positive untuk PDF variant lama | Test minimal 3 sample PDF berbeda (scanner, Word export, native PDF). Kalau ada miss, gunakan library `MimeDetective` sebagai fallback. |
| EF migration data loss (komponen jam truncate) | Pre-migration SQL check confirm zero row punya jam non-zero. Snapshot DB sebelum apply. |
| Razor view binding gagal untuk `DateOnly?` | Confirm .NET version project ≥ 6.0 (DateOnly support native). Kalau perlu, tambah model binder explicit. |
| Validator P03 break legitimate use case (back-dated renewal) | Konfirmasi via UAT: tidak ada workflow legit input renewal lebih awal dari source. Kalau ada → buat exception path. |
| IT promo migration gagal di Prod | Backup DB Prod sebelum apply. Tested di Dev dulu wajib. |
