### F4 — Sequence flow arrow di Versi X swimlane = text symbol `→` `⤳`, bukan SVG

Per plan Task 3-4 deliberate: pakai unicode arrow (`→` sequence, `⤳` message flow) sebagai inline text vs SVG line arrows untuk simplicity + print compatibility.

**Impact:** Visual kurang impactful vs slide PCP reference (page 8) yang punya SVG arrows colored merah/biru.
**Severity:** Low (per spec design decision — print-friendly priority).
**Fix:** Tidak applicable — design choice. Jika ingin upgrade ke SVG arrows, butuh task baru.

### F5 — Versi Z VSM Sesudah timeline visual impact (positive observation)

Timeline stripe Sesudah dengan shared linear scale 0-58,5j:
- VA segment: 0.43% (~5px lebar pada 1170px wrap) — nyaris tidak visible
- NVA segment: 0.17% (~2px) — invisible
- Wait segment: 13.68% — visible
- Empty striped fill: 85.72% — visible

**Impact:** Visual impact sangat brutal — Sesudah keliatan kecil banget dibanding Sebelum. **Sesuai spec design** (Q1=b shared linear scale).
**Severity:** ✅ Working as designed.

### F6 — Versi Z Issue table 2-col vs Improvement table 3-col (asymmetric)

Issue table: `No | Issue` (2 col)
Improvement table: `No | Improvement | Lane` (3 col)

Per plan Task 12 explicit: Issue table tanpa kolom Lokasi/Lane (compact), Improvement table dengan Lane.

**Impact:** Visual asymmetry side-by-side, tapi acceptable karena beda semantik (Issue konteks, Improvement lokasi penerapan).
**Severity:** ✅ Working as designed.
**Note:** Versi X punya 3-col untuk Issue (`No | Issue | Lokasi`) — inkonsisten antara X vs Z, tapi per spec.

### F7 — Improvement table column header "Applied" vs "Lokasi" inconsistency (Versi X)

Versi X Issue table column 3 = "Lokasi", Improvement table column 3 = "Applied".
Versi Z Improvement column 3 = "Lane".

**Impact:** Label inconsistency cross-table di Versi X, dan cross-file (X="Applied", Z="Lane").
**Severity:** Low (data sama, label berbeda).
**Fix:** Standardize ke "Lokasi" atau "Lane" konsisten. Plan eksplisit pakai "Applied" untuk X dan "Lane" untuk Z — design intent, tapi reviewer bisa bingung.

### F8 — Bootstrap Icons CDN online dependency

Semua icon (`bi-x-circle-fill`, `bi-check-circle-fill`, `bi-gear-fill`, dll) load via CDN `https://cdn.jsdelivr.net/npm/bootstrap-icons@1.11.3`.

**Impact:** File HTML butuh internet untuk render icon. Offline preview / air-gapped reviewer akan miss icons.
**Severity:** Low (PortalHC env biasanya ada internet).
**Fix:** Optional bundle Bootstrap Icons locally jika prioritas offline. Skip untuk sekarang.

### F9 — Marker tooltip text inconsistency (Versi X Sesudah)

Marker 3 muncul 2× di Sesudah panel:
- HC Portal lane Audit Log task: `title="3: Audit Log Lengkap"`
- Pekerja Submit task: `title="3: Audit Log"` (tanpa "Lengkap")

**Impact:** Tooltip slightly inconsistent untuk marker ID yang sama.
**Severity:** Trivial (different but compatible meanings).
**Fix:** Standardize keduanya ke `"3: Audit Log Lengkap"`.
