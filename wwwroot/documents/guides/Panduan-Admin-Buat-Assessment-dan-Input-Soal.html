⚠ Prerequisite
Sebelum membuat assessment, pastikan kategori sudah ada di sistem. Lihat Bagian A. Manage Kategori Assessment. Kalau kategori baru, buat dulu — wizard Buat Assessment hanya bisa pilih dari kategori yang sudah terdaftar.

A. Manage Kategori Assessment

Navigasi: Admin Panel → Manage Kategori Assessment

Halaman ini punya CRUD penuh untuk kategori assessment. Sebelum buat sesi assessment, kategori target harus sudah ada.

A.1 Tambah Parent Category

  1. Klik Tambah Kategori
  2. Isi Nama Kategori (contoh: "Assessment OJ", "Mandatory HSSE Training", "Sertifikasi Operator RFCC")
  3. Isi Pass Percentage Default (nilai minimum lulus untuk sesi di kategori ini; bisa di-override per sesi)
  4. Pilih Signatory User dari dropdown (pekerja yang akan menandatangani sertifikat hasil — biasanya manager unit terkait)
  5. Kosongkan Parent Category (parent = top-level)
  6. Klik Simpan

A.2 Tambah Sub-Kategori

  1. Klik Tambah Kategori
  2. Isi nama (contoh: "OJ - Operator", "OJ - Panelman")
  3. Pilih Parent Category dari dropdown (mis: "Assessment OJ")
  4. Signatory: kosongkan jika inherit parent, atau set sendiri jika override
  5. Klik Simpan
💡 Hierarki Sub-Kategori
Sistem mendukung 2 level (parent + child). Sub-kategori inherit signatory parent kalau field signatory dikosongkan. Berguna kalau satu rumpun kategori punya 1 manager yang TTD semua sub-nya.

A.3 Edit / Hapus Kategori

  • Edit: klik ikon pensil di baris kategori. Ubah nama, signatory, pass percentage. Klik Simpan.
  • Hapus: klik ikon sampah. Sistem cek apakah ada sesi assessment aktif yang masih pakai kategori ini. Kalau ada, hapus di-block dan muncul daftar sesi yang menghalangi — selesaikan dulu sesi tersebut atau pindahkan ke kategori lain sebelum hapus.
⚠ Hati-hati Hapus
Kategori yang sudah dipakai sesi assessment (aktif maupun arsip) berdampak ke histori records peserta. Lebih aman rename daripada hapus, kecuali kategori belum pernah dipakai sama sekali.

B. Buat Assessment Baru

Navigasi: Kelola Data → Manage Assessment & Training → klik tombol "Buat Assessment"

Halaman Manage Assessment & Training — klik tombol "Buat Assessment" di kanan atas

Anda akan masuk ke wizard 4 langkah:

Langkah 1 — Kategori & Judul

  1. Pilih Kategori dari dropdown (OJT, IHT, Training Licencor, OTS, Mandatory, Assessment Proton, dll)
  2. Isi Judul Assessment (contoh: "Assessment OJT - Operator RFCC Batch 1")
  3. Klik Selanjutnya
Langkah 1: Pilih kategori dan isi judul assessment
⚠ Khusus Assessment Proton
Jika kategori = "Assessment Proton", muncul field tambahan Track (Operator/Panelman, Tahun 1/2/3). Pilih sesuai penugasan coachee.

Langkah 2 — Pilih Peserta

  1. Gunakan filter bagian (dropdown) untuk memfilter per unit kerja
  2. Gunakan kolom pencarian untuk cari nama/email
  3. Centang karyawan yang akan menjadi peserta (minimal 1)
  4. Gunakan tombol Select All / Deselect All jika perlu
  5. Klik Selanjutnya
Langkah 2: Daftar karyawan dengan checkbox, filter bagian, dan pencarian

Langkah 3 — Settings

  2. Klik "Edit" pada masing-masing card jika ada yang perlu diubah

  4. Muncul modal sukses → klik "Manage Questions" untuk lanjut ke input soal

C. Buat Paket Soal

Setelah assessment dibuat, Anda perlu membuat paket soal sebelum peserta bisa mengerjakan.

  1. Di halaman Manage Packages, masukkan nama paket (contoh: "Paket A")
  2. Klik "Create Package"
  3. Ulangi jika ingin membuat lebih dari 1 paket (multi-package)
Halaman Manage Packages — form tambah paket, daftar paket, dan distribusi Elemen Teknis
💡 Single vs Multi-Package
Single Package = semua peserta dapat soal sama.
Multi-Package = peserta di-assign ke paket berbeda (distribusi otomatis), cocok untuk mengurangi kecurangan.

E. Preview & Verifikasi Soal

Klik tombol "Preview" pada paket untuk melihat soal seperti yang dilihat peserta.

Yang perlu dicek:

  • Teks soal tidak ada typo
  • Semua pilihan jawaban lengkap (A, B, C, D)
  • Jawaban bertanda "Correct" (hijau) memang benar
Preview soal — jawaban benar ditandai hijau dengan badge "Correct"
🔄 Soal Diacak Saat Ujian
Preview menampilkan soal dalam urutan import. Saat ujian, soal akan diacak otomatis untuk setiap peserta.
ℹ Preview untuk Essay & MA
  • Essay: preview tampilkan rubrik internal di samping textarea jawaban. Pastikan rubrik jelas — HC lain yang grade nanti pakai rubrik ini sebagai referensi.
  • Multiple Answer: preview tampilkan semua opsi yang ditandai sebagai benar (di-highlight). Cek tidak ada yang lupa ditandai.

F. Tipe Soal Detail (MC, MA, Essay)

Sistem mendukung tiga tipe soal. Pilih tipe saat input soal via Excel kolom QuestionType atau form manual.

F.1 Multiple Choice (MC)

  • Default semua soal lama (null = MC untuk backward compat).
  • Satu jawaban benar dari 4–5 opsi (radio button di UI peserta).
  • Auto-grading instan saat peserta submit.
  • Format Excel Correct: huruf opsi benar (mis: B).

F.2 Multiple Answer (MA)

  • Lebih dari satu jawaban benar (checkbox di UI peserta).
  • Peserta harus pilih semua opsi benar untuk dianggap betul. Kurang satu atau lebih satu = salah (binary scoring).
  • Auto-grading instan.
  • Format Excel Correct: huruf opsi dipisah koma tanpa spasi (mis: A,C,D).
  • Feature ditambahkan Phase 296.

F.3 Essay

  • Jawaban teks bebas (textarea di UI peserta).
  • Field tambahan saat input soal:
    • Rubrik — kunci jawaban referensi internal HC (TIDAK ditampilkan ke peserta).
    • MaxCharacters — batas karakter jawaban (default 2000).
  • Grading manual oleh HC — hasil tidak instan. Peserta lihat status "Menunggu Penilaian" di halaman hasil sampai HC selesai grade.
  • Workflow grading: HC buka sesi → tab Essay Grading → baca jawaban peserta → bandingkan dengan rubrik → input nilai (0–100% atau Pass/Fail tergantung config).

G. Manual Entry Sertifikat

Manual Entry adalah jalur alternatif untuk catat sertifikat peserta yang sudah punya hasil dari lembaga eksternal — tanpa harus bikin ujian online.

G.1 Kapan Pakai Manual Entry

  • Peserta sudah punya sertifikat sertifikasi dari BNSP, LSP, atau lembaga eksternal lain.
  • Pelatihan di-host vendor luar (mis: Schneider Electric, Honeywell) yang punya sistem ujian sendiri.
  • Historical record — input sertifikat lama yang belum tercatat di sistem.

G.2 Cara Buat Manual Entry

  1. Buat assessment baru seperti biasa (Bagian B)
  2. Di Langkah Settings, aktifkan toggle Manual Entry (field IsManualEntry=true)
  3. Field tambahan muncul:
    FieldKeterangan
    PenyelenggaraNama lembaga (mis: "BNSP", "PT Schneider Electric")
    KotaKota pelaksanaan pelatihan/ujian
    SubKategoriSub-kategori spesifik (mis: "Operator Listrik Tegangan Menengah")
    CertificateTypePilih: Kompetensi (BNSP/LSP), Profesi (asosiasi profesi), Pelatihan (sertifikat partisipasi)
  4. Pilih peserta (1 orang per Manual Entry — biasanya tidak batch)
  5. Upload file sertifikat (PDF/JPG) sebagai bukti
  6. Input nilai/status langsung tanpa peserta ujian online
  7. Submit — sertifikat masuk records peserta seperti hasil ujian online
ℹ Beda Manual Entry vs Regular
Manual Entry tidak generate sesi ujian online. Peserta tidak akan lihat sesi ini di tab Assessment-nya — sertifikat langsung muncul di tab Riwayat dan Sertifikat.

H. Operasional Sesi Berjalan

Fitur untuk monitor & intervensi sesi assessment yang sedang berjalan atau sudah selesai.

H.1 Real-Time Monitoring (SignalR)

  • Buka Admin Monitoring untuk sesi tertentu
  • Live dashboard tampilkan: siapa yang sudah masuk, soal ke berapa, sisa waktu, jumlah jawaban terisi
  • Update otomatis tanpa refresh (via AssessmentHub SignalR)
  • Bisa kirim notifikasi broadcast ke semua peserta sesi (mis: "Soal nomor 7 typo, jawab sesuai pemahaman terbaik")

H.2 Reshuffle Package

  • Kalau perlu ulang acak urutan opsi (mis: ada peserta yang bocor jawaban), klik Reshuffle di halaman package
  • Reshuffle Per-Sesi: acak ulang untuk semua peserta di 1 sesi
  • Reshuffle Bulk: acak ulang untuk semua sesi yang pakai paket ini
  • Hanya display A/B/C/D yang random — grading tetap konsisten via PackageOption.Id di backend

H.3 Akhiri Ujian

  • AkhiriUjian (per peserta): paksa submit semua jawaban yang sudah ada untuk 1 peserta. Berguna kalau peserta lupa submit atau koneksi peserta putus.
  • AkhiriSemuaUjian (bulk): paksa submit semua peserta sesi sekaligus. Berguna untuk keadaan darurat atau waktu habis sistemik.
  • Hasil tetap auto-graded (kecuali Essay yang masih perlu HC nilai manual)

H.4 Edit Jawaban Peserta

  • Setelah ujian selesai, Admin bisa override jawaban peserta via halaman Detail Sesi → klik nama peserta → edit jawaban per soal
  • Use case: koreksi typo sistem, adjudikasi soal yang ambigu, koreksi grading Essay
  • Setiap edit di-log otomatis ke tabel AssessmentEditLog dengan: timestamp, user yang edit, nilai lama, nilai baru, alasan (kalau diisi)
  • Audit trail lengkap — lihat Bagian J. Audit Log
⚠ Hati-hati Edit Jawaban
Edit jawaban post-submit adalah fitur sensitif — bisa mengubah nilai/kelulusan peserta. Gunakan hanya untuk koreksi yang benar-benar perlu. Selalu isi alasan di field Reason untuk audit trail.

I. Renewal Chain (Perpanjangan Sertifikat)

Untuk sertifikat yang perlu diperbarui berkala (mis: lisensi tahunan, sertifikasi yang expire), gunakan Renewal Chain untuk link sertifikat lama ke baru.

I.1 Cara Buat Renewal

  1. Buat sesi assessment baru seperti biasa (Bagian B)
  2. Di Langkah Settings, isi Renews Session ID (RenewsSessionId) dengan ID sesi sertifikat lama yang diperbarui
  3. Atau, isi Renews Training ID (RenewsTrainingId) kalau yang diperbarui adalah training record (bukan sesi assessment)
  4. Submit dan lanjut flow normal

I.2 Dampak di Sistem

  • Sertifikat baru link ke sertifikat lama — histori chain perpanjangan tersusun otomatis
  • Peserta lihat semua sertifikat di tab Riwayat, dengan marker Perpanjangan dari [sesi lama]
  • Sertifikat lama tetap valid sampai tanggal expire-nya, sertifikat baru ambil alih setelahnya
  • Laporan compliance bisa difilter berdasarkan chain (mis: "Pekerja yang sertifikatnya akan expire 3 bulan ke depan")
ℹ Use Case Renewal
Cocok untuk: lisensi K3, sertifikasi BNSP yang berlaku 3 tahun, sertifikat OPS yang perlu refresher tahunan. Feature ditambahkan Phase 200.

J. Audit Log

Sistem mencatat semua perubahan sensitif di assessment untuk audit trail & akuntabilitas.

J.1 AssessmentEditLog (Edit Jawaban Peserta)

  • Setiap kali Admin edit jawaban peserta post-submit, log otomatis dengan:
    • Timestamp (tanggal + jam edit)
    • User yang edit (email + nama Admin)
    • Soal nomor yang diedit
    • Nilai lama vs nilai baru
    • Alasan (kalau diisi)
  • Akses: halaman Detail Sesi → tab Edit Log

J.2 Audit Log Umum

  • Akses route terpisah: Admin Panel → Audit Log
  • Catatan aksi sensitif lain: create/delete sesi assessment, change kategori, change signatory, reshuffle, akhiri ujian bulk
  • Filter by: user, tanggal, tipe aksi, sesi target
  • Export Excel untuk laporan compliance / audit eksternal
⚠ Retention Policy
Audit log tidak boleh dihapus via UI. Retention permanen untuk kebutuhan audit internal/eksternal. Kalau ada constraint storage, koordinasi dengan Team IT.

Tips Penting

  • Siapkan file Excel soal sebelum membuat assessment agar proses lebih cepat
  • Untuk multi-package, pastikan jumlah soal tiap paket sama
  • Isi kolom Elemen Teknis untuk analisis kompetensi per area yang lebih baik
  • Selalu preview sebelum assessment dibuka — kesalahan jawaban benar langsung berdampak pada nilai
  • Gunakan status "Upcoming" untuk assessment yang belum waktunya dibuka
  • Token bisa di-Regenerate kapan saja dari menu Aksi di daftar assessment
  • Peserta tambahan bisa ditambah lewat halaman Edit Assessment
  • Essay grading workflow: selesaikan grading Essay dalam 5 hari kerja setelah ujian. Peserta yang menunggu lama bakal complain ke HC. Bagi grading antar HC kalau sesi besar (50+ peserta) supaya tidak bottleneck.
  • Signatory inheritance: kalau seluruh sub-kategori di satu rumpun pakai signatory yang sama, kosongkan signatory di sub-kategori — inherit dari parent. Lebih maintainable: ganti signatory di parent = semua sub ikut ganti.
  • Backup sebelum Reshuffle Bulk: reshuffle bulk affect semua sesi yang pakai paket ini, termasuk yang sudah selesai. Verify dampak sebelum klik — preferensi pakai reshuffle per-sesi untuk safety.
