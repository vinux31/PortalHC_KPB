Struktur Website Portal HC KPB

Arsitektur Aplikasi Human Capital Pertamina Kilang Balikpapan
Versi 1.0 · April 2026 · Platform: ASP.NET Core MVC (.NET)

1. Gambaran Umum

Portal HC KPB adalah aplikasi web terpusat untuk mengelola proses Human Capital di Pertamina Kilang Balikpapan — mencakup pengelolaan kompetensi pekerja (CMP), pengembangan karier (CDP), assessment online, coaching PROTON, sertifikasi, training budget, hingga administrasi organisasi. Aplikasi dibangun dengan pola Model-View-Controller (MVC) di atas .NET, menggunakan Entity Framework Core untuk akses data, SignalR untuk komunikasi real-time, dan autentikasi hybrid (LDAP Active Directory + lokal).

2. Arsitektur High-Level

Aplikasi mengikuti struktur standar ASP.NET Core MVC dengan pemisahan tanggung jawab yang jelas antar lapisan:

PortalHC_KPB/ ├── Controllers/ // Endpoint HTTP — menerima request, orkestrasi logika ├── Models/ // Entitas database & ViewModel untuk halaman ├── Views/ // Template UI (.cshtml) — HTML + Razor syntax ├── Services/ // Business logic terpisah (auth, grading, notifikasi, dsb.) ├── Data/ // DbContext (EF Core) + seed data awal ├── Hubs/ // SignalR hub — real-time event assessment ├── Middleware/ // Pipeline custom (maintenance mode, impersonation) ├── Helpers/ // Utility & extension methods ├── Migrations/ // Script perubahan skema database (EF Core) ├── ViewComponents/ // Komponen UI reusable (badge, banner, widget) ├── wwwroot/ // Asset statis: CSS, JS, gambar, dokumen panduan ├── Program.cs // Entry point — konfigurasi service & middleware ├── appsettings.json // Konfigurasi: connection string, LDAP, dsb. └── HcPortal.csproj // Definisi project & dependency

7. Infrastruktur Pendukung

7.1 Data Layer

Data/ApplicationDbContext.cs adalah jembatan antara kode C# dan database SQL Server via Entity Framework Core. Berisi DbSet<T> untuk semua entitas. File SeedData.cs mengisi data awal (role, admin default, kategori, dsb.) saat aplikasi pertama kali dijalankan.

7.2 Middleware

  • MaintenanceModeMiddleware — bila mode maintenance aktif, seluruh request non-Admin dialihkan ke halaman "sedang dalam perbaikan".
  • ImpersonationMiddleware — memastikan session impersonation terdeteksi di setiap request dan menampilkan banner.

7.3 Hubs (SignalR)

Hubs/AssessmentHub.cs menyediakan komunikasi dua arah realtime untuk monitoring ujian: admin dapat melihat peserta yang online, progress, waktu tersisa, dan kejadian suspicious (keluar fullscreen, switch tab) tanpa refresh halaman.

Akses halaman diatur berlapis berdasarkan role pengguna (disimpan di Models/UserRoles.cs). Urutan level (1 = tertinggi):

9. Alur Request — Bagaimana Halaman Muncul

  1. User membuka URL → request masuk ke Program.cs (konfigurasi pipeline).
  2. Middleware dijalankan berurutan: Authentication → MaintenanceMode → Impersonation.
  3. Routing mengarahkan ke Controller + Action sesuai URL (mis. /CMP/Records → CMPController.Records()).
  4. Controller memanggil Service untuk logika bisnis & DbContext untuk data.
  5. Data dikemas ke ViewModel, lalu dikirim ke View (.cshtml) untuk dirender jadi HTML.
  6. Browser menerima HTML final + asset dari wwwroot; event real-time berjalan via SignalR Hub bila dibutuhkan.
